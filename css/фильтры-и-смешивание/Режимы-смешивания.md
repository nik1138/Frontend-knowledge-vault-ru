# Режимы смешивания

Режимы смешивания (blending modes) в CSS позволяют комбинировать цвета двух или более элементов по различным алгоритмам, создавая сложные визуальные эффекты. Они особенно полезны при создании дизайна с наложением изображений, текстом на изображениях и другими эффектами.

## Основные понятия

Режимы смешивания определяют, как цвета одного элемента взаимодействуют с цветами элемента, находящегося под ним. В CSS есть два основных свойства для управления смешиванием:
- `mix-blend-mode` - для смешивания элемента с фоном
- `background-blend-mode` - для смешивания слоев фона одного элемента

## Свойство mix-blend-mode

Свойство `mix-blend-mode` позволяет элементу смешиваться с элементами, находящимися под ним, а также с фоном документа.

### Пример использования

```css
.blend-element {
  mix-blend-mode: multiply;
}
```

## Доступные режимы смешивания

### 1. normal - нормальный режим

```css
.normal-blend {
  mix-blend-mode: normal;
}
```

Стандартный режим, при котором верхний элемент полностью перекрывает нижний.

### 2. multiply - умножение

```css
.multiply-blend {
  mix-blend-mode: multiply;
}
```

Режим умножения умножает цвета верхнего и нижнего элементов, создавая более темные оттенки. Белый цвет становится прозрачным, черный остается черным.

### 3. screen - экранирование

```css
.screen-blend {
  mix-blend-mode: screen;
}
```

Обратный режим умножения - создает более светлые оттенки. Черный цвет становится прозрачным, белый остается белым.

### 4. overlay - наложение

```css
.overlay-blend {
  mix-blend-mode: overlay;
}
```

Комбинация умножения и экранирования, сохраняющая контрастность нижнего элемента. Светлые участки становятся светлее, темные - темнее.

### 5. darken - затемнение

```css
.darken-blend {
  mix-blend-mode: darken;
}
```

Выбирает более темный цвет из двух элементов для каждой точки.

### 6. lighten - осветление

```css
.lighten-blend {
  mix-blend-mode: lighten;
}
```

Выбирает более светлый цвет из двух элементов для каждой точки.

### 7. color-dodge - осветление основы

```css
.color-dodge-blend {
  mix-blend-mode: color-dodge;
}
```

Осветляет нижний цвет, чтобы отразить верхний цвет. Создает яркие, почти сияющие эффекты.

### 8. color-burn - затемнение основы

```css
.color-burn-blend {
  mix-blend-mode: color-burn;
}
```

Затемняет нижний цвет, чтобы отразить верхний цвет. Противоположный эффект осветления основы.

### 9. hard-light - жесткий свет

```css
.hard-light-blend {
  mix-blend-mode: hard-light;
}
```

Умножает или экранирует цвета в зависимости от верхнего цвета. Создает эффект направленного света.

### 10. soft-light - мягкий свет

```css
.soft-light-blend {
  mix-blend-mode: soft-light;
}
```

Похож на жесткий свет, но более мягкий. Создает эффект рассеянного света.

### 11. difference - разница

```css
.difference-blend {
  mix-blend-mode: difference;
}
```

Вычитает более темный цвет из более светлого. Белый инвертирует другой цвет, черный его оставляет без изменений.

### 12. exclusion - исключение

```css
.exclusion-blend {
  mix-blend-mode: exclusion;
}
```

Похож на разницу, но с меньшим контрастом. Создает более мягкие эффекты.

### 13. hue - оттенок

```css
.hue-blend {
  mix-blend-mode: hue;
}
```

Создает цвет с оттенком верхнего элемента и насыщенностью/яркостью нижнего.

### 14. saturation - насыщенность

```css
.saturation-blend {
  mix-blend-mode: saturation;
}
```

Создает цвет с насыщенностью верхнего элемента и оттенком/яркостью нижнего.

### 15. color - цвет

```css
.color-blend {
  mix-blend-mode: color;
}
```

Создает цвет с оттенком и насыщенностью верхнего элемента и яркостью нижнего.

### 16. luminosity - яркость

```css
.luminosity-blend {
  mix-blend-mode: luminosity;
}
```

Создает цвет с яркостью верхнего элемента и оттенком/насыщенностью нижнего.

## Свойство background-blend-mode

Свойство `background-blend-mode` позволяет смешивать слои фона одного элемента:

```css
.blended-background {
  background-image: url(image1.jpg), url(image2.jpg);
  background-blend-mode: multiply;
}
```

## Практические примеры использования

### Текст с эффектом смешивания с фоном

```css
.blend-text {
  mix-blend-mode: screen;
  color: white;
  position: absolute;
  top: 50%;
  left: 50%;
  transform: translate(-50%, -50%);
}
```

### Создание эффекта "светящихся" элементов

```css
.glowing-element {
  mix-blend-mode: screen;
  background: radial-gradient(circle, yellow, transparent 70%);
}
```

### Смешивание нескольких изображений

```css
.image-composite {
  background-image: url(texture1.png), url(texture2.png);
  background-blend-mode: overlay;
}
```

## Совместимость и производительность

Поддержка режимов смешивания в браузерах в 2025 году:
- Chrome: полная поддержка с версии 41+
- Firefox: полная поддержка с версии 32+
- Safari: полная поддержка с версии 8+
- Edge: полная поддержка с версии 79+

При использовании режимов смешивания важно учитывать:
- Они могут быть ресурсоемкими, особенно на мобильных устройствах
- Рекомендуется тестировать производительность на различных устройствах
- В российских проектах особенно важно учитывать производительность на бюджетных устройствах

## Применение в российских веб-проектах

В 2025 году российские дизайнеры и разработчики активно используют режимы смешивания для:
- Создания современного визуального дизайна
- Интеграции текста с изображениями
- Создания эффектов наложения при работе с брендированным контентом
- Адаптации дизайна под различные темы (светлая/темная)

## Заключение

Режимы смешивания предоставляют мощный инструмент для создания сложных визуальных эффектов в веб-дизайне. Их правильное использование позволяет создавать более выразительные и современные интерфейсы без необходимости в сложной обработке изображений вне браузера.

Для более глубокого понимания рекомендуется изучить [[CSS-фильтры]], [[Маски]] и [[Backdrop-filter]].

> [!tip]
> При работе с режимами смешивания в российских проектах рекомендуется тестировать визуальные эффекты на различных устройствах, особенно учитывая разнообразие используемых экранов в регионе.

> [!warning]
> Избегайте чрезмерного использования режимов смешивания, особенно в сочетании с анимациями, так как это может негативно сказаться на производительности и читаемости контента.
