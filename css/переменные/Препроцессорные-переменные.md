# Препроцессорные-переменные

Препроцессорные переменные — это переменные, определяемые в CSS-препроцессорах, таких как Sass/SCSS, Less и Stylus. Они позволяют хранить значения (цвета, размеры, шрифты и т.д.) и использовать их повторно в стилях, обеспечивая более высокий уровень абстракции и гибкости по сравнению с нативными CSS-переменными.

## Введение в препроцессорные переменные

CSS-препроцессоры расширяют возможности CSS, добавляя такие функции, как переменные, вложенные правила, миксины, функции и другие возможности, характерные для программных языков. Переменные в препроцессорах позволяют определять значения в одном месте и использовать их в других местах, что упрощает обслуживание и масштабирование CSS-кода.

## Типы препроцессоров и синтаксис переменных

### Sass/SCSS

В Sass/SCSS переменные определяются с помощью символа `$`:

```scss
$primary-color: #007bff;
$font-size-base: 16px;
$border-radius: 4px;

.button {
  background-color: $primary-color;
  font-size: $font-size-base;
  border-radius: $border-radius;
}
```

### Less

В Less переменные определяются с помощью символа `@`:

```less
@primary-color: #007bff;
@font-size-base: 16px;
@border-radius: 4px;

.button {
  background-color: @primary-color;
  font-size: @font-size-base;
  border-radius: @border-radius;
}
```

### Stylus

В Stylus переменные могут определяться с помощью символа `$` или без него:

```stylus
primary-color = #007bff
font-size-base = 16px
border-radius = 4px

.button
  background-color: primary-color
  font-size: font-size-base
  border-radius: border-radius
```

## Преимущества препроцессорных переменных

- **Повторное использование**: Определите значения один раз и используйте их везде.
- **Легкость поддержки**: Измените значение переменной в одном месте, и оно изменится везде, где используется.
- **Гибкость**: Возможность выполнять операции с переменными (например, lighten, darken, calc и т.д.).
- **Организация**: Группировка связанных переменных в одном месте (например, в отдельном файле `_variables.scss`).

## Практические примеры

### Пример 1: Цветовая палитра в SCSS

```scss
// Цвета бренда
$color-primary: #007bff;
$color-secondary: #6c757d;
$color-success: #28a745;
$color-danger: #dc3545;
$color-warning: #ffc107;
$color-info: #17a2b8;
$color-light: #f8f9fa;
$color-dark: #343a40;

// Цвета состояний
$color-hover: darken($color-primary, 10%);
$color-active: darken($color-primary, 20%);
```

### Пример 2: Размеры шрифтов и отступов в SCSS

```scss
// Размеры шрифтов
$font-size-xs: 0.75rem;
$font-size-sm: 0.875rem;
$font-size-base: 1rem;
$font-size-lg: 1.125rem;
$font-size-xl: 1.25rem;
$font-size-2xl: 1.5rem;
$font-size-3xl: 1.875rem;
$font-size-4xl: 2.25rem;
$font-size-5xl: 3rem;

// Отступы
$spacing-xs: 0.25rem;
$spacing-sm: 0.5rem;
$spacing-md: 1rem;
$spacing-lg: 1.5rem;
$spacing-xl: 2rem;
$spacing-2xl: 3rem;
$spacing-3xl: 4rem;
```

### Пример 3: Адаптивные размеры в SCSS

```scss
// Брейкпоинты
$breakpoint-xs: 0;
$breakpoint-sm: 576px;
$breakpoint-md: 768px;
$breakpoint-lg: 992px;
$breakpoint-xl: 1200px;
$breakpoint-xxl: 1400px;

// Адаптивные размеры шрифтов
$font-size-responsive: clamp(1rem, 2.5vw, 2rem);
```

## Особенности работы с переменными в препроцессорах

### Операции с переменными

В препроцессорах можно выполнять математические операции с переменными:

```scss
$base-font-size: 16px;
$multiplier: 1.2;
$calculated-font-size: $base-font-size * $multiplier; // 19.2px

.element {
  font-size: $calculated-font-size;
}
```

### Условия и логика

Некоторые препроцессоры (например, Sass) поддерживают условные операторы:

```scss
@mixin button-style($variant: 'primary') {
  padding: 10px 20px;
  border: none;
  border-radius: 4px;

  @if $variant == 'primary' {
    background-color: $color-primary;
    color: white;
  } @else if $variant == 'secondary' {
    background-color: $color-secondary;
    color: white;
  } @else {
    background-color: $color-light;
    color: $color-dark;
  }
}
```

### Функции

Можно создавать функции для обработки переменных:

```scss
@function calculate-rem($pixels, $context: 16px) {
  @return #{$pixels / $context}rem;
}

$font-size: calculate-rem(18px); // 1.125rem
```

## Структура файлов с переменными

Обычно переменные группируются в отдельных файлах (например, `_variables.scss`) и подключаются в начале основного файла стилей:

```
styles/
├── _variables.scss
├── _mixins.scss
├── _functions.scss
├── main.scss
```

```scss
// main.scss
@import 'variables';
@import 'mixins';
@import 'functions';

// Остальной CSS-код
```

## Совместимость с нативными CSS-переменными

Современные препроцессоры позволяют использовать нативные CSS-переменные в сочетании с препроцессорными переменными:

```scss
$primary-color: #007bff;

:root {
  --primary-color: #{$primary-color}; // Преобразование препроцессорной переменной в CSS-переменную
}

.button {
  background-color: var(--primary-color);
}
```

## Практические советы

- Используйте семантические имена переменных (например, `$color-primary` вместо `$blue`)
- Группируйте связанные переменные в отдельных файлах
- Документируйте переменные, особенно если они используются в разных частях проекта
- Используйте препроцессоры для сложных вычислений и логики, недоступных в нативном CSS
- Рассмотрите возможность совместного использования препроцессорных и нативных переменных для максимальной гибкости

## Заключение

Препроцессорные переменные — это мощный инструмент для улучшения структуры и поддержки CSS-кода. Они обеспечивают более высокий уровень абстракции и гибкости по сравнению с нативными CSS-переменными, особенно при выполнении сложных вычислений и логики. Использование препроцессоров особенно актуально в 2025 году, когда требования к сложности и функциональности веб-сайтов и приложений продолжают расти.
