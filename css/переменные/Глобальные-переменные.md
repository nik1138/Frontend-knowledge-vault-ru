# Глобальные-переменные

Глобальные переменные в CSS — это переменные, определенные на уровне всей системы стилей и доступные во всех частях проекта. Они служат основой для создания согласованной и масштабируемой архитектуры стилей, обеспечивая единообразие визуальных элементов и упрощая поддержку кода. В 2025 году глобальные переменные стали ключевым элементом современной CSS-архитектуры, особенно в крупных проектах и дизайн-системах.

## Введение в глобальные переменные

Глобальные переменные позволяют централизованно управлять значениями, используемыми по всему проекту. Это включает цвета бренда, размеры шрифтов, отступы, радиусы скруглений и другие часто используемые значения. Определение этих значений в одном месте упрощает внесение изменений и обеспечивает согласованность интерфейса.

## Архитектура глобальных переменных

### Определение в :root

Глобальные переменные обычно определяются в псевдоклассе `:root`, что делает их доступными на всех уровнях DOM:

```css
:root {
  /* Цвета бренда */
  --color-primary: #007bff;
  --color-secondary: #6c757d;
  --color-success: #28a745;
  --color-danger: #dc3545;
  --color-warning: #ffc107;
  --color-info: #17a2b8;
  --color-light: #f8f9fa;
  --color-dark: #343a40;
  
  /* Размеры шрифтов */
  --font-size-xs: 0.75rem;
  --font-size-sm: 0.875rem;
  --font-size-base: 1rem;
  --font-size-lg: 1.125rem;
  --font-size-xl: 1.25rem;
  --font-size-2xl: 1.5rem;
  --font-size-3xl: 1.875rem;
  --font-size-4xl: 2.25rem;
  --font-size-5xl: 3rem;
  
  /* Отступы */
  --spacing-xs: 0.25rem;
  --spacing-sm: 0.5rem;
  --spacing-md: 1rem;
  --spacing-lg: 1.5rem;
  --spacing-xl: 2rem;
  --spacing-2xl: 3rem;
  --spacing-3xl: 4rem;
  
  /* Радиусы скруглений */
  --radius-xs: 2px;
  --radius-sm: 4px;
  --radius-md: 8px;
  --radius-lg: 12px;
  --radius-xl: 16px;
  
  /* Тени */
  --shadow-sm: 0 1px 2px 0 rgba(0, 0, 0, 0.05);
  --shadow-md: 0 4px 6px -1px rgba(0, 0, 0, 0.1);
  --shadow-lg: 0 10px 15px -3px rgba(0, 0, 0, 0.1);
  --shadow-xl: 0 20px 25px -5px rgba(0, 0, 0, 0.1);
  
  /* Брейкпоинты */
  --breakpoint-xs: 0;
  --breakpoint-sm: 576px;
  --breakpoint-md: 768px;
  --breakpoint-lg: 992px;
  --breakpoint-xl: 1200px;
  --breakpoint-xxl: 1400px;
}
```

### Организация по категориям

Для лучшей структурированности глобальные переменные можно организовать по категориям:

```css
/* Цвета */
:root {
  --color-brand-primary: #007bff;
  --color-brand-secondary: #6f42c1;
  --color-status-success: #28a745;
  --color-status-warning: #ffc107;
  --color-status-danger: #dc3545;
  --color-neutral-light: #f8f9fa;
  --color-neutral-dark: #212529;
}

/* Типографика */
:root {
  --font-family-primary: 'Roboto', sans-serif;
  --font-family-secondary: 'Open Sans', sans-serif;
  --font-weight-light: 300;
  --font-weight-normal: 400;
  --font-weight-medium: 500;
  --font-weight-bold: 700;
  --line-height-tight: 1.25;
  --line-height-normal: 1.5;
  --line-height-relaxed: 1.75;
}

/* Размеры */
:root {
  --size-unit: 1rem;
  --size-container-sm: 540px;
  --size-container-md: 720px;
  --size-container-lg: 960px;
  --size-container-xl: 1140px;
  --size-container-xxl: 1320px;
}
```

```

├── globals/

│   ├── _typography.scss
│   ├── _spacing.scss
│   ├── _borders.scss
│   ├── _shadows.scss
│   └── _breakpoints.scss
├── components/
├── utilities/

```

## Использование глобальных переменных

### В компонентах

```css
.button {
  padding: var(--spacing-md) var(--spacing-lg);
  font-size: var(--font-size-base);
  background-color: var(--color-primary);
  border-radius: var(--radius-md);
  box-shadow: var(--shadow-md);
}

.card {
  padding: var(--spacing-lg);
  background-color: var(--color-gray-50);
  border-radius: var(--radius-lg);
  box-shadow: var(--shadow-sm);
}
```

### В адаптивных стилях

```css
:root {
  --layout-padding: var(--spacing-lg);
}

@media (min-width: 768px) {
  :root {
    --layout-padding: var(--spacing-xl);
  }
}

.layout {
  padding: var(--layout-padding);
}
```

## Преимущества глобальных переменных

### 1. Единообразие

Глобальные переменные обеспечивают единообразие визуальных элементов по всему проекту, так как все компоненты используют одни и те же значения.

### 2. Легкость поддержки

Изменение значения переменной в одном месте автоматически обновляет все элементы, использующие это значение, что значительно упрощает внесение изменений.

### 3. Масштабируемость

Глобальные переменные позволяют легко масштабировать проект, добавляя новые компоненты, которые автоматически соответствуют существующей системе.

### 4. Упрощение темизации

Глобальные переменные являются основой для реализации темизации, позволяя легко переключаться между различными визуальными темами.

## Практические рекомендации

### 1. Семантические имена

Используйте семантические имена переменных, которые отражают их назначение, а не конкретное значение:

```css
/* Хорошо */
--color-accent: #007bff;
--spacing-card-padding: 1rem;
--font-size-heading: 1.5rem;

/* Плохо */
--blue: #007bff;
--one-rem: 1rem;
--big-font: 1.5rem;
```

### 2. Иерархия именования

Создайте иерархическую систему именования для переменных:

```css
/* Структура: --category-variant-modifier */
--color-brand-primary
--color-brand-primary-dark
--color-status-success
--spacing-component-padding
--spacing-component-margin
--radius-card-small
--radius-card-large
```

### 3. Документирование

Документируйте глобальные переменные, особенно если они используются в разных частях проекта:

```css
:root {
  /* 
   * Основной цвет бренда
   * Используется для акцентных элементов, кнопок, ссылок
   */
  --color-brand-primary: #007bff;
  
  /* 
   * Базовый размер шрифта
   * Используется для основного текста на странице
   */
  --font-size-base: 1rem;
}
```

### 4. Ограничение количества

Не создавайте избыточное количество глобальных переменных. Только те значения, которые действительно используются в нескольких местах, должны быть глобальными.

### 5. Тестирование изменений

При изменении глобальных переменных тщательно тестируйте проект, чтобы убедиться, что изменения не повлияли негативно на другие компоненты.

## Совместимость с браузерами

CSS-переменные поддерживаются всеми современными браузерами (Chrome, Firefox, Safari, Edge). IE не поддерживает CSS-переменные, но для поддержки IE можно использовать полифиллы.

## Практические советы

- Используйте глобальные переменные для значений, которые повторяются в нескольких местах
- Создавайте иерархическую систему именования для переменных
- Документируйте переменные, особенно если они используются в разных частях проекта
- Регулярно пересматривайте и очищайте систему переменных от неиспользуемых значений
- Рассмотрите возможность использования инструментов для автоматической генерации переменных из дизайн-файлов
