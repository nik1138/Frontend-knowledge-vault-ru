# Организация файлов и папок в крупных проектах

Правильная организация файлов и папок в CSS-проектах критически важна для масштабируемости, поддерживаемости и эффективной работы команды разработчиков. Рассмотрим различные подходы к структурированию CSS-кода в крупных проектах.

## Общие принципы организации CSS-архитектуры

### 1. Модульность

Каждый файл должен содержать логически связанную функциональность:

```text
styles/
├── base/
│   ├── reset.css
│   ├── typography.css
│   └── variables.css
├── components/
│   ├── button.css
│   ├── card.css
│   ├── form.css
│   └── navigation.css
├── layout/
│   ├── header.css
│   ├── footer.css
│   ├── sidebar.css
│   └── grid.css
├── pages/
│   ├── home.css
│   ├── about.css
│   └── contact.css
├── themes/
│   ├── light.css
│   └── dark.css
└── utilities/
    ├── spacing.css
    ├── colors.css
    └── helpers.css
```

### 2. Последовательность подключения

Файлы должны подключаться в правильном порядке для корректного наследования и переопределения стилей:

```scss
// main.scss - основной файл стилей
@import 'base/reset';
@import 'base/variables';
@import 'base/typography';

@import 'layout/grid';
@import 'layout/header';
@import 'layout/footer';

@import 'components/button';
@import 'components/card';
@import 'components/form';

@import 'pages/home';
@import 'pages/about';

@import 'themes/light';
@import 'themes/dark';

@import 'utilities/helpers';
```

## Популярные методы организации CSS-файлов

### 1. Методология 7-1 (7 папок, 1 файл)

Этот подход, предложенный Hugo Giraudel, разделяет файлы по типу функциональности:

```text
scss/
├── abstracts/          # Переменные, миксины, функции
│   ├── _variables.scss
│   ├── _mixins.scss
│   └── _functions.scss
├── base/              # Сброс стилей, типографика
│   ├── _reset.scss
│   ├── _typography.scss
│   └── _base.scss
├── components/        # Компоненты UI
│   ├── _buttons.scss
│   ├── _carousel.scss
│   ├── _slider.scss
│   └── _media.scss
├── layout/            # Макетные компоненты
│   ├── _navigation.scss
│   ├── _grid.scss
│   ├── _header.scss
│   ├── _footer.scss
│   ├── _sidebar.scss
│   └── _forms.scss
├── pages/             # Специфические стили страниц
│   ├── _home.scss
│   ├── _contact.scss
│   └── _about.scss
├── themes/            # Различные темы
│   ├── _theme.scss
│   └── _admin.scss
├── vendors/           # Сторонние библиотеки
│   ├── _bootstrap.scss
│   ├── _jquery-ui.scss
│   └── _normalize.scss
└── main.scss          # Главный файл для импорта
```

### 2. Компонентно-ориентированная архитектура

Каждый компонент содержит свои стили, скрипты и шаблоны в одной папке:

```text
src/
├── components/
│   ├── button/

│   │   ├── Button.module.css
│   │   ├── Button.test.js
│   │   └── index.js
│   ├── card/
│   │   ├── Card.jsx
│   │   ├── Card.module.css
│   │   ├── Card.test.js
│   │   └── index.js
│   ├── navigation/
│   │   ├── Navigation.jsx
│   │   ├── Navigation.module.css
│   │   ├── Navigation.test.js
│   │   └── index.js
│   └── ui/
│       ├── Alert/
│       │   ├── Alert.jsx
│       │   ├── Alert.module.css
│       │   └── index.js
│       └── Modal/
│           ├── Modal.jsx
│           ├── Modal.module.css
│           └── index.js
├── pages/
│   ├── HomePage/
│   │   ├── HomePage.jsx
│   │   ├── HomePage.module.css
│   │   └── index.js
│   └── AboutPage/
│       ├── AboutPage.jsx
│       ├── AboutPage.module.css
│       └── index.js
└── styles/
    ├── globals.css
```

### 3. Архитектура ITCSS (Inverted Triangle CSS)

ITCSS организует CSS в перевернутый треугольник, где общие стили находятся вверху, а специфичные - внизу:

```text
src/
├── styles/
│   ├── settings/       # Переменные и настройки
│   │   ├── _colors.scss
│   │   ├── _breakpoints.scss
│   │   └── _z-index.scss
│   ├── tools/          # Миксины и функции
│   │   ├── _mixins.scss
│   │   └── _functions.scss
│   ├── generic/        # Сбросы и нормализация
│   │   ├── _box-sizing.scss
│   │   ├── _normalize.scss
│   │   └── _reset.scss
│   ├── elements/       # Базовые HTML элементы
│   │   ├── _page.scss
│   │   ├── _headings.scss
│   │   ├── _links.scss
│   │   └── _lists.scss
│   ├── objects/        # ООП CSS (OOCSS)
│   │   ├── _layout.scss
│   │   ├── _flag.scss
│   │   └── _media.scss
│   ├── components/     # Компоненты UI
│   │   ├── _buttons.scss
│   │   ├── _carousel.scss
│   │   └── _header.scss
│   ├── utilities/      # Вспомогательные классы
│   │   ├── _alignments.scss
│   │   ├── _spacings.scss
│   │   └── _helpers.scss
│   └── main.scss       # Главный файл
```

## Инструменты для управления структурой CSS

### 1. Автоматизация с Webpack

```javascript
// webpack.config.js
module.exports = {
  entry: './src/styles/main.scss',
  module: {
    rules: [
      {
        test: /\.scss$/,
        use: [
          'style-loader', // или MiniCssExtractPlugin.loader для продакшена
          'css-loader',
          'sass-loader'
        ]
      }
    ]
  },
  plugins: [
    new MiniCssExtractPlugin({
      filename: '[name].[contenthash].css'
    })
  ]
};
```

### 2. PostCSS и автопрефиксер

```javascript
// postcss.config.js
module.exports = {
  plugins: [
    require('autoprefixer'),
    require('postcss-nested'),
    require('postcss-import')
  ]
};
```

## Лучшие практики организации файлов

### 1. Использование файлов с подчеркиванием для импортируемых

```scss
// _variables.scss - файл будет импортироваться, но не компилироваться отдельно
$primary-color: #007bff;
$secondary-color: #6c757d;
$success-color: #28a745;

// main.scss - файл будет компилироваться
@import 'variables';
@import 'components/button';
```

### 2. Группировка связанных компонентов

```text
components/
├── form/
│   ├── _input.scss
│   ├── _textarea.scss
│   ├── _select.scss
│   └── _form-group.scss
├── navigation/
│   ├── _navbar.scss
│   ├── _sidebar.scss
│   └── _breadcrumb.scss
└── media/
    ├── _image.scss
    ├── _video.scss
    └── _gallery.scss
```

### 3. Использование namespace для предотвращения конфликтов

```scss
// _button.scss
.c-btn {  // "c" как префикс для компонента
  display: inline-block;
  padding: 0.5rem 1rem;
  border: 1px solid transparent;
  border-radius: 0.25rem;
  
  &--primary {
    background-color: $primary-color;
    color: white;
  }
  
  &--large {
    padding: 0.75rem 1.5rem;
    font-size: 1.25rem;
  }
}
```

## Масштабирование архитектуры

### 1. Для маленьких проектов

```text
styles/
├── main.css
└── components/
    └── button.css
```

### 2. Для средних проектов

```text
styles/
├── base/
├── components/
├── layout/
├── pages/
└── utils/
```

### 3. Для крупных проектов

```text
styles/
├── config/           # Конфигурация и переменные
├── foundation/       # Базовые стили
├── layout/           # Макетные компоненты
├── components/       # Повторно используемые компоненты
├── sections/         # Секции страниц
├── pages/            # Специфичные для страниц стили
├── themes/           # Темы оформления
├── vendor/           # Сторонние библиотеки
├── utils/            # Утилиты и миксины
└── main.scss         # Главный файл
```

> [!tip] Совет
> Используйте понятную и последовательную систему именования папок и файлов, чтобы любой член команды мог легко ориентироваться в структуре проекта.

> [!warning] Важно
> Избегайте глубокой вложенности папок (более 3-4 уровней), так как это затрудняет навигацию и увеличивает время поиска нужных файлов.
