# Стили для пользователей с нарушениями зрения

## Введение

Нарушения зрения включают в себя широкий спектр состояний - от частичной потери зрения до полной слепоты. Создание доступных веб-сайтов требует понимания этих различий и применения соответствующих CSS-техник. В этой статье рассмотрим ключевые аспекты создания стилей, которые улучшают доступность для пользователей с нарушениями зрения.

## Типы нарушений зрения

### 1. Снижение остроты зрения

Пользователи с частичной потерей зрения могут нуждаться в увеличенном размере текста, улучшенной контрастности и более крупных элементах управления.

### 2. Туннельное зрение

Пользователи с ограниченным полем зрения нуждаются в чистом, структурированном дизайне без лишних элементов.

### 3. Контузия и светочувствительность

Некоторые пользователи чувствительны к яркому свету и контрасту, что требует более мягких цветовых схем.

### 4. Цветовая слепота

Пользователи с различными формами цветовой слепоты не могут различать определенные цвета, что требует дополнительных визуальных индикаторов.

## Практические рекомендации

### 1. Гибкий размер текста

```css
/* Использование относительных единиц для размера шрифта */
html {
  font-size: 100%; /* Устанавливает базовый размер как 16px */
}

body {
  font-size: 1rem; /* 16px по умолчанию */
  line-height: 1.5; /* Обеспечивает хороший интерлиньяж */
}

/* Иерархия заголовков с относительными размерами */
h1 { font-size: 2.5rem; }
h2 { font-size: 2rem; }
h3 { font-size: 1.75rem; }
h4 { font-size: 1.5rem; }
h5 { font-size: 1.25rem; }
h6 { font-size: 1rem; }

/* Адаптация к пользовательским настройкам */
@media (min-resolution: 150dpi) {
  /* Для устройств с высоким разрешением */
  body {
    font-size: 1.125rem;
  }
}
```

### 2. Минимальные размеры интерактивных элементов

```css
/* Обеспечение минимального размера для элементов управления */
.button,
.link-button,
.input,
.select,
.checkbox,
.radio {
  min-height: 44px; /* Рекомендуемый размер касания по WCAG */
  min-width: 44px;
  padding: 12px 16px;
}

/* Для мобильных устройств */
@media (max-width: 768px) {
  .interactive-element {
    min-height: 48px;
    min-width: 48px;
  }
}

/* Обеспечение пространства между элементами управления */
.form-field {
  margin-bottom: 20px;
}

.control-group {
  display: flex;
  flex-direction: column;
  gap: 15px;
}
```

### 3. Улучшенная визуальная структура

```css
/* Четкая иерархия и структура */
.page-content {
  max-width: 80ch; /* Ограничивает ширину строки для лучшей читаемости */
  margin: 0 auto;
  padding: 20px;
}

/* Визуальные разделители */
.section {
  border-bottom: 1px solid #ccc;
  padding-bottom: 20px;
  margin-bottom: 30px;
}

/* Адаптация к увеличению масштаба */
@media (min-width: 1200px) {
  .container {
    max-width: 1000px; /* Предотвращает чрезмерное растягивание текста */
  }
}

/* Улучшенная читаемость */
.readable-text {
  font-family: system-ui, -apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto, sans-serif;
  font-size: 1.125rem;
  line-height: 1.6;
  text-align: left; /* Избегаем justify для лучшей читаемости */
  hyphens: auto; /* Автоматические переносы */
  overflow-wrap: break-word; /* Перенос длинных слов */
}
```

## Специальные стили для вспомогательных технологий

### 1. Скрытие элементов для зрения, но доступных для скринридеров

```css
/* Класс для скрытия элемента визуально, но доступного для скринридеров */
.visually-hidden {
  position: absolute;
  width: 1px;
  height: 1px;
  padding: 0;
  margin: -1px;
  overflow: hidden;
  clip: rect(0, 0, 0, 0);
  white-space: nowrap;
  border: 0;
}

/* Использование при фокусе */
.skip-link {
  position: absolute;
  top: -40px;
  left: 6px;
  color: white;
  background: #000;
  padding: 8px;
  text-decoration: none;
  transition: top 0.3s ease;
}

.skip-link:focus {
  top: 6px;
}
```

### 2. Улучшенные фокусные индикаторы

```css
/* Четкий фокусный индикатор */
.focusable {
  outline: 2px solid #4a90e2;
  outline-offset: 2px;
}

/* Альтернативный стиль фокуса */
.focusable-alt {
  box-shadow: 0 0 0 3px rgba(74, 144, 226, 0.5);
}

/* Уважение к пользовательским предпочтениям */
@media (prefers-reduced-motion: no-preference) {
  .focusable {
    transition: outline-offset 0.2s ease;
  }
}

/* Специфические стили для разных элементов */
button:focus,
input:focus,
select:focus,
textarea:focus,
a:focus {
  outline: 2px solid #4a90e2;
  outline-offset: 2px;
}

/* Исключение для элементов с собственными стилями фокуса */
.custom-focus {
  border: 2px solid #4a90e2;
  box-shadow: 0 0 0 2px rgba(74, 144, 226, 0.3);
  outline: none;
}
```

## Адаптация к пользовательским настройкам

### 1. Учет настроек пользователя

```css
/* Уважение к пользовательским настройкам */
@media (prefers-contrast: high) {
  body {
    background-color: #ffffff;
    color: #000000;
  }
  
  .card {
    border: 2px solid #000000;
  }
  
  .button {
    border: 2px solid #000000;
    background-color: #ffffff;
    color: #000000;
  }
}

/* Темная тема */
@media (prefers-color-scheme: dark) {
  body {
    background-color: #121212;
    color: #e0e0e0;
  }
  
  .card {
    background-color: #1e1e1e;
    border: 1px solid #444444;
  }
}

/* Уменьшение анимации */
@media (prefers-reduced-motion: reduce) {
  * {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
  }
}
```

### 2. Адаптация шрифтов

```css
/* Использование системных шрифтов для лучшей доступности */
.system-font {
  font-family: system-ui, -apple-system, BlinkMacSystemFont, 
               "Segoe UI", Roboto, "Helvetica Neue", Arial, sans-serif;
}

/* Альтернативные шрифты для улучшенной читаемости */
.readable-font {
  font-family: "Open Sans", "Roboto", "Segoe UI", sans-serif;
  font-weight: 400;
  /* Убедитесь, что шрифты загружаются корректно */
}

/* Учет пользовательских настроек шрифта */
@media (script: cursive) {
  /* Адаптация для различных систем письма */
  .content {
    font-family: ui-serif, serif;
  }
}
```

## Специфические техники для разных типов нарушений

### 1. Для пользователей с туннельным зрением

```css
/* Минимизация визуального шума */
.focus-mode {
  background-color: #ffffff;
  color: #000000;
  max-width: 65ch; /* Ограничивает ширину текста */
  margin: 0 auto;
  padding: 20px;
  border-left: 4px solid #4a90e2; /* Дополнительный визуальный индикатор */
}

/* Четкая структура без отвлекающих элементов */
.focus-content {
  display: flex;
  flex-direction: column;
  align-items: flex-start;
}

.focus-content h1,
.focus-content h2,
.focus-content h3 {
  margin-top: 1.5em;
  margin-bottom: 0.5em;
}

.focus-content p {
  margin-bottom: 1em;
}
```

### 2. Для пользователей с светочувствительностью

```css
/* Мягкие, неяркие цвета */
.low-stimulation {
  background-color: #f8f9fa; /* Мягкий серый фон */
  color: #212529; /* Темный, но не черный текст */
  /* Избегаем ярких цветов и высокого контраста */
}

/* Уменьшение анимации и мерцания */
.stable-content {
  contain: layout style paint; /* Ограничивает визуальные изменения */
}

/* Плавные переходы вместо резких изменений */
.smooth-transition {
  transition: background-color 0.3s ease, color 0.3s ease;
}
```

### 3. Для пользователей с астигматизмом

```css
/* Избегаем тонких линий и мелких деталей */
.astigmatism-friendly {
  border-width: 2px; /* Толстые линии вместо тонких */
  font-weight: 500; /* Избегаем тонких шрифтов */
  text-shadow: 0 0 1px rgba(0, 0, 0, 0.1); /* Мягкая тень для улучшения четкости */
}

/* Увеличенные интерактивные области */
.enlarged-hit-area {
  position: relative;
}

.enlarged-hit-area::before {
  content: '';
  position: absolute;
  top: -8px;
  left: -8px;
  right: -8px;
  bottom: -8px;
  /* Увеличивает область для нажатия */
}
```

## Тестирование и проверка

### 1. Инструменты для тестирования

```css
/* CSS для тестирования доступности */
.accessibility-test {
  /* Используйте CSS-переменные для легкой настройки */
  --test-font-size: 1rem;
  --test-line-height: 1.5;
  --test-max-width: 65ch;
  
  font-size: var(--test-font-size);
  line-height: var(--test-line-height);
  max-width: var(--test-max-width);
  margin: 0 auto;
  padding: 20px;
}
```

### 2. Проверка масштабирования

```css
/* Убедитесь, что контент корректно масштабируется */
.scalable-content {
  /* Используйте относительные единицы */
  font-size: 1rem;
  padding: 1rem;
  margin: 0.5rem;
  width: 90%; /* Вместо фиксированной ширины */
  max-width: 100%;
}

/* Избегайте фиксированных позиций, которые могут скрывать контент при масштабировании */
.avoid-fixed {
  /* position: fixed; - избегайте при масштабировании */
  position: static;
}
```

## Заключение

Создание стилей для пользователей с нарушениями зрения требует комплексного подхода, включающего:

1. Использование адекватных размеров текста и элементов управления
2. Обеспечение хорошей контрастности
3. Уважение к пользовательским настройкам
4. Четкие фокусные индикаторы
5. Адаптацию к различным типам нарушений зрения
6. Регулярное тестирование с вспомогательными технологиями

Следование этим принципам делает веб-сайты доступнее для всех пользователей, независимо от их зрительных возможностей.
