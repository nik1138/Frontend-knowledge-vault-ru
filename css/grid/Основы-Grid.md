# Основы CSS Grid

CSS Grid Layout (или просто Grid) - это мощная система макета, которая позволяет создавать двумерные сетки на веб-страницах. В отличие от Flexbox, который работает в одном направлении, Grid позволяет управлять как строками, так и столбцами одновременно, что делает его идеальным для создания сложных макетов.

## Что такое CSS Grid?

CSS Grid - это двумерная система макета, которая позволяет разработчикам создавать сетки с фиксированными и гибкими размерами. Он состоит из родительского элемента (grid container) и его дочерних элементов (grid items). Grid позволяет точно размещать элементы в строках и столбцах, что дает больше контроля над визуальной компоновкой.

## Основные концепции

### Grid Container

Чтобы создать сетку, нужно определить родительский элемент как grid container с помощью свойства `display: grid` или `display: inline-grid`:

```css
.grid-container {
  display: grid;
}
```

### Grid Items

Дочерние элементы grid container автоматически становятся grid items. Их можно позиционировать с помощью различных свойств Grid.

### Grid Lines

Сетка состоит из горизонтальных и вертикальных линий, которые определяют структуру. Линии нумеруются от 1, начиная с левого верхнего угла.

### Grid Tracks

Grid tracks - это пространства между линиями сетки. Горизонтальные треки - это строки, а вертикальные - столбцы.

### Grid Cells

Grid cell - это минимальная единица сетки, ограниченная четырьмя линиями сетки, как одна ячейка в таблице.

### Grid Areas

Grid area - это прямоугольная область, ограниченная четырьмя линиями сетки, которая может содержать несколько ячеек.

## Основные свойства Grid

### Свойства для Grid Container

#### `grid-template-columns` и `grid-template-rows`

Определяют размеры столбцов и строк сетки:

```css
.grid-container {
  display: grid;
  grid-template-columns: 100px 200px 150px;
  grid-template-rows: 80px 120px;
}
```

#### `grid-template-areas`

Позволяет определить именованные области сетки:

```css
.grid-container {
  display: grid;
  grid-template-areas: 
    "header header header"
    "sidebar main main"
    "footer footer footer";
}
```

#### `grid-gap` (устаревшее), `gap`

Определяет промежутки между ячейками:

```css
.grid-container {
  gap: 10px; /* или grid-gap: 10px; */
}
```

### Свойства для Grid Items

#### `grid-column` и `grid-row`

Позволяют элементу занимать несколько линий сетки:

```css
.grid-item {
  grid-column: 1 / 3; /* Занимает с 1 по 3 линию по горизонтали */
  grid-row: 1 / 2; /* Занимает с 1 по 2 линию по вертикали */
}
```

#### `grid-area`

Можно использовать для определения именованной области или как сокращение для `grid-row`, `grid-column`:

```css
.grid-item {
  grid-area: sidebar; /* Использует именованную область */
  /* или */
  grid-area: 1 / 1 / 3 / 2; /* row-start / column-start / row-end / column-end */
}
```

## Практические рекомендации

### Использование фракционных единиц (fr)

Фракционная единица `fr` позволяет распределять доступное пространство между треками:

```css
.grid-container {
  grid-template-columns: 1fr 2fr 1fr; /* Столбцы будут занимать 1/4, 1/2 и 1/4 ширины соответственно */
}
```

### Комбинация единиц измерения

Можно комбинировать различные единицы измерения:

```css
.grid-container {
  grid-template-columns: 200px 1fr 1fr 200px; /* Фиксированные и гибкие столбцы */
}
```

### Responsive Design

Grid отлично подходит для адаптивного дизайна. Используйте `minmax()` и `auto-fit`/`auto-fill`:

```css
.grid-container {
  grid-template-columns: repeat(auto-fit, minmax(250px, 1fr));
}
```

## Совместимость с браузерами

CSS Grid имеет хорошую поддержку в современных браузерах. В России, согласно статистике 2025 года, основные браузеры (Chrome, Firefox, Safari, Edge) поддерживают Grid Layout без необходимости использования префиксов.

## Заключение

CSS Grid - это мощный инструмент для создания сложных макетов. Понимание основных концепций и свойств позволяет эффективно использовать Grid для построения современных веб-сайтов. В сочетании с [[Flexbox]] и [[Media Queries]] Grid становится незаменимым инструментом для создания адаптивных и гибких интерфейсов.
