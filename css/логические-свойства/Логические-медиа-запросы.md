# Логические медиа-запросы в CSS

## Обзор

Логические медиа-запросы в CSS позволяют создавать адаптивные стили, которые учитывают направление письма и локализацию. В отличие от традиционных медиа-запросов, которые ориентируются на физические размеры экрана, логические медиа-запросы могут учитывать направление потока содержимого.

## Современные возможности

### CSS Logical Properties и Media Queries

С выходом новых спецификаций CSS в 2025 году появилась возможность использовать логические медиа-запросы для более точной настройки стилей в зависимости от направления письма и локализации интерфейса.

### Примеры использования

```css
/* Пример использования логических медиа-запросов */
@media (orientation: landscape) {
  .container {
    padding-inline-start: 20px;
    padding-inline-end: 10px;
  }
}

/* Медиа-запрос для RTL языков */
@media (direction: rtl) {
  .navigation {
    margin-inline-start: auto;
    margin-inline-end: 0;
  }
}
```

## Практические применения

### Адаптация интерфейса

Логические медиа-запросы позволяют создавать интерфейсы, которые автоматически адаптируются к различным направлениям письма без необходимости вручную изменять стили для каждого языка.

### Многоязычные сайты

Для российских разработчиков, создающих многоязычные сайты, логические медиа-запросы обеспечивают более чистый и поддерживаемый код при работе с различными системами письма.

## Совместимость

### Поддержка браузерами

На 2025 год большинство современных браузеров поддерживают логические медиа-запросы, что делает их пригодными для использования в продакшен-проектах, включая корпоративные решения в России.

### Резервные стратегии

Для обеспечения совместимости с устаревшими браузерами рекомендуется использовать прогрессивное улучшение и предоставление резервных стилей.
