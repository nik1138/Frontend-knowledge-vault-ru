# Формы

## Обзор

Формы — ключевой элемент взаимодействия между пользователем и веб-приложением. В 2025 году стилизация и функциональность форм играют решающую роль в конверсии и удовлетворенности пользователей. Особое внимание уделяется доступности, валидации и адаптивности.

## Основные элементы формы

- `input` — поля ввода (text, email, password и др.).
- `textarea` — многострочное поле ввода.
- `select` — выпадающий список.
- `button` — кнопки отправки или сброса.
- `label` — метки для полей ввода.
- `fieldset` и `legend` — группировка связанных элементов.

## Свойства CSS для стилизации

### 1. Визуальное оформление
- `border`, `border-radius` — границы и скругления.
- `background-color`, `color` — цвет фона и текста.
- `box-shadow` — тени для выделения активных полей.
- `padding`, `margin` — отступы внутри и снаружи элементов.

### 2. Размеры
- `width`, `height` — размеры полей.
- `min-width`, `max-width` — ограничения ширины.
- `font-size`, `line-height` — размер шрифта и высота строки.

### 3. Поведение
- `transition` — анимации при фокусе.
- `outline: none` — удаление стандартной рамки (заменить на `box-shadow`!).
- `appearance: none` — сброс стилей для `select`, `input[type="range"]` и др.

## Практические рекомендации

### 1. Структура и семантика
Используйте семантические теги:
- `form` — контейнер формы.
- `fieldset` — группа полей.
- `legend` — заголовок группы.
- `label` — связь с полем ввода через `for` и `id`.

### 2. Состояния полей
- `:default` — обычное состояние.
- `:focus` — при фокусе.
- `:hover` — при наведении.
- `:invalid`, `:valid` — состояния валидации.

Пример:
```css
.form-control {
  width: 100%;
  padding: 10px;
  border: 1px solid #ccc;
  border-radius: 4px;
  font-size: 16px;
  transition: border-color 0.3s, box-shadow 0.3s;
}

.form-control:focus {
  outline: none;
  border-color: #007bff;
  box-shadow: 0 0 0 3px rgba(0, 123, 255, 0.25);
}

.form-control:invalid {
  border-color: #dc3545;
}

.form-control:valid {
  border-color: #28a745;
}
```

### 3. Адаптивность
- Используйте `flexbox` или `grid` для выравнивания.
- `max-width` и `width` в процентах для адаптивности.
- `media queries` для изменения макета на мобильных устройствах.

### 4. Доступность
- Обязательно используйте `label` для каждого поля.
- `aria-describedby` для вспомогательного текста.
- `required`, `pattern`, `min`, `max` для валидации.
- Контрастность текста и фона по WCAG 2.1 AA.

## Примеры

### Адаптивная форма с валидацией
```css
.form-group {
  margin-bottom: 15px;
}

.form-label {
  display: block;
  margin-bottom: 5px;
  font-weight: bold;
}

.form-control {
  width: 100%;
  padding: 10px;
  border: 1px solid #ccc;
  border-radius: 4px;
  font-size: 16px;
  box-sizing: border-box;
}

.form-control:focus {
  outline: none;
  border-color: #007bff;
  box-shadow: 0 0 0 3px rgba(0, 123, 255, 0.25);
}

.form-error {
  color: #dc3545;
  font-size: 0.875em;
  margin-top: 5px;
  display: none;
}

.form-control:invalid ~ .form-error {
  display: block;
}

@media (min-width: 768px) {
  .form-row {
    display: flex;
    gap: 15px;
  }

  .form-group {
    flex: 1;
    margin-bottom: 0;
  }
}
```

## Заключение

Современные формы должны быть не только визуально привлекательными, но и интуитивно понятными, доступными и удобными для ввода данных. В 2025 году пользователи всё больше ценят качество взаимодействия, поэтому стилизация форм — это важный этап в создании успешного веб-приложения.
