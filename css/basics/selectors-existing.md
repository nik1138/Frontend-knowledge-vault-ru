# CSS-селекторы

CSS-селекторы — это шаблоны, которые определяют, к каким HTML-элементам применять CSS-правила. Они играют ключевую роль в каскадных стилях, позволяя точно указать, какие элементы страницы должны быть отформатированы определенным образом. Понимание селекторов необходимо для эффективной работы с CSS и создания адаптивных, поддерживаемых веб-страниц.

Селекторы являются основой для [[cascade]] и [[specificity|специфичности]] CSS, влияя на то, какие стили будут применены к элементам. Они также тесно связаны с концепцией [[inheritance]] стилей в HTML-документах.

## Типы CSS-селекторов

### 1. Селекторы тегов (элементов)

Селектор тега применяет стили ко всем элементам указанного типа в документе.

```css
p {
  color: blue;
}
```

Этот селектор применяет синий цвет текста ко всем `<p>` элементам.

### 2. Селекторы классов

Селектор класса применяет стили ко всем элементам, имеющим определенный класс. Начинается с точки (`.`).

```css
.highlight {
  background-color: yellow;
}
```

Этот селектор применяет желтый фон к элементам с классом `highlight`.

### 3. Селекторы ID

Селектор ID применяет стили к элементу с определенным идентификатором. Начинается с решетки (`#`).

```css
#header {
  font-size: 24px;
}
```

Этот селектор применяет размер шрифта 24px к элементу с ID `header`.

> [!warning] Важно
> ID должен быть уникальным в пределах документа. Используйте с осторожностью.

### 4. Селекторы атрибутов

Селектор атрибута применяет стили к элементам, у которых есть определенный атрибут или атрибут с определенным значением.

```css
input[type="text"] {
  border: 1px solid #ccc;
}

a[href^="https"] {
  color: green;
}
```

Первый селектор применяет рамку к текстовым полям, второй — зеленый цвет к ссылкам, начинающимся с `https`.

### 5. Псевдоклассы

Псевдоклассы позволяют применять стили к элементам в определенных состояниях.

```css
a:hover {
  color: red;
}

li:first-child {
  font-weight: bold;
}
```

Первый селектор меняет цвет ссылки при наведении, второй — делает жирным первый элемент списка.

### 6. Псевдоэлементы

Псевдоэлементы позволяют стилизовать определенные части элемента.

```css
p::first-line {
  font-weight: bold;
}

h1::before {
  content: "★ ";
}
```

Первый селектор делает первую строку параграфа жирной, второй добавляет звездочку перед заголовком.

## Комбинированные селекторы

### Селекторы потомков

Применяют стили к элементам, находящимся внутри другого элемента (на любом уровне вложенности).

```css
div p {
  margin: 10px;
}
```

Этот селектор применяет отступ к параграфам, находящимся внутри div.

### Дочерние селекторы

Применяют стили только к непосредственным дочерним элементам.

```css
ul > li {
  list-style: none;
}
```

Этот селектор убирает маркеры только у непосредственных дочерних элементов списка.

### Селекторы соседей

#### Соседи одного уровня (adjacent sibling)

```css
h1 + p {
  margin-top: 0;
}
```

Применяется к первому параграфу, следующему сразу за заголовком.

#### Общие соседи (general sibling)

```css
h1 ~ p {
  color: gray;
}
```

Применяется ко всем параграфам, следующим за заголовком на том же уровне.

## Современные селекторы

CSS3 и более поздние версии добавили множество новых селекторов:

```css
/* Структурные псевдоклассы */
tr:nth-child(odd) { background: #f2f2f2; }
p:nth-of-type(2) { font-style: italic; }

/* Атрибуты с частичным совпадением */
input[placeholder*="email"] { border-color: blue; }
a[href$=".pdf"] { color: red; }

/* Псевдоклассы состояния */
input:focus { outline: 2px solid blue; }
input:valid { border-color: green; }
```

## Лучшие практики

1. **Используйте понятные имена классов** - следуйте соглашениям (например, BEM)
2. **Избегайте избыточной специфичности** - не используйте ID, если можно обойтись классом
3. **Организуйте селекторы логически** - группируйте связанные стили
4. **Предпочитайте классы тегам** - для лучшей переиспользуемости
5. **Тестируйте селекторы на производительность** - избегайте чрезмерно сложных селекторов

## Сравнение подходов

| Подход | Преимущества | Недостатки |
|--------|--------------|------------|
| Селекторы тегов | Просты, быстрые | Низкая специфичность, ограниченная гибкость |
| Классы | Гибкие, переиспользуемые | Требуют разметки |
| ID | Высокая специфичность | Не переиспользуемы, сложно поддерживать |

## Инструменты для анализа селекторов

- DevTools в браузерах (Chrome, Firefox) - для отладки и анализа
- CSS Lint - проверяет качество CSS-кода
- SelectorGadget - визуальный инструмент для создания селекторов
- Specificity Calculator - для анализа специфичности селекторов

## Заключение

CSS-селекторы являются мощным инструментом для точного определения, к каким элементам применять стили. Понимание различных типов селекторов и их правильное использование позволяет создавать чистый, эффективный и поддерживаемый CSS-код. Важно следить за современными тенденциями в CSS и использовать новые возможности для улучшения разработки.
