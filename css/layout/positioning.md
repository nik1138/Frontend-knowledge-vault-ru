# Верстка - Позиционирование

Позиционирование в CSS позволяет точно контролировать расположение элементов на веб-странице. Это важный аспект верстки, который позволяет создавать сложные макеты и управлять потоком документа.

## Типы позиционирования

### static (по умолчанию)

Статическое позиционирование - это значение по умолчанию для всех элементов. Элементы следуют обычному потоку документа:

```css
.static-element {
  position: static;  /* Значение по умолчанию, можно не указывать */
  /* Свойства top, right, bottom, left игнорируются */
}
```

### relative (относительное)

Относительное позиционирование смещает элемент относительно его обычного положения в потоке документа:

```css
.relative-element {
  position: relative;
  top: 10px;        /* Смещение вниз от обычного положения */
  left: 20px;       /* Смещение вправо от обычного положения */
  /* Элемент остается в потоке документа */
  /* Другие элементы не изменяют свое положение */
}
```

### absolute (абсолютное)

Абсолютное позиционирование извлекает элемент из обычного потока документа:

```css
.absolute-element {
  position: absolute;
  top: 20px;        /* Расстояние от верхнего края содержащего блока */
  left: 30px;       /* Расстояние от левого края содержащего блока */
  /* Элемент извлекается из потока документа */
  /* Остальные элементы ведут себя так, как будто этого элемента нет */
}
```

### fixed (фиксированное)

Фиксированное позиционирование также извлекает элемент из потока документа, но позиционирует относительно области просмотра:

```css
.fixed-element {
  position: fixed;
  top: 0;           /* Фиксировано к верху области просмотра */
  right: 0;         /* Фиксировано к правому краю области просмотра */
  /* Элемент остается на месте при прокрутке страницы */
}
```

### sticky (липкое)

Липкое позиционирование - это гибрид относительного и фиксированного позиционирования:

```css
.sticky-element {
  position: sticky;
  top: 10px;        /* При прокрутке элемент "прилипает" к 10px от верха */
  /* Поведение зависит от прокрутки родительского контейнера */
}
```

## Свойства позиционирования

### top, right, bottom, left

Эти свойства определяют смещение позиционированного элемента:

```css
.positioned-element {
  position: absolute;
  top: 10px;        /* Расстояние от верхнего края содержащего блока */
  right: 15px;      /* Расстояние от правого края содержащего блока */
  bottom: 20px;     /* Расстояние от нижнего края содержащего блока */
  left: 25px;       /* Расстояние от левого края содержащего блока */
}
```

### z-index

Свойство z-index управляет стековым порядком элементов по оси Z:

```css
.element1 {
  position: relative;
  z-index: 1;       /* Ближе к пользователю */
}

.element2 {
  position: relative;
  z-index: -1;      /* Дальше от пользователя */
}

.element3 {
  position: relative;
  z-index: 0;       /* По умолчанию */
}
```

## Практические примеры

### Пример 1: Центрирование элемента

```css
.center-container {
  position: relative;
  height: 100vh;
}

.centered-element {
  position: absolute;
  top: 50%;
  left: 50%;
  transform: translate(-50%, -50%);
  /* Абсолютно позиционированный элемент в центре */
}
```

### Пример 2: Плавающее меню

```css
.floating-menu {
  position: fixed;
  top: 20px;
  right: 20px;
  background-color: white;
  box-shadow: 0 4px 6px rgba(0, 0, 0, 0.1);
  padding: 10px;
  border-radius: 4px;
  z-index: 1000;
  /* Меню остается на месте при прокрутке */
}
```

### Пример 3: Абсолютное позиционирование в контейнере

```css
.positioned-container {
  position: relative;
  width: 400px;
  height: 300px;
  border: 1px solid #ccc;
}

.overlay {
  position: absolute;
  top: 0;
  left: 0;
  width: 100%;
  height: 100%;
  background-color: rgba(0, 0, 0, 0.5);
  /* Перекрывает весь контейнер */
}

.corner-badge {
  position: absolute;
  top: 10px;
  right: 10px;
  background-color: red;
  color: white;
  padding: 5px 10px;
  border-radius: 12px;
  /* Угловая метка в контейнере */
}
```

### Пример 4: Липкая навигация

```css
.sticky-nav {
  position: sticky;
  top: 0;
  background-color: #333;
  color: white;
  padding: 10px;
  z-index: 100;
  /* Навигация "прилипает" к верху при прокрутке */
}

.nav-content {
  height: 1000px;  /* Добавляем высоту для демонстрации прокрутки */
}
```

## Содержащий блок

Содержащий блок (containing block) - это элемент, относительно которого позиционируется потомок:

- Для `position: relative` и `position: static` - содержащий блок определяется обычным потоком
- Для `position: absolute` - содержащий блок определяется ближайшим позиционированным предком (с `position: relative`, `absolute`, `fixed` или `sticky`)
- Для `position: fixed` - содержащий блок - область просмотра
- Для `position: sticky` - содержащий блок - ближайший прокручиваемый предок

```css
.parent {
  position: relative;  /* Создает содержащий блок для потомков */
  width: 500px;
  height: 400px;
}

.child {
  position: absolute;
  top: 20px;
  left: 30px;
  /* Позиционируется относительно .parent */
}
```

## Стековый контекст

Стековый контекст (stacking context) - это концепция, определяющая, как элементы стекаются друг над другом:

```css
.stack-context-1 {
  position: relative;
  z-index: 1;
}

.stack-context-2 {
  position: relative;
  z-index: 2;
  /* Элементы в этом контексте всегда будут над элементами в stack-context-1 */
}

.element-in-context-1 {
  position: absolute;
  z-index: 999;  /* Все равно будет под элементами из stack-context-2 */
}
```

## Распространенные ошибки

### 1. Забытый position:relative для содержащего блока

```css
/* Плохо: absolute элемент позиционируется относительно body */
.container {
  width: 500px;
  height: 400px;
}

.absolute-element {
  position: absolute;
  top: 10px;
  left: 10px;
}

/* Хорошо: добавляем position: relative */
.container {
  position: relative;
  width: 500px;
  height: 400px;
}

.absolute-element {
  position: absolute;
  top: 10px;
  left: 10px;
}
```

### 2. Неправильное использование z-index

```css
/* Проблема: z-index работает только для позиционированных элементов */
.element {
  z-index: 10;  /* Не будет работать без position */
}

/* Решение */
.element {
  position: relative;  /* или absolute, fixed, sticky */
  z-index: 10;
}
```

## Лучшие практики

1. **Используйте position: relative** для создания содержащих блоков
2. **Избегайте чрезмерного использования z-index** - это может усложнить отладку
3. **Тестируйте на разных размерах экрана** - убедитесь, что позиционирование работает корректно
4. **Используйте position: sticky** для элементов, которые должны "прилипать" при прокрутке
5. **Избегайте position: absolute** для основной верстки - лучше использовать flexbox или grid

## Совместимость и поддержка

Все типы позиционирования имеют отличную поддержку в современных браузерах. Значение `position: sticky` поддерживается во всех современных браузерах, но может требовать префиксов для старых версий.

## Заключение

Позиционирование - важный инструмент в арсенале CSS-разработчика. Правильное понимание и использование различных типов позиционирования позволяет создавать сложные и интерактивные веб-интерфейсы. Важно помнить о содержащих блоках, стековом контексте и правильном использовании z-index для достижения желаемого результата.
