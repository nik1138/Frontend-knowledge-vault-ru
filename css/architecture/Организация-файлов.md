# Организация файлов

Организация файлов в CSS-проекте — это фундаментальная часть создания масштабируемых и поддерживаемых веб-проектов. Правильная структура файлов позволяет командам разработчиков эффективно работать с кодом, упрощает внесение изменений и предотвращает конфликты.

## Общие принципы организации файлов

### 1. Логическая структура

Файлы должны быть организованы по логическим группам, отражающим архитектуру проекта:

```
styles/
├── settings/          # Переменные и настройки
├── tools/            # Миксины и функции
├── generic/          # Сбросы и нормализация
├── elements/         # Стили HTML-элементов
├── objects/          # Объекты без визуального оформления
├── components/       # Компоненты с визуальным оформлением
├── utilities/        # Вспомогательные классы
├── pages/            # Специфические стили страниц
├── themes/           # Темы оформления
├── vendors/          # Сторонние стили
└── main.scss         # Главный файл сборки
```

### 2. Последовательность подключения

Файлы должны подключаться в определенном порядке, чтобы избежать конфликтов специфичности:

```scss
// main.scss
@import 'settings/colors';
@import 'settings/typography';
@import 'settings/breakpoints';

@import 'tools/mixins';
@import 'tools/functions';

@import 'generic/reset';
@import 'generic/box-sizing';

@import 'elements/headings';
@import 'elements/links';

@import 'objects/layout';
@import 'objects/grid';

@import 'components/button';
@import 'components/card';
@import 'components/form';

@import 'utilities/helpers';
@import 'utilities/spacing';
```

## Методологии организации файлов

### 1. ITCSS (Inverted Triangle CSS)

ITCSS — это методология, которая организует CSS в виде перевернутого треугольника, где стили становятся всё более специфичными:

```
styles/
├── settings/
│   ├── _colors.scss
│   ├── _typography.scss
│   └── _breakpoints.scss
├── tools/
│   ├── _mixins.scss
│   └── _functions.scss
├── generic/
│   ├── _normalize.scss
│   ├── _reset.scss
│   └── _box-sizing.scss
├── elements/
│   ├── _headings.scss
│   ├── _links.scss
│   └── _lists.scss
├── objects/
│   ├── _layout.scss
│   ├── _grid.scss
│   └── _hide.scss
├── components/
│   ├── _button.scss
│   ├── _card.scss
│   ├── _form.scss
│   └── _navigation.scss
├── utilities/
│   ├── _spacing.scss
│   ├── _typography.scss
│   └── _helpers.scss
└── main.scss
```

#### Пример файлов в ITCSS:

```scss
// settings/_colors.scss
$color-primary: #007bff;
$color-secondary: #6c757d;
$color-success: #28a745;
$color-danger: #dc3545;
$color-warning: #ffc107;
$color-info: #17a2b8;
$color-light: #f8f9fa;
$color-dark: #343a40;
```

```scss
// components/_button.scss
.c-btn {
  display: inline-block;
  font-weight: 400;
  color: #212529;
  text-align: center;
  vertical-align: middle;
  user-select: none;
  background-color: transparent;
  border: 1px solid transparent;
  padding: 0.375rem 0.75rem;
  font-size: 1rem;
  line-height: 1.5;
  border-radius: 0.25rem;
  transition: color 0.15s ease-in-out, background-color 0.15s ease-in-out, 
              border-color 0.15s ease-in-out, box-shadow 0.15s ease-in-out;
}

.c-btn--primary {
  @extend .c-btn;
  color: #fff;
  background-color: $color-primary;
  border-color: $color-primary;
}

.c-btn--secondary {
  @extend .c-btn;
  color: #fff;
  background-color: $color-secondary;
  border-color: $color-secondary;
}
```

### 2. Atomic Design

Atomic Design организует файлы по уровням компонентов:

```
styles/
├── atoms/
│   ├── _buttons.scss
│   ├── _inputs.scss
│   ├── _typography.scss
│   └── _colors.scss
├── molecules/
│   ├── _search-form.scss
│   ├── _user-card.scss
│   └── _product-card.scss
├── organisms/
│   ├── _header.scss
│   ├── _footer.scss
│   ├── _navigation.scss
│   └── _sidebar.scss
├── templates/
│   ├── _main-layout.scss
│   └── _blog-layout.scss
├── pages/
│   ├── _home.scss
│   ├── _about.scss
│   └── _contact.scss
└── main.scss
```

## Структура файлов в российских реалиях 2025 года

В 2025 году в российской веб-разработке организация файлов должна учитывать следующие факторы:

### 1. Соответствие требованиям доступности

Файлы должны включать стили для обеспечения доступности:

```
styles/
├── settings/
│   ├── _colors.scss          # Цветовые переменные с учетом контрастности
│   └── _accessibility.scss   # Настройки доступности
├── components/
│   ├── _button.scss          # Стили кнопок с фокусом и ARIA
│   └── _form.scss            # Стили форм с метками и валидацией
└── utilities/
    └── _screen-reader.scss   # Вспомогательные классы для скринридеров
```

### 2. Поддержка тем оформления

С учетом требований к адаптации под различные условия использования:

```
styles/
├── themes/
│   ├── _light.scss           # Светлая тема
│   ├── _dark.scss            # Темная тема
│   └── _high-contrast.scss   # Высокая контрастность
├── settings/
│   └── _theme-variables.scss # Переменные для тем
└── components/
    └── _themeable-components.scss # Компоненты, поддерживающие темы
```

### 3. Локализация и международизация

Файлы должны учитывать особенности разных языков:

```
styles/
├── settings/
│   ├── _typography.scss      # Базовые шрифты
│   └── _rtl.scss             # Поддержка RTL-языков
├── utilities/
│   └── _text-direction.scss  # Утилиты для направления текста
```

## Практические рекомендации

### 1. Именование файлов

Следуйте единообразным правилам именования:

- Используйте kebab-case для имен файлов: `navigation-menu.scss`
- Добавляйте префиксы для указания типа компонента: `c-card.scss`, `m-search-form.scss`
- Используйте подчеркивание для частичных файлов: `_variables.scss`

### 2. Структура компонентов

Каждый компонент должен быть изолирован и содержать все необходимые стили:

```scss
// components/_user-profile.scss
.c-user-profile {
  display: flex;
  align-items: center;
  gap: 1rem;
  padding: 1rem;
  border: 1px solid #e9ecef;
  border-radius: 0.25rem;
  background-color: white;
  
  &__avatar {
    width: 3rem;
    height: 3rem;
    border-radius: 50%;
    object-fit: cover;
  }
  
  &__info {
    flex: 1;
  }
  
  &__name {
    font-weight: bold;
    margin-bottom: 0.25rem;
  }
  
  &__role {
    color: #6c757d;
    font-size: 0.875rem;
  }
}
```

### 3. Общие и специфические стили

Разделяйте общие и странице-специфичные стили:

```
styles/
├── base/                   # Общие стили
│   ├── _typography.scss
│   └── _layout.scss
├── components/            # Переиспользуемые компоненты
│   ├── _button.scss
│   └── _card.scss
├── pages/                 # Специфические стили страниц
│   ├── _home.scss
│   ├── _product.scss
│   └── _checkout.scss
└── vendors/               # Сторонние библиотеки
    ├── _bootstrap.scss
    └── _fontawesome.scss
```

### 4. Документирование структуры

Добавляйте комментарии и документацию к структуре файлов:

```scss
// ========================================
// COMPONENT: User Card
// ========================================
//
// Description: A card component for displaying user information
//
// Components:
// - c-user-card: Main card container
// - c-user-card__image: User avatar
// - c-user-card__content: Card content area
// - c-user-card__name: User name
// - c-user-card__role: User role
//
// Modifiers:
// - c-user-card--featured: Featured card variant
//
// ========================================

.c-user-card {
  // Styles here
}
```

## Зависимости и импорты

Управление зависимостями между файлами критично для избежания конфликтов:

```scss
// settings/_variables.scss
// Базовые переменные - не зависят ни от чего
$primary-color: #007bff;

// settings/_colors.scss
// Зависит от _variables.scss
$text-color: #212529;
$background-color: #fff;

// components/_button.scss
// Зависит от переменных
.c-btn {
  color: $text-color;
  background-color: $primary-color;
}
```

## Современные тенденции

### 1. CSS-переменные в файлах

Использование CSS Custom Properties для динамических тем:

```scss
// settings/_theme-variables.scss
:root {
  --color-primary: #007bff;
  --color-secondary: #6c757d;
  --font-size-base: 16px;
  --border-radius: 0.25rem;
}

// Для темной темы
[data-theme="dark"] {
  --color-primary: #0d6efd;
  --color-secondary: #5a6268;
}
```

### 2. Модульные стили

Использование CSS Modules или scoped стилей в современных фреймворках:

```
components/
├── Button/
│   ├── index.js
│   ├── Button.module.scss  # Модульные стили
│   └── Button.test.js
```

## Заключение

Правильная организация файлов в CSS-проекте — это ключ к созданию масштабируемых, поддерживаемых и понятных стилей. В условиях российского рынка 2025 года, с учетом требований к доступности, производительности и локализации, продуманная структура файлов становится особенно важной.

Независимо от выбранной методологии ([[ITCSS]], [[Atomic-Design]], компонентная архитектура), важно придерживаться принципов последовательности, предсказуемости и документированности. Это обеспечит эффективную работу команды разработчиков и упростит поддержку проекта в долгосрочной перспективе.
