# Viewport-единицы в CSS

## Определение

Viewport-единицы (единицы области просмотра) в CSS - это относительные единицы измерения, которые основаны на размерах области просмотра (viewport) браузера. Эти единицы позволяют создавать элементы, размер которых пропорционален размерам окна браузера.

## Виды viewport-единиц

### vw (viewport width)
- 1vw = 1% от ширины области просмотра
- Пример: при ширине окна 800px, 1vw = 8px
- Используется для создания элементов, масштабирующихся по ширине

### vh (viewport height)
- 1vh = 1% от высоты области просмотра
- Пример: при высоте окна 600px, 1vh = 6px
- Используется для создания элементов, масштабирующихся по высоте

### vmin (viewport minimum)
- 1vmin = 1% от меньшей стороны области просмотра (ширины или высоты)
- Пример: если ширина 800px, высота 600px, то 1vmin = 6px
- Полезно для обеспечения консистентности на устройствах с разными пропорциями

### vmax (viewport maximum)
- 1vmax = 1% от большей стороны области просмотра (ширины или высоты)
- Пример: если ширина 800px, высота 600px, то 1vmax = 8px
- Используется редко, но может быть полезно в специфических сценариях

## Практическое применение

### Создание полноэкранных секций

```css
.full-screen-section {
  width: 100vw;
  height: 100vh;
  display: flex;
  align-items: center;
  justify-content: center;
}

.hero-section {
  height: 90vh; /* Занимает 90% высоты экрана */
  background: linear-gradient(135deg, #667eea 0%, #764ba2 100%);
}
```

### Адаптивная типографика

```css
.responsive-title {
  font-size: 4vw; /* Размер шрифта изменяется с шириной экрана */
  font-size: clamp(1.5rem, 4vw, 3rem); /* Совмещение с clamp() для ограничения */
}

.subtitle {
  font-size: 2.5vh; /* Размер шрифта изменяется с высотой экрана */
}
```

### Адаптивные размеры элементов

```css
.responsive-card {
  width: 80vw; /* 80% от ширины окна */
  height: 40vh; /* 40% от высоты окна */
  margin: 5vh auto; /* Вертикальные отступы 5% от высоты окна */
}

.square-box {
  width: 20vmin; /* Квадратный элемент, размер зависит от меньшей стороны */
  height: 20vmin;
  background-color: #3498db;
}
```

### Создание адаптивных сеток

```css
.viewport-grid {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(25vw, 1fr));
  gap: 2vw;
}

.adaptive-item {
  min-height: 30vh;
  padding: 3vw;
}
```

## Современные возможности (2025 год)

### Совмещение с clamp()

```css
.modern-responsive-text {
  /* Минимум 16px, идеал 4vw, максимум 24px */
  font-size: clamp(1rem, 4vw, 1.5rem);
}

.adaptive-container {
  width: clamp(300px, 90vw, 1200px);
}
```

### Использование с calc()

```css
.calculated-size {
  width: calc(100vw - 20px); /* Ширина за вычетом 20px */
  height: calc(50vh - 2rem); /* Высота за вычетом 2rem */
}

.offset-element {
  left: calc(50% - 50vw); /* Выравнивание по левому краю */
}
```

## Особенности и подводные камни

### Проблемы с мобильными устройствами

На мобильных устройствах адресная строка может изменять размер viewport, что приводит к непредсказуемому поведению:

```css
/* Решение для мобильных устройств */
.mobile-friendly {
  height: 100dvh; /* Использование dvh вместо vh (если поддерживается) */
  height: 100vh; /* Резервный вариант */
}

/* Альтернативное решение */
:root {
  --vh: 1vh;
}

.adaptive-height {
  height: calc(var(--vh, 1vh) * 100);
}

/* Обновление переменной через JavaScript */
/* document.documentElement.style.setProperty('--vh', window.innerHeight * 0.01 + 'px'); */
```

### Проблемы с масштабированием

Viewport-единицы не изменяются при масштабировании страницы, в отличие от `px`, что может создать проблемы с доступностью.

## Рекомендации по использованию в 2025 году

- Широкое использование `vw` и `vh` для создания адаптивных интерфейсов
- Применение `vmin` и `vmax` для обеспечения консистентности на устройствах с разными пропорциями
- Комбинация с `clamp()` и `calc()` для более точного контроля
- Учет особенностей мобильных устройств при использовании `vh`
- В российской практике viewport-единицы активно используются в современных SPA-приложениях и адаптивных сайтах

## Преимущества и недостатки

### Преимущества
- Создание действительно адаптивных интерфейсов
- Простота масштабирования элементов
- Хорошая интеграция с современными CSS-функциями
- Эффективность при создании полноэкранных компонентов

### Недостатки
- Проблемы с мобильными браузерами из-за изменения размера viewport
- Невозможность корректировки при масштабировании страницы
- Потенциальные проблемы с доступностью
- Сложность предсказания точных размеров

## Современные тенденции

В 2025 году в российской веб-разработке наблюдается:
- Увеличение использования viewport-единиц в комбинации с CSS Grid и Flexbox
- Интеграция с системами компонентов для создания адаптивных элементов
- Активное использование `clamp()` для ограничения диапазона viewport-единиц
- Рост популярности новых единиц, таких как `dvh`, `dvw` (dynamic viewport units)
