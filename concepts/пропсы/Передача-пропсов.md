# Передача пропсов

**Передача пропсов** (props passing) — это фундаментальный механизм во фронтенд-фреймворках, позволяющий передавать данные от родительских компонентов к дочерним. Пропсы (props) представляют собой неизменяемые (immutable) значения, которые компонент получает извне и использует для настройки своего поведения или отображения.

## Основы передачи пропсов

В большинстве современных фронтенд-фреймворков (React, Vue, Angular и др.) компоненты могут принимать входные параметры, называемые пропсами. Это позволяет создавать переиспользуемые и гибкие компоненты.

### Пример в React

```jsx
// Родительский компонент
function App() {
  return (
    <div>
      <Greeting name="Иван" age={25} isActive={true} />
    </div>
  );
}

// Дочерний компонент
function Greeting(props) {
  return (
    <h1>
      Привет, {props.name}! Тебе {props.age} лет.
      {props.isActive ? ' Активный пользователь' : ' Неактивный пользователь'}
    </h1>
  );
}
```

### Пример в Vue

```vue
<!-- Родительский компонент -->
<template>
  <div>
    <Greeting :name="userName" :age="userAge" :isActive="isUserActive" />
  </div>
</template>

<script>
import Greeting from './Greeting.vue';

export default {
  components: {
    Greeting
  },
  data() {
    return {
      userName: 'Иван',
      userAge: 25,
      isUserActive: true
    };
  }
};
</script>
```

```vue
<!-- Дочерний компонент Greeting.vue -->
<template>
  <h1>
    Привет, {{ name }}! Тебе {{ age }} лет.
    <span v-if="isActive">Активный пользователь</span>
    <span v-else>Неактивный пользователь</span>
  </h1>
</template>

<script>
export default {
  props: {
    name: String,
    age: Number,
    isActive: Boolean
  }
};
</script>
```

## Типы данных в пропсах

Пропсы могут передавать различные типы данных:

- **Примитивы**: строки, числа, булевы значения
- **Объекты**: сложные структуры данных
- **Функции**: колбэки для взаимодействия с родительским компонентом
- **JSX/HTML-элементы**: для передачи содержимого (children)

### Передача функции как пропса

```jsx
function Parent() {
  const handleClick = () => {
    console.log('Кнопка нажата!');
  };

  return <Child onClick={handleClick} />;
}

function Child({ onClick }) {
  return <button onClick={onClick}>Нажми меня</button>;
}
```

## Важные особенности

- Пропсы передаются только сверху вниз (от родителя к потомку)
- Пропсы неизменяемы внутри компонента
- Пропсы могут иметь значения по умолчанию
- Пропсы можно валидировать для обеспечения целостности данных

## Вложенные компоненты и передача пропсов

При создании сложных интерфейсов часто возникает необходимость передавать пропсы через несколько уровней компонентов. Это называется "prop drilling" и может усложнить архитектуру приложения.

```jsx
// Проблема: передача пропсов через несколько уровней
function App() {
  return <Level1 user={{ name: 'Иван' }} />;
}

function Level1({ user }) {
  return <Level2 user={user} />;
}

function Level2({ user }) {
  return <Level3 user={user} />;
}

function Level3({ user }) {
  return <div>Пользователь: {user.name}</div>;
}
```

Для решения этой проблемы используются такие подходы, как [[Контекст]] или [[Глобальное состояние]].

## Лучшие практики

- Передавайте только необходимые данные
- Используйте осмысленные имена пропсов
- Валидируйте пропсы с помощью PropTypes или TypeScript
- Избегайте передачи лишних данных, особенно больших объектов
