# Документирование компонентов

## Обзор

Документирование компонентов — это процесс создания описания пользовательских интерфейсных компонентов, их свойств, состояний и использования. Это особенно важно в современных фронтенд-фреймворках и библиотеках, где компонентный подход является основой архитектуры.

## Важность документирования компонентов

Документирование компонентов необходимо для:
- [[Совместная разработка]] - унификация подхода к использованию компонентов
- [[UI-библиотеки]] - создание библиотек повторно используемых компонентов
- [[Поддержка интерфейсов]] - упрощение обслуживания и обновления UI
- [[Тестирование компонентов]] - обеспечение качества через документированные ожидания

## Типы документирования компонентов

### 1. Примеры использования компонентов

Каждый компонент должен иметь примеры использования:

```jsx
// Button.jsx
import React from 'react';

/**
 * Компонент кнопки
 * @param {string} variant - стиль кнопки ('primary', 'secondary', 'danger')
 * @param {boolean} disabled - заблокирована ли кнопка
 * @param {ReactNode} children - содержимое кнопки
 * @param {Function} onClick - обработчик клика
 * @example
 * <Button variant="primary" onClick={handleClick}>
 *   Нажми меня
 * </Button>
 */
export const Button = ({ variant = 'primary', disabled = false, children, onClick }) => {
  const baseClass = 'btn';
  const className = `${baseClass} btn-${variant} ${disabled ? 'btn-disabled' : ''}`;
  
  return (
    <button className={className} disabled={disabled} onClick={onClick}>
      {children}
    </button>
  );
};
```

### 2. Документирование пропсов (props)

Для каждого компонента важно документировать его свойства:

```jsx
// UserProfile.jsx
import React from 'react';

/**
 * Компонент профиля пользователя
 * 
 * Отображает информацию о пользователе с возможностью редактирования
 * 
 * @component
 * @example
 * return (
 *   <UserProfile 
 *     user={{ 
 *       name: 'Иван Петров', 
 *       email: 'ivan@example.com',
 *       avatar: '/path/to/avatar.jpg'
 *     }}
 *     editable={true}
 *     onEdit={handleEdit}
 *   />
 * )
 * 
 * @param {Object} props - Свойства компонента
 * @param {Object} props.user - Объект пользователя
 * @param {string} props.user.name - Имя пользователя
 * @param {string} props.user.email - Email пользователя
 * @param {string} [props.user.avatar] - URL аватара пользователя
 * @param {boolean} [props.editable=false] - Доступно ли редактирование
 * @param {Function} [props.onEdit] - Обработчик события редактирования
 * @returns {JSX.Element} Компонент профиля пользователя
 */
export const UserProfile = ({ user, editable = false, onEdit }) => {
  return (
    <div className="user-profile">
      {user.avatar && <img src={user.avatar} alt="Аватар" className="avatar" />}
      <div className="user-info">
        <h3>{user.name}</h3>
        <p>{user.email}</p>
      </div>
      {editable && (
        <button onClick={onEdit} className="edit-btn">
          Редактировать
        </button>
      )}
    </div>
  );
};
```

### 3. Документирование состояний компонентов

Компоненты могут иметь различные состояния, которые также нужно документировать:

```jsx
// LoadingSpinner.jsx
import React from 'react';

/**
 * Компонент индикатора загрузки
 * 
 * ## Состояния:
 * - `visible`: индикатор отображается
 * - `hidden`: индикатор скрыт
 * - `error`: состояние ошибки (если указан errorText)
 * 
 * @param {Object} props
 * @param {boolean} [props.visible=true] - Видимость индикатора
 * @param {string} [props.size='medium'] - Размер ('small', 'medium', 'large')
 * @param {string} [props.errorText] - Текст ошибки (если применимо)
 * @param {string} [props.className] - Дополнительный CSS класс
 */
export const LoadingSpinner = ({ 
  visible = true, 
  size = 'medium', 
  errorText, 
  className = '' 
}) => {
  if (!visible && !errorText) return null;
  
  const spinnerClass = `spinner spinner-${size} ${className}`;
  
  if (errorText) {
    return <div className="error-state">{errorText}</div>;
  }
  
  return <div className={spinnerClass}>Загрузка...</div>;
};
```

## Структура документирования компонентов

### 1. Общее описание

Начинайте с описания назначения компонента:

```jsx
/**
 * Компонент формы входа
 * 
 * Отображает форму для аутентификации пользователя.
 * Включает поля для ввода email и пароля, а также кнопку входа.
 * Обрабатывает валидацию и отправку данных.
 */
```

### 2. Примеры использования

Предоставьте один или несколько примеров:

```jsx
/**
 * @example
 * // Простое использование
 * <LoginForm onLogin={handleLogin} />
 * 
 * @example
 * // С предустановленным email
 * <LoginForm 
 *   defaultEmail="user@example.com"
 *   onLogin={handleLogin}
 *   onForgotPassword={handleForgotPassword}
 * />
 */
```

### 3. Документирование пропсов

Полностью документируйте все пропсы:

```jsx
/**
 * @param {Object} props - Свойства компонента
 * @param {Function} props.onLogin - Обработчик успешного входа
 * @param {Function} [props.onForgotPassword] - Обработчик "Забыли пароль?"
 * @param {string} [props.defaultEmail] - Предустановленный email
 * @param {boolean} [props.rememberMe=true] - Показывать чекбокс "Запомнить меня"
 */
```

### 4. Возвращаемое значение

Укажите, что возвращает компонент:

```jsx
/**
 * @returns {JSX.Element} Компонент формы входа
 */
```

## Документирование сложных компонентов

### Компоненты с дочерними элементами

```jsx
// Card.jsx
/**
 * Компонент карточки
 * 
 * Универсальный компонент карточки с поддержкой заголовка, 
 * содержимого и дополнительных действий.
 * 
 * @param {Object} props
 * @param {ReactNode} props.title - Заголовок карточки
 * @param {ReactNode} props.children - Содержимое карточки
 * @param {Array<Object>} [props.actions] - Массив действий для карточки
 * @param {string} [props.variant='default'] - Стиль карточки
 * 
 * @example
 * <Card title="Заголовок карточки" variant="primary">
 *   <p>Содержимое карточки</p>
 *   <Card.Actions>
 *     <Button>Действие 1</Button>
 *     <Button variant="secondary">Действие 2</Button>
 *   </Card.Actions>
 * </Card>
 */
export const Card = ({ title, children, actions, variant = 'default' }) => {
  return (
    <div className={`card card-${variant}`}>
      <div className="card-header">
        <h3>{title}</h3>
      </div>
      <div className="card-body">
        {children}
      </div>
      {actions && (
        <div className="card-footer">
          {actions.map((action, index) => (
            <button key={index} onClick={action.handler}>
              {action.text}
            </button>
          ))}
        </div>
      )}
    </div>
  );
};

/**
 * Компонент действий карточки
 * 
 * Используется внутри компонента Card для отображения действий.
 * 
 * @param {Object} props
 * @param {ReactNode} props.children - Кнопки или другие элементы действий
 */
Card.Actions = ({ children }) => {
  return <div className="card-actions">{children}</div};
};
```

### Компоненты с состоянием

```jsx
// Accordion.jsx
import React, { useState } from 'react';

/**
 * Компонент аккордеона
 * 
 * Интерактивный компонент, позволяющий скрывать и показывать 
 * содержимое разделов.
 * 
 * @param {Object} props
 * @param {Array<Object>} props.items - Массив элементов аккордеона
 * @param {string} items[].title - Заголовок элемента
 * @param {ReactNode} items[].content - Содержимое элемента
 * @param {number} [props.defaultActiveIndex] - Индекс активного элемента по умолчанию
 * @param {boolean} [props.multiple=false] - Разрешить открытие нескольких элементов
 * 
 * @example
 * const items = [
 *   { title: 'Заголовок 1', content: <p>Содержимое 1</p> },
 *   { title: 'Заголовок 2', content: <p>Содержимое 2</p> }
 * ];
 * 
 * <Accordion items={items} defaultActiveIndex={0} />
 */
export const Accordion = ({ items, defaultActiveIndex, multiple = false }) => {
  const [activeIndexes, setActiveIndexes] = useState(
    defaultActiveIndex !== undefined ? [defaultActiveIndex] : []
  );

  const toggleItem = (index) => {
    if (multiple) {
      setActiveIndexes(prev =>
        prev.includes(index)
          ? prev.filter(i => i !== index)
          : [...prev, index]
      );
    } else {
      setActiveIndexes(prev =>
        prev[0] === index ? [] : [index]
      );
    }
  };

  return (
    <div className="accordion">
      {items.map((item, index) => (
        <div key={index} className="accordion-item">
          <button 
            className="accordion-header"
            onClick={() => toggleItem(index)}
          >
            {item.title}
            <span className={`arrow ${activeIndexes.includes(index) ? 'up' : 'down'}`} />
          </button>
          {activeIndexes.includes(index) && (
            <div className="accordion-content">
              {item.content}
            </div>
          )}
        </div>
      ))}
    </div>
  );
};
```

## Инструменты для документирования компонентов

### Storybook

Storybook — один из самых популярных инструментов для документирования компонентов:

```jsx
// Button.stories.jsx
import React from 'react';
import { Button } from './Button';

export default {
  title: 'UI/Button',
  component: Button,
  argTypes: {
    backgroundColor: { control: 'color' },
    size: {
      control: { type: 'select' },
      options: ['small', 'medium', 'large'],
    },
  },
};

const Template = (args) => <Button {...args} />;

export const Primary = Template.bind({});
Primary.args = {
  variant: 'primary',
  label: 'Button',
};

export const Secondary = Template.bind({});
Secondary.args = {
  variant: 'secondary',
  label: 'Button',
};
```

  // реализация

|------|------|---------|-------------|

## Заключение

Документирование компонентов — важная часть разработки пользовательских интерфейсов. Хорошо документированные компоненты упрощают их повторное использование, улучшают совместную работу и уменьшают количество ошибок при внедрении. Использование стандартов документирования и автоматических инструментов помогает поддерживать высокое качество документации на протяжении всего жизненного цикла проекта.
