# Алгоритмы-сортировки

## Обзор

Алгоритмы сортировки — это методы упорядочивания элементов в коллекции по заданному критерию. В контексте фронтенд-разработки сортировка часто используется для упорядочивания данных перед отображением в интерфейсе, таких как списки товаров, таблицы с данными, комментарии и т.д.

## Важность в фронтенд-разработке

Сортировка играет ключевую роль в пользовательском опыте:
- Улучшает восприятие информации
- Позволяет пользователям находить нужные элементы быстрее
- Необходима для эффективной работы с большими наборами данных
- Используется в фильтрации и поиске

## Основные алгоритмы сортировки

### 1. Пузырьковая сортировка (Bubble Sort)

Самый простой алгоритм сортировки, который работает путем многократного прохода по списку, сравнивая соседние элементы и меняя их местами, если они находятся в неправильном порядке.

```javascript
function bubbleSort(arr) {
    const n = arr.length;
    for (let i = 0; i < n - 1; i++) {
        for (let j = 0; j < n - i - 1; j++) {
            if (arr[j] > arr[j + 1]) {
                // Меняем местами
                [arr[j], arr[j + 1]] = [arr[j + 1], arr[j]];
            }
        }
    }
    return arr;
}

// Пример использования
const numbers = [64, 34, 25, 12, 22, 11, 90];
console.log(bubbleSort(numbers)); // [11, 12, 22, 25, 34, 64, 90]
```

**Сложность:**
- Время: O(n²) в худшем и среднем случаях
- Память: O(1)

> [!tip] Применение в фронтенде
> Пузырьковая сортировка может использоваться для сортировки небольших списков, например, сортировка пунктов меню или фильтров, где производительность не является критичной.

### 2. Сортировка выбором (Selection Sort)

Алгоритм находит минимальный элемент и помещает его в начало массива, затем находит следующий минимальный элемент и помещает его на вторую позицию и т.д.

```javascript
function selectionSort(arr) {
    const n = arr.length;
    for (let i = 0; i < n - 1; i++) {
        let minIdx = i;
        for (let j = i + 1; j < n; j++) {
            if (arr[j] < arr[minIdx]) {
                minIdx = j;
            }
        }
        // Меняем местами
        [arr[i], arr[minIdx]] = [arr[minIdx], arr[i]];
    }
    return arr;
}

// Пример использования
const numbers = [64, 34, 25, 12, 22, 11, 90];
console.log(selectionSort(numbers)); // [11, 12, 22, 25, 34, 64, 90]
```

**Сложность:**
- Время: O(n²)
- Память: O(1)

### 3. Сортировка вставками (Insertion Sort)

Алгоритм строит отсортированный массив по одному элементу за раз, беря каждый элемент и вставляя его в правильное место в уже отсортированной части массива.

```javascript
function insertionSort(arr) {
    for (let i = 1; i < arr.length; i++) {
        let current = arr[i];
        let j = i - 1;
        
        while (j >= 0 && arr[j] > current) {
            arr[j + 1] = arr[j];
            j--;
        }
        arr[j + 1] = current;
    }
    return arr;
}

// Пример использования
const numbers = [64, 34, 25, 12, 22, 11, 90];
console.log(insertionSort(numbers)); // [11, 12, 22, 25, 34, 64, 90]
```

**Сложность:**
- Время: O(n²) в худшем случае, O(n) в лучшем
- Память: O(1)

> [!tip] Применение в фронтенде
> Сортировка вставками эффективна для небольших наборов данных или почти отсортированных массивов, таких как сортировка недавно добавленных элементов в список.

### 4. Быстрая сортировка (Quick Sort)

Эффективный алгоритм сортировки, который использует принцип "разделяй и властвуй". Выбирает опорный элемент и разделяет массив на две части: элементы меньше опорного и элементы больше или равны опорному.

```javascript
function quickSort(arr, low = 0, high = arr.length - 1) {
    if (low < high) {
        const pi = partition(arr, low, high);
        quickSort(arr, low, pi - 1);
        quickSort(arr, pi + 1, high);
    }
    return arr;
}

function partition(arr, low, high) {
    const pivot = arr[high];
    let i = low - 1;
    
    for (let j = low; j < high; j++) {
        if (arr[j] < pivot) {
            i++;
            [arr[i], arr[j]] = [arr[j], arr[i]];
        }
    }
    [arr[i + 1], arr[high]] = [arr[high], arr[i + 1]];
    return i + 1;
}

// Пример использования
const numbers = [64, 34, 25, 12, 22, 11, 90];
console.log(quickSort([...numbers])); // [11, 12, 22, 25, 34, 64, 90]
```

**Сложность:**
- Время: O(n log n) в среднем случае, O(n²) в худшем
- Память: O(log n)

### 5. Сортировка слиянием (Merge Sort)

Алгоритм сортировки, который также использует принцип "разделяй и властвуй". Делит массив на две половины, рекурсивно сортирует каждую половину, а затем объединяет отсортированные половины.

```javascript
function mergeSort(arr) {
    if (arr.length <= 1) {
        return arr;
    }
    
    const mid = Math.floor(arr.length / 2);
    const left = mergeSort(arr.slice(0, mid));
    const right = mergeSort(arr.slice(mid));
    
    return merge(left, right);
}

function merge(left, right) {
    let result = [];
    let leftIndex = 0;
    let rightIndex = 0;
    
    while (leftIndex < left.length && rightIndex < right.length) {
        if (left[leftIndex] < right[rightIndex]) {
            result.push(left[leftIndex]);
            leftIndex++;
        } else {
            result.push(right[rightIndex]);
            rightIndex++;
        }
    }
    
    return result
        .concat(left.slice(leftIndex))
        .concat(right.slice(rightIndex));
}

// Пример использования
const numbers = [64, 34, 25, 12, 22, 11, 90];
console.log(mergeSort(numbers)); // [11, 12, 22, 25, 34, 64, 90]
```

**Сложность:**
- Время: O(n log n) во всех случаях
- Память: O(n)

> [!warning] Важно
> Merge Sort всегда имеет O(n log n) время выполнения, что делает его предсказуемым выбором, но он требует больше памяти по сравнению с Quick Sort.

## Практические примеры в фронтенде

### Сортировка списка товаров

```javascript
// Пример сортировки товаров по цене
const products = [
    { name: "Телефон", price: 500 },
    { name: "Ноутбук", price: 1200 },
    { name: "Планшет", price: 300 },
    { name: "Часы", price: 200 }
];

// Сортировка по возрастанию цены
const sortedByPrice = products.sort((a, b) => a.price - b.price);
console.log(sortedByPrice);

// Сортировка по убыванию цены
const sortedByPriceDesc = products.sort((a, b) => b.price - a.price);
console.log(sortedByPriceDesc);

// Сортировка по названию
const sortedByName = products.sort((a, b) => a.name.localeCompare(b.name));
console.log(sortedByName);
```

### Сортировка с кастомной логикой

```javascript
// Сортировка с учетом нескольких критериев
const users = [
    { name: "Иван", age: 25, salary: 50000 },
    { name: "Мария", age: 30, salary: 60000 },
    { name: "Алексей", age: 25, salary: 55000 }
];

// Сначала по возрасту, затем по зарплате
const sortedUsers = users.sort((a, b) => {
    if (a.age !== b.age) {
        return a.age - b.age;
    }
    return b.salary - a.salary; // по убыванию зарплаты при одинаковом возрасте
});

console.log(sortedUsers);
```

## Встроенные методы сортировки JavaScript

JavaScript предоставляет встроенный метод `sort()`, который по умолчанию преобразует элементы в строки и сортирует их в порядке UTF-16:

```javascript
const numbers = [10, 5, 40, 25, 1000, 1];
console.log(numbers.sort()); // [1, 10, 1000, 25, 40, 5] - неправильный порядок!

// Правильная сортировка чисел
console.log(numbers.sort((a, b) => a - b)); // [1, 5, 10, 25, 40, 1000]
```

## Сравнение алгоритмов

| Алгоритм | Лучший случай | Средний случай | Худший случай | Память | Стабильность |
|----------|---------------|----------------|---------------|--------|--------------|
| Bubble Sort | O(n) | O(n²) | O(n²) | O(1) | Да |
| Selection Sort | O(n²) | O(n²) | O(n²) | O(1) | Нет |
| Insertion Sort | O(n) | O(n²) | O(n²) | O(1) | Да |
| Quick Sort | O(n log n) | O(n log n) | O(n²) | O(log n) | Нет |
| Merge Sort | O(n log n) | O(n log n) | O(n log n) | O(n) | Да |

## Заключение

Понимание различных алгоритмов сортировки важно для фронтенд-разработчиков, особенно при работе с большими наборами данных. Выбор правильного алгоритма может значительно повлиять на производительность приложения.

Для большинства случаев в фронтенде рекомендуется использовать встроенный метод `sort()` с кастомной функцией сравнения, так как он оптимизирован в браузерах. Однако знание основных алгоритмов полезно для понимания производительности и для случаев, когда требуется специфическая логика сортировки.
