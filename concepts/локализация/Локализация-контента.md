# Локализация контента

## Обзор

**Локализация** (сокращённо l10n) — это процесс адаптации приложения к конкретной языковой и культурной среде. В отличие от [[Интернационализация|интернационализации]], которая готовит приложение к локализации, локализация включает в себя фактическое создание переводов и адаптацию контента для конкретной аудитории.

## Основные аспекты локализации

### 1. Перевод пользовательского интерфейса

Перевод всех элементов интерфейса, включая:

- Заголовки и подписи
- Кнопки и ссылки
- Сообщения об ошибках и уведомления
- Помощь и документация

### 2. Адаптация культурных особенностей

- Изображения и иконки, соответствующие культуре
- Цвета, ассоциации и символы
- Примеры данных и контексты использования

## Структура файлов локализации

### JSON-файлы

Наиболее распространённый формат для хранения переводов:

```json
{
  "common": {
    "save": "Сохранить",
    "cancel": "Отмена",
    "delete": "Удалить"
  },
  "navigation": {
    "home": "Главная",
    "about": "О нас",
    "contact": "Контакты"
  },
  "errors": {
    "required": "Это поле обязательно",
    "email": "Пожалуйста, введите действительный адрес электронной почты"
  }
}
```

### YAML-файлы

Альтернативный формат, который может быть более удобочитаемым:

```yaml
common:
  save: "Сохранить"
  cancel: "Отмена"
  delete: "Удалить"
  
navigation:
  home: "Главная"
  about: "О нас"
  contact: "Контакты"
  
errors:
  required: "Это поле обязательно"
  email: "Пожалуйста, введите действительный адрес электронной почты"
```

## Работа с переводами

### Извлечение строк для перевода

Создание системы, которая позволяет легко извлекать строки для передачи переводчикам:

```javascript
// Использование ключей вместо жёстко закодированных строк
const buttonText = t('common.save');
const errorMessage = t('errors.required');
```

### Обновление переводов

Система должна поддерживать:

- Добавление новых строк
- Обновление существующих переводов
- Управление устаревшими строками

## Инструменты локализации

### Веб-инструменты

- **Transifex** — платформа для управления переводами
- **Crowdin** — инструмент для коллаборативной локализации
- **Phrase** — облачная платформа для локализации

### Интеграция с системой контроля версий

```bash
# Пример workflow с использованием git
git checkout -b feature/translations
# Добавление новых переводов
git add locales/
git commit -m "feat: add French translations"
git push origin feature/translations
```

## Качество переводов

### Проверка согласованности

- Единый стиль и терминология
- Соответствие контексту использования
- Проверка грамматики и орфографии

### Тестирование локализации

```javascript
// Тестирование с разными локалями
describe('Localization tests', () => {
  test('should display correct Russian translation', () => {
    i18n.changeLanguage('ru');
    expect(getByText('Сохранить')).toBeInTheDocument();
  });

  test('should display correct English translation', () => {
    i18n.changeLanguage('en');
    expect(getByText('Save')).toBeInTheDocument();
  });
});
```

## Работа с динамическим контентом

### Перевод динамических данных

Для контента, поступающего из API или базы данных, может потребоваться специальная обработка:

```javascript
// Пример с категориями товаров
const categories = [
  { id: 1, name: 'electronics', labelKey: 'category.electronics' },
  { id: 2, name: 'books', labelKey: 'category.books' }
];

const localizedCategories = categories.map(cat => ({
  ...cat,
  label: t(cat.labelKey)
}));
```

### Интерполяция в переводах

Использование переменных в переводах:

```javascript
// В файле переводов
{
  "welcome_message": "Добро пожаловать, {{userName}}! У вас {{count}} новых сообщений."
}

// В коде
const message = t('welcome_message', { 
  userName: user.name, 
  count: user.unreadMessages 
});
```

## Работа с RTL языками

Для языков с направлением справа налево (например, арабский, иврит) необходима специальная обработка:

```css
/* Установка направления текста */
[dir="rtl"] {
  text-align: right;
  direction: rtl;
}

/* Корректировка отступов и позиционирования */
[dir="rtl"] .button {
  margin-right: 0;
  margin-left: 10px;
}
```

```javascript
// Определение направления языка
const isRTL = (locale) => {
  const rtlLocales = ['ar', 'he', 'fa', 'ur'];
  return rtlLocales.includes(locale.split('-')[0]);
};

// Применение класса направления
document.documentElement.dir = isRTL(currentLocale) ? 'rtl' : 'ltr';
```

## Практические рекомендации

### 1. Организация команды локализации

- Технический специалист (интернационализация)
- Переводчики
- Редакторы
- Тестировщики

### 2. Автоматизация процессов

- Автоматическое извлечение строк
- Проверка на наличие непереведённых строк
- Валидация форматов дат, чисел и т.д.

### 3. Мониторинг качества

- Регулярные проверки переведённого контента
- Сбор обратной связи от пользователей
- А/Б тестирование разных вариантов переводов

## Заключение

Локализация контента — это ключевой элемент успеха приложения на международных рынках. Качественная локализация не только переводит слова, но и адаптирует опыт использования под культурные особенности целевой аудитории.

> [!tip]
> Используйте систему управления переводами (TMS) для эффективной организации процесса локализации.

> [!warning]
> Не забывайте учитывать юридические и культурные особенности при локализации контента для разных регионов.
