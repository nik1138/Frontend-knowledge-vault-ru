# Семантическое версионирование (SemVer)

## Определение

**Семантическое версионирование** (SemVer) — это система нумерации версий программного обеспечения, состоящая из трех чисел: `MAJOR.MINOR.PATCH` (например, `2.3.1`). Эта система позволяет разработчикам понимать влияние обновлений на их проекты.

## Структура версии

Формат: `MAJOR.MINOR.PATCH`

- **MAJOR** (первая цифра): увеличивается при внесении несовместимых изменений API
- **MINOR** (вторая цифра): увеличивается при добавлении обратно совместимого функционала
- **PATCH** (третья цифра): увеличивается при добавлении обратно совместимых исправлений багов

### Примеры

- `1.0.0` → `2.0.0`: несовместимые изменения API
- `1.0.0` → `1.1.0`: добавление нового функционала
- `1.0.0` → `1.0.1`: исправление багов

## Правила SemVer

1. **Начальная версия**: `0.1.0` или `1.0.0`
2. **Версия 0.x.y**: предварительная стадия, изменения могут быть несовместимыми
3. **Версия 1.0.0**: стабильный API, после этого несовместимые изменения увеличивают MAJOR
4. **Пре-релизные версии**: `1.0.0-alpha`, `1.0.0-beta.1`
5. **Постфикс сборки**: `1.0.0+build.1`

## Практическое применение в фронтенде

### В package.json

```json
{
  "name": "my-react-app",
  "version": "1.2.3",
  "dependencies": {
    "react": "^18.2.0",
    "react-dom": "^18.2.0",
    "lodash": "~4.17.21"
  }
}
```

### Типы зависимостей

- `^` — позволяет обновления до следующей мажорной версии
- `~` — позволяет обновления до следующей минорной версии
- `>` или `<` — задает диапазон версий

### Работа с зависимостями

```bash
# Установка конкретной версии
npm install lodash@4.17.21

# Обновление до последней минорной/патч-версии
npm update lodash

# Проверка устаревших зависимостей
npm outdated
```

## Преимущества SemVer

- **Предсказуемость**: разработчики знают, какие изменения принесет обновление
- **Управление зависимостями**: упрощает процесс обновления пакетов
- **Совместимость**: помогает избежать проблем с совместимостью

## Примеры из реальной жизни

### React

- `16.8.0` — добавление хуков (MINOR)
- `17.0.0` — архитектурные изменения (MAJOR)
- `17.0.1` — исправление багов (PATCH)

### Vue.js

- `2.0.0` — несовместимые изменения (MAJOR)
- `2.6.0` — добавление функционала (MINOR)
- `2.6.11` — исправление багов (PATCH)

## Распространенные ошибки

1. **Нарушение принципов SemVer**: изменения в патч-версии, ломающие совместимость
2. **Отсутствие документации изменений**: разработчики не знают, что изменилось
3. **Частые мажорные обновления**: усложняет поддержку проектов

## Заключение

Семантическое версионирование — ключевой инструмент для управления зависимостями и обеспечения стабильности проектов. Правильное использование SemVer упрощает разработку и поддержку приложений, особенно в экосистемах с большим количеством зависимостей, как в современном фронтенд-развитии.
