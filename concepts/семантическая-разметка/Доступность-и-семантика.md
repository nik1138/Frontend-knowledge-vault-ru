# Доступность и семантика

Семантическая разметка играет ключевую роль в создании доступных веб-сайтов. Использование правильных HTML-элементов помогает вспомогательным технологиям, таким как скринридеры, правильно интерпретировать содержимое и предоставить его пользователям с ограниченными возможностями.

## Основы доступности веб-контента

Доступность (accessibility) — это практика создания веб-сайтов, которые могут использоваться всеми людьми, включая тех, у кого есть физические или когнитивные ограничения. К ним относятся:

- Люди с нарушениями зрения (слепые или слабовидящие)
- Люди с нарушениями слуха
- Люди с нарушениями моторики
- Люди с когнитивными расстройствами

Семантическая разметка является одним из основных инструментов для обеспечения доступности.

## Как семантика влияет на доступность

### Структура документа

Семантические теги создают логическую структуру документа, которую могут понять вспомогательные технологии:

```html
<!-- Хорошо: Использование семантических тегов -->
<header>
    <h1>Название сайта</h1>
    <nav>
        <ul>
            <li><a href="/">Главная</a></li>
            <li><a href="/about">О нас</a></li>
        </ul>
    </nav>
</header>

<main>
    <h2>Основной заголовок</h2>
    <p>Основное содержимое страницы...</p>
</main>

<aside>
    <h3>Боковая панель</h3>
    <p>Дополнительная информация</p>
</aside>

<footer>
    <p>&copy; 2025 Все права защищены</p>
</footer>
```

```html
<!-- Плохо: Использование только div-элементов -->
<div class="header">
    <div class="title">Название сайта</div>
    <div class="navigation">
        <ul>
            <li><a href="/">Главная</a></li>
            <li><a href="/about">О нас</a></li>
        </ul>
    </div>
</div>

<div class="main-content">
    <div class="main-title">Основной заголовок</div>
    <div class="content">Основное содержимое страницы...</div>
</div>

<div class="sidebar">
    <div class="sidebar-title">Боковая панель</div>
    <div class="sidebar-content">Дополнительная информация</div>
</div>

<div class="footer">
    <div class="copyright">&copy; 2025 Все права защищены</div>
</div>
```

### Заголовки и иерархия

Правильное использование заголовков (`<h1>` - `<h6>`) создает логическую иерархию документа:

```html
<!-- Правильная иерархия заголовков -->
<main>
    <h1>Основной заголовок страницы</h1>
    
    <section>
        <h2>Раздел 1</h2>
        <p>Содержимое раздела 1...</p>
        
        <h3>Подраздел 1.1</h3>
        <p>Содержимое подраздела 1.1...</p>
    </section>
    
    <section>
        <h2>Раздел 2</h2>
        <p>Содержимое раздела 2...</p>
        
        <h3>Подраздел 2.1</h3>
        <p>Содержимое подраздела 2.1...</p>
        
        <h4>Подраздел 2.1.1</h4>
        <p>Содержимое подраздела 2.1.1...</p>
    </section>
</main>
```

> [!tip] Совет
> Используйте заголовки последовательно, без пропусков уровней. Не переходите от `<h1>` сразу к `<h3>`, минуя `<h2>`.

### Семантические элементы и их роль

Каждый семантический элемент несет определенную информацию для вспомогательных технологий:

- `<nav>`: Обозначает навигационные области
- `<main>`: Указывает на основное содержимое страницы
- `<article>`: Обозначает независимый фрагмент контента
- `<section>`: Группирует связанные по смыслу элементы
- `<aside>`: Содержит сопутствующую информацию
- `<figure>` и `<figcaption>`: Представляют медиа-контент с описанием

## Атрибуты доступности

Хотя семантические теги сами по себе улучшают доступность, дополнительные атрибуты могут предоставить еще больше контекста:

### `aria-label` и `aria-labelledby`

Эти атрибуты позволяют предоставить текстовую метку элементу, который не имеет видимой метки:

```html
<button aria-label="Закрыть окно">×</button>

<div id="dialog-title">Заголовок диалога</div>
<div role="dialog" aria-labelledby="dialog-title">
    <!-- Содержимое диалога -->
</div>
```

### `role` атрибуты

Хотя семантические теги уже имеют встроенные роли, иногда бывает необходимо их явно указать:

```html
<!-- Хотя бы и не рекомендуется, но иногда может быть полезно -->
<nav role="navigation">
    <ul>
        <li><a href="/">Главная</a></li>
        <li><a href="/about">О нас</a></li>
    </ul>
</nav>
```

> [!warning] Важно
> Используйте встроенные семантические теги вместо добавления ролей, когда это возможно. Избегайте дублирования ролей, которые уже есть у элементов по умолчанию.

## Практические рекомендации

### 1. Используйте правильные теги для содержимого

```html
<!-- Хорошо: Использование семантических тегов -->
<article>
    <header>
        <h2>Заголовок статьи</h2>
        <time datetime="2025-01-15">15 января 2025</time>
    </header>
    <p>Содержимое статьи...</p>
    <footer>
        <p>Автор: Иван Иванов</p>
    </footer>
</article>
```

### 2. Обеспечьте альтернативный текст для изображений

```html
<!-- Хорошо: Описательный альтернативный текст -->
<img src="chart.png" alt="График продаж за последние 6 месяцев, показывающий рост на 25%">

<!-- Плохо: Общий или отсутствующий alt-текст -->
<img src="chart.png" alt="Изображение">
<img src="chart.png" alt="">
```

### 3. Используйте списки для группировки элементов

```html
<!-- Хорошо: Использование семантических списков -->
<nav>
    <ul>
        <li><a href="/">Главная</a></li>
        <li><a href="/about">О нас</a></li>
        <li><a href="/contact">Контакты</a></li>
    </ul>
</nav>

<!-- Плохо: Использование div-элементов для списка -->
<div>
    <div><a href="/">Главная</a></div>
    <div><a href="/about">О нас</a></div>
    <div><a href="/contact">Контакты</a></div>
</div>
```

### 4. Обеспечьте контрастность текста

Хотя это не связано напрямую с семантикой, хорошая контрастность важна для пользователей с нарушениями зрения:

```css
/* Хорошо: Достаточный контраст */
body {
    color: #000;
    background-color: #fff;
}

/* Плохо: Недостаточный контраст */
.low-contrast {
    color: #888;
    background-color: #eee;
}
```

## Тестирование доступности

### Инструменты для проверки

- **axe DevTools**: Расширение для браузера, которое проверяет доступность
- **WAVE**: Веб-инструмент для оценки доступности
- **Lighthouse**: Интегрированный инструмент в Chrome DevTools

### Проверка с помощью скринридера

Проверьте, как ваш сайт читается с помощью скринридера, например:
- NVDA (для Windows)
- VoiceOver (для macOS)
- TalkBack (для Android)

## Примеры доступного кода

### Форма с правильной семантикой

```html
<form>
    <fieldset>
        <legend>Личная информация</legend>
        
        <label for="name">Имя:</label>
        <input type="text" id="name" name="name" required>
        
        <label for="email">Email:</label>
        <input type="email" id="email" name="email" required>
        
        <label for="message">Сообщение:</label>
        <textarea id="message" name="message"></textarea>
    </fieldset>
    
    <button type="submit">Отправить</button>
</form>
```

### Интерактивный элемент с доступностью

```html
<details>
    <summary>Часто задаваемые вопросы</summary>
    <div>
        <h3>Вопрос 1</h3>
        <p>Ответ на вопрос 1...</p>
        
        <h3>Вопрос 2</h3>
        <p>Ответ на вопрос 2...</p>
    </div>
</details>
```

## Заключение

Семантическая разметка и доступность тесно связаны. Использование правильных HTML-элементов создает основу для доступного веб-сайта, который может быть использован всеми пользователями, независимо от их возможностей. Это не только этично, но и часто требуется по закону в некоторых юрисдикциях.
