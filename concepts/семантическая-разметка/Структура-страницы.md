# Структура страницы

Структура веб-страницы — это организация и расположение элементов на странице с использованием семантических HTML-тегов. Правильная структура делает веб-сайт более доступным, понятным для поисковых систем и легким для сопровождения.

## Основные компоненты структуры страницы

Правильная структура веб-страницы включает в себя несколько ключевых элементов:

### Общая структура HTML-документа

```html
<!DOCTYPE html>
<html lang="ru">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Заголовок страницы</title>
    <meta name="description" content="Описание страницы">
</head>
<body>
    <!-- Структура основных блоков -->
    <header>
        <!-- Заголовок сайта и навигация -->
    </header>
    
    <nav>
        <!-- Основная навигация -->
    </nav>
    
    <main>
        <!-- Основное содержимое -->
    </main>
    
    <aside>
        <!-- Дополнительная информация -->
    </aside>
    
    <footer>
        <!-- Нижний колонтитул -->
    </footer>
</body>
</html>
```

## Семантические блоки страницы

### `<header>` - заголовок

Элемент `<header>` обычно содержит логотип, навигацию и другие вступительные элементы:

```html
<header>
    <div class="container">
        <h1>Название сайта</h1>
        <p>Слоган или краткое описание</p>
        
        <nav>
            <ul>
                <li><a href="/">Главная</a></li>
                <li><a href="/about">О нас</a></li>
                <li><a href="/services">Услуги</a></li>
                <li><a href="/contact">Контакты</a></li>
            </ul>
        </nav>
    </div>
</header>
```

> [!note] Примечание
> Элемент `<header>` может использоваться не только в начале документа, но и внутри других элементов, таких как `<article>` или `<section>`.

### `<nav>` - навигация

Тег `<nav>` предназначен для основных навигационных блоков:

```html
<nav>
    <ul>
        <li><a href="/" aria-current="page">Главная</a></li>
        <li><a href="/products">Продукты</a></li>
        <li><a href="/services">Услуги</a></li>
        <li><a href="/about">О нас</a></li>
        <li><a href="/contact">Контакты</a></li>
    </ul>
</nav>

<!-- Навигация внутри статьи -->
<nav aria-label="Навигация по статье">
    <ul>
        <li><a href="#introduction">Введение</a></li>
        <li><a href="#main-content">Основное содержимое</a></li>
        <li><a href="#conclusion">Заключение</a></li>
    </ul>
</nav>
```

### `<main>` - основное содержимое

Элемент `<main>` содержит уникальное содержимое страницы:

```html
<main>
    <h1>Заголовок текущей страницы</h1>
    <p>Основное содержимое страницы...</p>
    
    <section>
        <h2>Раздел 1</h2>
        <p>Содержимое раздела 1...</p>
    </section>
    
    <section>
        <h2>Раздел 2</h2>
        <p>Содержимое раздела 2...</p>
    </section>
</main>
```

> [!caution] Ограничение
> На странице должно быть не более одного элемента `<main>`, и он не должен находиться внутри `<article>`, `<aside>`, `<footer>`, `<header>` или `<nav>`.

### `<section>` - логическая группа

Тег `<section>` группирует связанные по смыслу элементы:

```html
<section>
    <h2>Наши услуги</h2>
    <p>Мы предлагаем широкий спектр услуг...</p>
    
    <article>
        <h3>Веб-разработка</h3>
        <p>Создание современных веб-сайтов...</p>
    </article>
    
    <article>
        <h3>Дизайн</h3>
        <p>Создание привлекательного дизайна...</p>
    </article>
</section>
```

### `<article>` - независимый контент

Элемент `<article>` содержит контент, который может существовать независимо:

```html
<article>
    <header>
        <h2>Заголовок статьи</h2>
        <p>Автор: <span>Иван Иванов</span></p>
        <p>Опубликовано: <time datetime="2025-01-15">15 января 2025</time></p>
    </header>
    
    <p>Содержимое статьи...</p>
    
    <footer>
        <p>Теги: <a href="/tag/web">Веб</a>, <a href="/tag/development">Разработка</a></p>
    </footer>
</article>
```

### `<aside>` - сопутствующая информация

Тег `<aside>` содержит информацию, косвенно связанную с основным содержимым:

```html
<aside>
    <h3>Популярные статьи</h3>
    <ul>
        <li><a href="/article1">Статья 1</a></li>
        <li><a href="/article2">Статья 2</a></li>
        <li><a href="/article3">Статья 3</a></li>
    </ul>
    
    <h3>Реклама</h3>
    <div class="ad">
        <!-- Рекламный блок -->
    </div>
</aside>
```

### `<footer>` - нижний колонтитул

Элемент `<footer>` содержит информацию о разделе или документе:

```html
<footer>
    <div class="container">
        <p>&copy; 2025 Все права защищены</p>
        <nav>
            <ul>
                <li><a href="/privacy">Политика конфиденциальности</a></li>
                <li><a href="/terms">Условия использования</a></li>
                <li><a href="/sitemap">Карта сайта</a></li>
            </ul>
        </nav>
        <address>
            Контактная информация: <a href="mailto:info@example.com">info@example.com</a>
        </address>
    </div>
</footer>
```

## Лучшие практики структурирования страницы

### 1. Используйте семантические теги по назначению

```html
<!-- Хорошо: Правильное использование семантических тегов -->
<main>
    <h1>Основной заголовок</h1>
    <section>
        <h2>Раздел</h2>
        <p>Содержимое...</p>
    </section>
</main>

<!-- Плохо: Неправильное использование тегов -->
<div class="main-content">
    <div class="main-title">Основной заголовок</div>
    <div class="section">
        <div class="section-title">Раздел</div>
        <div class="content">Содержимое...</div>
    </div>
</div>
```

### 2. Поддерживайте логическую иерархию заголовков

```html
<!-- Хорошо: Последовательная иерархия -->
<h1>Основной заголовок страницы</h1>

<h2>Раздел 1</h2>
<p>Содержимое раздела 1...</p>

<h3>Подраздел 1.1</h3>
<p>Содержимое подраздела 1.1...</p>

<h2>Раздел 2</h2>
<p>Содержимое раздела 2...</p>
```

### 3. Обеспечьте доступность структуры

```html
<!-- Хорошо: Структура с учетом доступности -->
<nav aria-label="Основная навигация">
    <ul>
        <li><a href="/">Главная</a></li>
        <li><a href="/services">Услуги</a></li>
    </ul>
</nav>

<main>
    <h1>Заголовок текущей страницы</h1>
    <p>Основное содержимое...</p>
</main>

<aside aria-labelledby="sidebar-title">
    <h2 id="sidebar-title">Боковая панель</h2>
    <p>Дополнительная информация...</p>
</aside>
```

### 4. Используйте вспомогательные атрибуты

```html
<!-- Хорошо: Использование ARIA-атрибутов для улучшения структуры -->
<main role="main">
    <h1>Основной заголовок</h1>
    <p>Содержимое страницы...</p>
</main>

<aside role="complementary">
    <h2>Дополнительная информация</h2>
    <p>Сопутствующий контент...</p>
</aside>

<footer role="contentinfo">
    <p>Информация об авторских правах...</p>
</footer>
```

## Проверка структуры страницы

### Инструменты для анализа

- **HTML Validator**: Проверяет валидность HTML-кода
- **WAVE**: Инструмент для проверки доступности
- **axe DevTools**: Проверяет доступность и структуру
- **Chrome DevTools**: Позволяет анализировать DOM-структуру

### Что проверять

1. Правильность использования семантических тегов
2. Иерархию заголовков
3. Соответствие структуры содержимому
4. Наличие основных структурных элементов

## Заключение

Правильная структура веб-страницы с использованием семантических HTML-тегов является фундаментом для создания доступных, SEO-оптимизированных и легко поддерживаемых веб-сайтов. Она помогает поисковым системам, вспомогательным технологиям и разработчикам лучше понимать и взаимодействовать с содержимым страницы.
