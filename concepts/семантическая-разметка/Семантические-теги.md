# Семантические теги

Семантические теги — это HTML-элементы, которые явно описывают свое назначение и тип содержимого для браузера и разработчика. Они позволяют структурировать веб-страницу логически, что улучшает доступность, SEO и поддерживаемость кода.

## Основные семантические теги HTML5

HTML5 представил множество новых семантических тегов, которые позволяют более точно описывать структуру документа:

### `<header>`
Элемент `<header>` представляет собой группу вступительного содержимого или навигационных ссылок. Он может содержать заголовки, логотип, поисковую форму или информацию об авторе.

```html
<header>
    <h1>Название сайта</h1>
    <nav>
        <ul>
            <li><a href="/">Главная</a></li>
            <li><a href="/about">О нас</a></li>
        </ul>
    </nav>
</header>
```

> [!note] Примечание
> Элемент `<header>` может встречаться несколько раз на странице, например, один в начале документа и по одному в каждом `<article>` или `<section>`.

### `<nav>`
Тег `<nav>` определяет навигационные ссылки. Он предназначен для основных блоков навигации.

```html
<nav>
    <ul>
        <li><a href="#services">Услуги</a></li>
        <li><a href="#portfolio">Портфолио</a></li>
        <li><a href="#contact">Контакты</a></li>
    </ul>
</nav>
```

> [!warning] Важно
> Не все группы ссылок должны быть обернуты в `<nav>`. Этот тег рекомендуется использовать только для основных навигационных блоков.

### `<main>`
Элемент `<main>` содержит основное содержимое документа, которое уникально для конкретной страницы и не повторяется на других страницах сайта.

```html
<main>
    <h1>Основной заголовок страницы</h1>
    <p>Это основное содержимое страницы.</p>
</main>
```

> [!caution] Ограничение
> На странице должно быть не более одного элемента `<main>`, и он не должен находиться внутри `<article>`, `<aside>`, `<footer>`, `<header>` или `<nav>`.

### `<article>`
Тег `<article>` представляет собой независимый фрагмент контента, который может быть перепубликован или распространён отдельно.

```html
<article>
    <h2>Новость дня</h2>
    <p>Содержимое новости...</p>
    <footer>
        <p>Автор: Иван Иванов</p>
    </footer>
</article>
```

### `<section>`
Элемент `<section>` представляет собой логическую группу содержимого, обычно с заголовком.

```html
<section>
    <h2>О компании</h2>
    <p>Информация о компании...</p>
</section>
```

> [!tip] Совет
> Используйте `<section>` для логического разделения содержимого, когда у вас есть заголовок для каждой секции.

### `<aside>`
Тег `<aside>` используется для содержимого, которое косвенно связано с основным содержимым.

```html
<aside>
    <h3>Реклама</h3>
    <p>Рекламный блок</p>
</aside>
```

### `<footer>`
Элемент `<footer>` содержит информацию об авторе, копирайты, ссылки на связанные документы или любую другую информацию, относящуюся к нижнему колонтитулу.

```html
<footer>
    <p>&copy; 2025 Все права защищены</p>
    <address>Контактная информация: info@example.com</address>
</footer>
```

## Другие важные семантические теги

### `<figure>` и `<figcaption>`
Эти теги используются для группировки медиа-контента (изображений, диаграмм, видео) с их описанием.

```html
<figure>
    <img src="chart.png" alt="График продаж">
    <figcaption>График продаж за последние 6 месяцев</figcaption>
</figure>
```

### `<time>`
Тег `<time>` используется для представления даты или времени.

```html
<time datetime="2025-01-15">15 января 2025</time>
```

### `<mark>`
Элемент `<mark>` выделяет текст, который может быть релевантным или важным в заданном контексте.

```html
<p>Поиск по запросу "<mark>семантический HTML</mark>" дал 100 результатов.</p>
```

### `<details>` и `<summary>`
Эти теги позволяют создавать интерактивные элементы, которые могут быть развернуты или свернуты.

```html
<details>
    <summary>Часто задаваемые вопросы</summary>
    <p>Здесь находятся ответы на часто задаваемые вопросы...</p>
</details>
```

## Примеры правильного использования

### Структура статьи

```html
<article>
    <header>
        <h1>Заголовок статьи</h1>
        <p>Опубликовано: <time datetime="2025-01-15">15 января 2025</time></p>
    </header>
    
    <section>
        <h2>Введение</h2>
        <p>Введение к статье...</p>
    </section>
    
    <section>
        <h2>Основная часть</h2>
        <p>Основное содержимое статьи...</p>
    </section>
    
    <footer>
        <p>Автор: Иван Иванов</p>
    </footer>
</article>
```

### Структура блога

```html
<main>
    <article>
        <h2>Заголовок первой статьи</h2>
        <p>Содержимое первой статьи...</p>
    </article>
    
    <article>
        <h2>Заголовок второй статьи</h2>
        <p>Содержимое второй статьи...</p>
    </article>
</main>

<aside>
    <h3>Архив</h3>
    <ul>
        <li><a href="#">Январь 2025</a></li>
        <li><a href="#">Декабрь 2024</a></li>
    </ul>
</aside>
```

## Сравнение с несемантическими тегами

| Семантический тег | Несемантический аналог | Назначение |
|------------------|------------------------|------------|
| `<header>` | `<div class="header">` | Заголовок секции или документа |
| `<nav>` | `<div class="nav">` | Навигационные ссылки |
| `<main>` | `<div class="main">` | Основное содержимое |
| `<article>` | `<div class="article">` | Независимый контент |
| `<section>` | `<div class="section">` | Логическая группа контента |
| `<aside>` | `<div class="aside">` | Сопутствующая информация |
| `<footer>` | `<div class="footer">` | Нижний колонтитул |

## Заключение

Использование семантических тегов — это фундаментальный аспект современной веб-разработки. Они не только улучшают структуру документа, но и делают его более доступным и индексируемым. Правильное применение этих тегов способствует созданию более понятного и поддерживаемого кода.
