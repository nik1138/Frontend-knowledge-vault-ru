# Алгоритмы сортировки

Алгоритмы сортировки — это методы упорядочивания элементов в коллекции по определенному критерию. Это одна из фундаментальных задач в компьютерных науках и программировании. [[concepts/algorithms/algorithms|Алгоритмы]]

## Сравнительная таблица сложности

| Алгоритм | Лучший случай | Средний случай | Худший случай | Пространственная сложность | Стабильность |
|----------|---------------|----------------|---------------|-----------------------------|--------------|
| Пузырьковая сортировка | O(n) | O(n²) | O(n²) | O(1) | Да |
| Сортировка выбором | O(n²) | O(n²) | O(n²) | O(1) | Нет |
| Сортировка вставками | O(n) | O(n²) | O(n²) | O(1) | Да |
| Сортировка слиянием | O(n log n) | O(n log n) | O(n log n) | O(n) | Да |
| Быстрая сортировка | O(n log n) | O(n log n) | O(n²) | O(log n) | Нет |
| Пирамидальная сортировка | O(n log n) | O(n log n) | O(n log n) | O(1) | Нет |
| Сортировка подсчетом | O(n+k) | O(n+k) | O(n+k) | O(k) | Да |
| Поразрядная сортировка | O(d(n+k)) | O(d(n+k)) | O(d(n+k)) | O(n+k) | Да |

> [!note] Обозначения
> - n: количество элементов
> - k: диапазон значений
> - d: количество разрядов

## Пузырьковая сортировка (Bubble Sort)

Пузырьковая сортировка — один из самых простых алгоритмов сортировки. Он работает, многократно проходя по списку, сравнивая соседние элементы и меняя их местами, если они находятся в неправильном порядке.

```javascript
function bubbleSort(arr) {
    const n = arr.length;
    for (let i = 0; i < n - 1; i++) {
        for (let j = 0; j < n - i - 1; j++) {
            if (arr[j] > arr[j + 1]) {
                [arr[j], arr[j + 1]] = [arr[j + 1], arr[j]];
            }
        }
    }
    return arr;
}
```

### Когда использовать:
- Когда список очень мал
- Когда простота реализации важнее производительности
- Когда нужно минимальное количество изменений в памяти

### Преимущества:
- Простота реализации
- Не требует дополнительной памяти
- Стабильный

### Недостатки:
- Низкая эффективность O(n²)
- Много обменов элементов

## Сортировка выбором (Selection Sort)

Сортировка выбором работает, находя минимальный элемент в неотсортированной части массива и помещая его в начало.

```javascript
function selectionSort(arr) {
    const n = arr.length;
    for (let i = 0; i < n - 1; i++) {
        let minIdx = i;
        for (let j = i + 1; j < n; j++) {
            if (arr[j] < arr[minIdx]) {
                minIdx = j;
            }
        }
        [arr[i], arr[minIdx]] = [arr[minIdx], arr[i]];
    }
    return arr;
}
```

### Когда использовать:
- Когда размер списка небольшой
- Когда обмен элементов дорогой

### Преимущества:
- Простота реализации
- Мало обменов (O(n))

### Недостатки:
- Всегда O(n²) независимо от входных данных
- Не стабильный

## Сортировка вставками (Insertion Sort)

Сортировка вставками строит конечный отсортированный массив по одному элементу за раз, как люди часто сортируют игральные карты.

```javascript
function insertionSort(arr) {
    for (let i = 1; i < arr.length; i++) {
        let current = arr[i];
        let j = i - 1;
        while (j >= 0 && arr[j] > current) {
            arr[j + 1] = arr[j];
            j--;
        }
        arr[j + 1] = current;
    }
    return arr;
}
```

### Когда использовать:
- Когда список почти отсортирован
- Для малых наборов данных
- Как часть гибридных алгоритмов

### Преимущества:
- Эффективна для малых наборов данных
- Адаптивна (быстрее для частично отсортированных данных)
- Стабильна

### Недостатки:
- O(n²) в худшем случае
- Много сдвигов элементов

## Сортировка слиянием (Merge Sort)

Сортировка слиянием — это алгоритм "разделяй и властвуй", который делит массив на половины, сортирует каждую половину, а затем объединяет их.

```javascript
function mergeSort(arr) {
    if (arr.length <= 1) return arr;
    
    const mid = Math.floor(arr.length / 2);
    const left = mergeSort(arr.slice(0, mid));
    const right = mergeSort(arr.slice(mid));
    
    return merge(left, right);
}

function merge(left, right) {
    let result = [];
    let leftIdx = 0, rightIdx = 0;
    
    while (leftIdx < left.length && rightIdx < right.length) {
        if (left[leftIdx] < right[rightIdx]) {
            result.push(left[leftIdx++]);
        } else {
            result.push(right[rightIdx++]);
        }
    }
    
    return result.concat(left.slice(leftIdx)).concat(right.slice(rightIdx));
}
```

### Когда использовать:
- Когда требуется стабильная сортировка
- Когда важна гарантированная производительность O(n log n)
- Для сортировки связных списков

### Преимущества:
- Гарантированная O(n log n)
- Стабильна
- Предсказуема

### Недостатки:
- Требует O(n) дополнительной памяти
- Медленнее на практике, чем быстрая сортировка

## Быстрая сортировка (Quick Sort)

Быстрая сортировка — это алгоритм "разделяй и властвуй", который выбирает опорный элемент и разделяет массив так, что элементы меньше опорного находятся слева, а больше — справа.

```javascript
function quickSort(arr, low = 0, high = arr.length - 1) {
    if (low < high) {
        const pi = partition(arr, low, high);
        quickSort(arr, low, pi - 1);
        quickSort(arr, pi + 1, high);
    }
    return arr;
}

function partition(arr, low, high) {
    const pivot = arr[high];
    let i = low - 1;
    
    for (let j = low; j < high; j++) {
        if (arr[j] < pivot) {
            i++;
            [arr[i], arr[j]] = [arr[j], arr[i]];
        }
    }
    
    [arr[i + 1], arr[high]] = [arr[high], arr[i + 1]];
    return i + 1;
}
```

### Когда использовать:
- Когда нужна хорошая средняя производительность
- Когда ограничена память
- Для общего использования

### Преимущества:
- Хорошая средняя производительность O(n log n)
- На месте (мало дополнительной памяти)
- Практически эффективна

### Недостатки:
- Худшая производительность O(n²)
- Не стабильна
- Зависит от выбора опорного элемента

## Пирамидальная сортировка (Heap Sort)

Пирамидальная сортировка использует структуру данных куча для сортировки элементов.

```javascript
function heapSort(arr) {
    const n = arr.length;
    
    // Построение кучи
    for (let i = Math.floor(n / 2) - 1; i >= 0; i--) {
        heapify(arr, n, i);
    }
    
    // Извлечение элементов из кучи
    for (let i = n - 1; i > 0; i--) {
        [arr[0], arr[i]] = [arr[i], arr[0]];
        heapify(arr, i, 0);
    }
    
    return arr;
}

function heapify(arr, n, i) {
    let largest = i;
    let left = 2 * i + 1;
    let right = 2 * i + 2;
    
    if (left < n && arr[left] > arr[largest]) {
        largest = left;
    }
    
    if (right < n && arr[right] > arr[largest]) {
        largest = right;
    }
    
    if (largest !== i) {
        [arr[i], arr[largest]] = [arr[largest], arr[i]];
        heapify(arr, n, largest);
    }
}
```

### Когда использовать:
- Когда нужна гарантированная O(n log n)
- Когда важна предсказуемость
- Когда нужна сортировка на месте

### Преимущества:
- Гарантированная O(n log n)
- На месте
- Нечувствительна к входным данным

### Недостатки:
- Не стабильна
- Не адаптивна
- Медленнее быстрой сортировки в среднем случае

## Сортировка подсчетом (Counting Sort)

Сортировка подсчетом — это алгоритм сортировки, который работает за линейное время при определенных условиях, подсчитывая количество вхождений каждого элемента.

```javascript
function countingSort(arr, max) {
    const count = new Array(max + 1).fill(0);
    const output = new Array(arr.length);
    
    // Подсчет количества вхождений
    for (let i = 0; i < arr.length; i++) {
        count[arr[i]]++;
    }
    
    // Изменение count[i] так, что count[i] содержит фактическую
    // позицию этого элемента в выходном массиве
    for (let i = 1; i <= max; i++) {
        count[i] += count[i - 1];
    }
    
    // Построение выходного массива
    for (let i = arr.length - 1; i >= 0; i--) {
        output[count[arr[i]] - 1] = arr[i];
        count[arr[i]]--;
    }
    
    return output;
}
```

### Когда использовать:
- Когда диапазон значений ограничен
- Для сортировки целых чисел
- Когда нужна линейная производительность

### Преимущества:
- O(n+k) производительность
- Стабильна
- Эффективна для малого диапазона

### Недостатки:
- Требует дополнительной памяти
- Неэффективна при большом диапазоне
- Работает только с целыми числами

## Поразрядная сортировка (Radix Sort)

Поразрядная сортировка сортирует числа по разрядам, начиная с младшего разряда к старшему.

```javascript
function radixSort(arr) {
    if (arr.length === 0) return arr;
    
    const max = Math.max(...arr);
    
    for (let exp = 1; Math.floor(max / exp) > 0; exp *= 10) {
        countingSortByDigit(arr, exp);
    }
    
    return arr;
}

function countingSortByDigit(arr, exp) {
    const output = new Array(arr.length);
    const count = new Array(10).fill(0);
    
    for (let i = 0; i < arr.length; i++) {
        const digit = Math.floor(arr[i] / exp) % 10;
        count[digit]++;
    }
    
    for (let i = 1; i < 10; i++) {
        count[i] += count[i - 1];
    }
    
    for (let i = arr.length - 1; i >= 0; i--) {
        const digit = Math.floor(arr[i] / exp) % 10;
        output[count[digit] - 1] = arr[i];
        count[digit]--;
    }
    
    for (let i = 0; i < arr.length; i++) {
        arr[i] = output[i];
    }
}
```

### Когда использовать:
- Для сортировки целых чисел
- Когда числа имеют ограниченное количество разрядов
- Когда нужна линейная производительность

### Преимущества:
- O(d(n+k)) производительность
- Стабильна
- Эффективна для целых чисел

### Недостатки:
- Работает только с целыми числами
- Требует дополнительной памяти
- Неэффективна при большом количестве разрядов

## Стабильность сортировки

Стабильность — важное свойство сортировки. Стабильный алгоритм сохраняет относительный порядок элементов с одинаковыми ключами. Это важно при сортировке сложных структур данных.

## Заключение

Выбор алгоритма сортировки зависит от конкретной задачи:
- Для малых массивов подойдут простые алгоритмы (вставками, выбором)
- Для гарантированной производительности — сортировка слиянием
- Для лучшей средней производительности — быстрая сортировка
- Для специфических случаев — сортировка подсчетом или поразрядная

> [!tip] Совет
> Понимание различных алгоритмов сортировки помогает выбрать наиболее подходящий для конкретной задачи, учитывая ограничения по времени и памяти.
