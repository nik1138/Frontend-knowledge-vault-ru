# Асинхронное программирование

**Асинхронное программирование** — это парадигма программирования, при которой выполнение операций не блокирует основной поток выполнения. В контексте фронтенд-разработки это позволяет выполнять длительные операции (например, сетевые запросы, работу с файлами) без "замораживания" пользовательского интерфейса.

## Основные понятия

В традиционном **синхронном** программировании каждая операция должна завершиться перед тем, как начнется следующая. В асинхронном программировании выполнение может продолжаться, даже если предыдущая операция еще не завершена.

## Преимущества в фронтенд-разработке

- **Плавный пользовательский интерфейс**: UI не "зависает" во время выполнения длительных операций
- **Улучшенная производительность**: возможность выполнять несколько операций параллельно
- **Лучший пользовательский опыт**: интерактивность приложения сохраняется во время загрузки данных

## Основные подходы в JavaScript

### 1. Callback-функции

```javascript
// Пример callback-функции
function fetchData(callback) {
  setTimeout(() => {
    const data = 'Данные загружены';
    callback(data);
  }, 1000);
}

fetchData((result) => {
  console.log(result); // 'Данные загружены' через 1 секунду
});
```

> [!warning] Внимание
> Callback-функции могут привести к "аду callback'ов" при сложных сценариях

### 2. Promise

```javascript
// Пример с Promise
function fetchDataAsync() {
  return new Promise((resolve, reject) => {
    setTimeout(() => {
      const success = Math.random() > 0.2; // 80% шанс успеха
      if (success) {
        resolve('Данные успешно загружены');
      } else {
        reject(new Error('Ошибка загрузки данных'));
      }
    }, 1000);
  });
}

fetchDataAsync()
  .then(result => console.log(result))
  .catch(error => console.error(error));
```

### 3. async/await

```javascript
// Пример с async/await
async function handleData() {
  try {
    const result = await fetchDataAsync();
    console.log(result);
  } catch (error) {
    console.error(error);
  }
}

handleData();
```

## Практическое применение в фронтенде

### Загрузка данных с сервера

```javascript
// Пример загрузки данных с API
async function loadUserData(userId) {
  try {
    const response = await fetch(`/api/users/${userId}`);
    if (!response.ok) {
      throw new Error(`Ошибка: ${response.status}`);
    }
    const userData = await response.json();
    return userData;
  } catch (error) {
    console.error('Ошибка загрузки данных пользователя:', error);
    throw error;
  }
}

// Использование
loadUserData(123).then(userData => {
  // Обновление UI с новыми данными
  document.getElementById('user-name').textContent = userData.name;
});
```

### Работа с событиями

```javascript
// Пример асинхронной обработки события
document.getElementById('load-button').addEventListener('click', async () => {
  const button = event.target;
  button.disabled = true;
  button.textContent = 'Загрузка...';
  
  try {
    const data = await loadUserData(123);
    // Обновление UI с новыми данными
    displayUserData(data);
  } catch (error) {
    showError('Не удалось загрузить данные');
  } finally {
    button.disabled = false;
    button.textContent = 'Загрузить данные';
  }
});
```

## Лучшие практики

1. **Используйте async/await** вместо Promise.then() для лучшей читаемости
2. **Обрабатывайте ошибки** с помощью try/catch
3. **Не забывайте отменять запросы** при размонтировании компонентов
4. **Используйте индикаторы загрузки** для улучшения UX
5. **Избегайте чрезмерного вложенного асинхронного кода**
