# Паттерн Медиатор

## Обзор

Паттерн **Медиатор** (Mediator) - это поведенческий паттерн проектирования, который определяет объект, инкапсулирующий способ взаимодействия между набором объектов. Медиатор способствует слабой связанности системы, избегая прямых ссылок между объектами и позволяя изменять их взаимодействие независимо.

### Назначение

- **Цель**: Централизовать сложные взаимодействия между объектами, делая их более управляемыми.
- **Проблема**: При сильной связанности объектов изменение одного объекта может повлиять на другие, что затрудняет сопровождение кода.
- **Решение**: Введение медиатора, который управляет взаимодействием между объектами.

### Основные компоненты

- **Медиатор (Mediator)**: Интерфейс для общения с компонентами.
- **Конкретный медиатор (Concrete Mediator)**: Реализует интерфейс медиатора и координирует взаимодействие между компонентами.
- **Коллеги (Colleagues)**: Объекты, которые взаимодействуют друг с другом через медиатор.

## Принципы работы

Медиатор выступает в роли посредника, через который проходят все взаимодействия между коллегами. Это позволяет:

- Снизить связанность между компонентами
- Упростить архитектуру системы
- Облегчить тестирование и сопровождение

## Преимущества

- **Снижение связанности**: Объекты не зависят напрямую друг от друга.
- **Централизованное управление**: Логика взаимодействия сосредоточена в одном месте.
- **Легкость модификации**: Изменения в логике взаимодействия не затрагивают сами компоненты.

## Недостатки

- **Центральная точка сбоя**: Медиатор может стать "узким местом" системы.
- **Сложность реализации**: Для простых случаев может быть избыточным.

## Когда использовать

- Когда нужно уменьшить связанность между множеством объектов.
- Когда сложно изменять существующие классы из-за их зависимости от других классов.
- Когда логика взаимодействия между объектами сложна и требует централизованного управления.

## Связь с другими паттернами

## Заключение

Паттерн Медиатор эффективно решает проблему сложных взаимодействий между объектами, обеспечивая гибкость и масштабируемость системы. Он особенно полезен в сложных приложениях, где множество компонентов должны взаимодействовать друг с другом.
