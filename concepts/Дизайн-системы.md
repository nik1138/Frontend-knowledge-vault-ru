# Дизайн-системы

Дизайн-система - это организованный набор компонентов, шаблонов и стандартов, которые используются для создания пользовательских интерфейсов. Это важная часть компонентной архитектуры, обеспечивающая согласованность и масштабируемость пользовательского интерфейса.

## Определение и Цель

Дизайн-система - это единый источник истины для команды разработки и дизайна, которая определяет:
- Визуальные компоненты и их варианты
- Правила их использования
- Принципы и паттерны проектирования
- Документацию и примеры использования

## Основные Элементы Дизайн-Системы

### 1. Атомарные компоненты
- Цвета (палитра)
- Шрифты и типографика
- Пространственные единицы (отступы, размеры)
- Тени и эффекты
- Иконки и изображения

### 2. Молекулы
- Комбинации атомарных компонентов (например, кнопка с иконкой)
- Простые интерактивные элементы (например, инпут с лейблом)

### 3. Организмы
- Сложные компоненты, состоящие из молекул (например, навигационная панель)
- Повторно используемые секции интерфейса

### 4. Шаблоны
- Макеты страниц
- Раскладки
- Паттерны навигации

## Архитектура Компонентов в Дизайн-Системе

### Слои Дизайн-Системы

#### 1. Фундаментальный слой (Foundation)
- Цвета, шрифты, размеры, эффекты
- Дизайн-токены (design tokens)

#### 2. Компонентный слой (Components)
- Атомарные компоненты
- Универсальные компоненты UI

#### 3. Страницы/Шаблоны (Pages/Templates)
- Комбинации компонентов
- Готовые решения для часто используемых сценариев

### Пример реализации компонента Button:

```jsx
// components/atoms/Button/Button.jsx
import React from 'react';
import PropTypes from 'prop-types';
import './Button.css';

const Button = ({ 
  children, 
  variant = 'primary', 
  size = 'medium', 
  disabled = false, 
  onClick 
}) => {
  const buttonClass = `btn btn--${variant} btn--${size} ${
    disabled ? 'btn--disabled' : ''
  }`;

  return (
    <button className={buttonClass} disabled={disabled} onClick={onClick}>
      {children}
    </button>
  );
};

Button.propTypes = {
  children: PropTypes.node.isRequired,
  variant: PropTypes.oneOf(['primary', 'secondary', 'danger', 'outline']),
  size: PropTypes.oneOf(['small', 'medium', 'large']),
  disabled: PropTypes.bool,
  onClick: PropTypes.func
};

export default Button;
```

## Преимущества Дизайн-Системы

1. **Единообразие** - все компоненты выглядят и ведут себя одинаково
2. **Экономия времени** - повторное использование готовых компонентов
3. **Согласованность UX** - единый опыт для пользователей
4. **Масштабируемость** - легкость внесения изменений во всей системе
5. **Снижение когнитивной нагрузки** - команды работают с понятной системой

## Инструменты для Создания Дизайн-Систем

### Storybook
- Популярный инструмент для создания и документирования компонентов
- Позволяет создавать "истории" для каждого компонента
- Поддерживает несколько фреймворков (React, Vue, Angular)

### Figma
- Инструмент для дизайна с поддержкой компонентов
- Позволяет создавать библиотеки компонентов
- Поддерживает варианты компонентов

### Zeroheight
- Инструмент для документирования дизайн-систем
- Создает живые стайлгайды

## Лучшие Практики

### 1. Дизайн-токены
Используйте дизайн-токены для абстрагирования визуальных значений:

```javascript
// tokens/colors.js
export const colors = {
  primary: {
    100: '#e3f2fd',
    500: '#2196f3',
    900: '#0d47a1'
  },
  secondary: {
    500: '#ff4081'
  },
  neutral: {
    50: '#fafafa',
    100: '#f5f5f5',
    900: '#212121'
  }
};
```

### 2. Темизация
Обеспечьте поддержку тем в дизайн-системе:

```jsx
// utils/theme.js
export const theme = {
  colors: {
    primary: tokens.colors.primary[500],
    background: tokens.colors.neutral[50],
    text: tokens.colors.neutral[900]
  },
  spacing: {
    small: tokens.spacing[4],
    medium: tokens.spacing[8],
    large: tokens.spacing[16]
  }
};
```

### 3. Документация
Для каждого компонента создавайте:
- Описание использования
- Примеры кода
- Варианты использования
- Доступность (a11y) рекомендации

### 4. Тестирование
- Тестирование компонентов (unit-тесты)
- Визуальное тестирование (snapshot testing)
- Тестирование доступности

## Миграция к Дизайн-Системе

### 1. Аудит существующих компонентов
- Сбор всех компонентов в приложении
- Классификация и анализ
- Выявление дубликатов и несоответствий

### 2. Создание стандартизированных компонентов
- Разработка компонентов на основе существующих
- Обеспечение обратной совместимости
- Поэтапная замена старых компонентов

### 3. Обновление документации
- Создание руководства по использованию
- Обучение команды
- Внедрение новых процессов
