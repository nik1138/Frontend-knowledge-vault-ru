# Роутинг

## Обзор

**Роутинг** (маршрутизация) — это механизм, который позволяет управлять навигацией между различными представлениями (страницами, компонентами) в веб-приложении без полной перезагрузки страницы. Роутинг особенно важен в **одностраничных приложениях (SPA)**, где пользовательский интерфейс обновляется динамически на основе текущего URL.

## Основные понятия

Роутинг включает в себя:

- **Маршруты (routes)** — определенные пути URL, связанные с определенными компонентами или представлениями
- **Навигацию** — переход между маршрутами
- **Параметры маршрута** — динамические части URL (например, `user/123`)
- **Промежуточное ПО маршрутизации (guards)** — логика, определяющая, может ли пользователь перейти на определенный маршрут

## Типы роутинга

### Клиентский роутинг

Клиентский роутинг обрабатывается на стороне браузера. URL обновляется с использованием `History API` без запроса к серверу. Это стандартный подход для SPA.

### Серверный роутинг

Серверный роутинг обрабатывается на стороне сервера. Каждый URL запрашивает новый HTML-документ. Используется в традиционных веб-приложениях.

## Примеры реализации

### Роутинг в React (React Router)

```jsx
import { BrowserRouter as Router, Routes, Route, Link } from 'react-router-dom';

function App() {
  return (
    <Router>
      <nav>
        <Link to="/">Главная</Link>
        <Link to="/about">О нас</Link>
        <Link to="/users">Пользователи</Link>
      </nav>

      <Routes>
        <Route path="/" element={<Home />} />
        <Route path="/about" element={<About />} />
        <Route path="/users" element={<Users />} />
        <Route path="/users/:id" element={<UserProfile />} />
      </Routes>
    </Router>
  );
}

function Home() {
  return <h2>Главная страница</h2>;
}

function About() {
  return <h2>О нас</h2>;
}

function Users() {
  return <h2>Список пользователей</h2>;
}

function UserProfile({ match }) {
  const userId = match.params.id;
  return <h2>Профиль пользователя {userId}</h2>;
}
```

### Роутинг в Vue.js (Vue Router)

```vue
<template>
  <div id="app">
    <router-link to="/">Главная</router-link>
    <router-link to="/about">О нас</router-link>
    <router-link to="/users">Пользователи</router-link>

    <router-view></router-view>
  </div>
</template>

<script>
import { createRouter, createWebHistory } from 'vue-router'
import Home from './components/Home.vue'
import About from './components/About.vue'
import Users from './components/Users.vue'

const routes = [
  { path: '/', component: Home },
  { path: '/about', component: About },
  { path: '/users', component: Users }
]

const router = createRouter({
  history: createWebHistory(),
  routes
})

export default {
  name: 'App',
  router
}
</script>
```

### Роутинг в Angular

```typescript
import { NgModule } from '@angular/core';
import { RouterModule, Routes } from '@angular/router';
import { HomeComponent } from './home/home.component';
import { AboutComponent } from './about/about.component';

const routes: Routes = [
  { path: '', component: HomeComponent },
  { path: 'about', component: AboutComponent },
  { path: 'users/:id', component: UserProfileComponent }
];

@NgModule({
  imports: [RouterModule.forRoot(routes)],
  exports: [RouterModule]
})
export class AppRoutingModule { }
```

## Лучшие практики

- **Используйте семантически понятные URL** — они улучшают UX и SEO
- **Обрабатывайте ошибки навигации** — показывайте страницу 404 при неправильном маршруте
- **Оптимизируйте загрузку компонентов** — используйте lazy loading для улучшения производительности
- **Защищайте маршруты** — реализуйте авторизацию и аутентификацию для приватных маршрутов
