# Классы и объекты

**Классы и объекты** являются фундаментальными концепциями объектно-ориентированного программирования (ООП). Класс - это шаблон или чертеж для создания объектов, а объект - это экземпляр класса. В контексте фронтенд-разработки понимание этих концепций позволяет создавать более структурированный, поддерживаемый и переиспользуемый код.

## Что такое класс?

Класс в JavaScript и TypeScript - это особый способ определения шаблона для объектов. Он позволяет инкапсулировать данные и методы, которые работают с этими данными, в одной сущности. Классы были добавлены в JavaScript начиная с ES6, хотя под капотом они по-прежнему используют прототипы.

### Основные элементы класса:

- **Свойства (properties)** - данные, которые хранит объект
- **Методы (methods)** - функции, которые работают с этими данными
- **Конструктор (constructor)** - специальный метод, вызываемый при создании экземпляра класса
- **Статические методы** - методы, принадлежащие классу, а не экземпляру

## Что такое объект?

Объект - это экземпляр класса. Когда вы создаете объект, вы используете класс как шаблон для создания конкретного экземпляра с собственными значениями свойств. В JavaScript объекты могут быть созданы как с использованием классов, так и без них (литералы объектов).

## Примеры в JavaScript и TypeScript

### Простой класс в JavaScript

```javascript
class User {
  constructor(name, email) {
    this.name = name;
    this.email = email;
    this.isActive = true;
  }

  // Метод класса
  getInfo() {
    return `Пользователь: ${this.name}, Email: ${this.email}`;
  }

  // Метод для изменения состояния
  deactivate() {
    this.isActive = false;
  }
}

// Создание объекта (экземпляра класса)
const user1 = new User('Алексей', 'alexey@example.com');
console.log(user1.getInfo()); // Пользователь: Алексей, Email: alexey@example.com
user1.deactivate();
console.log(user1.isActive); // false
```

### Класс в TypeScript с типами

```typescript
class Product {
  // Объявление свойств с типами
  private id: number;
  public name: string;
  protected price: number;

  constructor(id: number, name: string, price: number) {
    this.id = id;
    this.name = name;
    this.price = price;
  }

  // Метод с возвращаемым типом
  public getInfo(): string {
    return `Товар: ${this.name}, Цена: ${this.price} руб.`;
  }

  // Метод для изменения цены
  public updatePrice(newPrice: number): void {
    if (newPrice > 0) {
      this.price = newPrice;
    }
  }
}

// Создание экземпляра класса
const laptop = new Product(1, 'Ноутбук', 50000);
console.log(laptop.getInfo()); // Товар: Ноутбук, Цена: 50000 руб.
```

### Использование геттеров и сеттеров

```typescript
class Counter {
  private _count: number = 0;

  // Геттер
  get count(): number {
    return this._count;
  }

  // Сеттер
  set count(value: number) {
    if (value >= 0) {
      this._count = value;
    } else {
      console.error('Значение счетчика не может быть отрицательным');
    }
  }

  increment(): void {
    this._count++;
  }
}

const counter = new Counter();
counter.increment();
console.log(counter.count); // 1
counter.count = 5;
console.log(counter.count); // 5
```

## Практическое применение в фронтенд-разработке

Классы особенно полезны при создании компонентов пользовательского интерфейса, управления состоянием приложения и работы с API. Вот несколько примеров:

### 1. Компонент формы

```typescript
class FormComponent {
  private formElement: HTMLFormElement;
  private inputs: HTMLInputElement[];

  constructor(formId: string) {
    this.formElement = document.getElementById(formId) as HTMLFormElement;
    this.inputs = Array.from(this.formElement.querySelectorAll('input'));
  }

  public validate(): boolean {
    return this.inputs.every(input => input.value.trim() !== '');
  }

  public reset(): void {
    this.formElement.reset();
  }

  public getData(): { [key: string]: string } {
    const data: { [key: string]: string } = {};
    this.inputs.forEach(input => {
      data[input.name] = input.value;
    });
    return data;
  }
}
```

### 2. Сервис для работы с API

```typescript
class ApiService {
  private baseUrl: string;

  constructor(baseUrl: string) {
    this.baseUrl = baseUrl;
  }

  public async getData<T>(endpoint: string): Promise<T> {
    const response = await fetch(`${this.baseUrl}/${endpoint}`);
    if (!response.ok) {
      throw new Error(`Ошибка при запросе: ${response.status}`);
    }
    return response.json();
  }

  public async postData<T>(endpoint: string, data: any): Promise<T> {
    const response = await fetch(`${this.baseUrl}/${endpoint}`, {
      method: 'POST',
      headers: {
        'Content-Type': 'application/json',
      },
      body: JSON.stringify(data),
    });
    if (!response.ok) {
      throw new Error(`Ошибка при отправке данных: ${response.status}`);
    }
    return response.json();
  }
}
```

## Ключевые особенности классов в JavaScript/TypeScript

- **Конструктор** - специальный метод, вызываемый при создании экземпляра класса
- **Наследование** - возможность создания класса на основе другого класса (см. [[Наследование]])
- **Инкапсуляция** - сокрытие внутренней реализации (см. [[Инкапсуляция]])
- **Полиморфизм** - возможность объектов разных классов иметь одинаковый интерфейс (см. [[Полиморфизм]])

## Заключение

Классы и объекты - основа объектно-ориентированного программирования. Они позволяют структурировать код, делать его более читаемым и поддерживаемым. В контексте фронтенд-разработки, особенно при использовании фреймворков, понимание и правильное использование классов помогает создавать более надежные и масштабируемые приложения.

Для более глубокого понимания ООП рекомендуется ознакомиться с другими концепциями: [[Наследование]], [[Инкапсуляция]], [[Полиморфизм]] и [[Абстракция]].
