# Адаптивность

## Обзор

Адаптивность (responsive design) — это подход к веб-дизайну, при котором веб-страницы автоматически адаптируются к размеру экрана устройства, на котором они отображаются. Это позволяет обеспечить оптимальный пользовательский опыт независимо от того, используется ли десктоп, планшет или смартфон.

## Основные принципы адаптивности

### 1. Гибкая сетка (Fluid Grid)

Вместо фиксированных значений в пикселях используются относительные единицы измерения:

```css
/* Вместо фиксированной ширины */
.container {
  width: 960px;
}

/* Используем относительные единицы */
.container {
  width: 100%;
  max-width: 960px;
  margin: 0 auto;
}

/* Используем flexbox для гибкой сетки */
.flex-container {
  display: flex;
  flex-wrap: wrap;
}

.flex-item {
  flex: 1 1 300px; /* grow, shrink, basis */
  margin: 10px;
}
```

### 2. Гибкие изображения и медиа-элементы

Изображения и видео должны масштабироваться в зависимости от размера контейнера:

```css
/* Гибкие изображения */
img, video, iframe {
  max-width: 100%;
  height: auto;
}

/* Адаптивные изображения с сохранением соотношения сторон */
.ratio-container {
  position: relative;
  width: 100%;
  height: 0;
  padding-bottom: 56.25%; /* 16:9 Aspect Ratio */
}

.ratio-container iframe {
  position: absolute;
  top: 0;
  left: 0;
  width: 100%;
  height: 100%;
}
```

### 3. Медиа-запросы (Media Queries)

Медиа-запросы позволяют применять разные стили в зависимости от характеристик устройства:

```css
/* Базовые стили для мобильных устройств */
.grid {
  display: flex;
  flex-direction: column;
}

.item {
  width: 100%;
  padding: 10px;
}

/* Стили для планшетов */
@media (min-width: 768px) {
  .grid {
    flex-direction: row;
    flex-wrap: wrap;
  }
  
  .item {
    width: calc(50% - 20px);
  }
}

/* Стили для десктопов */
@media (min-width: 1024px) {
  .item {
    width: calc(33.333% - 20px);
  }
}

/* Пример медиа-запроса для ориентации экрана */
@media (orientation: landscape) {
  .mobile-nav {
    display: none;
  }
}
```

## Типичные точки останова (Breakpoints)

Общепринятые точки останова для адаптивности:

```css
/* Мобильные устройства (до 767px) */
@media (max-width: 767px) {
  /* Стили для мобильных */
}

/* Планшеты (от 768px до 1023px) */
@media (min-width: 768px) and (max-width: 1023px) {
  /* Стили для планшетов */
}

/* Десктопы (от 1024px) */
@media (min-width: 1024px) {
  /* Стили для десктопов */
}

/* Большие экраны (от 1200px) */
@media (min-width: 1200px) {
  /* Стили для больших экранов */
}
```

## Flexbox и Grid для адаптивности

### Flexbox

```css
/* Адаптивная навигация */
.nav {
  display: flex;
  flex-wrap: wrap;
  justify-content: space-between;
  align-items: center;
}

.nav-item {
  flex: 0 0 auto;
  margin: 5px;
}

/* Адаптивная карточка */
.card-container {
  display: flex;
  flex-direction: column;
}

@media (min-width: 768px) {
  .card-container {
    flex-direction: row;
  }
  
  .card-image {
    flex: 1;
  }
  
  .card-content {
    flex: 2;
  }
}
```

### CSS Grid

```css
/* Адаптивная сетка с использованием Grid */
.grid-container {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(300px, 1fr));
  gap: 20px;
}

/* Адаптивная сетка с фиксированным количеством колонок */
@media (min-width: 768px) {
  .grid-container {
    grid-template-columns: repeat(2, 1fr);
  }
}

@media (min-width: 1024px) {
  .grid-container {
    grid-template-columns: repeat(3, 1fr);
  }
}
```

## Практические рекомендации

### 1. Mobile First подход

Начинайте разработку с мобильных устройств и постепенно добавляйте стили для больших экранов:

```css
/* Стили для мобильных устройств */
.main-content {
  padding: 10px;
  font-size: 16px;
}

/* Добавляем стили для планшетов */
@media (min-width: 768px) {
  .main-content {
    padding: 20px;
    font-size: 18px;
  }
}

/* Добавляем стили для десктопов */
@media (min-width: 1024px) {
  .main-content {
    padding: 30px;
    font-size: 20px;
    max-width: 1200px;
    margin: 0 auto;
  }
}
```

### 2. Использование относительных единиц

```css
/* Используем относительные единицы вместо фиксированных */
.text {
  font-size: 1.2rem;        /* Относительно размера шрифта root элемента */
  padding: 1em;             /* Относительно размера шрифта текущего элемента */
  width: 80%;               /* Относительно ширины родительского элемента */
  margin: 2vw;              /* Относительно ширины окна просмотра */
  height: 30vh;             /* Относительно высоты окна просмотра */
}
```

### 3. Адаптивные шрифты

```css
/* Использование clamp() для адаптивных шрифтов */
h1 {
  font-size: clamp(1.5rem, 4vw, 3rem);
}

/* Адаптивные шрифты с использованием calc() */
.adaptive-text {
  font-size: calc(1rem + 0.5vw);
}
```

## Адаптивность и производительность

### Оптимизация изображений

```html
<!-- Использование srcset для адаптивных изображений -->
<img 
  src="image-400.jpg" 
  srcset="image-400.jpg 400w, 
          image-800.jpg 800w, 
          image-1200.jpg 1200w"
  sizes="(max-width: 768px) 100vw, 
         (max-width: 1024px) 50vw, 
         33vw"
  alt="Описание изображения">
```

### Адаптивные видео

```html
<!-- Адаптивное встраивание видео -->
<div class="video-wrapper">
  <iframe 
    src="https://www.youtube.com/embed/VIDEO_ID" 
    frameborder="0" 
    allowfullscreen>
  </iframe>
</div>

<style>
.video-wrapper {
  position: relative;
  padding-bottom: 56.25%; /* 16:9 Aspect Ratio */
  height: 0;
  overflow: hidden;
}

.video-wrapper iframe {
  position: absolute;
  top: 0;
  left: 0;
  width: 100%;
  height: 100%;
}
</style>
```

## Проверка адаптивности

### Инструменты для тестирования

1. **DevTools** в браузерах (Chrome, Firefox) - режим адаптивности
2. **Responsinator** - онлайн-инструмент для проверки адаптивности
3. **BrowserStack** - тестирование на реальных устройствах

### Практические проверки

- Проверка на разных размерах экрана
- Проверка при повороте устройства
- Проверка с включенным/выключенным масштабированием
- Проверка на различных устройствах (реальных или эмуляторах)

## Заключение

Адаптивность является ключевым аспектом современной веб-разработки. Она обеспечивает доступность контента для пользователей на различных устройствах и улучшает пользовательский опыт. Правильное использование гибких сеток, медиа-запросов и современных CSS-инструментов позволяет создавать эффективные адаптивные решения.
