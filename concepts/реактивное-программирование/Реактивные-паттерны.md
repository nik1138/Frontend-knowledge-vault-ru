# Реактивные паттерны

## Общее понятие

Реактивные паттерны - это архитектурные и программные шаблоны, которые помогают реализовать принципы реактивного программирования в реальных приложениях. Эти паттерны позволяют создавать системы, которые являются отзывчивыми, устойчивыми, гибкими и ориентированными на потоки данных.

## Основные принципы реактивного программирования

1. **Отзывчивость (Responsive)** - система должна отвечать быстро
2. **Устойчивость (Resilient)** - система должна оставаться отзывчивой при сбоях
3. **Гибкость (Elastic)** - система должна эффективно использовать ресурсы при изменении нагрузки
4. **Управляемость (Message Driven)** - система должна использовать асинхронные сообщения для обеспечения слабой связанности

## Ключевые реактивные паттерны

### 1. Паттерн "Наблюдатель" (Observer Pattern)

Паттерн "Наблюдатель" позволяет объекту (наблюдателю) подписываться на изменения другого объекта (субъекта). В реактивном программировании это реализуется через Observable и Observer.

```javascript
import { Observable } from 'rxjs';

class DataSource {
  constructor() {
    this.data$ = new Observable(observer => {
      this.observer = observer;
      // Инициализация источника данных
    });
  }

  updateData(newData) {
    this.observer.next(newData);
  }
}

const dataSource = new DataSource();

// Подписка на изменения данных
dataSource.data$.subscribe(data => {
  console.log('Новые данные:', data);
});
```

### 2. Паттерн "Публикатор-Подписчик" (Pub-Sub)

Паттерн Pub-Sub позволяет компонентам системы общаться через посредника, избегая прямой зависимости. В RxJS это реализуется через Subject и его вариации.

```javascript
import { Subject, BehaviorSubject } from 'rxjs';

// Создание шины событий
const eventBus = new Subject();

// Публикация события
function publishEvent(eventType, data) {
  eventBus.next({ type: eventType, payload: data });
}

// Подписка на события
eventBus.subscribe(event => {
  switch(event.type) {
    case 'USER_LOGIN':
      console.log('Пользователь вошел:', event.payload);
      break;
    case 'USER_LOGOUT':
      console.log('Пользователь вышел:', event.payload);
      break;
  }
});

// Использование BehaviorSubject для хранения состояния
const userState$ = new BehaviorSubject({ isLoggedIn: false, user: null });

// Обновление состояния
function login(user) {
  userState$.next({ isLoggedIn: true, user });
}

function logout() {
  userState$.next({ isLoggedIn: false, user: null });
}
```

### 3. Паттерн "Хранилище" (Store Pattern)

Паттерн Store централизует управление состоянием приложения. Он обеспечивает единый источник правды и позволяет наблюдать за изменениями состояния.

```javascript
import { BehaviorSubject, distinctUntilChanged } from 'rxjs';

class Store {
  constructor(initialState = {}) {
    this.state$ = new BehaviorSubject(initialState);
  }

  getState() {
    return this.state$.getValue();
  }

  updateState(newState) {
    const currentState = this.getState();
    this.state$.next({ ...currentState, ...newState });
  }

  select(selector) {
    return this.state$.pipe(
      map(selector),
      distinctUntilChanged()
    );
  }
}

// Использование Store
const appStore = new Store({
  user: { name: '', email: '' },
  theme: 'light',
  notifications: []
});

// Выбор конкретной части состояния
const user$ = appStore.select(state => state.user);
const theme$ = appStore.select(state => state.theme);

user$.subscribe(user => {
  console.log('Пользователь изменился:', user);
});
```

### 4. Паттерн "Поток данных" (Data Flow Pattern)

Паттерн Data Flow устанавливает направленный поток данных от источника к потребителю, минимизируя мутации состояния.

```javascript
import { fromEvent, combineLatest } from 'rxjs';
import { map, startWith, debounceTime } from 'rxjs/operators';

// Поток ввода пользователя
const input$ = fromEvent(document.getElementById('search'), 'input').pipe(
  map(event => event.target.value),
  debounceTime(300),
  startWith('')
);

// Поток фильтров
const filter$ = fromEvent(document.querySelectorAll('.filter'), 'change').pipe(
  map(event => event.target.value),
  startWith('all')
);

// Комбинирование потоков данных
const searchResults$ = combineLatest([input$, filter$]).pipe(
  map(([query, filter]) => ({ query, filter })),
  switchMap(params => performSearch(params))
);

searchResults$.subscribe(results => {
  updateUI(results);
});
```

### 5. Паттерн "Обработка ошибок" (Error Handling Pattern)

Правильная обработка ошибок в реактивных системах критически важна для устойчивости приложения.

```javascript
import { of, throwError } from 'rxjs';
import { catchError, retry, finalize } from 'rxjs/operators';

// Паттерн с повторными попытками и обработкой ошибок
function apiCallWithErrorHandling() {
  return apiCall().pipe(
    retry(3), // Повторить до 3 раз при ошибке
    catchError(error => {
      console.error('Ошибка API:', error);
      return of({ error: true, message: error.message });
    }),
    finalize(() => {
      // Очистка ресурсов
      hideLoadingSpinner();
    })
  );
}

// Паттерн с резервным вариантом
function apiCallWithFallback() {
  return primaryApiCall().pipe(
    catchError(() => {
      console.log('Первый API не работает, используем резервный');
      return secondaryApiCall();
    })
  );
}
```

## Практические примеры использования

### Реактивное управление формами

```javascript
import { fromEvent, combineLatest } from 'rxjs';
import { map, filter, debounceTime, distinctUntilChanged } from 'rxjs/operators';

class ReactiveForm {
  constructor(formElement) {
    this.form$ = fromEvent(formElement, 'input').pipe(
      map(() => this.getFormData()),
      debounceTime(300),
      distinctUntilChanged((prev, curr) => JSON.stringify(prev) === JSON.stringify(curr))
    );

    this.validation$ = this.form$.pipe(
      map(data => this.validateForm(data)),
      share()
    );

    // Обновление UI при изменениях
    this.validation$.subscribe(validation => {
      this.updateValidationUI(validation);
      this.toggleSubmitButton(validation.isValid);
    });
  }

  getFormData() {
    // Сбор данных формы
    return {
      email: document.getElementById('email').value,
      password: document.getElementById('password').value
    };
  }

  validateForm(data) {
    return {
      isValid: this.isValidEmail(data.email) && data.password.length >= 8,
      errors: {
        email: !this.isValidEmail(data.email) ? 'Неверный email' : null,
        password: data.password.length < 8 ? 'Пароль слишком короткий' : null
      }
    };
  }
}
```

### Реактивная загрузка данных

```javascript
import { BehaviorSubject, switchMap, startWith, catchError } from 'rxjs';
import { map, tap } from 'rxjs/operators';

class ReactiveDataLoader {
  constructor() {
    this.refresh$ = new BehaviorSubject(null);
    this.loading$ = new BehaviorSubject(false);
  }

  loadData() {
    return this.refresh$.pipe(
      tap(() => this.loading$.next(true)),
      switchMap(() => this.fetchData()),
      map(data => ({ data, error: null })),
      catchError(error => of({ data: null, error: error.message })),
      tap(() => this.loading$.next(false))
    );
  }

  refresh() {
    this.refresh$.next(Date.now());
  }

  fetchData() {
    return fetch('/api/data').then(response => response.json());
  }
}
```

## Практические рекомендации

1. **Используйте подходящие Subject'ы** - понимайте разницу между Subject, BehaviorSubject, ReplaySubject и AsyncSubject
2. **Управляйте жизненным циклом подписок** - всегда отписывайтесь от потоков, чтобы избежать утечек памяти
3. **Декомпозируйте сложные потоки** - разбивайте сложные цепочки на более простые и понятные части

> [!tip] Совет
> Используйте оператор share() или shareReplay() для мультиподписки на один и тот же поток, чтобы избежать дублирования операций.

## Заключение

Реактивные паттерны предоставляют проверенные решения для распространенных задач в реактивном программировании. Их правильное применение позволяет создавать надежные, масштабируемые и легко поддерживаемые приложения, которые эффективно реагируют на изменения и взаимодействуют с асинхронными источниками данных.
