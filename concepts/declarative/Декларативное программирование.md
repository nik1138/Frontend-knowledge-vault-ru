# Декларативное программирование

Декларативное программирование — это парадигма программирования, при которой программа описывает *что* должно быть вычислено, а не *как* это сделать. В декларативном программировании акцент делается на логике и желаемом результате, а не на последовательности шагов для достижения этого результата.

## Основные характеристики

### Описание желаемого результата
Программа специфицирует, что должно быть вычислено, а не как это сделать.

### Минимизация изменяемого состояния
Декларативные языки стремятся минимизировать или полностью избегать изменяемого состояния.

### Высокоуровневые абстракции
Используются абстракции, которые скрывают детали реализации.

### Фокус на логике, а не на потоке управления
Основное внимание уделяется логике решения задачи, а не последовательности выполнения операций.

## Примеры на различных языках

### SQL
```sql
-- Декларативный подход
SELECT name, age 
FROM users 
WHERE age > 18 
ORDER BY age DESC;
```

### HTML
```html
<!-- Декларативный подход -->
<div class="container">
  <h1 class="title">Заголовок страницы</h1>
  <p class="description">Описание страницы</p>
</div>
```

### JavaScript (функциональный стиль)
```javascript
// Декларативный подход
const numbers = [1, 2, 3, 4, 5];
const doubled = numbers.map(x => x * 2);
const evens = numbers.filter(x => x % 2 === 0);
const sum = numbers.reduce((acc, x) => acc + x, 0);
```

### React (JSX)
```jsx
// Декларативный подход
function UserList({ users }) {
  return (
    <ul>
      {users.map(user => (
        <li key={user.id}>
          {user.name} - {user.email}
        </li>
      ))}
    </ul>
  );
}
```

## Преимущества декларативного программирования

### Более высокий уровень абстракции
Декларативный подход позволяет сосредоточиться на логике, а не на деталях реализации.

### Легкость понимания логики
Программы легче читать и понимать, так как описывают *что* нужно сделать.

### Лучшая поддержка параллелизма
Минимизация изменяемого состояния упрощает параллельное выполнение кода.

### Меньше склонность к ошибкам
Из-за отсутствия изменяемого состояния меньше возможностей для ошибок.

### Повторное использование
Декларативные конструкции часто более универсальны и переиспользуемы.

## Недостатки декларативного программирования

### Меньший контроль над выполнением
Разработчик меньше контролирует, как именно выполняется программа.

### Потенциально менее эффективное выполнение
Абстракции могут привести к менее эффективному коду по сравнению с императивным подходом.

### Сложность для некоторых задач
Некоторые задачи сложнее выразить в декларативной форме.

### Кривая обучения
Для разработчиков, привыкших к императивному стилю, декларативный подход может быть сложнее для освоения.

## Декларативное программирование в контексте других парадигм

### Отличие от императивного программирования
В то время как императивное программирование описывает *как* достичь результата, декларативное программирование описывает *что* должно быть вычислено.

```javascript
// Императивный подход
const numbers = [1, 2, 3, 4, 5];
const evenNumbers = [];
for (let i = 0; i < numbers.length; i++) {
  if (numbers[i] % 2 === 0) {
    evenNumbers.push(numbers[i]);
  }
}

// Декларативный подход
const numbers = [1, 2, 3, 4, 5];
const evenNumbers = numbers.filter(n => n % 2 === 0);
```

### Сочетание с функциональным программированием
Декларативное программирование тесно связано с функциональным программированием, которое является одной из его форм.

### Сочетание с логическим программированием
Логическое программирование (например, Prolog) является формой декларативного программирования.

## Практическое применение

### Реляционные базы данных
SQL является классическим примером декларативного языка программирования.

### Пользовательские интерфейсы
HTML, CSS и современные фреймворки для UI (React, Vue) используют декларативный подход.

### Конфигурационные файлы
YAML, JSON и другие форматы конфигурации являются декларативными.

### Регулярные выражения
Регулярные выражения описывают *что* искать, а не *как* искать.

## Сравнение подходов

### Обработка данных
```javascript
// Императивный подход
function getAdultUsers(users) {
  const result = [];
  for (let i = 0; i < users.length; i++) {
    if (users[i].age >= 18) {
      result.push(users[i]);
    }
  }
  return result;
}

// Декларативный подход
const getAdultUsers = users => users.filter(user => user.age >= 18);
```

### Стилизация
```css
/* Декларативный подход */
.button {
  background-color: blue;
  color: white;
  padding: 10px 20px;
  border: none;
  border-radius: 4px;
}
```

## Применение в современных технологиях

### В [[js]]
JavaScript поддерживает декларативное программирование через:
- Методы массивов (map, filter, reduce)
- Промисы и async/await
- Декларативные библиотеки (React, Vue)

### В [[ts]]
TypeScript расширяет возможности декларативного программирования в JavaScript:
- Интерфейсы для описания структуры данных
- Дженерики для создания универсальных функций
- Типы для декларативного описания ограничений

### В [[html]]
HTML является декларативным языком разметки, описывающим структуру документа.

### В [[css]]
CSS является декларативным языком стилей, описывающим оформление документа.
