# Доступность веб-контента

## Обзор

Доступность веб-контента (Web Accessibility) - это практика разработки веб-сайтов и приложений таким образом, чтобы они были доступны для людей с ограниченными возможностями. Это включает в себя людей с нарушениями зрения, слуха, моторики, когнитивных способностей и других ограничений.

Доступность не только помогает людям с ограниченными возможностями, но и улучшает общий пользовательский опыт, делает сайт более удобным для всех пользователей и повышает SEO.

## Основные принципы доступности (WCAG)

Стандарты доступности определяются [[WCAG 2.1]] (Web Content Accessibility Guidelines), которые включают четыре основных принципа:

- **Воспринимаемость** (Perceivable) - информация должна быть доступна для восприятия всеми пользователями
- **Управляемость** (Operable) - интерфейс должен быть удобен для управления
- **Понятность** (Understandable) - информация и работа интерфейса должны быть понятны
- **Надежность** (Robust) - контент должен быть надежным для различных вспомогательных технологий

## Ключевые аспекты доступности

### Семантическая разметка

Использование правильных HTML-элементов помогает вспомогательным технологиям понимать структуру страницы:

```html
<!-- Хорошо: семантическая разметка -->
<header>
  <nav>
    <ul>
      <li><a href="/">Главная</a></li>
      <li><a href="/about">О нас</a></li>
    </ul>
  </nav>
</header>

<main>
  <article>
    <h1>Заголовок статьи</h1>
    <p>Содержимое статьи...</p>
  </article>
</main>

<aside>
  <section>
    <h2>Боковая панель</h2>
    <p>Дополнительная информация...</p>
  </section>
</aside>

<footer>
  <p>© 2025 Все права защищены</p>
</footer>
```

```html
<!-- Плохо: использование div для всего -->
<div>
  <div>
    <div>
      <div><a href="/">Главная</a></div>
      <div><a href="/about">О нас</a></div>
    </div>
  </div>
</div>

<div>
  <div>
    <div>Заголовок статьи</div>
    <div>Содержимое статьи...</div>
  </div>
</div>
```

### Атрибуты доступности

#### ARIA (Accessible Rich Internet Applications)

ARIA-атрибуты предоставляют дополнительную информацию о роли, состоянии и свойствах элементов:

```html
<!-- Кнопка с ARIA-описанием -->
<button 
  aria-label="Закрыть модальное окно"
  onclick="closeModal()">
  ✕
</button>

<!-- Вспомогательный текст для скринридера -->
<span class="sr-only">Заголовок формы</span>
<input type="text" aria-describedby="help-text" />

<div id="help-text" class="help-text">
  Введите ваше имя в формате Имя Фамилия
</div>

<!-- Роль и состояние для кастомного компонента -->
<div 
  role="tablist" 
  aria-label="Панель вкладок">
  <button 
    role="tab" 
    aria-selected="true" 
    aria-controls="panel-1">
    Вкладка 1
  </button>
  <div 
    role="tabpanel" 
    id="panel-1" 
    hidden>
    Содержимое первой вкладки
  </div>
</div>
```

### Контрастность цветов

Цветовой контраст между текстом и фоном должен соответствовать минимальным требованиям WCAG:

```css
/* Хороший контраст для основного текста */
.text-primary {
  color: #000000; /* Темный текст */
  background-color: #ffffff; /* Белый фон */
  /* Коэффициент контрастности: 21:1 */
}

/* Минимальный контраст для крупного текста (3:1) */
.text-large {
  color: #767676; /* Светло-серый */
  background-color: #ffffff; /* Белый фон */
  font-size: 1.5em;
  /* Коэффициент контрастности: 4.5:1 */
}

/* Плохо: низкий контраст */
.low-contrast {
  color: #cccccc; /* Светло-серый */
  background-color: #ffffff; /* Белый фон */
  /* Коэффициент контрастности: 1.6:1 - НЕДОСТАТОЧНО */
}
```

### Клавиатурная навигация

Все интерактивные элементы должны быть доступны через клавиатуру:

```html
<!-- Правильная последовательность табуляции -->
<nav>
  <a href="/">Главная</a>
  <a href="/about">О нас</a>
  <a href="/contact">Контакты</a>
</nav>

<button onclick="doSomething()">Действие</button>
<input type="text" placeholder="Введите текст" />
<select>
  <option value="1">Опция 1</option>
  <option value="2">Опция 2</option>
</select>

<!-- Кастомный компонент с клавиатурной поддержкой -->
<div 
  tabindex="0" 
  role="button" 
  aria-pressed="false"
  onkeydown="handleKeyDown(event)"
  onclick="handleClick()">
  Кастомная кнопка
</div>

<script>
function handleKeyDown(event) {
  // Обработка клавиш пробела и Enter для активации
  if (event.key === 'Enter' || event.key === ' ') {
    event.preventDefault();
    handleClick();
  }
}
</script>
```

### Фокус управления

Управление фокусом особенно важно для динамических интерфейсов:

```javascript
// Перенаправление фокуса при открытии модального окна
function openModal() {
  const modal = document.getElementById('modal');
  modal.style.display = 'block';
  
  // Установка фокуса на первый интерактивный элемент
  const firstFocusable = modal.querySelector('input, button, select, textarea, [tabindex]:not([tabindex="-1"])');
  if (firstFocusable) {
    firstFocusable.focus();
  }
}

// Ограничение фокуса внутри модального окна
function trapFocus(event) {
  const modal = document.getElementById('modal');
  const focusableElements = modal.querySelectorAll('input, button, select, textarea, [tabindex]:not([tabindex="-1"])');
  const firstElement = focusableElements[0];
  const lastElement = focusableElements[focusableElements.length - 1];

  if (event.key === 'Tab') {
    if (event.shiftKey) {
      if (document.activeElement === firstElement) {
        lastElement.focus();
        event.preventDefault();
      }
    } else {
      if (document.activeElement === lastElement) {
        firstElement.focus();
        event.preventDefault();
      }
    }
  }
}
```

### Для разработчиков

1. **Используйте семантические HTML-элементы** - [[HTML семантика]]
2. **Добавляйте альтернативный текст для изображений** - [[Альтернативный текст]]
3. **Обеспечивайте достаточный контраст цветов** - [[Цветовой контраст]]

5. **Тестируйте с вспомогательными технологиями** - [[Тестирование доступности]]

### Для дизайнеров

1. **Проектируйте с учетом доступности с самого начала**
2. **Используйте понятные иконки и символы**
3. **Предоставляйте визуальные индикаторы фокуса**
4. **Рассматривайте различные состояния элементов (hover, focus, active)**

## Заключение

Доступность - это не опциональная функция, а основной аспект хорошего веб-дизайна и разработки. Интеграция принципов доступности с самого начала процесса разработки делает веб-контент более инклюзивным и улучшает опыт для всех пользователей.

> [!tip] Совет
> Регулярно тестируйте ваш сайт с помощью скринридеров и инструментов автоматического анализа. Доступность - это непрерывный процесс, а не разовое мероприятие.

> [!warning] Важно
> Доступность может быть требованием по закону в некоторых юрисдикциях. Несоблюдение стандартов доступности может привести к юридическим последствиям и репутационным потерям.
