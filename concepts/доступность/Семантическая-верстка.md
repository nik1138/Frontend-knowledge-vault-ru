# Семантическая верстка

## Обзор

Семантическая верстка - это подход к созданию HTML-разметки, при котором используются элементы, соответствующие их смысловому содержанию и функциональности. Правильное использование семантических HTML-элементов критически важно для доступности, так как позволяет вспомогательным технологиям (например, скринридерам) правильно интерпретировать и представлять контент пользователям с ограниченными возможностями.

## Основные семантические элементы

### 1. Структурные элементы

#### `<header>`
Представляет вводную информацию для раздела или документа.

```html
<header>
  <h1>Заголовок сайта</h1>
  <nav>
    <ul>
      <li><a href="/">Главная</a></li>
      <li><a href="/about">О нас</a></li>
    </ul>
  </nav>
</header>
```

#### `<nav>`
Определяет навигационные ссылки.

```html
<nav aria-label="Основная навигация">
  <ul>
    <li><a href="/">Главная</a></li>
    <li><a href="/products">Продукты</a></li>
    <li><a href="/contact">Контакты</a></li>
  </ul>
</nav>

<!-- Для вспомогательных навигаций -->
<nav aria-label="Навигация по разделам">
  <ol>
    <li><a href="#section1">Раздел 1</a></li>
    <li><a href="#section2">Раздел 2</a></li>
  </ol>
</nav>
```

#### `<main>`
Представляет основное содержимое страницы, уникальное для конкретной страницы.

```html
<main>
  <article>
    <h1>Основная статья</h1>
    <p>Содержимое статьи...</p>
  </article>
</main>
```

#### `<section>`
Группирует связанные по смыслу элементы.

```html
<section>
  <h2>О компании</h2>
  <p>Информация о компании...</p>
</section>

<section>
  <h2>Наши услуги</h2>
  <p>Описание услуг...</p>
</section>
```

#### `<article>`
Представляет самостоятельный, завершенный фрагмент контента.

```html
<article>
  <header>
    <h2>Заголовок статьи</h2>
    <time datetime="2023-11-23">23 ноября 2023</time>
  </header>
  <p>Содержимое статьи...</p>
  <footer>
    <p>Автор: Иван Иванов</p>
  </footer>
</article>
```

#### `<aside>`
Содержит контент, косвенно связанный с основным содержимым.

```html
<aside>
  <h3>Связанные статьи</h3>
  <ul>
    <li><a href="/related1">Связанная статья 1</a></li>
    <li><a href="/related2">Связанная статья 2</a></li>
  </ul>
</aside>
```

#### `<footer>`
Представляет нижний колонтитул раздела или документа.

```html
<footer>
  <p>&copy; 2023 Компания. Все права защищены.</p>
  <address>
    <a href="mailto:info@example.com">info@example.com</a>
  </address>
</footer>
```

### 2. Элементы для текстового содержимого

#### Заголовки (`<h1>` - `<h6>`)
Используются для создания иерархии заголовков.

```html
<!-- Правильная иерархия заголовков -->
<main>
  <h1>Основной заголовок страницы</h1>
  
  <section>
    <h2>Раздел 1</h2>
    <p>Содержимое раздела 1...</p>
    
    <h3>Подраздел 1.1</h3>
    <p>Содержимое подраздела 1.1...</p>
  </section>
  
  <section>
    <h2>Раздел 2</h2>
    <p>Содержимое раздела 2...</p>
  </section>
</main>
```

#### `<p>`, `<br>`, `<hr>`
Для параграфов, разрывов строк и разделителей.

```html
<p>Это первый параграф с <br> разрывом строки внутри.</p>

<p>Это второй параграф.</p>

<hr>

<p>Это третий параграф после разделителя.</p>
```

#### `<blockquote>`, `<q>`, `<cite>`
Для цитат и указания источников.

```html
<blockquote cite="https://example.com/source">
  <p>Это длинная цитата, которая занимает несколько строк и может содержать несколько параграфов.</p>
  <footer>— <cite>Имя автора, <a href="https://example.com/source">Название источника</a></cite></footer>
</blockquote>

<p>Короткая цитата: <q>Вот так выглядит короткая цитата</q> в пределах обычного текста.</p>
```

#### `<figure>` и `<figcaption>`
Для изображений с подписями.

```html
<figure>
  <img src="chart.png" alt="Диаграмма роста продаж за последние 5 лет">
  <figcaption>Рис. 1: Рост продаж с 2018 по 2023 год</figcaption>
</figure>
```

### 3. Элементы для списков

#### `<ul>`, `<ol>`, `<li>`
Для неупорядоченных и упорядоченных списков.

```html
<ul>
  <li>Первый элемент списка</li>
  <li>Второй элемент списка
    <ul>
      <li>Вложенный элемент</li>
      <li>Еще один вложенный элемент</li>
    </ul>
  </li>
</ul>

<ol>
  <li>Первый шаг</li>
  <li>Второй шаг</li>
  <li>Третий шаг</li>
</ol>
```

#### `<dl>`, `<dt>`, `<dd>`
Для списков определений.

```html
<dl>
  <dt>Доступность</dt>
  <dd>Способность веб-сайта быть использованным всеми людьми, включая тех, кто имеет ограниченные возможности.</dd>
  
  <dt>Семантика</dt>
  <dd>Смысловое значение HTML-элементов, которое помогает вспомогательным технологиям понимать структуру документа.</dd>
</dl>
```

### 4. Элементы для встраивания контента

#### `<img>` с атрибутом `alt`

```html
<!-- Изображение как иллюстрация -->
<img src="sunset.jpg" alt="Закат над океаном с пальмами на переднем плане">

<!-- Декоративное изображение -->
<img src="decoration.png" alt="" role="presentation">

<!-- Изображение как часть предложения -->
<p>На картинке <img src="logo.png" alt="Логотип компании"> видно наш логотип.</p>
```

#### `<iframe>` с атрибутом `title`

```html
<iframe src="map.html" 
        title="Карта с местоположением офиса компании"
        width="600" 
        height="400">
</iframe>
```

## Практические рекомендации

### 1. Правильная иерархия заголовков

> [!tip] Совет
> Используйте заголовки последовательно, без пропусков уровней. Это помогает пользователям скринридеров навигировать по странице.

```html
<!-- Хорошо: последовательная иерархия -->
<h1>Основной заголовок</h1>
<h2>Подзаголовок</h2>
<h3>Раздел</h3>
<h4>Подраздел</h4>

<!-- Плохо: пропуск уровней -->
<h1>Основной заголовок</h1>
<h3>Подзаголовок</h3> <!-- Пропущен h2 -->
<h5>Раздел</h5> <!-- Пропущен h4 -->
```

### 2. Альтернативный текст для изображений

```html
<!-- Информативные изображения -->
<img src="chart.png" alt="Диаграмма показывает увеличение пользователей на 25% за последний квартал">

<!-- Декоративные изображения -->
<img src="decoration.png" alt="" role="presentation">

<!-- Функциональные изображения (в ссылках или кнопках) -->
<a href="/product">
  <img src="product-image.png" alt="Умный телефон XYZ-1000">
</a>

<button onclick="deleteItem()">
  <img src="delete-icon.png" alt="Удалить">
</button>
```

### 3. Семантические элементы для таблиц

```html
<table>
  <caption>Продажи по кварталам 2023 года</caption>
  <thead>
    <tr>
      <th scope="col">Квартал</th>
      <th scope="col">Продажи</th>
      <th scope="col">Прибыль</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <th scope="row">Q1</th>
      <td>100,000</td>
      <td>25,000</td>
    </tr>
    <tr>
      <th scope="row">Q2</th>
      <td>120,000</td>
      <td>30,000</td>
    </tr>
  </tbody>
</table>
```

### 4. Правильное использование форм

```html
<form action="/submit" method="post">
  <fieldset>
    <legend>Информация о пользователе</legend>
    
    <label for="name">Имя:</label>
    <input type="text" id="name" name="name" required>
    
    <label for="email">Email:</label>
    <input type="email" id="email" name="email" required>
    
    <label for="message">Сообщение:</label>
    <textarea id="message" name="message"></textarea>
    
    <button type="submit">Отправить</button>
  </fieldset>
</form>
```

## Проверка семантической структуры

### Инструменты для проверки

1. **Скринридеры** (NVDA, JAWS, VoiceOver)
2. **Расширения браузера** (axe, WAVE)
3. **Разработческие инструменты** - проверка DOM-структуры

### Ручная проверка

1. Проверьте иерархию заголовков
2. Убедитесь, что все изображения имеют подходящий `alt`-текст
3. Проверьте правильное использование элементов формы
4. Убедитесь, что навигация логична и последовательна

## Пример комплексной семантической разметки

```html
<!DOCTYPE html>
<html lang="ru">
<head>
  <meta charset="UTF-8">
  <title>Пример семантической разметки</title>
</head>
<body>
  <header role="banner">
    <h1>Мой веб-сайт</h1>
    <nav aria-label="Основная навигация">
      <ul>
        <li><a href="/">Главная</a></li>
        <li><a href="/about">О нас</a></li>
        <li><a href="/contact">Контакты</a></li>
      </ul>
    </nav>
  </header>

  <main>
    <article>
      <header>
        <h1>Интересная статья</h1>
        <p>Опубликовано: <time datetime="2023-11-23">23 ноября 2023</time></p>
      </header>
      
      <section>
        <h2>Введение</h2>
        <p>Вступительный абзац статьи...</p>
      </section>
      
      <section>
        <h2>Основная часть</h2>
        <p>Основное содержимое статьи...</p>
        
        <figure>
          <img src="diagram.png" alt="Диаграмма показывает статистику роста">
          <figcaption>Рис. 1: Статистика роста за последние 5 лет</figcaption>
        </figure>
      </section>
      
      <footer>
        <p>Автор: <address>Иван Иванов</address></p>
      </footer>
    </article>
    
    <aside>
      <h2>Связанные материалы</h2>
      <ul>
        <li><a href="/related1">Связанная статья 1</a></li>
        <li><a href="/related2">Связанная статья 2</a></li>
      </ul>
    </aside>
  </main>

  <footer role="contentinfo">
    <p>&copy; 2023 Компания. Все права защищены.</p>
    <nav aria-label="Дополнительная навигация">
      <ul>
        <li><a href="/privacy">Политика конфиденциальности</a></li>
        <li><a href="/terms">Условия использования</a></li>
      </ul>
    </nav>
  </footer>
</body>
</html>
```

## Заключение

Семантическая верстка является фундаментом доступного веб-контента. Правильное использование семантических HTML-элементов не только улучшает доступность для пользователей с ограниченными возможностями, но и улучшает SEO, поддерживаемость кода и общее качество веб-сайта.

См. также:
- [[Стандарты-WCAG]]
- [[ARIA-атрибуты]]
- [[Доступные-компоненты]]

> [!warning] Важно
> Никогда не используйте семантические элементы только для стилизации. Если вам нужно стилизовать элемент, используйте CSS-классы, а не семантические элементы, которые не соответствуют содержимому.
