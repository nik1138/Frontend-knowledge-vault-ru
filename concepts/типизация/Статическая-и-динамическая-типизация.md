# Статическая и динамическая типизация

## Определение

**Статическая типизация** — это система типов, в которой типы переменных проверяются на этапе компиляции, до выполнения программы. Типы переменных определяются во время написания кода и не могут изменяться в процессе выполнения.

**Динамическая типизация** — это система типов, в которой типы переменных проверяются во время выполнения программы. Тип переменной определяется в момент присвоения значения и может изменяться в процессе выполнения.

## Сравнение

| Характеристика | Статическая типизация | Динамическая типизация |
|----------------|----------------------|------------------------|
| Проверка типов | На этапе компиляции | Во время выполнения |
| Ошибки типов | Обнаруживаются до запуска | Обнаруживаются при запуске |
| Производительность | Выше (меньше проверок во время выполнения) | Ниже (проверки во время выполнения) |
| Гибкость | Ниже | Выше |
| Отладка | Проще | Сложнее |
| Объём кода | Больше (нужны аннотации типов) | Меньше |

## Примеры в JavaScript и TypeScript

### JavaScript (динамическая типизация):
```javascript
let userName = "Алексей";  // строка
console.log(userName);     // "Алексей"

userName = 42;             // теперь число
console.log(userName);     // 42

userName = true;           // теперь булевое значение
console.log(userName);     // true
```

В JavaScript переменная `userName` может принимать значения разных типов, и это не вызовет ошибки на этапе написания кода. Ошибки могут возникнуть только во время выполнения, если мы попытаемся использовать значение не по назначению.

### TypeScript (статическая типизация):
```typescript
let userName: string = "Алексей";  // явно указываем, что это строка
console.log(userName);             // "Алексей"

// userName = 42;                  // ОШИБКА: Type 'number' is not assignable to type 'string'
```

В TypeScript при попытке присвоить переменной другого типа значение мы получим ошибку на этапе компиляции, до запуска программы.

## Преимущества статической типизации

> [!tip] Преимущества статической типизации
> - **Раннее обнаружение ошибок**: Ошибки типов обнаруживаются до запуска программы
> - **Улучшенная автодополнение**: IDE может лучше понимать структуру кода
> - **Более безопасный рефакторинг**: При изменении интерфейсов компилятор подскажет, где нужно внести изменения
> - **Более понятный код**: Типы служат документацией для других разработчиков

## Преимущества динамической типизации

> [!tip] Преимущества динамической типизации
> - **Гибкость**: Быстрое прототипирование и эксперименты
> - **Меньше кода**: Не нужно писать аннотации типов
> - **Выразительность**: Некоторые паттерны легче реализовать без строгой системы типов

## Примеры для фронтенд-разработчиков

### Пример с API-запросом в JavaScript (без типов):
```javascript
// Без типизации
function displayUser(userData) {
  document.getElementById('name').textContent = userData.name; // Ошибка, если нет поля name
  document.getElementById('age').textContent = userData.age;
  document.getElementById('email').textContent = userData.email;
}

// Вызов функции с неправильными данными
displayUser({ username: 'Алексей', years: 25, mail: 'alex@example.com' }); // Ошибки не будет до выполнения
```

### То же в TypeScript:
```typescript
interface User {
  name: string;
  age: number;
  email: string;
}

function displayUser(userData: User) {
  document.getElementById('name').textContent = userData.name;
  document.getElementById('age').textContent = userData.age.toString();
  document.getElementById('email').textContent = userData.email;
}

// displayUser({ username: 'Алексей', years: 25, mail: 'alex@example.com' }); 
// ОШИБКА: Property 'name', 'age', 'email' are missing in type
```

## Вывод

Для фронтенд-разработчиков, особенно при работе с крупными приложениями, статическая типизация (например, через TypeScript) может значительно улучшить качество кода, снизить количество багов и упростить сопровождение проекта. Однако для небольших проектов или прототипов динамическая типизация может быть более подходящей из-за своей гибкости.
