# REST API

REST (Representational State Transfer) - это архитектурный стиль для разработки распределенных систем, особенно веб-сервисов. REST API использует стандартные HTTP методы (GET, POST, PUT, DELETE и др.) для взаимодействия с ресурсами.

## Основные принципы REST

- **Stateless** - каждый запрос от клиента к серверу должен содержать всю информацию, необходимую для обработки запроса
- **Кэширование** - ответы должны быть кэшируемыми или не кэшируемыми
- **Единообразие интерфейса** - единый интерфейс между компонентами системы
- **Клиент-серверная архитектура** - разделение между клиентом и сервером
- **Слоистая система** - клиент может не знать, взаимодействует ли он с конечным сервером или промежуточным агентом

## HTTP методы в REST

- **GET** - для получения данных
- **POST** - для создания новых ресурсов
- **PUT** - для обновления существующих ресурсов
- **DELETE** - для удаления ресурсов
- **PATCH** - для частичного обновления ресурсов

## Преимущества REST API

- Простота и понятность
- Масштабируемость
- Поддержка кэширования
- Независимость от языка программирования и платформы

## Сравнение с другими архитектурами

REST API часто сравнивают с [[GraphQL]], [[SOAP]] и [[gRPC]], каждая из которых имеет свои особенности и области применения.
