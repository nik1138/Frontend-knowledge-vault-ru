# GraphQL

GraphQL - это язык запросов для API и серверный runtime для выполнения этих запросов с использованием типа системы, которую вы определяете для своих данных. GraphQL был разработан Facebook и позволяет клиентам запрашивать только те данные, которые им необходимы.

## Основные особенности

- **Точная выборка данных** - клиенты могут запрашивать только те поля, которые им нужны
- **Одиночный endpoint** - в отличие от REST, GraphQL обычно использует один endpoint для всех операций
- **Сильная типизация** - GraphQL имеет встроенную систему типов для ваших данных
- **Интроспекция** - возможность запросить схему API для получения информации о доступных операциях

## Типы операций

- **Query** - для получения данных (аналог GET в REST)
- **Mutation** - для изменения данных (аналог POST, PUT, DELETE в REST)
- **Subscription** - для получения обновлений в реальном времени

## Преимущества GraphQL

- Устранение проблемы "over-fetching" и "under-fetching" данных
- Автоматическая документация API через схему
- Гибкость в запросах данных
- Мощные инструменты разработки (например, GraphiQL)

## Недостатки и ограничения

- Сложность кэширования по сравнению с REST
- Сложность реализации аутентификации и авторизации
- Не все инструменты и библиотеки поддерживают GraphQL
- Сложность реализации ограничения скорости запросов

## GraphQL vs REST

GraphQL и [[REST API]] являются альтернативными подходами к созданию API, каждый из которых имеет свои сценарии использования.
