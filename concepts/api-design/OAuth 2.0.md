# OAuth 2.0

OAuth 2.0 (Open Authorization) - это протокол авторизации, который позволяет приложениям получать ограниченный доступ к пользовательским аккаунтам на HTTP-сервисе. OAuth 2.0 широко используется для аутентификации и авторизации в API.

## Основные компоненты

- **Resource Owner** - пользователь, владеющий ресурсами
- **Client** - приложение, запрашивающее доступ к ресурсам
- **Resource Server** - сервер, защищающий ресурсы
- **Authorization Server** - сервер, проверяющий подлинность владельца ресурса и выдающий токены

## Типы грантов OAuth 2.0

- **Authorization Code** - наиболее распространенный тип, используемый для веб-приложений
- **Implicit** - используется для клиентских приложений, выполняемых в браузере
- **Resource Owner Password Credentials** - для доверенных приложений
- **Client Credentials** - для машинной аутентификации

## Токены

OAuth 2.0 использует токены доступа (access tokens) для авторизации запросов. Эти токены обычно имеют ограниченное время жизни и могут быть обновлены с помощью refresh tokens.

## Безопасность

OAuth 2.0 требует HTTPS для защиты передаваемых токенов. Также важна правильная обработка и хранение токенов на стороне клиента.

## Сравнение с JWT

OAuth 2.0 и [[JWT]] часто используются вместе, но это разные технологии: OAuth 2.0 - это протокол авторизации, а JWT - формат токенов.
