# gRPC

gRPC - это высокопроизводительный RPC (Remote Procedure Call) фреймворк, разработанный Google. gRPC использует HTTP/2 для транспорта, Protocol Buffers (protobuf) в качестве интерфейсного определения языка и обеспечивает межъязыковую поддержку.

## Основные особенности

- **Использование HTTP/2** - обеспечивает мультиплексирование запросов и двунаправленную передачу данных
- **Protocol Buffers** - эффективный механизм сериализации данных, более компактный, чем JSON
- **Кроссплатформенность** - поддержка более чем 10 языков программирования
- **Типизация** - строгая типизация через определения сервисов в .proto файлах

## Типы gRPC вызовов

- **Unary RPC** - клиент отправляет один запрос, сервер возвращает один ответ
- **Server streaming RPC** - клиент отправляет запрос, сервер возвращает поток ответов
- **Client streaming RPC** - клиент отправляет поток запросов, сервер возвращает один ответ
- **Bidirectional streaming RPC** - клиент и сервер обмениваются потоками сообщений

## Преимущества gRPC

- **Высокая производительность** - бинарная сериализация и эффективное использование сети
- **Компактные сообщения** - Protocol Buffers более компактны, чем JSON
- **Генерация кода** - автоматическая генерация клиентского и серверного кода
- **Поддержка потоковой передачи** - встроенная поддержка потоковой передачи данных

## Недостатки gRPC

- **Сложность отладки** - бинарный формат затрудняет отладку по сравнению с текстовыми форматами
- **Ограниченная поддержка браузерами** - требует дополнительных инструментов для работы в браузере
- **Кривая обучения** - необходимость изучения Protocol Buffers и концепции сервисов

## Сравнение с REST и GraphQL

gRPC часто сравнивают с [[REST API]] и [[GraphQL]], особенно в контексте микросервисной архитектуры.
