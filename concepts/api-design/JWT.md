# JWT (JSON Web Token)

JWT (JSON Web Token) - это открытый стандарт (RFC 7519), который определяет компактный и независимый способ передачи информации между сторонами в виде JSON-объекта. JWT часто используется для аутентификации и авторизации в веб-приложениях и API.

## Структура JWT

JWT состоит из трех частей, разделенных точками:

- **Header** - содержит информацию о типе токена и алгоритме шифрования
- **Payload** - содержит утверждения (claims) - информацию о пользователе и другие данные
- **Signature** - используется для проверки целостности токена

Пример: `header.payload.signature`

## Типы утверждений (Claims)

- **Registered claims** - предопределенные утверждения (iss, exp, sub, aud и др.)
- **Public claims** - общедоступные утверждения, определенные в IANA
- **Private claims** - пользовательские утверждения, согласованные между сторонами

## Преимущества JWT

- Компактность - JWT содержит всю необходимую информацию
- Самодостаточность - не требует хранения состояния на сервере
- Гибкость - может содержать произвольные данные

## Недостатки JWT

- Невозможность отзыва токена до истечения срока действия
- Потенциальные проблемы с безопасностью при неправильной реализации
- Размер токена может быть значительным при большом количестве данных

## Безопасность JWT

JWT должны быть защищены с использованием надежных алгоритмов шифрования. Также важно правильно управлять сроком действия токенов и использовать HTTPS.

## Сравнение с OAuth 2.0

JWT и [[OAuth 2.0]] часто используются совместно: OAuth 2.0 определяет протокол авторизации, а JWT - формат токенов, используемых в этом протоколе.
