# SOLID Principles

## Определение
SOLID Principles — это пять основных принципов объектно-ориентированного программирования и проектирования, сформулированные Робертом Мартином. Эти принципы помогают создавать более гибкие, сопровождаемые и расширяемые системы.

## Пять принципов SOLID

### 1. Принцип единственной ответственности (SRP - Single Responsibility Principle)
Каждый класс должен иметь только одну причину для изменения, т.е. выполнять только одну функцию.

### 2. Принцип открытости/закрытости (OCP - Open/Closed Principle)
Программные сущности должны быть открыты для расширения, но закрыты для модификации.

### 3. Принцип подстановки Барбары Лисков (LSP - Liskov Substitution Principle)
Объекты в программе должны быть заменяемыми экземплярами их подтипов без нарушения корректности программы.

### 4. Принцип разделения интерфейса (ISP - Interface Segregation Principle)
Много специализированных интерфейсов лучше, чем один универсальный.

### 5. Принцип инверсии зависимостей (DIP - Dependency Inversion Principle)
Зависимости должны строиться на абстракциях, а не на деталях.

## Применение в различных языках
- В [[ts]] и [[js]] принципы SOLID реализуются особенно эффективно благодаря системе типов
- В других языках применяются с учетом специфики языка

## Преимущества
- Повышенная сопровождаемость кода
- Лучшая тестируемость
- Снижение зацепления между компонентами
- Более гибкая архитектура
