# Кодирование на доске

## Обзор

Кодирование на доске (whiteboard coding) - это форма технического собеседования, при которой кандидат решает алгоритмические задачи на доске или в блокноте без использования компьютера. Несмотря на то, что в 2025 году многие компании переходят к онлайн-платформам, умение решать задачи на доске остается важным навыком, особенно в крупных российских IT-компаниях.

## Подготовка к кодированию на доске

### Психологическая подготовка

**Преодоление стресса:**
- Практика перед зеркалом
- Репетиция с друзьями
- Дыхательные техники
- Понимание, что ошибки - часть процесса

> [!tip] Совет
> В российских компаниях особенно ценится умение рассуждать вслух и объяснять процесс решения. Не молчите во время решения задачи!

### Техническая подготовка

**Основные темы для повторения:**
- [[Алгоритмы-и-структуры-данных]]
- Сортировка и поиск
- Динамическое программирование
- Графовые алгоритмы
- Работа со строками

**Языки программирования:**

- Популярные выборы: Java, Python, C++, JavaScript
- Знайте стандартные библиотеки и структуры данных

## Структура решения задачи на доске

### 1. Понимание задачи (2-3 минуты)

**Шаги:**
- Внимательно прочитайте задачу
- Задайте уточняющие вопросы:
  - Какие ограничения на входные данные?
  - Какие граничные случаи возможны?
  - Какие типы данных используются?

**Пример:**
```
Задача: Найти два числа в массиве, сумма которых равна заданному числу.
Вопросы:
- Могут ли быть отрицательные числа?
- Может ли быть несколько решений?
- Нужно ли вернуть индексы или значения?
```

### 2. Обсуждение подхода (3-5 минут)

**Анализ сложности:**
- Временная сложность
- Пространственная сложность
- Возможные оптимизации

**Обсуждение нескольких решений:**
- Наивное решение O(n²)
- Оптимизированное решение O(n) с использованием хеш-таблицы

### 3. Псевдокод (2-3 минуты)

**Запись логики до написания кода:**
```
1. Создать хеш-таблицу для хранения чисел
2. Пройти по массиву
3. Для каждого числа вычислить complement = target - current_number
4. Если complement есть в хеш-таблице, вернуть индексы
5. Иначе добавить current_number в хеш-таблицу
```

### 4. Написание кода (5-10 минут)

**Пример реализации на Python:**
```python
def two_sum(nums, target):
    """
    Найти два числа в массиве, сумма которых равна target
    Возвращает индексы этих чисел
    """
    num_to_index = {}
    
    for i, num in enumerate(nums):
        complement = target - num
        if complement in num_to_index:
            return [num_to_index[complement], i]
        num_to_index[num] = i
    
    return []  # Если решения нет
```

### 5. Тестирование (2-3 минуты)

**Тестирование с примерами:**
- Обычный случай: [2, 7, 11, 15], target = 9 → [0, 1]
- Граничный случай: [3, 3], target = 6 → [0, 1]
- Отрицательный случай: [1, 2, 3], target = 7 → []

## Популярные категории задач

### Массивы и строки

**Задачи:**
- Двухуказательный метод
- Подмассивы с заданной суммой
- Палиндромы
- Анаграммы
- Поиск подстроки

**Пример задачи:**
```
Задача: Найти самую длинную подстроку без повторяющихся символов
Решение: Sliding window с использованием хеш-таблицы
Сложность: O(n)
```

### Деревья и графы

**Задачи:**
- Обход деревьев (DFS, BFS)
- Поиск в бинарном дереве поиска
- Топологическая сортировка
- Кратчайший путь

**Пример задачи:**
```
Задача: Проверить, является ли бинарное дерево BST
Решение: Рекурсивная проверка границ для каждого узла
Сложность: O(n)
```

### Динамическое программирование

**Задачи:**
- Максимальная подпоследовательность
- Задача о рюкзаке
- Путь в сетке
- Редакционное расстояние

**Пример задачи:**
```
Задача: Максимальная сумма непересекающихся подмассивов длины k
Решение: DP с предварительным вычислением сумм
Сложность: O(n)
```

## Частые ошибки и как их избежать

### Технические ошибки

1. **Забытые граничные случаи**
   - Пустой массив
   - Один элемент
   - Все элементы одинаковы

2. **Неправильная сложность**
   - Неоценка временной сложности
   - Игнорирование пространственной сложности

3. **Ошибки в реализации**
   - Офсеты (off-by-one)
   - Неправильные условия остановки
   - Неправильная инициализация

### Коммуникативные ошибки

1. **Молчание во время решения**
2. **Слишком быстрое написание кода без обсуждения**
3. **Отказ от обсуждения альтернатив**
4. **Нежелание принимать подсказки**

> [!warning] Важно
> В российских компаниях часто оценивают не только правильность решения, но и способность кандидата объяснять код на русском языке. Практикуйтесь в объяснении алгоритмов и решений на родном языке.

## Примеры задач и решений

### Задача 1: Сортировка цветов (Sort Colors)

**Условие:** Дан массив с элементами 0, 1, 2, представляющими красный, белый и синий цвета. Отсортировать массив так, чтобы элементы одного цвета были рядом, в порядке красный, белый, синий.

**Решение:**
```python
def sort_colors(nums):
    """
    Сортировка массива с тремя значениями за один проход
    Используем трехуказательный подход (Dutch National Flag)
    """
    left, current = 0, 0
    right = len(nums) - 1
    
    while current <= right:
        if nums[current] == 0:
            nums[left], nums[current] = nums[current], nums[left]
            left += 1
            current += 1
        elif nums[current] == 1:
            current += 1
        else:  # nums[current] == 2
            nums[current], nums[right] = nums[right], nums[current]
            right -= 1
            # Не увеличиваем current, потому что мы не знаем, 
            # что получили после обмена с right
```

**Сложность:** O(n) по времени, O(1) по памяти

### Задача 2: Сумма двух чисел II (Two Sum II)

**Условие:** Дан отсортированный массив и целевое число. Найти два числа, сумма которых равна целевому, и вернуть их индексы (1-индексация).

**Решение:**
```python
def two_sum_sorted(nums, target):
    """
    Используем двухуказательный метод на отсортированном массиве
    """
    left, right = 0, len(nums) - 1
    
    while left < right:
        current_sum = nums[left] + nums[right]
        if current_sum == target:
            return [left + 1, right + 1]  # 1-индексация
        elif current_sum < target:
            left += 1
        else:
            right -= 1
    
    return []  # Решения нет
```

**Сложность:** O(n) по времени, O(1) по памяти

## Заключение

Кодирование на доске требует не только технических знаний, но и коммуникативных навыков. Успех на таком собеседовании зависит от:

1. **Технической подготовки** - знание структур данных и алгоритмов
2. **Практики** - регулярное решение задач
3. **Коммуникации** - умение объяснять процесс решения
4. **Управления стрессом** - спокойствие и уверенность

Помните, что даже в 2025 году, несмотря на развитие онлайн-платформ, многие российские компании продолжают использовать доску как способ оценки мышления кандидата в реальном времени.
