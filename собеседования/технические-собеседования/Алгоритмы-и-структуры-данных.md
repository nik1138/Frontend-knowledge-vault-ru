# Алгоритмы и структуры данных

## Обзор

Алгоритмы и структуры данных - это фундаментальные понятия программирования, которые играют ключевую роль в технических собеседованиях. Знание различных структур данных и алгоритмов позволяет эффективно решать задачи и оптимизировать производительность программного обеспечения.

## Основные структуры данных

### Массивы и строки

**Массивы** - это упорядоченные коллекции элементов, доступ к которым осуществляется по индексу. Важные особенности:

- Время доступа: O(1)
- Время вставки/удаления: O(n) в среднем случае
- Популярные задачи: [[Двухуказательный метод]], [[Сортировка]], [[Поиск подстроки]]

**Строки** - это специальный тип массива, состоящий из символов. Важные методы работы со строками включают:

- Работа с подстроками
- Регулярные выражения
- Хеширование строк
- Палиндромы и анаграммы

> [!tip] Совет
> Знание различных алгоритмов работы со строками (например, KMP, Рабин-Карп) может быть полезным для решения сложных задач на собеседованиях.

### Связные списки

**Односвязный список** состоит из узлов, каждый из которых содержит данные и указатель на следующий узел.

**Двусвязный список** дополнительно содержит указатель на предыдущий узел.

**Основные операции:**
- Вставка: O(1) при наличии указателя
- Удаление: O(1) при наличии указателя
- Поиск: O(n)

**Часто встречающиеся задачи:**
- Разворот списка
- Обнаружение цикла (алгоритм Флойда "черепаха и заяц")
- Нахождение среднего элемента
- Удаление дубликатов

### Стеки и очереди

**Стек (Stack)** - структура данных, работающая по принципу LIFO (Last In, First Out).

**Основные операции:**
- push(x) - добавление элемента
- pop() - удаление последнего элемента
- peek()/top() - просмотр последнего элемента

**Применение:**
- Обработка скобочных выражений
- Обратная польская нотация
- Рекурсия и вызовы функций

**Очередь (Queue)** - структура данных, работающая по принципу FIFO (First In, First Out).

**Двусторонняя очередь (Deque)** - позволяет добавлять и удалять элементы с обеих сторон.

### Хеш-таблицы

**Хеш-таблица** - структура данных, обеспечивающая быстрый доступ к данным по ключу.

**Время выполнения:**
- Поиск: O(1) в среднем случае
- Вставка: O(1) в среднем случае
- Удаление: O(1) в среднем случае

**Применение:**
- Подсчет частоты элементов
- Проверка на дубликаты
- Кеширование результатов

### Деревья

**Бинарное дерево** - структура данных, в которой каждый узел имеет не более двух дочерних узлов.

**Бинарное дерево поиска (BST)** - особый тип бинарного дерева, в котором левый потомок меньше родителя, а правый - больше.

**Обходы дерева:**
- Прямой (pre-order): корень → левый → правый
- Центрированный (in-order): левый → корень → правый
- Концевой (post-order): левый → правый → корень

**Специальные деревья:**
- AVL-деревья
- Красно-черные деревья
- Кучи (Min-heap и Max-heap)

## Основные алгоритмы

### Сортировка

| Алгоритм | Средняя сложность | Лучшая сложность | Худшая сложность | Стабильность |
|----------|-------------------|------------------|------------------|--------------|
| Пузырьковая | O(n²) | O(n) | O(n²) | Да |
| Слиянием | O(n log n) | O(n log n) | O(n log n) | Да |
| Быстрая | O(n log n) | O(n log n) | O(n²) | Нет |
| Кучей | O(n log n) | O(n log n) | O(n log n) | Нет |

### Поиск

**Линейный поиск** - O(n)
**Бинарный поиск** - O(log n) для отсортированного массива

### Графовые алгоритмы

**Обход в глубину (DFS)** - использует стек или рекурсию
**Обход в ширину (BFS)** - использует очередь

**Алгоритмы на графах:**
- Поиск в топологическом порядке
- Поиск компонент сильной связанности
- Алгоритм Дейкстры
- Алгоритм Флойда-Уоршелла
- MST (минимальное остовное дерево)
