# Конфигурация

Файл `tsconfig.json` - это конфигурационный файл TypeScript, который определяет параметры компиляции для проекта. Он указывает компилятору TypeScript, какие файлы нужно включить в компиляцию, какие параметры использовать и как обрабатывать различные аспекты кода.

## Создание tsconfig.json

Для создания базового файла конфигурации выполните команду в корне проекта:

```bash
tsc --init
```

Это создаст файл `tsconfig.json` с часто используемыми настройками и комментариями к ним.

## Структура файла конфигурации

Пример базовой конфигурации:

```json
{
  "compilerOptions": {
    /* Основные параметры */
    "target": "ES2020",
    "module": "commonjs",
    "lib": ["ES2020"],
    "outDir": "./dist",
    "rootDir": "./src",
    
    /* Проверка типов */
    "strict": true,
    "noImplicitAny": true,
    "strictNullChecks": true,
    
    /* Дополнительные проверки */
    "noUnusedLocals": true,
    "noUnusedParameters": true,
    "noImplicitReturns": true,
    "noFallthroughCasesInSwitch": true,
    
    /* Модули */
    "moduleResolution": "node",
    "baseUrl": "./",
    "esModuleInterop": true,
    "allowSyntheticDefaultImports": true,
    
    /* Выходные файлы */
    "sourceMap": true,
    "declaration": true,
    "declarationMap": true
  },
  "include": [
    "src/**/*"
  ],
  "exclude": [
    "node_modules",
    "**/*.spec.ts"
  ]
}
```

## Основные параметры компиляции

### target
Определяет версию ECMAScript, в которую будет транспилирован код:

```json
{
  "compilerOptions": {
    "target": "ES2020"  // или "ES5", "ES2015", "ES2016", "ES2017", "ES2018", "ES2019", "ES2020", "ES2021", "ESNext"
  }
}
```

### module
Определяет систему модулей для генерируемого JavaScript:

```json
{
  "compilerOptions": {
    "module": "commonjs"  // или "amd", "system", "umd", "es2015", "es2020", "esnext"
  }
}
```

### lib
Указывает библиотеки, которые будут включены в компиляцию:

```json
{
  "compilerOptions": {
    "lib": ["ES2020", "DOM", "DOM.Iterable"]
  }
}
```

### outDir и rootDir
Определяют папки для вывода и источника:

```json
{
  "compilerOptions": {
    "outDir": "./dist",
    "rootDir": "./src"
  }
}
```

## Параметры строгой проверки типов

### strict
Включает все строгие проверки:

```json
{
  "compilerOptions": {
    "strict": true
  }
}
```

### noImplicitAny
Запрещает неявное использование типа `any`:

```json
{
  "compilerOptions": {
    "noImplicitAny": true
  }
}
```

### strictNullChecks
Включает строгую проверку null и undefined:

```json
{
  "compilerOptions": {
    "strictNullChecks": true
  }
}
```

## Параметры модулей

### moduleResolution
Определяет стратегию разрешения модулей:

```json
{
  "compilerOptions": {
    "moduleResolution": "node"  // или "classic"
  }
}
```

### baseUrl
Базовый путь для разрешения неабсолютных имен модулей:

```json
{
  "compilerOptions": {
    "baseUrl": "./"
  }
}
```

### paths
Псевдонимы для путей к модулям:

```json
{
  "compilerOptions": {
    "baseUrl": ".",
    "paths": {
      "@src/*": ["src/*"],
      "@components/*": ["src/components/*"],
      "@utils/*": ["src/utils/*"]
    }
  }
}
```

## Параметры вывода

### sourceMap
Генерирует source map файлы для отладки:

```json
{
  "compilerOptions": {
    "sourceMap": true
  }
}
```

### declaration
Генерирует файлы определений (.d.ts):

```json
{
  "compilerOptions": {
    "declaration": true
  }
}
```

## Секции include, exclude и files

### include
Указывает файлы или паттерны для включения:

```json
{
  "include": [
    "src/**/*",
    "types/**/*"
  ]
}
```

### exclude
Указывает файлы или паттерны для исключения:

```json
{
  "exclude": [
    "node_modules",
    "dist",
    "**/*.spec.ts"
  ]
}
```

### files
Явно указывает файлы для включения:

```json
{
  "files": [
    "src/main.ts",
    "src/utils.ts"
  ]
}
```

## Пример продвинутой конфигурации

```json
{
  "compilerOptions": {
    /* Базовые параметры */
    "target": "ES2020",
    "module": "ESNext",
    "lib": ["ES2020", "DOM", "DOM.Iterable"],
    "allowJs": false,
    "skipLibCheck": true,
    "esModuleInterop": true,
    "allowSyntheticDefaultImports": true,
    "strict": true,
    "forceConsistentCasingInFileNames": true,
    "noFallthroughCasesInSwitch": true,
    "moduleResolution": "node",
    "resolveJsonModule": true,
    "isolatedModules": true,
    "noEmit": false,
    
    /* Параметры вывода */
    "outDir": "./dist",
    "rootDir": "./src",
    "removeComments": true,
    "sourceMap": true,
    
    /* Проверки типов */
    "strictNullChecks": true,
    "strictFunctionTypes": true,
    "noImplicitReturns": true,
    "noImplicitOverride": true,
    "noUnusedLocals": true,
    "noUnusedParameters": true,
    
    /* Дополнительные параметры */
    "incremental": true,
    "tsBuildInfoFile": "./build/tsbuildinfo",
    "composite": true,
    
    /* Псевдонимы путей */
    "baseUrl": ".",
    "paths": {
      "@/*": ["src/*"],
      "@components/*": ["src/components/*"],
      "@utils/*": ["src/utils/*"],
      "@assets/*": ["src/assets/*"]
    }
  },
  "include": [
    "src/**/*"
  ],
  "exclude": [
    "node_modules",
    "dist",
    "**/*.test.ts",
    "**/*.spec.ts"
  ]
}
```

## Наследование конфигурации

Вы можете наследовать конфигурацию из другого файла:

```json
{
  "extends": "./base-config.json",
  "compilerOptions": {
    "outDir": "./custom-dist"
  }
}
```

## Компиляция с разными конфигурациями

Для разных целей (например, разработка и продакшн) можно использовать разные файлы конфигурации:

- `tsconfig.json` - основная конфигурация
- `tsconfig.dev.json` - конфигурация для разработки
- `tsconfig.prod.json` - конфигурация для продакшена
- `tsconfig.node.json` - конфигурация для серверного кода

Пример наследования:

```json
// tsconfig.prod.json
{
  "extends": "./tsconfig.json",
  "compilerOptions": {
    "removeComments": true,
    "sourceMap": false
  }
}
```

## Заключение

Правильная настройка файла `tsconfig.json` - ключ к эффективной работе с TypeScript. Он позволяет настроить процесс компиляции под конкретные нужды проекта, обеспечивая баланс между производительностью, безопасностью типов и удобством разработки.
