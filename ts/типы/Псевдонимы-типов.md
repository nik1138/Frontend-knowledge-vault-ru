# Псевдонимы типов (Type Aliases)

Псевдонимы типов в TypeScript позволяют создавать новые имена для типов. Это удобный способ создания синонимов для сложных или часто используемых типов, что улучшает читаемость кода и позволяет повторно использовать определения типов.

## Основы

Псевдоним типа создается с помощью ключевого слова `type`, за которым следует имя псевдонима, знак равно и определение типа:

```ts
type Name = string;
type Age = number;
type Person = {
  name: Name;
  age: Age;
};
```

## Примеры использования

### 1. Простые псевдонимы

```ts
type ID = string | number;
type Callback = () => void;
type StringOrNumberArray = (string | number)[];
```

### 2. Псевдонимы для сложных объектов

```ts
type Address = {
  street: string;
  city: string;
  zipCode: string;
};

type User = {
  id: ID;
  name: string;
  email: string;
  address: Address;
};
```

### 3. Псевдонимы с дженериками

```ts
type ApiResponse<T> = {
  status: number;
  data: T;
  message: string;
};

const userResponse: ApiResponse<User> = {
  status: 200,
  data: { id: 1, name: "John", email: "john@example.com" },
  message: "Success"
};
```

## Преимущества использования псевдонимов

- **Читаемость**: Улучшает понимание кода, особенно при работе со сложными типами
- **Повторное использование**: Позволяет определять типы один раз и использовать их в нескольких местах
- **Поддержка**: Облегчает внесение изменений в определения типов, так как изменения нужно вносить только в одном месте
- **Документация**: Псевдонимы могут служить формой документации, объясняя назначение типов

## Сравнение с интерфейсами

Псевдонимы типов часто сравнивают с интерфейсами [[Интерфейсы]]. В то время как оба могут использоваться для определения формата объектов, у них есть различия:

- Псевдонимы могут представлять любые типы (примитивы, объединения, пересечения и т.д.), а интерфейсы主要用于 для объектов
- Интерфейсы поддерживают наследование и расширение, тогда как псевдонимы нет (хотя можно использовать пересечения типов [[Пересечения-типов]])

## Практические советы

- Используйте осмысленные имена для псевдонимов типов
- Рассмотрите использование псевдонимов для сложных типов, чтобы улучшить читаемость
- Избегайте слишком вложенных или сложных псевдонимов, которые могут усложнить понимание кода
- Псевдонимы особенно полезны при работе с API-ответами и сложными структурами данных
