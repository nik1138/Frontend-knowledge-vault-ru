# Рефлексия в TypeScript

## Обзор

Рефлексия — это способность программы исследовать свои собственные структуры во время выполнения. В TypeScript, как и в JavaScript, рефлексия реализована через объект `Reflect` и встроенные возможности языка для интроспекции объектов, классов и функций.

## Основы

TypeScript предоставляет объект `Reflect`, который содержит статические методы для операций, аналогичных операциям, выполняемым в прототипах объектов. Эти методы могут быть использованы для выполнения операций с объектами и их свойствами.

## Примеры использования

### Получение свойств объекта

```typescript
const obj = { name: 'Alice', age: 30 };

// Получение всех ключей объекта
const keys = Reflect.ownKeys(obj);
console.log(keys); // ['name', 'age']

// Проверка наличия свойства
console.log(Reflect.has(obj, 'name')); // true
```

### Работа с методами объекта

```typescript
class Person {
  name: string;

  constructor(name: string) {
    this.name = name;
  }

  greet() {
    return `Hello, my name is ${this.name}`;
  }
}

const person = new Person('Bob');

// Вызов метода через Reflect
const greeting = Reflect.apply(person.greet, person, []);
console.log(greeting); // 'Hello, my name is Bob'
```

## Практическое применение

Рефлексия особенно полезна при создании фреймворков, библиотек для валидации, DI-контейнеров и других инструментов, требующих анализа структуры объектов во время выполнения.
