# Экспорт-и-импорт

## Обзор

Экспорт и импорт являются основными механизмами в TypeScript (и JavaScript), позволяющими создавать модульные приложения. Они позволяют разделять код на логические части, упрощая его поддержку, тестирование и повторное использование. В этом документе рассматриваются различные способы экспорта и импорта, включая именованный экспорт, экспорт по умолчанию и деструктуризацию при импорте.

## Именованный экспорт

Именованный экспорт позволяет экспортировать несколько значений из модуля. Это могут быть переменные, функции, классы или интерфейсы. При импорте необходимо использовать фигурные скобки и указать точные имена экспортируемых значений.

```typescript
// mathUtils.ts
export const PI = 3.14159;

export function add(a: number, b: number): number {
  return a + b;
}

export function subtract(a: number, b: number): number {
  return a - b;
}

// Также можно экспортировать интерфейсы и типы
export interface Calculator {
  add: (a: number, b: number) => number;
  subtract: (a: number, b: number) => number;
}
```

```typescript
// main.ts
import { PI, add, subtract, Calculator } from './mathUtils';

console.log(PI); // 3.14159
console.log(add(2, 3)); // 5
console.log(subtract(5, 2)); // 3

const calc: Calculator = {
  add: (a, b) => a + b,
  subtract: (a, b) => a - b
};
```

## Экспорт по умолчанию

Каждый модуль может иметь только один экспорт по умолчанию. Это удобно для экспорта основного значения, например, класса или функции, которая является "главным" экспортом модуля.

```typescript
// calculator.ts
class Calculator {
  add(a: number, b: number): number {
    return a + b;
  }

  subtract(a: number, b: number): number {
    return a - b;
  }
}

export default Calculator;
```

```typescript
// main.ts
import Calculator from './calculator';

const calc = new Calculator();
console.log(calc.add(2, 3)); // 5
```

## Комбинированный экспорт

Можно комбинировать именованный экспорт и экспорт по умолчанию в одном модуле.

```typescript
// utils.ts
export const MAX_VALUE = 100;

export function multiply(a: number, b: number): number {
  return a * b;
}

class Helper {
  static format(value: string): string {
    return value.trim().toUpperCase();
  }
}

export default Helper;
```

```typescript
// main.ts
import Helper, { MAX_VALUE, multiply } from './utils';

console.log(MAX_VALUE); // 100
console.log(multiply(4, 5)); // 20
console.log(Helper.format('  hello world  ')); // 'HELLO WORLD'
```

## Перенаправляющий экспорт

Перенаправляющий экспорт позволяет экспортировать значения из другого модуля, как будто они были определены в текущем модуле.

```typescript
// allMath.ts
export { add, subtract } from './mathUtils';
export { multiply } from './utils';
export { default as Calculator } from './calculator';
```

```typescript
// main.ts
import { add, subtract, multiply, Calculator } from './allMath';

console.log(add(1, 2)); // 3
console.log(subtract(5, 3)); // 2
console.log(multiply(2, 4)); // 8
const calc = new Calculator();
```

## Псевдонимы при импорте

При импорте можно использовать псевдонимы для избежания конфликта имен.

```typescript
// shapes.ts
export const circle = { radius: 5 };
export const square = { side: 10 };
```

```typescript
// main.ts
import { circle as circleShape, square as squareShape } from './shapes';

console.log(circleShape.radius); // 5
console.log(squareShape.side); // 10
```

## Импорт всего модуля

Можно импортировать весь модуль под одним именем с помощью `import * as`.

```typescript
// mathUtils.ts
export const PI = 3.14159;
export function add(a: number, b: number): number {
  return a + b;
}
export function subtract(a: number, b: number): number {
  return a - b;
}
```

```typescript
// main.ts
import * as math from './mathUtils';

console.log(math.PI); // 3.14159
console.log(math.add(2, 3)); // 5
console.log(math.subtract(5, 2)); // 3
```

## Импорт без присвоения

Иногда нужно импортировать модуль только ради его побочных эффектов, например, для выполнения кода при загрузке модуля.

```typescript
// sideEffect.ts
console.log('Этот код выполнится при импорте модуля');

export const value = 42;
```

```typescript
// main.ts
import './sideEffect'; // Выполнит код в sideEffect.ts
import { value } from './sideEffect';

console.log(value); // 42
```

## Лучшие практики

- Используйте именованный экспорт для наборов связанных функций, интерфейсов или констант.
- Используйте экспорт по умолчанию для одного основного значения, которое экспортирует модуль.
- Будьте последовательны в использовании именованных экспортов и экспорта по умолчанию в проекте.
- Используйте перенаправляющие экспорт для создания удобных точек входа (entry points) в библиотеки или модули.
- Избегайте чрезмерного использования `import * as`, так как это может затруднить понимание того, какие конкретно значения используются из модуля.
