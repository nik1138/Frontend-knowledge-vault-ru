# Управление состоянием

## Введение

Управление состоянием (state management) - это процесс организации, хранения и обновления данных в приложении. Это одна из ключевых аспектов разработки современных приложений, особенно в экосистеме React и других фреймворков.

## Основные концепции управления состоянием

### Что такое состояние?

Состояние приложения - это любые данные, которые могут измениться в течение жизненного цикла приложения. Это могут быть:

- Данные пользователя (имя, профиль, настройки)
- Данные интерфейса (открытые вкладки, состояние модальных окон)
- Данные приложения (списки, фильтры, настройки)
- Асинхронные данные (данные API, загрузка и т.д.)

### Типы состояния

1. **Локальное состояние** - данные, специфичные для одного компонента
2. **Глобальное состояние** - данные, доступные во всем приложении
3. **Временное состояние** - данные, которые существуют только во время сессии
4. **Персистентное состояние** - данные, сохраняемые между сессиями
5. **Асинхронное состояние** - данные, полученные из внешних источников

## Архитектурные паттерны управления состоянием

### Flux-архитектура

Flux - это архитектурный паттерн, предложенный Facebook, который использует однонаправленный поток данных:

```
Action → Dispatcher → Store → View
```

- **Actions** - объекты, описывающие, что произошло
- **Dispatcher** - центральный регистр для хранения обратных вызовов
- **Stores** - содержат состояние и логику приложения
- **Views** - компоненты пользовательского интерфейса

### Redux

Redux реализует концепции Flux с упором на предсказуемость:

```typescript
import { createStore, combineReducers, applyMiddleware } from 'redux';
import { composeWithDevTools } from 'redux-devtools-extension';

// Определение типов
interface UserState {
  id: number | null;
  name: string;
  email: string;
}

interface AppState {
  user: UserState;
  theme: string;
}

// Типизированные действия
const SET_USER = 'SET_USER' as const;
const LOGOUT = 'LOGOUT' as const;
const SET_THEME = 'SET_THEME' as const;

interface SetUserAction {
  type: typeof SET_USER;
  payload: { id: number; name: string; email: string };
}

interface LogoutAction {
  type: typeof LOGOUT;
}

interface SetThemeAction {
  type: typeof SET_THEME;
  payload: string;
}

type AppAction = SetUserAction | LogoutAction | SetThemeAction;

// Редьюсеры
const userReducer = (
  state: UserState = { id: null, name: '', email: '' },
  action: AppAction
): UserState => {
  switch (action.type) {
    case SET_USER:
      return action.payload;
    case LOGOUT:
      return { id: null, name: '', email: '' };
    default:
      return state;
  }
};

const themeReducer = (
  state: string = 'light',
  action: AppAction
): string => {
  switch (action.type) {
    case SET_THEME:
      return action.payload;
    default:
      return state;
  }
};

// Комбинация редьюсеров
const rootReducer = combineReducers<AppState>({
  user: userReducer,
  theme: themeReducer,
});

// Создание хранилища
const store = createStore(
  rootReducer,
  composeWithDevTools(applyMiddleware())
);

// Типизация состояния
type RootState = ReturnType<typeof rootReducer>;
```

### Context API

React Context API предоставляет способ передачи данных через дерево компонентов без необходимости передавать пропсы на каждом уровне:

```typescript
import React, { createContext, useContext, useReducer, ReactNode } from 'react';

interface AppState {
  user: { id: number; name: string } | null;
  theme: 'light' | 'dark';
  notifications: string[];
}

type AppAction = 
  | { type: 'SET_USER'; payload: { id: number; name: string } }
  | { type: 'SET_THEME'; payload: 'light' | 'dark' }
  | { type: 'ADD_NOTIFICATION'; payload: string }
  | { type: 'CLEAR_NOTIFICATIONS' }
  | { type: 'LOGOUT' };

const initialState: AppState = {
  user: null,
  theme: 'light',
  notifications: [],
};

const AppContext = createContext<{
  state: AppState;
  dispatch: React.Dispatch<AppAction>;
} | undefined>(undefined);

const appReducer = (state: AppState, action: AppAction): AppState => {
  switch (action.type) {
    case 'SET_USER':
      return { ...state, user: action.payload };
    case 'SET_THEME':
      return { ...state, theme: action.payload };
    case 'ADD_NOTIFICATION':
      return {
        ...state,
        notifications: [...state.notifications, action.payload],
      };
    case 'CLEAR_NOTIFICATIONS':
      return { ...state, notifications: [] };
    case 'LOGOUT':
      return { ...state, user: null };
    default:
      return state;
  }
};

interface AppProviderProps {
  children: ReactNode;
}

export const AppProvider: React.FC<AppProviderProps> = ({ children }) => {
  const [state, dispatch] = useReducer(appReducer, initialState);

  return (
    <AppContext.Provider value={{ state, dispatch }}>
      {children}
    </AppContext.Provider>
  );
};

export const useAppContext = () => {
  const context = useContext(AppContext);
  if (!context) {
    throw new Error('useAppContext must be used within an AppProvider');
  }
  return context;
};
```

## Современные библиотеки управления состоянием

### Zustand

Zustand - это легковесная библиотека для управления состоянием:

```typescript
import { create } from 'zustand';
import { persist, subscribeWithSelector } from 'zustand/middleware';

interface User {
  id: number;
  name: string;
  email: string;
}

interface Notification {
  id: number;
  message: string;
  type: 'info' | 'success' | 'warning' | 'error';
}

interface AppState {
  user: User | null;
  theme: 'light' | 'dark';
  notifications: Notification[];
  setUser: (user: User) => void;
  logout: () => void;
  setTheme: (theme: 'light' | 'dark') => void;
  addNotification: (notification: Omit<Notification, 'id'>) => void;
  removeNotification: (id: number) => void;
}

const useAppStore = create<AppState>()(
  persist(
    subscribeWithSelector((set, get) => ({
      user: null,
      theme: 'light',
      notifications: [],
      
      setUser: (user) => set({ user }),
      
      logout: () => set({ user: null }),
      
      setTheme: (theme) => set({ theme }),
      
      addNotification: (notification) => {
        const id = Date.now();
        const newNotification = { ...notification, id };
        set((state) => ({
          notifications: [...state.notifications, newNotification],
        }));
        
        // Автоматическое удаление уведомления через 5 секунд
        setTimeout(() => {
          get().removeNotification(id);
        }, 5000);
      },
      
      removeNotification: (id) =>
        set((state) => ({
          notifications: state.notifications.filter(
            (n) => n.id !== id
          ),
        })),
    })),
    {
      name: 'app-storage', // имя ключа в localStorage
    }
  )
);
```

### Jotai

Jotai - это атомарная библиотека управления состоянием:

```typescript
import { atom, useAtom, Provider } from 'jotai';
import { atomWithStorage } from 'jotai/utils';

interface User {
  id: number;
  name: string;
  email: string;
}

// Атомы для состояния
const userAtom = atomWithStorage<User | null>('user', null);
const themeAtom = atom<'light' | 'dark'>('light');
const themeSetterAtom = atom(
  null,
  (get, set, theme: 'light' | 'dark') => {
    set(themeAtom, theme);
  }
);

// Композитный атом для проверки аутентификации
const isAuthenticatedAtom = atom((get) => {
  const user = get(userAtom);
  return user !== null;
});

// Компонент, использующий атомы
const UserProfile: React.FC = () => {
  const [user, setUser] = useAtom(userAtom);
  const [theme] = useAtom(themeAtom);
  const [, setTheme] = useAtom(themeSetterAtom);
  const isAuthenticated = useAtomValue(isAuthenticatedAtom);

  return (
    <div className={`profile ${theme}`}>
      {isAuthenticated ? (
        <div>
          <h2>Привет, {user?.name}!</h2>
          <button onClick={() => setUser(null)}>Выйти</button>
        </div>
      ) : (
        <p>Пожалуйста, войдите в систему</p>
      )}
      <button onClick={() => setTheme(theme === 'light' ? 'dark' : 'light')}>
        Переключить тему
      </button>
    </div>
  );
};
```

## Практические рекомендации

### Когда использовать глобальное состояние?

- Данные используются в нескольких компонентах
- Необходимо синхронизировать состояние между компонентами
- Данные должны сохраняться между маршрутами/экранами
- Сложная бизнес-логика требует централизованного управления

### Когда использовать локальное состояние?

- Данные специфичны для одного компонента
- Не влияют на другие части приложения
- Простые изменения UI (открытие/закрытие элементов интерфейса)
- Временные данные, не требующие сохранения

### Оптимизация производительности

1. **Избегайте избыточного рендеринга** - используйте `React.memo`, `useMemo`, `useCallback`
2. **Разделяйте состояние** - разделяйте часто и редко изменяемые данные
3. **Используйте селекторы** - извлекайте только необходимые данные из состояния
4. **Избегайте глубоких изменений** - используйте иммутабельные структуры данных

## Заключение

Управление состоянием - ключевой аспект разработки современных приложений. Выбор подходящей стратегии управления состоянием зависит от сложности приложения, требований к производительности и предпочтений команды разработчиков.
