# Определение классов в TypeScript

## Введение

Классы в TypeScript являются центральным элементом объектно-ориентированного программирования (ООП). Они предоставляют удобный способ инкапсуляции данных и функций в одном объекте с возможностью наследования и полиморфизма. TypeScript расширяет возможности ES6 классов, добавляя поддержку типизации и других функций.

## Основы определения класса

Класс в TypeScript определяется с использованием ключевого слова `class`, за которым следует имя класса. В теле класса можно определять свойства, методы, конструкторы и другие члены класса.

```typescript
class Car {
  brand: string;
  model: string;
  year: number;

  constructor(brand: string, model: string, year: number) {
    this.brand = brand;
    this.model = model;
    this.year = year;
  }

  getFullInfo(): string {
    return `${this.year} ${this.brand} ${this.model}`;
  }
}

const myCar = new Car("Toyota", "Camry", 2022);
console.log(myCar.getFullInfo()); // Вывод: "2022 Toyota Camry"
```

> [!tip] Совет
> Используйте конструктор для инициализации свойств экземпляра. Это помогает избежать ошибок, связанных с неинициализированными переменными.

## Свойства класса

Свойства класса могут быть объявлены с различными модификаторами доступа: `public`, `private`, и `protected`. По умолчанию свойства являются `public`.

```typescript
class Person {
  public name: string;
  private age: number;
  protected address: string;

  constructor(name: string, age: number, address: string) {
    this.name = name;
    this.age = age;
    this.address = address;
  }
}
```

## Методы класса

Методы класса определяются внутри тела класса и могут быть как экземплярами, так и статическими (см. [[Статические-члены]]).

```typescript
class Calculator {
  add(a: number, b: number): number {
    return a + b;
  }

  subtract(a: number, b: number): number {
    return a - b;
  }
}
```

## Конструктор

Конструктор - это специальный метод, который вызывается при создании нового экземпляра класса. Он используется для инициализации свойств экземпляра.

```typescript
class Rectangle {
  width: number;
  height: number;

  constructor(width: number, height: number) {
    this.width = width;
    this.height = height;
  }

  getArea(): number {
    return this.width * this.height;
  }
}
```

## Параметрический синтаксис конструктора

TypeScript поддерживает сокращенный синтаксис для инициализации свойств через конструктор:

```typescript
class Student {
  constructor(
    public name: string,
    private id: number,
    protected course: string
  ) {}
}
```

В этом примере `name`, `id`, и `course` автоматически становятся свойствами класса с соответствующими модификаторами доступа.

## Практические рекомендации

1. **Используйте конкретные типы** для свойств класса вместо `any`
2. **Инициализируйте свойства в конструкторе** для обеспечения согласованности
3. **Следуйте соглашениям об именовании** (PascalCase для имен классов)
4. **Разделяйте ответственность** между классами по принципу единственной ответственности
5. **Документируйте классы** с помощью JSDoc для лучшей читаемости

## Заключение

Классы в TypeScript предоставляют мощный способ организации кода с использованием объектно-ориентированного подхода. Понимание основ определения классов является фундаментом для более продвинутых концепций, таких как наследование и абстракция.

Для более глубокого изучения рекомендуется ознакомиться с разделами [[Наследование-классов]] и [[Модификаторы-доступа]].
