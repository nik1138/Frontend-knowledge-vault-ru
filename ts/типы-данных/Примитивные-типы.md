# Примитивные типы данных в TypeScript

## Введение

Примитивные типы данных являются основой системы типов TypeScript. Это наиболее простые типы, которые не являются объектами и не содержат методов. В TypeScript, как и в JavaScript, существует 7 примитивных типов данных, которые мы можем использовать для определения типов переменных.

## Основные примитивные типы

### String (Строка)

Тип `string` представляет собой последовательность символов, заключенную в кавычки (одинарные, двойные или обратные). В TypeScript строки являются неизменяемыми объектами.

**Пример:**
```typescript
let firstName: string = "Александр";
let lastName: string = 'Иванов';
let fullName: string = `${firstName} ${lastName}`; // Шаблонная строка
```

**Практические рекомендации:**
- Используйте шаблонные строки (обратные кавычки) для форматирования сложных строк
- Всегда используйте один стиль кавычек в проекте для согласованности

### Number (Число)

Тип `number` представляет как целые, так и дробные числа. В TypeScript все числа представлены как числа с плавающей точкой двойной точности (64-битные).

**Пример:**
```typescript
let age: number = 25;
let price: number = 99.99;
let scientific: number = 1e6; // 1000000
let binary: number = 0b1010; // 10 в двоичной системе
let hex: number = 0xff; // 255 в шестнадцатеричной системе
let octal: number = 0o744; // 484 в восьмеричной системе
```

**Практические рекомендации:**
- Используйте `number` для всех числовых значений, включая целые числа
- Будьте осторожны с операциями с плавающей точкой из-за потенциальных ошибок округления

### Boolean (Логический тип)

Тип `boolean` может принимать только два значения: `true` или `false`.

**Пример:**
```typescript
let isActive: boolean = true;
let isCompleted: boolean = false;
let hasPermission: boolean = age >= 18; // Результат выражения
```

**Практические рекомендации:**
- Используйте логические переменные для проверки условий
- Избегайте использования строк или других типов для логических проверок

### Null и Undefined

Типы `null` и `undefined` представляют отсутствие значения. `null` - это присвоенное значение, которое означает "намеренно пустое", а `undefined` означает переменную, которой не было присвоено значение.

**Пример:**
```typescript
let user: string = null; // Требуется включить strictNullChecks
let config: undefined = undefined;

// С null и undefined в strict mode
let name: string | null = null;
let email: string | undefined = undefined;
```

**Практические рекомендации:**
- Используйте `strictNullChecks` для более безопасной типизации
- Явно указывайте `| null` или `| undefined` в типах, если переменная может не иметь значения
- Используйте `undefined` для неинициализированных переменных, `null` для намеренно пустых значений

### Symbol (Символ)

Тип `symbol` представляет уникальные и неизменяемые значения, которые могут использоваться в качестве идентификаторов для свойств объектов.

**Пример:**
```typescript
let sym1: symbol = Symbol("key1");
let sym2: symbol = Symbol("key2");

console.log(sym1 === sym2); // false - каждый символ уникален

// Символы с одинаковым описанием все равно уникальны
let symA: symbol = Symbol("test");
let symB: symbol = Symbol("test");
console.log(symA === symB); // false
```

**Практические рекомендации:**
- Используйте символы для создания уникальных ключей свойств
- Символы полезны для создания приватных свойств в объектах
- Символы не могут быть преобразованы в строки автоматически

### BigInt

Тип `bigint` представляет целые числа произвольной длины. Это позволяет работать с очень большими числами, которые не помещаются в стандартный `number`.

**Пример:**
```typescript
let bigNumber: bigint = 1234567890123456789012345678901234567890n;
let anotherBig: bigint = BigInt(1000); // Используя конструктор
let fromString: bigint = BigInt("9999999999999999999999999999999999");

// Операции с bigint
let result: bigint = bigNumber + 100n;
```

**Практические рекомендации:**
- Используйте `bigint` только при необходимости работы с очень большими целыми числами
- Не используйте операторы сравнения между `number` и `bigint`
- BigInt несовместим с `number` - они не могут быть использованы вместе в арифметических операциях

## Особенности и практические применения

### Строгая типизация

TypeScript позволяет точно определять типы переменных, что помогает избежать ошибок в процессе разработки:

```typescript
let message: string = "Привет";
// message = 42; // Ошибка: нельзя присвоить число строке
```

### Совместимость с JavaScript

Все примитивные типы TypeScript совместимы с JavaScript, что позволяет использовать их в существующих проектах без изменений.

### Лучшие практики

1. **Явное указание типов**: Хотя TypeScript может выводить типы автоматически, рекомендуется явно указывать их для улучшения читаемости кода.

2. **Использование union типов**: Для переменных, которые могут принимать несколько значений, используйте union типы:

```typescript
let status: "active" | "inactive" | "pending" = "active";
```

3. **Проверка типов**: Используйте проверки типов для безопасной работы с данными:

```typescript
function processValue(value: string | number) {
  if (typeof value === "string") {
    return value.toUpperCase(); // Здесь TypeScript знает, что это строка
  }
  return value * 2; // Здесь TypeScript знает, что это число
}
```
