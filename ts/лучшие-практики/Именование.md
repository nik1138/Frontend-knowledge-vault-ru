# Именование TypeScript

## Обзор

Соглашения об именовании в TypeScript играют ключевую роль в создании читаемого и поддерживаемого кода. Правильные имена переменных, функций, классов и других сущностей помогают разработчикам быстро понимать назначение кода и облегчают сопровождение.

## Основные принципы именования

- **Ясность**: Имя должно ясно отражать назначение сущности
- **Консистентность**: Следуйте единообразным правилам во всей кодовой базе
- **Согласованность**: Используйте одинаковые шаблоны для схожих сущностей
- **Семантичность**: Имя должно быть описательным и понятным

## Стили именования

### PascalCase (Верблюжий регистр с заглавной буквы)

Используется для:
- Имен классов
- Имен интерфейсов
- Имен типов
- Имен enum'ов
- Имен пространств имен

```typescript
class UserService {
  // реализация
}

interface User {
  id: string;
  name: string;
}

type UserRole = 'admin' | 'user' | 'moderator';

enum Status {
  Active = 'ACTIVE',
  Inactive = 'INACTIVE',
  Pending = 'PENDING'
}

namespace Validation {
  export function validateEmail(email: string): boolean {
    // реализация
  }
}
```

### camelCase (Верблюжий регистр)

Используется для:
- Имен переменных
- Имен функций
- Имен параметров
- Имен методов
- Имен свойств

```typescript
const userName = 'John';
const userAge = 30;

function calculateTotal(items: Item[]): number {
  return items.reduce((sum, item) => sum + item.price, 0);
}

class User {
  id: string;
  name: string;

  updateName(newName: string): void {
    this.name = newName;
  }
}
```

### UPPER_SNAKE_CASE (Верхний змеиный регистр)

Используется для:
- Констант
- Ключей конфигурации
- Кодов ошибок

```typescript
const API_URL = 'https://api.example.com';
const MAX_RETRY_COUNT = 3;

const ERROR_CODES = {
  USER_NOT_FOUND: 'USER_NOT_FOUND',
  INVALID_CREDENTIALS: 'INVALID_CREDENTIALS'
};
```

### kebab-case (дефисный регистр)

Используется для:
- Имен файлов
- Имен папок
- URL-адресов

## Именование переменных

### Используйте описательные имена

```typescript
// Хорошо: описательное имя
const userRegistrationDate = new Date('2023-01-15');
const itemsInCart = 5;

// Плохо: неинформативное имя
const d = new Date('2023-01-15');
const n = 5;
```

### Избегайте сокращений

```typescript
// Хорошо: полные имена
const numberOfActiveUsers = users.filter(user => user.isActive).length;
const calculateUserScore = (user: User): number => { /* реализация */ };

// Плохо: сокращенные имена
const numActiveUsers = users.filter(user => user.isActive).length;
const calcUserScore = (user: User): number => { /* реализация */ };
```

### Для булевых значений используйте префиксы

```typescript
// Хорошо: префиксы для булевых значений
const isActive: boolean = true;
const isUserLoggedIn: boolean = false;
const hasPermission: boolean = true;
const canEdit: boolean = false;
const shouldRefresh: boolean = true;

// Плохо: неясные имена для булевых значений
const active: boolean = true;
const logged: boolean = false;
const permission: boolean = true;
```

### Используйте соглашения для коллекций

```typescript
// Хорошо: соглашения для коллекций
const userNames: string[] = ['John', 'Jane', 'Bob'];
const activeUsers: User[] = users.filter(user => user.isActive);
const userMap: Map<string, User> = new Map();

// Плохо: неясные имена для коллекций
const names: string[] = ['John', 'Jane', 'Bob'];
const filtered: User[] = users.filter(user => user.isActive);
const map: Map<string, User> = new Map();
```

```typescript

  // реализация

function validateEmail(email: string): boolean {
  // реализация
}

function calculateTotalPrice(items: CartItem[]): number {
  // реализация

// Хорошо: глаголы для методов
class UserService {
  createUser(userData: UserInput): Promise<User> {
    // реализация
  }

  updateUser(userId: string, updates: Partial<User>): Promise<User> {
    // реализация
  }

  deleteUser(userId: string): Promise<void> {
    // реализация
  }

```

### Используйте префиксы для специфичных функций

```typescript
// Хорошо: префиксы для различных типов функций
function isUserValid(user: User): boolean {
  return user.name !== '' && user.email.includes('@');
}

function getUserName(user: User): string {
  return user.name;
}

function createNewUser(userData: UserInput): User {
  return new User(userData);
}

function handleUserClick(event: MouseEvent): void {
  // обработка клика

function formatUserDate(date: Date): string {
  return date.toLocaleDateString();
}
```

## Именование классов и интерфейсов

### Используйте существительные для классов

```typescript
// Хорошо: существительные для классов
class User {
  // реализация
}

class PaymentProcessor {
  // реализация
}

class EmailService {
  // реализация
}

class DatabaseConnection {
  // реализация
}
```

### Используйте описательные имена для интерфейсов

```typescript
// Хорошо: описательные имена для интерфейсов
interface User {
  id: string;
  name: string;
  email: string;
}

interface PaymentProcessor {
  processPayment(amount: number, method: string): Promise<boolean>;
}

interface ConfigurableService {
  configure(options: Record<string, any>): void;

// Альтернатива: префикс I для интерфейсов (менее рекомендуется в современном TypeScript)

  id: string;
  name: string;
}
```

## Именование констант

### Используйте UPPER_SNAKE_CASE для констант

```typescript
// Хорошо: UPPER_SNAKE_CASE для констант
const API_BASE_URL = 'https://api.example.com';
const MAX_FILE_SIZE = 1024 * 1024 * 5; // 5MB
const SUPPORTED_FORMATS = ['jpg', 'png', 'gif'];

// Для объектов-констант используйте camelCase для свойств

  apiUrl: 'https://api.example.com',
  timeout: 5000,
  maxRetries: 3

```

## Специфичные соглашения

### Приватные члены класса

Для приватных членов класса используйте префикс подчеркивания:

```typescript
class UserService {
  private _currentUser: User | null = null;
  private _userRepository: UserRepository;

  private _validateUser(user: User): boolean {
    // внутренняя валидация
    return user.name !== '';
  }

  public getCurrentUser(): User | null {
    return this._currentUser;
  }
}
```

> **Примечание**: В современном TypeScript предпочтительно использовать модификатор `private` вместо соглашения с подчеркиванием, но соглашение все еще распространено.

### Обработчики событий

Для обработчиков событий используйте префикс `handle`:

```typescript
class UserComponent {
  private handleUserClick = (event: MouseEvent): void => {
    // обработка клика пользователя
  }

  private handleUserInput = (event: KeyboardEvent): void => {
    // обработка ввода пользователя
  }

  private handleUserChange = (newUser: User): void => {
    // обработка изменения пользователя
  }
}
```

## Общие рекомендации

### Избегайте неинформативных имен

```typescript
// Плохо: неинформативные имена
const a = 'John';
const obj = { name: 'Jane' };
const list = [1, 2, 3];
const fn = (x: number) => x * 2;

// Хорошо: информативные имена
const userName = 'John';
const user = { name: 'Jane' };
const numbers = [1, 2, 3];
const doubleNumber = (num: number) => num * 2;
```

### Используйте единообразные шаблоны для схожих сущностей

```typescript
// Хорошо: единообразные шаблоны
class UserService {
  createUser(): Promise<User> { /* реализация */ }
  getUser(): Promise<User> { /* реализация */ }
  updateUser(): Promise<User> { /* реализация */ }
  deleteUser(): Promise<boolean> { /* реализация */ }
}

// Плохо: несогласованные шаблоны
class UserService {
  makeUser(): Promise<User> { /* реализация */ }
  findUser(): Promise<User> { /* реализация */ }
  modifyUser(): Promise<User> { /* реализация */ }
  removeUser(): Promise<boolean> { /* реализация */ }
}
```

### Используйте контекстно-зависимые имена

```typescript
// Хорошо: имена, учитывающие контекст
class PaymentService {
  process(): boolean { /* реализация */ }
  validate(): boolean { /* реализация */ }
  calculate(): number { /* реализация */ }
}

// Хорошо: более явные имена при необходимости
class PaymentService {
  processPayment(): boolean { /* реализация */ }
  validatePayment(): boolean { /* реализация */ }
  calculateTotal(): number { /* реализация */ }
}
```
