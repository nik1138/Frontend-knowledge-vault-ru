# Расширение интерфейсов в TypeScript

TypeScript позволяет расширять интерфейсы с помощью ключевого слова `extends`. Это позволяет создавать более специфичные интерфейсы на основе более общих, реализуя принцип наследования.

## Базовое расширение интерфейсов

```typescript
interface Animal {
  name: string;
  age: number;
}

interface Dog extends Animal {
  breed: string;
  bark(): void;
}

const myDog: Dog = {
  name: "Рекс",
  age: 3,
  breed: "Лабрадор",
  bark() {
    console.log("Гав!");
  }
};
```

## Расширение нескольких интерфейсов

Интерфейс может расширять несколько других интерфейсов:

```typescript
interface CanFly {
  fly(): void;
}

interface CanSwim {
  swim(): void;
}

interface Duck extends CanFly, CanSwim {
  name: string;
}

const donald: Duck = {
  name: "Дональд",
  fly() {
    console.log("Летит как утка");
  },
  swim() {
    console.log("Плавает как утка");
  }
};
```

## Перекрытие свойств при расширении

При расширении интерфейсов можно уточнить типы свойств, но новый тип должен быть совместим с базовым:

```typescript
interface Shape {
  color: string;
}

interface Circle extends Shape {
  color: "red" | "blue" | "green"; // Более узкий тип
  radius: number;
}
```

## Расширение интерфейсов с методами

```typescript
interface Vehicle {
  start(): void;
  stop(): void;
}

interface Car extends Vehicle {
  drive(): void;
  openTrunk(): void;
}

interface ElectricCar extends Car {
  charge(): void;
  getBatteryLevel(): number;
}
```

## Практические применения

Расширение интерфейсов особенно полезно при работе с API, где базовые интерфейсы могут определять общие свойства, а расширенные - специфичные для конкретных сущностей:

```typescript
interface ApiResponse {
  status: number;
  message: string;
}

interface UserResponse extends ApiResponse {
  user: {
    id: number;
    name: string;
    email: string;
  };
}

interface PostResponse extends ApiResponse {
  post: {
    id: number;
    title: string;
    content: string;
  };
}
```

> [!tip] Совет
> Используйте расширение интерфейсов для создания иерархий типов, отражающих реальные отношения между сущностями в вашем приложении.

> [!warning] Важно
> При расширении нескольких интерфейсов убедитесь, что в них нет конфликтующих определений свойств с одинаковыми именами, но несовместимыми типами.
