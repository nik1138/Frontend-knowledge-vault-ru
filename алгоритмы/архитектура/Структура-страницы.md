# Структура страницы

Структура страницы определяет архитектуру и организацию контента на веб-странице. Это основа, на которой строится вся визуальная иерархия и функциональность сайта.

## Основные элементы структуры

### Семантические теги HTML5

Семантические теги позволяют структурировать контент логически и делают его более доступным:

- `<header>` - заголовок страницы или раздела
- `<nav>` - навигационные элементы
- `<main>` - основное содержимое страницы
- `<article>` - независимый контент
- `<section>` - логическая группа контента
- `<aside>` - сопутствующий контент
- `<footer>` - нижний колонтитул

### Пример базовой структуры

```html
<!DOCTYPE html>
<html lang="ru">
<head>
    <[Meta_признанная_экстремистской_организацией_в_РФ] charset="UTF-8">
    <[Meta_признанная_экстремистской_организацией_в_РФ] name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Структура страницы</title>
</head>
<body>
    <div class="page-wrapper">
        <header class="header">
            <h1>Заголовок сайта</h1>
            <nav class="navigation">
                <ul>
                    <li><a href="#home">Главная</a></li>
                    <li><a href="#about">О нас</a></li>
                    <li><a href="#contact">Контакты</a></li>
                </ul>
            </nav>
        </header>

        <main class="main-content">
            <section class="hero">
                <h2>Заголовок основного раздела</h2>
                <p>Описание основного контента</p>
            </section>

            <section class="content-grid">
                <article class="content-item">
                    <h3>Подзаголовок статьи</h3>
                    <p>Содержимое статьи</p>
                </article>
            </section>
        </main>

        <aside class="sidebar">
            <h3>Боковая панель</h3>
            <ul>
                <li>Ссылка 1</li>
                <li>Ссылка 2</li>
            </ul>
        </aside>

        <footer class="footer">
            <p>&copy; 2025 Все права защищены</p>
        </footer>
    </div>
</body>
</html>
```

## Иерархия и логика структуры

### Логическая последовательность

Структура страницы должна следовать логической последовательности:

1. Заголовок (header) - содержит основную информацию о сайте
2. Навигация - пути к основным разделам
3. Основной контент (main) - главное содержимое
4. Дополнительная информация (aside) - сопутствующий контент
5. Нижний колонтитул (footer) - дополнительные ссылки и информация

### Визуальная иерархия

- Размер шрифта и вес для создания визуального акцента
- Пространственное распределение контента
- Цветовые акценты и контрастность
- Использование белого пространства

## Адаптивная структура

### Responsive дизайн

Современная структура страницы должна быть адаптивной:

- Использование flexbox и grid для создания адаптивных сеток
- Медиа-запросы для различных размеров экрана
- Масштабируемый дизайн для мобильных устройств

```css
/* Пример адаптивной сетки */
.page-wrapper {
    display: grid;
    grid-template-areas: 
        "header header"
        "main sidebar"
        "footer footer";
    grid-template-columns: 1fr 300px;
    grid-gap: 20px;
}

@media (max-width: 768px) {
    .page-wrapper {
        grid-template-areas: 
            "header"
            "main"
            "sidebar"
            "footer";
        grid-template-columns: 1fr;
    }
}
```

## Архитектурные паттерны структуры

### F-образная структура

- Основан на поведении пользователей при чтении контента
- Глаза пользователей сканируют страницу в форме буквы F
- Подходит для контент-ориентированных сайтов

### Z-образная структура

- Используется для одностраничных лендингов
- Глаза пользователей перемещаются по диагонали
- Эффективна для призыва к действию

### Grid-структура

- Используется для сайтов с большим количеством контента
- Позволяет эффективно организовать визуальные элементы
- Подходит для галерей, каталогов и новостных сайтов

## Доступность и SEO

### Структура для скринридеров

- Правильная иерархия заголовков (h1-h6)
- Использование ARIA-атрибутов
- Логический порядок чтения элементов

### Оптимизация для поисковых систем

- Заголовок h1 должен отражать основную тему страницы
- Подзаголовки h2-h6 структурируют контент
- Использование семантических тегов улучшает понимание контента

## Российские особенности 2025

### Требования к доступности

В России действуют строгие требования к доступности веб-сайтов:

- ГОСТ Р 52872-2012 для веб-сайтов
- Обеспечение доступности для людей с ограниченными возможностями
- Использование альтернативного текста для изображений

### Языковые особенности

- Правильное указание языка (lang="ru")
- Учет особенностей русского языка в верстке
- Поддержка кириллических шрифтов

## Практические рекомендации

### Тестирование структуры

- Использование инструментов валидации HTML
- Тестирование с помощью скринридеров
- Проверка на различных устройствах и браузерах

### Оптимизация производительности

- Минимизация DOM-элементов
- Эффективное использование CSS-селекторов
- Оптимизация загрузки ресурсов

## Заключение

Правильная структура страницы - это основа качественного веб-сайта. Она влияет на доступность, SEO, производительность и пользовательский опыт. При проектировании структуры важно учитывать не только визуальные аспекты, но и функциональные требования, требования доступности и особенности целевой аудитории.

В российской экосистеме 2025 года особое внимание уделяется соответствию законодательным требованиям, что делает правильную структуру страницы не просто рекомендацией, а необходимостью.
