# Связные списки

## Введение

Связный список - это линейная структура данных, в которой элементы (узлы) связаны между собой через указатели. Связные списки остаются важной структурой данных, особенно в системах с ограниченной памятью, embedded системах и задачах, требующих частых вставок и удалений.

## Определение

**Связный список** - это структура данных, состоящая из узлов, где каждый узел содержит данные и ссылку (указатель) на следующий узел в последовательности. В отличие от массивов, элементы не хранятся в непрерывной области памяти.

## Типы связных списков

### Односвязный список
Каждый узел содержит данные и ссылку только на следующий узел.

### Двусвязный список
Каждый узел содержит данные и ссылки как на следующий, так и на предыдущий узел.

### Кольцевой (циклический) список
Последний узел ссылается на первый узел, образуя кольцо.

## Структура узла

```javascript
// Узел односвязного списка
class ListNode {
    constructor(val, next = null) {
        this.val = val;
        this.next = next;
    }
}

// Узел двусвязного списка
class DoublyListNode {
    constructor(val, prev = null, next = null) {
        this.val = val;
        this.prev = prev;
        this.next = next;
    }
}
```

```

    ВЫХОД: новая голова списка

    новый_узел = НОВЫЙ УзелОдносвязного(значение)

    ВЫХОД: вставленный узел

    новый_узел = НОВЫЙ УзелДвусвязного(значение)
    новый_узел.следующий = узел.следующий

function insertAtTail(head, val) {
    const newNode = new ListNode(val);

    if (head === null) {
        return newNode;

    let current = head;
    while (current.next !== null) {
        current = current.next;

    return head;

    const newNode = new DoublyListNode(val);
    newNode.next = node.next;
    newNode.prev = node;

        node.next.prev = newNode;
    }
    node.next = newNode;

    return newNode;
}

### Удаление элемента

#### Псевдокод для удаления элемента из связного списка:

```
АЛГОРИТМ УдалитьПоЗначению(голова, значение)
    ВХОД: голова - первый узел списка, значение - значение для удаления
    ВЫХОД: новая голова списка

    ЕСЛИ голова = NULL ТОГДА
        ВОЗВРАТ NULL
    КОНЕЦ ЕСЛИ

    ЕСЛИ голова.значение = значение ТОГДА
        ВОЗВРАТ голова.следующий
    КОНЕЦ ЕСЛИ

        текущий = текущий.следующий
    КОНЕЦ ПОКА

    ЕСЛИ текущий.следующий ≠ NULL ТОГДА
        текущий.следующий = текущий.следующий.следующий

        ВОЗВРАТ голова.следующий

        текущий = текущий.следующий
        i = i + 1
    КОНЕЦ ПОКА

    ЕСЛИ текущий = NULL ИЛИ текущий.следующий = NULL ТОГДА
        ВОЗВРАТ голова
    КОНЕЦ ЕСЛИ

    текущий.следующий = текущий.следующий.следующий
    ВОЗВРАТ голова

// Удаление по значению в JavaScript
function deleteNode(head, val) {
    if (head === null) return null;

    if (head.val === val) {
        return head.next;
    }

    let current = head;
    while (current.next !== null && current.next.val !== val) {
        current = current.next;
    }

    if (current.next !== null) {
        current.next = current.next.next;
    }

    return head;
}

// Удаление по позиции в JavaScript
function deleteAtPosition(head, position) {
    if (head === null || position < 0) return head;

    if (position === 0) {
        return head.next;
    }

    let current = head;
    for (let i = 0; current !== null && i < position - 1; i++) {
        current = current.next;
    }

    if (current === null || current.next === null) {
        return head;
    }

    current.next = current.next.next;
    return head;
}

    current.next = current.next.next;

### Поиск элемента

```java
// Поиск элемента по значению
public boolean search(ListNode head, int val) {
    ListNode current = head;
    while (current != null) {
        if (current.val == val) {
            return true;
        }
        current = current.next;
    }
    return false;
}

// Поиск элемента по индексу
public ListNode getNodeAt(ListNode head, int index) {
    ListNode current = head;
    int count = 0;
    
    while (current != null && count < index) {
        current = current.next;
        count++;
    }
    
    return current;
}
```

## Реализация связного списка

```java
// Полная реализация односвязного списка
class LinkedList {
    private ListNode head;
    private int size;
    
    public LinkedList() {
        head = null;
        size = 0;
    }
    
    public void addFirst(int val) {
        ListNode newNode = new ListNode(val);
        newNode.next = head;
        head = newNode;
        size++;
    }
    
    public void addLast(int val) {
        ListNode newNode = new ListNode(val);
        
        if (head == null) {
            head = newNode;
        } else {
            ListNode current = head;
            while (current.next != null) {
                current = current.next;
            }
            current.next = newNode;
        }
        size++;
    }
    
    public boolean remove(int val) {
        if (head == null) return false;
        
        if (head.val == val) {
            head = head.next;
            size--;
            return true;
        }
        
        ListNode current = head;
        while (current.next != null && current.next.val != val) {
            current = current.next;
        }
        
        if (current.next != null) {
            current.next = current.next.next;
            size--;
            return true;
        }
        
        return false;
    }
    
    public boolean contains(int val) {
        ListNode current = head;
        while (current != null) {
            if (current.val == val) {
                return true;
            }
            current = current.next;
        }
        return false;
    }
    
    public int size() {
        return size;
    }
    
    public boolean isEmpty() {
        return head == null;
    }
}
```

## Сложность операций

| Операция | Время | Примечание |
|----------|-------|------------|
| Доступ | O(n) | Нет прямого доступа по индексу |
| Поиск | O(n) | В худшем случае |
| Вставка в начало | O(1) | Быстрая вставка |
| Вставка в конец | O(n) | Требует обхода |
| Удаление | O(n) | Требует поиска элемента |

## Преимущества и недостатки

### Преимущества:
- Динамический размер
- Эффективная вставка и удаление в начале списка
- Нет необходимости в предварительном выделении памяти
- Эффективное использование памяти (только необходимые узлы)

### Недостатки:
- Нет прямого доступа к элементам по индексу
- Требует дополнительной памяти для хранения указателей
- Плохая кэш-локальность (узлы могут быть разбросаны по памяти)
- Сложнее реализации по сравнению с массивами

### Разворот списка

```javascript
// Разворот односвязного списка
function reverse(head) {
    let prev = null;
    let current = head;
    let next;

    while (current !== null) {
        next = current.next;
        current.next = prev;
        prev = current;
        current = next;
    }

    return prev;
}
```

### Обнаружение цикла

```javascript
// Алгоритм Флойда (черепаха и заяц)
function hasCycle(head) {
    if (head === null || head.next === null) return false;

    let slow = head;
    let fast = head;

    while (fast !== null && fast.next !== null) {
        slow = slow.next;
        fast = fast.next.next;

        if (slow === fast) {
            return true;
        }
    }

    return false;
}
```

### Нахождение начала цикла

```javascript
function detectCycle(head) {
    if (head === null || head.next === null) return null;

    let slow = head;
    let fast = head;

    // Найти точку встречи
    while (fast !== null && fast.next !== null) {
        slow = slow.next;
        fast = fast.next.next;

        if (slow === fast) break;
    }

    if (fast === null || fast.next === null) return null;

    // Найти начало цикла
    let ptr1 = head;
    let ptr2 = slow;

    while (ptr1 !== ptr2) {
        ptr1 = ptr1.next;
        ptr2 = ptr2.next;
    }

    return ptr1;
}
```

## Заключение

Связные списки являются важной структурой данных, обеспечивающей гибкость при частых вставках и удалениях. Знание связных списков и умение работать с ними остается критически важным как для прохождения технических собеседований, так и для решения задач, где важна динамическая структура данных.
