# Алгоритмы сортировки

## Общее описание

Алгоритмы сортировки - это фундаментальные алгоритмы, которые упорядочивают элементы в списке или массиве в соответствии с определенным критерием (обычно по возрастанию или убыванию). Знание различных алгоритмов сортировки остается важным для прохождения технических собеседований и оптимизации производительности приложений.

## Основные типы алгоритмов сортировки

### 1. Сравнительные сортировки
- [[Сортировка-пузырьком]]
- [[Быстрая-сортировка]]
- [[Сортировка-слиянием]]
- [[Пирамидальная-сортировка]]

### 2. Несравнительные сортировки
- Поразрядная сортировка (Radix Sort)
- Подсчетом (Counting Sort)
- Поразрядная сортировка (Bucket Sort)

## Классификация по характеристикам

| Алгоритм | Сложность (средний случай) | Сложность (худший случай) | Память | Устойчивость |
|----------|----------------------------|---------------------------|---------|--------------|
| [[Сортировка-пузырьком]] | O(n²) | O(n²) | O(1) | Да |
| [[Быстрая-сортировка]] | O(n log n) | O(n²) | O(log n) | Нет |
| [[Сортировка-слиянием]] | O(n log n) | O(n log n) | O(n) | Да |
| [[Пирамидальная-сортировка]] | O(n log n) | O(n log n) | O(1) | Нет |

## Общая реализация на JavaScript

Поскольку этот файл представляет собой обзорный документ, ниже приведен пример общей реализации функции сортировки на JavaScript, которая может использоваться для различных алгоритмов:

```javascript
// Общая функция для демонстрации подхода к сортировке
function demonstrateSort(arr, sortFunction) {
    const arrCopy = [...arr]; // Создаем копию массива, чтобы не изменять оригинал
    const startTime = performance.now();

    const sortedArr = sortFunction(arrCopy);

    const endTime = performance.now();
    console.log(`Сортировка заняла: ${endTime - startTime} миллисекунд`);

    return sortedArr;
}

// Пример использования различных встроенных методов сортировки
function defaultSort(arr) {
    return arr.sort((a, b) => a - b); // Сортировка по возрастанию
}

// Пример псевдокода для общей сортировки:
/*
АЛГОРИТМ ОбщаяСортировка(массив)
    ВХОД: массив - неотсортированный массив элементов
    ВЫХОД: отсортированный массив элементов

    1. КОПИРОВАТЬ(массив) -> рабочийМассив
    2. ПРИМЕНИТЬ_АЛГОРИТМ_СОРТИРОВКИ(рабочийМассив)
    3. ВОЗВРАТИТЬ рабочийМассив
КОНЕЦ АЛГОРИТМА
*/
```

## Применение

Знание алгоритмов сортировки остается критически важным для:

- Прохождения технических собеседований
- Оптимизации производительности приложений
- Работы с большими объемами данных
- Разработки высоконагруженных систем

## Важные аспекты

> [!tip] Практические рекомендации
> - Для малых массивов (n < 50) часто эффективна сортировка вставками
> - [[Быстрая-сортировка]] - выбор по умолчанию для общего использования
> - [[Сортировка-слиянием]] предпочтительна, когда нужна стабильность и гарантированная производительность
> - [[Пирамидальная-сортировка]] полезна, когда ограничена дополнительная память

> [!warning] Важно
> В промышленной разработке редко приходится реализовывать алгоритмы сортировки с нуля - обычно используются встроенные реализации в языках программирования. Однако понимание принципов работы этих алгоритмов критически важно для оптимизации и анализа производительности.
