# Линейный поиск

## Определение

Линейный поиск (Linear Search) - это простой алгоритм поиска, который проверяет каждый элемент в списке последовательно до тех пор, пока не будет найден искомый элемент или не будут проверены все элементы.

## Принцип работы

Алгоритм начинает с первого элемента массива и последовательно сравнивает каждый элемент с искомым значением. Если элемент найден, алгоритм возвращает его индекс. Если весь массив пройден без нахождения элемента, возвращается индикатор отсутствия элемента (обычно -1 или null).

## Псевдокод

```
АЛГОРИТМ Линейный_Поиск(массив, цель)
    ВХОД: массив - список элементов для поиска, цель - искомое значение
    ВЫХОД: индекс элемента или -1, если элемент не найден

    FOR i = 0 ДО ДЛИНА(массив) - 1:
        ЕСЛИ массив[i] = цель:
            ВОЗВРАТИТЬ i  // Возвращаем индекс найденного элемента
        КОНЕЦ ЕСЛИ
    КОНЕЦ FOR

    ВОЗВРАТИТЬ -1  // Элемент не найден
КОНЕЦ АЛГОРИТМА
```

## Реализация на Python

```python
def linear_search(arr, target):
    """
    Линейный поиск элемента в массиве
    :param arr: список элементов для поиска
    :param target: искомое значение
    :return: индекс элемента или -1, если элемент не найден
    """
    for i in range(len(arr)):
        if arr[i] == target:
            return i  # Возвращаем индекс найденного элемента
    return -1  # Элемент не найден

# Пример использования
numbers = [64, 34, 25, 12, 22, 11, 90]
target = 22
result = linear_search(numbers, target)

if result != -1:
    print(f"Элемент {target} найден по индексу {result}")
else:
    print(f"Элемент {target} не найден в массиве")
```

## Реализация на JavaScript

```javascript
function linearSearch(arr, target) {
    for (let i = 0; i < arr.length; i++) {
        if (arr[i] === target) {
            return i;  // Возвращаем индекс найденного элемента
        }
    }
    return -1;  // Элемент не найден
}

// Пример использования
const numbers = [64, 34, 25, 12, 22, 11, 90];
const target = 22;
const result = linearSearch(numbers, target);

if (result !== -1) {
    console.log(`Элемент ${target} найден по индексу ${result}`);
} else {
    console.log(`Элемент ${target} не найден в массиве`);
}
```

## Временная сложность

- **Лучший случай**: O(1) - когда искомый элемент находится в начале массива
- **Средний случай**: O(n) - когда искомый элемент находится в середине массива
- **Худший случай**: O(n) - когда искомый элемент находится в конце массива или отсутствует

## Пространственная сложность

O(1) - алгоритм не требует дополнительной памяти, кроме нескольких переменных.

## Преимущества

- Прост в понимании и реализации
- Работает с любыми типами данных, для которых определено сравнение
- Не требует предварительной сортировки данных
- Работает с любыми структурами данных (массивы, списки, и т.д.)
- Устойчив к ошибкам в данных

## Недостатки

- Низкая эффективность на больших массивах из-за линейной сложности
- Не использует информацию о структуре данных
- Всегда требует в худшем случае n сравнений

## Практическое применение

Линейный поиск используется:

- В образовательных целях для понимания основ алгоритмов поиска
- В системах, где данные несортированы и объем мал (n < 100)
- В embedded системах с ограниченными ресурсами
- При поиске в небольших коллекциях данных
- В прототипировании и начальных этапах разработки

> [!tip] Практический совет
> Линейный поиск может быть приемлемым решением для поиска в небольших коллекциях данных или когда сортировка массива для более эффективного поиска обойдется дороже, чем выполнение линейного поиска.

> [!warning] Важно
> Не рекомендуется использовать линейный поиск для больших наборов данных. В таких случаях предпочтительнее использовать [[Бинарный-поиск]] (если данные отсортированы) или хэш-таблицы.

## Оптимизации

1. **Ранний выход при нахождении**:
```python
def optimized_linear_search(arr, target):
    for i, value in enumerate(arr):
        if value == target:
            return i
    return -1
```

2. **Поиск нескольких вхождений**:
```python
def find_all_occurrences(arr, target):
    """
    Поиск всех вхождений элемента
    :param arr: список элементов для поиска
    :param target: искомое значение
    :return: список индексов всех вхождений
    """
    indices = []
    for i in range(len(arr)):
        if arr[i] == target:
            indices.append(i)
    return indices
```

3. **Сквозной поиск с условием**:
```python
def linear_search_with_condition(arr, condition):
    """
    Поиск элемента по условию
    :param arr: список элементов для поиска
    :param condition: функция-условие для поиска
    :return: индекс первого элемента, удовлетворяющего условию
    """
    for i in range(len(arr)):
        if condition(arr[i]):
            return i
    return -1
```

## Сравнение с другими алгоритмами

- [[Бинарный-поиск]] - более эффективен с O(log n) сложностью, но требует отсортированных данных
- [[Поиск-в-глубину]] - используется для поиска в графах и деревьях
- [[Поиск-в-ширину]] - также используется для поиска в графах, но с другим подходом

## Применение в реальных системах

Линейный поиск может применяться:

1. **В системах управления** - для поиска элементов в небольших списках
2. **В embedded системах** - где важна простота реализации
3. **В прототипах** - как временное решение до оптимизации
4. **В системах безопасности** - для проверки небольших списков разрешений

## Заключение

Линейный поиск - это простой, но важный алгоритм, который служит основой для понимания более сложных методов поиска. Знание этого алгоритма важно как для понимания основ, так и для принятия обоснованных решений о его применимости в конкретных ситуациях.
