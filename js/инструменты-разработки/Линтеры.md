# Линтеры

## Введение

Линтеры - это инструменты статического анализа кода, которые помогают выявлять потенциальные проблемы, несоответствия стилю и возможные ошибки в JavaScript-коде до его выполнения. В 2025 году в российской разработке линтеры стали неотъемлемой частью процесса разработки и сборки проектов, обеспечивая консистентность кода и предотвращая распространенные ошибки.

## Основные линтеры JavaScript

### ESLint

ESLint остается самым популярным и гибким линтером для JavaScript и TypeScript. Он позволяет настраивать правила под конкретные проекты и команды, поддерживает плагины и может интегрироваться с различными IDE и сборщиками.

#### Установка ESLint

```bash
npm install eslint --save-dev
npx eslint --init
```

#### Пример конфигурации (.eslintrc.js)

```javascript
module.exports = {
    env: {
        browser: true,
        es2021: true,
        node: true
    },
    extends: [
        'eslint:recommended',
        '@typescript-eslint/recommended', // для TypeScript проектов
        'plugin:react/recommended' // для React проектов
    ],
    parserOptions: {
        ecmaVersion: 'latest',
        sourceType: 'module'
    },
    plugins: [
        'react',
        '@typescript-eslint'
    ],
    rules: {
        'no-console': 'warn',
        'no-unused-vars': 'error',
        'indent': ['error', 2],
        'quotes': ['error', 'single'],
        'semi': ['error', 'always']
    },
    settings: {
        react: {
            version: 'detect'
        }
    }
};
```

> [!tip] Совет
> В российской практике разработки часто используются предустановленные конфигурации линтеров от крупных компаний, таких как Яндекс (arui-presets) или Тинькофф (eslint-config-tinkoff), чтобы обеспечить единообразие кода в крупных командах.

### JSHint

JSHint - один из старейших линтеров, который предоставляет базовую проверку синтаксиса и распространенных ошибок. Хотя он менее гибкий, чем ESLint, все еще используется в некоторых проектах.

### JSLint

Созданный Дугласом Крокфордом, JSLint был первым JavaScript-линтером. Имеет строгие правила и минимальную настраиваемость, что делает его подходящим для проектов с жесткими стандартами кодирования.

## Интеграция с IDE

### VS Code

Для VS Code доступны расширения для всех популярных линтеров:

```json
// settings.json для VS Code
{
    "eslint.enable": true,
    "eslint.run": "onSave",
    "editor.codeActionsOnSave": {
        "source.fixAll.eslint": true
    }
}
```

### WebStorm/IntelliJ IDEA

Веб-шторм имеет встроенную поддержку линтеров и может автоматически настраивать их при обнаружении конфигурационных файлов в проекте.

## Популярные конфигурации и пресеты

### Airbnb JavaScript Style Guide

Один из самых известных пресетов, обеспечивающий единообразие кода в команде:

```bash
npm install eslint-config-airbnb-base eslint-plugin-import --save-dev
```

```javascript
// .eslintrc.js
module.exports = {
    extends: ['airbnb-base'],
    rules: {
        // переопределение правил при необходимости
    }
};
```

### Standard JS

Конфигурация без настроек - "JavaScript, стиль кодирования без настройки":

```bash
npm install standard --save-dev
```

## Лучшие практики использования линтеров

### 1. Интеграция в процесс сборки

```json
// package.json
{
    "scripts": {
        "lint": "eslint src/**/*.js",
        "lint:fix": "eslint src/**/*.js --fix",
        "prebuild": "npm run lint"
    }
}
```

### 2. Использование pre-commit хуков

```bash
npm install --save-dev husky lint-staged
```

```json
// package.json
{
    "husky": {
        "hooks": {
            "pre-commit": "lint-staged"
        }
    },
    "lint-staged": {
        "*.js": ["eslint --fix", "git add"]
    }
}
```

### 3. Интеграция с CI/CD

```yaml
# GitHub Actions
name: Lint
on: [push, pull_request]
jobs:
  lint:
    runs-on: ubuntu-latest
    steps:
    - uses: actions/checkout@v2
    - name: Setup Node.js
      uses: actions/setup-node@v2
      with:
        node-version: '18'
    - run: npm ci
    - run: npm run lint
```

## Плагины и расширения

### Для React-проектов

```bash
npm install eslint-plugin-react eslint-plugin-react-hooks --save-dev
```

```javascript
// .eslintrc.js
module.exports = {
    extends: [
        'plugin:react/recommended',
        'plugin:react-hooks/recommended'
    ],
    plugins: ['react', 'react-hooks'],
    rules: {
        'react/react-in-jsx-scope': 'off' // для React 17+
    }
};
```

### Для TypeScript-проектов

```bash
npm install @typescript-eslint/eslint-plugin @typescript-eslint/parser --save-dev
```

```javascript
// .eslintrc.js
module.exports = {
    parser: '@typescript-eslint/parser',
    plugins: ['@typescript-eslint'],
    extends: [
        '@typescript-eslint/recommended'
    ]
};
```

## Автоматическое исправление

ESLint может автоматически исправлять некоторые ошибки:

```bash
# Исправить все возможные ошибки
npx eslint src/**/*.js --fix

# Исправить ошибки в VS Code при сохранении
```

|--------|----------|-------------------|---------------------|
