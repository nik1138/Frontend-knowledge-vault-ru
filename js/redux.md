# Redux

Redux — это предсказуемое хранилище состояния для JavaScript-приложений. Он помогает управлять глобальным состоянием приложения и особенно популярен в экосистеме React.

## Основные Концепции

### Store (Хранилище)
Единственный источник истины для состояния приложения. Хранилище содержит всё состояние приложения в одном месте.

### Actions (Действия)
Объекты, описывающие, что произошло в приложении. Actions всегда имеют тип (type) и могут содержать дополнительные данные.

### Reducers (Редьюсеры)
Чистые функции, которые определяют, как состояние приложения изменяется в ответ на действия. Они принимают текущее состояние и действие, а затем возвращают новое состояние.

### Dispatch
Механизм отправки действий в хранилище для изменения состояния.

## Архитектура

Redux следует однонаправленному потоку данных:
1. Представления (Views) вызывают действия (Actions)
2. Действия отправляются в хранилище через dispatch
3. Редьюсеры обновляют состояние на основе действий
4. Приложение перерисовывается на основе нового состояния

## Плюсы Redux

- Предсказуемость: состояние может быть изменено только через действия
- Отладка: Redux DevTools позволяет отслеживать изменения состояния
- Тестируемость: редьюсеры легко тестировать как чистые функции
- Масштабируемость: подходит для больших приложений с сложным состоянием

## Минусы Redux

- Сложность: требует изучения концепций и шаблонов
- Бойлерплейт: необходимость создания действий, редьюсеров и т.д.
- Производительность: может быть избыточным для простых приложений
