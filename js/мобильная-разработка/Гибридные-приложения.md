# Гибридные приложения

Гибридные приложения — это приложения, которые используют веб-технологии (HTML, CSS, JavaScript) для создания интерфейса, но упакованы в нативный контейнер. В 2025 году гибридные приложения остаются популярным выбором для быстрой разработки кроссплатформенных решений.

## Основные фреймворки

### Apache Cordova/PhoneGap

Один из первых и самых известных фреймворков для создания гибридных приложений.

### Ionic

Фреймворк на основе Angular, React или Vue для создания гибридных приложений с нативным внешним видом.

### Capacitor

Современная альтернатива Cordova от команды Ionic, с лучшей поддержкой нативных функций.

## Преимущества гибридных приложений

- Единая кодовая база для iOS и Android
- Быстрая разработка и развертывание
- Использование знакомых веб-технологий
- Возможность доступа к нативным API через плагины

## Недостатки гибридных приложений

- Производительность может быть ниже, чем у нативных приложений
- Ограниченный доступ к некоторым нативным функциям
- Зависимость от фреймворков и плагинов

## Архитектура гибридного приложения

Гибридное приложение состоит из:

- Веб-просмотра (WebView), который отображает HTML/CSS/JavaScript
- Нативного контейнера, который управляет доступом к устройству
- Плагинов, которые обеспечивают связь между веб-частью и нативными API

## Практические примеры

### Создание простого гибридного приложения с использованием Capacitor

```bash
npm install -g @capacitor/cli
capacitor create MyApp
cd MyApp
npm install @capacitor/core @capacitor/cli
capacitor add android
capacitor add ios
```

### Доступ к нативным API

```javascript
import { Plugins } from '@capacitor/core';

const { Geolocation } = Plugins;

const coordinates = await Geolocation.getCurrentPosition();
```
