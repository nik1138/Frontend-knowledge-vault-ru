# Паттерны проектирования в JavaScript

## Обзор

Паттерны проектирования - это проверенные решения для часто встречающихся проблем в разработке программного обеспечения. Они позволяют создавать более гибкий, поддерживаемый и масштабируемый код. В JavaScript особенно важны паттерны, адаптированные под динамическую природу языка.

## Категории паттернов

### Порождающие паттерны

#### Фабричный метод (Factory Method)

Паттерн предоставляет интерфейс для создания экземпляров классов, при этом позволяет подклассам изменять тип создаваемых объектов.

```javascript
class User {
  constructor(name, role) {
    this.name = name;
    this.role = role;
  }
}

class UserFactory {
  static createUser(type, name) {
    switch(type) {
      case 'admin':
        return new User(name, 'administrator');
      case 'moderator':
        return new User(name, 'moderator');
      case 'user':
        return new User(name, 'regular user');
      default:
        throw new Error('Неизвестный тип пользователя');
    }
  }
}

// Использование
const admin = UserFactory.createUser('admin', 'Иван Петров');
```

#### Синглтон (Singleton)

Паттерн гарантирует, что у класса будет только один экземпляр, и предоставляет глобальную точку доступа к нему.

```javascript
class DatabaseConnection {
  constructor() {
    if (DatabaseConnection.instance) {
      return DatabaseConnection.instance;
    }
    
    this.connection = this.createConnection();
    DatabaseConnection.instance = this;
    return this;
  }
  
  createConnection() {
    // Логика создания подключения к базе данных
    console.log('Создание подключения к базе данных');
    return { connected: true };
  }
  
  query(sql) {
    console.log(`Выполнение запроса: ${sql}`);
  }
}

// Использование
const db1 = new DatabaseConnection();
const db2 = new DatabaseConnection();
console.log(db1 === db2); // true
```

### Структурные паттерны

#### Декоратор (Decorator)

Паттерн позволяет динамически добавлять новую функциональность объектам, не изменяя их структуру.

```javascript
// Базовый класс
class Coffee {
  cost() {
    return 100;
  }
  
  description() {
    return 'Обычный кофе';
  }
}

// Декоратор
class MilkDecorator {
  constructor(coffee) {
    this.coffee = coffee;
  }
  
  cost() {
    return this.coffee.cost() + 30;
  }
  
  description() {
    return this.coffee.description() + ', с молоком';
  }
}

class SugarDecorator {
  constructor(coffee) {
    this.coffee = coffee;
  }
  
  cost() {
    return this.coffee.cost() + 10;
  }
  
  description() {
    return this.coffee.description() + ', с сахаром';
  }
}

// Использование
let coffee = new Coffee();
coffee = new MilkDecorator(coffee);
coffee = new SugarDecorator(coffee);

console.log(coffee.description()); // Обычный кофе, с молоком, с сахаром
console.log(coffee.cost()); // 140
```

### Поведенческие паттерны

#### Наблюдатель (Observer)

Паттерн определяет зависимость "один ко многим" между объектами, так что при изменении состояния одного объекта все зависимые объекты уведомляются и обновляются автоматически.

```javascript
class EventEmitter {
  constructor() {
    this.events = {};
  }
  
  on(event, callback) {
    if (!this.events[event]) {
      this.events[event] = [];
    }
    this.events[event].push(callback);
  }
  
  emit(event, data) {
    if (this.events[event]) {
      this.events[event].forEach(callback => callback(data));
    }
  }
  
  off(event, callback) {
    if (this.events[event]) {
      this.events[event] = this.events[event].filter(cb => cb !== callback);
    }
  }
}

// Использование
const emitter = new EventEmitter();

emitter.on('userLogin', (userData) => {
  console.log(`Пользователь ${userData.name} вошел в систему`);
});

emitter.on('userLogin', (userData) => {
  console.log(`Запись в лог: ${userData.name} вошел в систему в ${new Date().toISOString()}`);
});

emitter.emit('userLogin', { name: 'Алексей', id: 123 });
```
