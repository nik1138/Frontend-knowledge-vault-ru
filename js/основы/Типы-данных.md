# Типы данных в JavaScript

JavaScript имеет динамическую типизацию, что означает, что переменные не привязаны к определенному типу данных при объявлении. В 2025 году в JavaScript существует 8 основных типов данных, которые делятся на примитивные и объектные типы.

## Примитивные типы данных

### Number (Число)
Представляет как целые, так и дробные числа. В JavaScript все числа представлены в формате IEEE 754 (64-битный формат с плавающей точкой).

```javascript
let integer = 42;
let float = 3.14;
let negative = -10;
let scientific = 1.23e5; // 123000
let infinity = Infinity;
let notANumber = NaN;
```

> [!tip] Интересный факт
> В JavaScript существует специальное значение `NaN` (Not-a-Number), которое возвращается при попытке выполнить недопустимые математические операции, но оно имеет тип `number`.

### BigInt
Представляет целые числа произвольной длины. Введен в ES2020.

```javascript
let bigNumber = 1234567890123456789012345678901234567890n;
let anotherBig = BigInt("123456789012345678901234567890");
```

### String (Строка)
Представляет текстовые данные. В JavaScript строки неизменяемы.

```javascript
let singleQuote = 'Привет';
let doubleQuote = "Мир";
let templateLiteral = `Значение: ${40 + 2}`;
```

### Boolean (Логический тип)
Представляет логическое значение: `true` или `false`.

```javascript
let isTrue = true;
let isFalse = false;
```

### Undefined
Тип с единственным значением `undefined`, которое присваивается переменной, которой не было присвоено значение.

```javascript
let uninitializedVar;
console.log(uninitializedVar); // undefined
```

### Null
Тип с единственным значением `null`, которое представляет собой "ничего" или "пустое значение".

```javascript
let emptyValue = null;
```

### Symbol (Символ)
Уникальный и неизменяемый тип данных, введенный в ES6. Используется для создания уникальных идентификаторов для свойств объектов.

```javascript
let symbol1 = Symbol('описание');
let symbol2 = Symbol('описание');
console.log(symbol1 === symbol2); // false
```

### Object (Объект)
Объекты являются ссылочными типами данных и включают в себя обычные объекты, массивы, функции и другие специальные типы.

```javascript
let obj = { name: 'Иван', age: 30 };
let arr = [1, 2, 3];
let func = function() { return 'Привет'; };
```

## Проверка типа данных

### Оператор typeof
```javascript
typeof 42;                // "number"
typeof 'hello';           // "string"
typeof true;              // "boolean"
typeof undefined;         // "undefined"
typeof {a: 1};            // "object"
typeof [1, 2, 3];         // "object"
typeof function() {};     // "function"
typeof null;              // "object" (исторический баг)
typeof Symbol();          // "symbol"
typeof 123n;              // "bigint"
```

> [!warning] Важно
> `typeof null` возвращает `"object"`, что является историческим багом JavaScript. Для проверки на `null` используйте строгое равенство: `value === null`.

### Проверка на null
```javascript
function isNull(value) {
    return value === null;
}

function isNullOrUndefined(value) {
    return value == null; // проверяет на null или undefined
}
```

## Преобразование типов

JavaScript автоматически преобразует типы данных в некоторых ситуациях:

```javascript
// Явное преобразование
String(123);        // "123"
Number("456");      // 456
Boolean(0);         // false

// Неявное преобразование
"5" - 3;            // 2 (строка преобразуется в число)
"5" + 3;            // "53" (число преобразуется в строку)
true + 1;           // 2 (true преобразуется в 1)
```

## Ложные значения (falsy values)

Следующие значения интерпретируются как `false` в логическом контексте:

- `false`
- `0`
- `-0`
- `0n` (BigInt)
- `""` (пустая строка)
- `null`
- `undefined`
- `NaN`

```javascript
if ("") {
    console.log("Это не выполнится");
} else {
    console.log("Пустая строка - ложное значение");
}
```

## Особенности работы с типами в 2025 году

С развитием TypeScript и строгой типизации, важно понимать динамическую природу JavaScript. Однако, в 2025 году все еще актуальна проверка типов при работе с внешними API и пользовательским вводом.

```javascript
// Пример проверки типа валидации
function validateInput(input) {
    if (typeof input !== 'string') {
        throw new Error('Ожидается строка');
    }
    return input.trim();
}
```

## Практические рекомендации

1. Используйте `===` вместо `==` для сравнения, чтобы избежать неявного преобразования типов
2. Проверяйте на `null` и `undefined` отдельно, так как `null == undefined` возвращает `true`
3. При работе с числами используйте `Number.isNaN()` вместо глобального `isNaN()` для более точной проверки
4. В российских проектах часто сталкиваются с необходимостью обработки строк в разных кодировках - учитывайте это при работе с текстовыми данными

> [!tip] Совет
> Используйте `Object.prototype.toString.call()` для точной проверки типа, особенно когда работаете с объектами: `Object.prototype.toString.call([])` возвращает `"[object Array]"`.
