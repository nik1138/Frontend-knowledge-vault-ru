# Архитектура серверного рендеринга (SSR)

## Общее понятие

Серверный рендеринг (Server-Side Rendering, SSR) — это подход к разработке веб-приложений, при котором HTML-разметка формируется на сервере и отправляется клиенту полностью готовой для отображения. Это контрастирует с клиентским рендерингом (CSR), где браузер загружает минимальный HTML и JavaScript, который затем генерирует содержимое страницы.

## Преимущества SSR

### SEO оптимизация
- Поисковые роботы получают полностью сформированный HTML
- Контент индексируется корректно без необходимости в сложных системах рендеринга
- В 2025 году в России по-прежнему важна оптимизация под Яндекс, который активно развивает алгоритмы обработки динамического контента

### Улучшенная производительность
- Быстрая первоначальная загрузка страницы (First Contentful Paint)
- Улучшенный пользовательский опыт на медленных соединениях
- Важно для российских регионов с ограниченной пропускной способностью

### Лучшая доступность
- Контент доступен сразу после загрузки
- Улучшенная поддержка скринридеров и вспомогательных технологий

## Архитектурные паттерны

### Традиционный SSR
```
Клиент → Запрос → Сервер → Рендеринг → HTML → Клиент
```
- Подходит для статического контента
- Простая реализация
- Высокая нагрузка на сервер при высоком трафике

### Изоморфный (универсальный) рендеринг
```
Сервер: Рендеринг HTML
Клиент: Гидрация и дальнейшее управление
```
- Одинаковый код для сервера и клиента
- Повторное использование компонентов
- Сложность в управлении состоянием

### Статическая генерация (SSG) + SSR
- Статические страницы генерируются заранее
- Динамический контент рендерится на лету
- Компромисс между производительностью и гибкостью

## Архитектурные компоненты

### Серверный рендерер
- Отвечает за генерацию HTML
- Интеграция с шаблонизаторами или фреймворками
- Обработка запросов и формирование ответа

### Модуль гидрации [[Гидрация]]
- Инициализация клиентской части приложения
- Привязка обработчиков событий
- Восстановление состояния приложения

### Кэширование
- Кэширование готовых HTML-страниц
- Стратегии инвалидации кэша
- CDN интеграция

## Вызовы API в SSR архитектуре

### Серверные API вызовы
- Данные извлекаются на сервере
- Объединяются с шаблонами
- Результат отправляется клиенту

### Клиентские API вызовы
- Дополнительные данные загружаются после гидрации
- Интерактивность приложения
- Обновление состояния в реальном времени
