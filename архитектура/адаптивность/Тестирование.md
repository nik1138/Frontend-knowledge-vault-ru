# Тестирование

Тестирование адаптивности - это процесс проверки корректной работы веб-сайтов и приложений на различных устройствах, размерах экрана и разрешениях. В условиях разнообразия устройств в 2025 году качественное тестирование адаптивности становится критически важным для обеспечения положительного пользовательского опыта.

## Виды тестирования адаптивности

### 1. Визуальное тестирование

Проверка корректного отображения элементов интерфейса на разных размерах экрана:

- Отсутствие обрезания контента
- Сохранение читаемости текста
- Корректное позиционирование элементов
- Сохранение пропорций изображений

### 2. Функциональное тестирование

Проверка работоспособности интерактивных элементов:

- Кликабельность кнопок и ссылок
- Работа форм и валидация
- Навигация по сайту
- Работа JavaScript-функций

### 3. Производительность

Оценка скорости загрузки и отзывчивости:

- Время загрузки страницы
- Время отклика на действия пользователя
- Использование памяти
- Энергопотребление (особенно для мобильных)

#### Selenium WebDriver с Grid
```javascript
const { Builder, Capabilities } = require('selenium-webdriver');

async function testResponsive() {
  // Тестирование на разных размерах окна
  const capabilities = [
    Capabilities.chrome().set('windowSize', { width: 375, height: 667 }), // iPhone
    Capabilities.chrome().set('windowSize', { width: 768, height: 1024 }), // iPad
    Capabilities.chrome().set('windowSize', { width: 1920, height: 1080 }) // Desktop
  ];

  for (const cap of capabilities) {
    const driver = await new Builder().withCapabilities(cap).build();
    try {
      await driver.get('https://example.com');
      // Проверки адаптивности
      await checkElementVisibility(driver);
      await checkLayout(driver);
    } finally {
      await driver.quit();
    }
  }
}
```

#### Puppeteer

async function testOnMultipleDevices() {

## Ключевые точки проверки (breakpoints)

### Стандартные breakpoints для российского рынка (2025)

| Устройство | Ширина экрана | Соотношение | Назначение |
|------------|---------------|-------------|------------|
| Мобильные (малые) | 320px - 480px | 16:9, 4:3 | Бюджетные смартфоны |
| Мобильные (большие) | 481px - 768px | 19.5:9, 18:9 | Современные смартфоны |
| Планшеты | 769px - 1024px | 4:3, 3:2 | Планшеты в портретной ориентации |
| Ноутбуки | 1025px - 1440px | 16:9, 16:10 | Ноутбуки и мониторы |
| Десктопы | 1441px+ | 16:9, 21:9 | Широкоформатные мониторы |

## Практические сценарии тестирования

### 1. Тестирование типографики

```javascript
// Проверка читаемости текста на разных устройствах
async function testTypography(page) {
  // Проверка минимального размера шрифта
  const fontSize = await page.evaluate(() => {
    const body = document.querySelector('body');
    const style = window.getComputedStyle(body);
    return parseFloat(style.fontSize);
  });
  
  // Рекомендуемый минимальный размер для мобильных - 16px
  expect(fontSize).toBeGreaterThanOrEqual(16);
}
```

### 2. Тестирование сенсорных элементов

```javascript
// Проверка размера touch targets
async function testTouchTargets(page) {
  const smallTargets = await page.evaluate(() => {
    const buttons = Array.from(document.querySelectorAll('button, a'));
    return buttons.filter(el => {
      const rect = el.getBoundingClientRect();
      return rect.width < 44 || rect.height < 44; // Рекомендуемый размер 44px
    });
  });
  
  expect(smallTargets.length).toBe(0);
}
```

### 3. Тестирование производительности

```javascript
// Измерение метрик производительности
async function testPerformance(page) {
  const metrics = await page.metrics();
  
  // Проверка использования памяти
  expect(metrics.JSHeapUsedSize).toBeLessThan(100 * 1024 * 1024); // 100MB
  
  // Время загрузки
  const navigationStart = performance.timing.navigationStart;
  const loadComplete = performance.timing.loadEventEnd;
  const loadTime = loadComplete - navigationStart;
  
  expect(loadTime).toBeLessThan(3000); // Максимум 3 секунды
}
```

## Автоматизированные тесты адаптивности

### 1. Визуальное регрессионное тестирование

```javascript
// Пример с использованием Percy
const percySnapshot = require('@percy/puppeteer');

describe('Responsive Design Tests', () => {
  it('should render correctly on different screen sizes', async () => {
    await page.goto('https://example.com');
    
    // Установка разных размеров экрана
    await page.setViewport({ width: 375, height: 667 });
    await percySnapshot(page, 'Mobile View');
    
    await page.setViewport({ width: 768, height: 1024 });
    await percySnapshot(page, 'Tablet View');
    
    await page.setViewport({ width: 1920, height: 1080 });
    await percySnapshot(page, 'Desktop View');
  });
});
```

### 2. Unit-тесты для адаптивных компонентов

```javascript
// Тестирование компонента с использованием Jest и Testing Library
import { render, screen } from '@testing-library/react';
import ResponsiveComponent from './ResponsiveComponent';

describe('ResponsiveComponent', () => {
  it('renders correctly on mobile view', () => {
    // Мокаем размер экрана
    Object.defineProperty(window, 'innerWidth', {
      writable: true,
      configurable: true,
      value: 375,
    });
    
    render(<ResponsiveComponent />);
    
    // Проверяем мобильную версию
    expect(screen.getByTestId('mobile-layout')).toBeInTheDocument();
  });
  
  it('renders correctly on desktop view', () => {
    Object.defineProperty(window, 'innerWidth', {
      writable: true,
      configurable: true,
      value: 1200,
    });
    
    render(<ResponsiveComponent />);
    
    // Проверяем десктопную версию
    expect(screen.getByTestId('desktop-layout')).toBeInTheDocument();
  });
});
```

## Метрики и KPI

### 1. Core Web Vitals (2025)

- **Largest Contentful Paint (LCP)**: < 2.5 секунд
- **First Input Delay (FID)**: < 100 миллисекунд
- **Cumulative Layout Shift (CLS)**: < 0.1

### 2. Дополнительные метрики

- **First Contentful Paint (FCP)**: < 1.5 секунд
- **Time to Interactive (TTI)**: < 3.8 секунд
- **Total Blocking Time (TBT)**: < 300 миллисекунд

## Практические рекомендации

### 1. План тестирования

1. **Подготовительный этап**:
   - Определение целевых устройств
   - Создание списка проверок
   - Подготовка тестовых данных

2. **Исполнительный этап**:
   - Тестирование на эмуляторах/симуляторах
   - Тестирование на реальных устройствах
   - Автоматизированные проверки

3. **Анализ результатов**:
   - Сбор и анализ данных
   - Идентификация проблем
   - Приоритизация исправлений

### 2. Чек-лист адаптивности

- [ ] Отсутствие горизонтального скролла на мобильных
- [ ] Корректное отображение текста (не менее 16px)

- [ ] Правильное отображение изображений
- [ ] Работа всех интерактивных элементов
