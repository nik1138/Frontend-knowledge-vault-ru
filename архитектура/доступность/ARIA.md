# ARIA - Доступные веб-приложения с расширенной семантикой

## Обзор

ARIA (Accessible Rich Internet Applications) - это набор атрибутов, которые позволяют разработчикам создавать более доступные веб-приложения, особенно в тех случаях, когда стандартного HTML-семантического значения недостаточно. В российской практике 2025 года знание и правильное применение ARIA становится критически важным для соответствия требованиям доступности.

## Основные категории ARIA

### 1. Роли (Roles)
Определяют тип элемента и его функцию в интерфейсе:

- `role="button"` - для элементов, которые визуально функционируют как кнопки, но не используют тег `<button>`
- `role="navigation"` - для областей навигации
- `role="main"` - для основного содержимого страницы
- `role="banner"` - для заголовка сайта
- `role="contentinfo"` - для информационной области (футер)
- `role="complementary"` - для дополнительного контента
- `role="search"` - для поисковой формы

### 2. Свойства (Properties)
Описывают характеристики элемента:

- `aria-label` - предоставляет текстовую метку элементу
- `aria-labelledby` - связывает элемент с другим элементом, который служит его меткой
- `aria-describedby` - указывает на элемент, описывающий текущий элемент
- `aria-haspopup` - указывает, что элемент имеет всплывающее окно
- `aria-expanded` - показывает состояние развернутости/свернутости элемента

### 3. Состояния (States)
Отражают текущее состояние элемента:

- `aria-checked` - состояние чекбокса или радиокнопки
- `aria-disabled` - отключенное состояние элемента
- `aria-hidden` - скрытие элемента от вспомогательных технологий
- `aria-invalid` - указание на ошибку ввода
- `aria-required` - обязательность поля ввода

## Практические примеры

### Кнопка с ARIA-ролью
```html
<div role="button" tabindex="0" aria-label="Закрыть окно" onclick="closeWindow()">
  ×
</div>
```

### Вспомогательная информация
```html
<label for="email">Email:</label>
<input type="email" id="email" aria-describedby="email-help">
<div id="email-help" class="help-text">
  Пожалуйста, введите действительный email адрес
</div>
```

### Прогресс-бар
```html
<div role="progressbar" 
     aria-valuenow="50" 
     aria-valuemin="0" 
     aria-valuemax="100" 
     aria-label="Прогресс загрузки">
  <div style="width: 50%"></div>
</div>
```

### Модальное окно
```html
<div role="dialog" 
     aria-labelledby="dialog-title" 
     aria-modal="true" 
     class="modal">
  <h2 id="dialog-title">Подтверждение действия</h2>
  <p>Вы действительно хотите удалить этот элемент?</p>
  <button>Да</button>
  <button>Нет</button>
</div>
```

## Лучшие практики

### 1. Используйте семантический HTML в первую очередь
ARIA должен использоваться как дополнение к семантическому HTML, а не замена ему:
- Используйте `<button>` вместо `role="button"`
- Используйте `<nav>` вместо `role="navigation"`
- Используйте `<header>`, `<main>`, `<footer>` вместо соответствующих ролей

### 2. Избегайте конфликта ролей
Не используйте ARIA-роли, которые противоречат семантике HTML-элемента:
```html
<!-- НЕПРАВИЛЬНО -->
<button role="link">Нажми меня</button>

<!-- ПРАВИЛЬНО -->
<a href="#" role="button">Нажми меня</a>
```

### 3. Обновляйте ARIA-атрибуты динамически
При изменении состояния элемента обновляйте соответствующие ARIA-атрибуты:
```javascript
// Обновление состояния чекбокса
checkbox.setAttribute('aria-checked', isChecked ? 'true' : 'false');

// Обновление прогресс-бара
progressBar.setAttribute('aria-valuenow', currentValue);
```

### 4. Тестируйте с вспомогательными технологиями
Регулярно проверяйте, как ARIA-атрибуты интерпретируются скринридерами:
- NVDA (популярный в России)
- JAWS
- VoiceOver (на Mac)
- TalkBack (на Android)

## Распространенные ошибки

1. **Избыточные роли** - добавление ролей к элементам, которые уже имеют соответствующую семантику
2. **Неправильные имена** - использование `aria-label` с пустыми или бессмысленными значениями
3. **Нарушение иерархии** - неправильное использование заголовков с ARIA-ролями
4. **Отсутствие фокуса** - элементы с ARIA-ролями, которые недоступны с клавиатуры

## Российские особенности

В 2025 году в России:
- Увеличилось количество пользователей скринридеров NVDA и JAWS
- Ужесточились требования к доступности государственных сайтов
- Повысилась осведомленность о доступности среди разработчиков
- Введены штрафные санкции за несоответствие сайтов требованиям доступности

## Заключение

ARIA - мощный инструмент для создания доступных веб-приложений, но требует тщательного понимания и осторожного применения. Правильное использование ARIA-атрибутов значительно улучшает опыт пользователей с ограниченными возможностями и помогает соответствовать требованиям российского законодательства о доступности.
