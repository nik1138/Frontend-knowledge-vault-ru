# Доступная навигация в веб-приложениях

## Обзор

Доступная навигация - это критический аспект веб-доступности, обеспечивающий возможность использования веб-приложений пользователями, которые не используют мышь или сенсорные устройства. Это включает пользователей со скринридерами, пользователей с моторными ограничениями, а также тех, кто предпочитает клавиатурную навигацию. В российском контексте 2025 года доступная навигация является обязательным требованием для соответствия законодательству о доступности.

## Принципы доступной навигации

### 1. Клавиатурная доступность
Все функции интерфейса должны быть доступны с клавиатуры без использования мыши:
- Каждый интерактивный элемент должен быть доступен через Tab
- Порядок фокуса должен быть логичным и последовательным
- Состояние фокуса должно быть визуально очевидным

### 2. Визуальное обозначение фокуса
- Обязательно визуальное выделение элемента, находящегося в фокусе
- Использование контрастного выделения (минимум 3:1 по отношению к фону)
- Не отключать стандартное обозначение фокуса без адекватной замены

### 3. Пропуск навигации
- Предоставление возможности пропустить навигацию к основному содержимому
- Использование якорных ссылок для быстрого перехода
- Реализация "skip links" для улучшения эффективности навигации

## Технические аспекты

### Порядок фокуса
```html
<!-- Правильный порядок фокуса соответствует логической структуре страницы -->
<header>
  <nav>
    <a href="/">Главная</a>
    <a href="/about">О нас</a>
    <a href="/contact">Контакты</a>
  </nav>
</header>
<main>
  <article>
    <h1>Заголовок статьи</h1>
    <p>Содержимое статьи...</p>
  </article>
</main>
<aside>
  <h2>Боковая панель</h2>
  <a href="/related">Связанные статьи</a>
</aside>
```

### Управление фокусом в интерактивных компонентах

#### Аккордеон
```html
<div class="accordion">
  <button aria-expanded="false" aria-controls="panel1" id="accordion1">
    Заголовок аккордеона
  </button>
  <div id="panel1" role="region" aria-labelledby="accordion1" hidden>
    <p>Содержимое аккордеона</p>
  </div>
</div>
```

```javascript
// Управление фокусом в аккордеоне
const accordionButton = document.getElementById('accordion1');
const accordionPanel = document.getElementById('panel1');

accordionButton.addEventListener('click', function() {
  const expanded = this.getAttribute('aria-expanded') === 'true';
  
  this.setAttribute('aria-expanded', !expanded);
  accordionPanel.hidden = expanded;
  
  if (!expanded) {
    // При открытии фокус переходит к содержимому
    accordionPanel.focus();
  }
});
```

#### Модальные окна
```html
<button id="open-modal">Открыть модальное окно</button>

<div id="modal" role="dialog" aria-modal="true" aria-labelledby="modal-title" hidden>
  <div class="modal-content">
    <h2 id="modal-title">Заголовок модального окна</h2>
    <p>Содержимое модального окна</p>
    <button id="close-modal">Закрыть</button>
  </div>
</div>
```

```javascript
// Управление фокусом в модальном окне
const modal = document.getElementById('modal');
const openButton = document.getElementById('open-modal');
const closeButton = document.getElementById('close-modal');
let previousActiveElement;

openButton.addEventListener('click', function() {
  previousActiveElement = document.activeElement;
  modal.hidden = false;
  closeButton.focus(); // Устанавливаем фокус на первый интерактивный элемент
  
  // Ограничиваем фокус в пределах модального окна
  trapFocus(modal);
});

closeButton.addEventListener('click', function() {
  modal.hidden = true;
  previousActiveElement.focus(); // Возвращаем фокус на элемент, который открыл модальное окно
});

function trapFocus(element) {
  const focusableElements = element.querySelectorAll(
    'button, [href], input, select, textarea, [tabindex]:not([tabindex="-1"])'
  );
  const firstElement = focusableElements[0];
  const lastElement = focusableElements[focusableElements.length - 1];

  element.addEventListener('keydown', function(e) {
    if (e.key === 'Tab') {
      if (e.shiftKey && document.activeElement === firstElement) {
        e.preventDefault();
        lastElement.focus();
      } else if (!e.shiftKey && document.activeElement === lastElement) {
        e.preventDefault();
        firstElement.focus();
      }
    }
  });
}
```

## Типы навигации

### 1. Линейная навигация
- Последовательное перемещение по элементам страницы
- Использование Tab/Shift+Tab для навигации
- Поддержка клавиш со стрелками для дополнительной навигации

### 2. Деревовидная навигация
- Использование стрелок для навигации по вложенным элементам
- Поддержка клавиш Home/End для быстрого перехода
- Применение в меню и списках с вложенными элементами

### 3. Карточная навигация
- Навигация между карточками контента
- Использование стрелок для перемещения по карточкам
- Поддержка Page Up/Page Down для быстрого пролистывания

## Пропуск навигации

### Skip links
```html
<a href="#main-content" class="skip-link">Перейти к основному содержимому</a>
<a href="#main-navigation" class="skip-link">Перейти к навигации</a>

<nav id="main-navigation">
  <!-- Навигационные ссылки -->
</nav>

<main id="main-content">
  <!-- Основное содержимое -->
</main>
```

```css
.skip-link {
  position: absolute;
  top: -40px;
  left: 6px;
  background: #000;
  color: #fff;
  padding: 8px;
  text-decoration: none;
  border-radius: 4px;
  z-index: 1000;
}

.skip-link:focus {
  top: 6px;
}
```

## Практические рекомендации

### 1. Использование tabindex
- `tabindex="0"` - элемент становится доступным для фокуса в нормальном порядке
- `tabindex="-1"` - элемент доступен программно, но не в нормальном порядке фокуса
- Избегайте `tabindex` больше 0 (нарушает логический порядок фокуса)

### 2. Семантические элементы навигации
```html
<!-- Используйте семантические элементы для навигации -->
<nav aria-label="Основная навигация">
  <ul>
    <li><a href="/">Главная</a></li>
    <li><a href="/services">Услуги</a></li>
    <li><a href="/about">О нас</a></li>
  </ul>
</nav>

<nav aria-label="Навигация по странице">
  <ol>
    <li><a href="#section1">Раздел 1</a></li>
    <li><a href="#section2">Раздел 2</a></li>
  </ol>
</nav>
```

### 3. Интерактивные компоненты
- Все кнопки, ссылки и интерактивные элементы должны быть доступны с клавиатуры
- Используйте `role="button"` только если нельзя использовать тег `<button>`
- Обрабатывайте события клавиатуры (Enter, Space, Escape) в интерактивных компонентах

### 4. Визуальная обратная связь
```css
/* Обеспечьте видимое обозначение фокуса */
button:focus,
input:focus,
a:focus {
  outline: 2px solid #005fcc;
  outline-offset: 2px;
}

/* Альтернативный стиль фокуса для пользователей, предпочитающих уменьшенную анимацию */
@media (prefers-reduced-motion: reduce) {
  *:focus {
    transition: none;
  }
}
```

## Российские особенности

В 2025 году в России:
- Увеличилось количество пользователей, использующих клавиатурную навигацию
- Ужесточились требования к доступности государственных сайтов
- Повысилась осведомленность о важности доступной навигации среди разработчиков
- Ведущие российские компании внедряют стандарты доступной навигации

## Тестирование доступной навигации

### Ручное тестирование
1. Навигация только с клавиатуры (без мыши)
2. Проверка порядка фокуса
3. Проверка видимости фокуса
4. Проверка функциональности всех интерактивных элементов
5. Проверка работы "skip links"

### Инструменты для тестирования
- **axe-core** - библиотека для автоматического тестирования доступности
- **WAVE** - веб-инструмент для анализа доступности
- **Lighthouse** - встроенная проверка в Chrome DevTools
- **NVDA** - скринридер для тестирования с точки зрения пользователя

### Критерии успеха
- Все интерактивные элементы доступны с клавиатуры
- Порядок фокуса логичен и последователен
- Состояние фокуса визуально очевидно
- Отсутствуют "ловушки" фокуса
- Работают "skip links" для обхода навигации

## Заключение

Доступная навигация - это не только требование законодательства, но и важный аспект инклюзивного дизайна. Правильная реализация навигации с клавиатуры улучшает опыт всех пользователей, а не только тех, кто зависит от клавиатурной навигации. В условиях российского законодательства 2025 года, обеспечение доступной навигации становится обязательным элементом разработки современных веб-приложений.
