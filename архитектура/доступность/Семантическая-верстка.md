# Семантическая верстка для доступных веб-приложений

## Обзор

Семантическая верстка - это подход к созданию HTML-разметки, при котором элементы используются в соответствии с их смысловым значением. Это основа доступности веб-приложений, так как позволяет вспомогательным технологиям правильно интерпретировать и представлять контент пользователям. В российском контексте 2025 года семантическая верстка является обязательным требованием для соответствия ГОСТ Р 52872-2012 и федеральному закону №419-ФЗ.

## Основные семантические элементы HTML

### Структурные элементы
- `<header>` - заголовок раздела или документа
- `<nav>` - навигационные ссылки
- `<main>` - основное содержимое документа
- `<article>` - самостоятельный контент
- `<section>` - тематическая группа контента
- `<aside>` - дополнительный контент
- `<footer>` - нижний колонтитул раздела или документа

### Текстовые элементы
- `<h1>`-`<h6>` - заголовки различных уровней
- `<p>` - абзацы
- `<strong>` - важный текст
- `<em>` - выделенный текст
- `<mark>` - выделенный маркером текст
- `<time>` - дата и время
- `<address>` - контактная информация

### Интерактивные элементы
- `<button>` - кнопка
- `<input>` - поле ввода
- `<select>` - выпадающий список
- `<textarea>` - многострочное поле ввода
- `<a>` - ссылка
- `<label>` - метка для элемента формы

### Группировка элементов
- `<ul>` - неупорядоченный список
- `<ol>` - упорядоченный список
- `<li>` - элемент списка
- `<dl>` - список определений
- `<dt>` - термин в списке определений
- `<dd>` - определение в списке определений

## Правильная иерархия заголовков

### Принципы структурирования
1. Используйте только один `<h1>` на странице - это основной заголовок
2. Не пропускайте уровни заголовков (например, переход от `<h2>` к `<h4>`)
3. Каждый заголовок должен отражать структуру контента
4. Заголовки должны быть информативными и краткими

### Пример правильной иерархии
```html
<h1>Основной заголовок страницы</h1>
  <h2>Раздел 1</h2>
    <h3>Подраздел 1.1</h3>
    <h3>Подраздел 1.2</h3>
  <h2>Раздел 2</h2>
    <h3>Подраздел 2.1</h3>
      <h4>Подраздел 2.1.1</h4>
```

## Семантические паттерны

### Навигация
```html
<nav aria-label="Основная навигация">
  <ul>
    <li><a href="/">Главная</a></li>
    <li><a href="/about">О нас</a></li>
    <li><a href="/services">Услуги</a></li>
    <li><a href="/contact">Контакты</a></li>
  </ul>
</nav>
```

### Форма с семантической разметкой
```html
<form action="/submit" method="post">
  <fieldset>
    <legend>Личная информация</legend>
    <div>
      <label for="name">Имя:</label>
      <input type="text" id="name" name="name" required>
    </div>
    <div>
      <label for="email">Email:</label>
      <input type="email" id="email" name="email" required>
    </div>
  </fieldset>
</form>
```

### Статья с комментариями
```html
<article>
  <header>
    <h1>Заголовок статьи</h1>
    <p>Опубликовано: <time datetime="2025-01-15">15 января 2025</time></p>
  </header>
  <p>Содержимое статьи...</p>
  <footer>
    <p>Автор: Иван Иванов</p>
  </footer>
</article>

<section aria-labelledby="comments-title">
  <h2 id="comments-title">Комментарии</h2>
  <article>
    <header>
      <h3>Комментарий пользователя</h3>
      <p>Опубликовано: <time datetime="2025-01-16T10:30">16 января 2025</time></p>
    </header>
    <p>Текст комментария...</p>
  </article>
</section>
```

## Лучшие практики

### 1. Используйте правильные элементы для правильных целей
- Используйте `<button>` для кнопок, а не `<div>` или `<span>`
- Используйте `<nav>` для навигации
- Используйте `<article>` для самостоятельного контента
- Используйте `<section>` для тематических разделов

### 2. Обеспечьте доступность форм
- Всегда используйте `<label>` для полей ввода
- Используйте `fieldset` и `legend` для группировки связанных полей
- Указывайте типы ввода (`type="email"`, `type="tel"` и т.д.)
- Используйте `required`, `min`, `max` и другие атрибуты для валидации

### 3. Создавайте осмысленные альтернативы
- Используйте осмысленные `alt` тексты для изображений
- Для декоративных изображений используйте пустой `alt=""`
- Для информативных изображений описывайте содержимое

### 4. Обеспечьте контекст для ссылок
- Не используйте общие тексты вроде "нажмите здесь" или "подробнее"
- Сделайте текст ссылки информативным
- Добавьте контекст с помощью `aria-label` при необходимости

## Примеры хорошей семантической верстки

### Структура веб-страницы
```html
<!DOCTYPE html>
<html lang="ru">
<head>
  <meta charset="UTF-8">
  <title>Заголовок страницы</title>
</head>
<body>
  <header>
    <h1>Название сайта</h1>
    <nav aria-label="Основная навигация">
      <ul>
        <li><a href="/">Главная</a></li>
        <li><a href="/catalog">Каталог</a></li>
      </ul>
    </nav>
  </header>

  <main>
    <article>
      <header>
        <h1>Заголовок статьи</h1>
      </header>
      <section>
        <h2>Первый раздел</h2>
        <p>Содержимое раздела...</p>
      </section>
      <section>
        <h2>Второй раздел</h2>
        <p>Содержимое второго раздела...</p>
      </section>
    </article>
  </main>

  <aside>
    <h2>Связанные статьи</h2>
    <ul>
      <li><a href="/related1">Статья 1</a></li>
      <li><a href="/related2">Статья 2</a></li>
    </ul>
  </aside>

  <footer>
    <p>&copy; 2025 Название компании. Все права защищены.</p>
  </footer>
</body>
</html>
```

## Российские особенности

В 2025 году в России:
- Увеличилось внимание к доступности государственных и муниципальных сайтов
- Повысилась осведомленность разработчиков о важности семантической верстки
- Появились дополнительные требования к семантической структуре в обновленных ГОСТах
- Российские скринридеры лучше поддерживают современную семантическую разметку

## Проверка семантической разметки

### Инструменты для проверки
1. **WAVE** - веб-инструмент для анализа доступности
2. **axe DevTools** - браузерное расширение для проверки семантики
3. **Lighthouse** - встроенная проверка в Chrome DevTools
4. **HTML Validator** - проверка валидности HTML-разметки

### Ручная проверка
1. Использование скринридера (NVDA, JAWS)
2. Проверка навигации с клавиатуры
3. Анализ структуры заголовков
4. Проверка корректности использования семантических элементов

## Заключение

Семантическая верстка - это фундамент доступного веб-приложения. Правильное использование семантических HTML-элементов делает контент понятным для всех пользователей, включая тех, кто использует вспомогательные технологии. В условиях ужесточения требований к доступности в России, семантическая верстка становится не просто хорошей практикой, а юридической необходимостью.
