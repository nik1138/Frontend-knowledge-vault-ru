# API-интеграция

## Обзор

API-интеграция — это процесс подключения фронтенд-приложения к внешним API для получения и отправки данных. В 2025 году это стало стандартной практикой при разработке веб-приложений, особенно в условиях роста микросервисной архитектуры и необходимости интеграции с различными внешними сервисами.

## Виды API

### REST API
REST (Representational State Transfer) — архитектурный стиль, основанный на использовании HTTP-методов (GET, POST, PUT, DELETE) для взаимодействия с ресурсами. REST API популярны благодаря своей простоте и широкому распространению.

```javascript
// Пример запроса к REST API
fetch('/api/users')
  .then(response => response.json())
  .then(data => console.log(data));
```

### GraphQL
GraphQL — язык запросов, позволяющий клиенту запрашивать только нужные данные. Популярен в 2025 году благодаря гибкости и эффективности при работе с большими объемами данных.

```javascript
// Пример GraphQL-запроса
const query = `
  query {
    user(id: "1") {
      name
      email
    }
  }
`;
```

### gRPC
gRPC — высокопроизводительный RPC-фреймворк от Google, использующий Protocol Buffers. Подходит для микросервисов и систем с высокими требованиями к производительности.

## Безопасность API

### Аутентификация
- **OAuth 2.0** — стандарт для авторизации, часто используется с внешними сервисами.
- **JWT (JSON Web Tokens)** — токены, используемые для аутентификации и передачи информации между клиентом и сервером.
- **API-ключи** — простой способ аутентификации, но менее безопасный.

### Защита от атак
- **CORS (Cross-Origin Resource Sharing)** — настройка политики доступа к ресурсам с других доменов.
- **Rate Limiting** — ограничение количества запросов для предотвращения DDoS-атак.
- **Валидация данных** — проверка входных данных на клиенте и сервере.

## Управление ошибками

### Обработка HTTP-статусов
```javascript
fetch('/api/data')
  .then(response => {
    if (!response.ok) {
      throw new Error(`HTTP error! status: ${response.status}`);
    }
    return response.json();
  })
  .catch(error => console.error('Error:', error));
```

### Повторные попытки запросов
Использование библиотек или собственных реализаций для автоматических повторных попыток при временных ошибках.

## Российские реалии 2025

### Локализация и требования к данным
- Соблюдение закона о персональных данных (152-ФЗ).
- Необходимость хранения персональных данных на серверах, расположенных в РФ.
- Требования к защите информации (ФЗ-187).

### Работа с отечественными API
- Интеграция с API отечественных сервисов (например, Госуслуги, Сбер, Яндекс).
- Использование российских CDN и облачных решений.

## Лучшие практики

- Использование промежуточного слоя для управления API-запросами.
- Кэширование данных для улучшения производительности.
- Документирование API с использованием OpenAPI/Swagger.
- Мониторинг и логирование запросов.
