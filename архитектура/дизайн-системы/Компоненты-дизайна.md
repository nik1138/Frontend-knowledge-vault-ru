# Компоненты дизайна

## Введение

Компоненты дизайна — это повторно используемые визуальные элементы интерфейса, которые формируют основу пользовательского опыта. В рамках дизайн-системы они обеспечивают консистентность, ускоряют разработку и упрощают поддержку интерфейсов. В российских реалиях 2025 года особое внимание уделяется локализации, доступности и соответствию отечественным стандартам.

## Классификация компонентов

### Примитивные (атомарные) компоненты

- **Button** (Кнопка): Основной элемент взаимодействия.
- **Input** (Поле ввода): Для ввода текста, чисел и т.д.
- **Label** (Метка): Текст, описывающий назначение другого элемента.
- **Icon** (Иконка): Визуальное обозначение функции или информации.
- **Text** (Текст): Элементы типографики (заголовки, параграфы, ссылки).

### Молекулы (составные элементы)

- **Input Group** (Группа полей): Комбинация полей ввода с метками.
- **Button Group** (Группа кнопок): Несколько кнопок, объединённых по функционалу.
- **Form Field** (Поле формы): Инпут с меткой, подсказкой и валидацией.

### Организмы (сложные блоки)

- **Card** (Карточка): Контейнер с информацией.
- **Modal** (Модальное окно): Всплывающее окно для взаимодействия.
- **Navigation** (Навигация): Меню и навигационные элементы.
- **Header** (Шапка): Верхняя часть страницы с логотипом и навигацией.
- **Footer** (Футер): Нижняя часть страницы.

### Шаблоны и страницы

- **Page Templates** (Шаблоны страниц): Каркасы для различных типов страниц.
- **Full Page Examples** (Примеры страниц): Полные реализации страниц с использованием компонентов.

## Принципы разработки компонентов

### 1. Атомарный дизайн

Следование методологии атомарного дизайна:
- Атомы → Молекулы → Организмы → Шаблоны → Страницы

### 2. Доступность (Accessibility)

- Соответствие стандартам [[WCAG]].
- Поддержка клавиатурной навигации.
- Корректная семантика HTML.
- ARIA-атрибуты для улучшения доступности.

### 3. Адаптивность и отзывчивость

- Работа на различных устройствах и размерах экрана.
- Использование гибких сеток и медиа-запросов.

### 4. Тестируемость

- Юнит-тесты для логики компонентов.
- Скриншотные тесты для визуальной регрессии.
- Тесты доступности.

### 5. Документирование

- Примеры использования в [[Storybook]].
- Описание свойств (props) и событий.
- Рекомендации по использованию.

## Архитектура компонентов

### Структура файлов

```
components/
├── Button/
│   ├── Button.tsx
│   ├── Button.styles.ts
│   ├── Button.test.tsx
│   └── index.ts
├── Input/
│   ├── Input.tsx
│   ├── Input.styles.ts
│   ├── Input.test.tsx
│   └── index.ts
└── ...
```

### Импорт и экспорт

- Использование barrel-файлов (`index.ts`) для удобства импорта.
- Экспорт компонентов через центральный файл дизайн-системы.

## Практические рекомендации

### 1. Использование TypeScript

- Типизация свойств компонентов для предотвращения ошибок.
- Создание интерфейсов для сложных компонентов.

### 2. Стилизация

- CSS-in-JS (styled-components, Emotion) для динамических стилей.
- CSS-модули для изоляции стилей.
- Использование [[Токены-дизайна]] для значений цветов, размеров и т.д.

### 3. Управление состоянием

- Внутреннее состояние компонентов (например, состояние фокуса).
- Интеграция с внешними системами управления состоянием (Redux, Zustand и т.д.).

## Заключение

Компоненты дизайна — ключевой элемент современной фронтенд-архитектуры. Их правильная разработка и использование позволяют создавать консистентные, доступные и масштабируемые интерфейсы. В условиях российского рынка 2025 года особое значение приобретают локализация, доступность и соответствие отечественным стандартам.
