# Токены дизайна

## Введение

Токены дизайна — это атомарные значения, используемые для определения визуальных стилей в дизайн-системе. Они представляют собой именованные переменные для цветов, размеров, шрифтов, теней и других стилевых свойств. В российских реалиях 2025 года токены дизайна играют важную роль в обеспечении консистентности, доступности и возможности быстрой персонализации интерфейсов в соответствии с национальными стандартами и требованиями импортозамещения.

## Типы токенов

### Цветовые токены

- **Base Colors** (Базовые цвета): Атомарные цвета (например, `red-500`, `blue-300`).
- **Semantic Colors** (Семантические цвета): Цвета с определённым значением (например, `primary`, `secondary`, `success`, `error`).
- **Theme Colors** (Цвета темы): Цвета, специфичные для светлой/тёмной темы.

### Типографические токены

- **Font Family**: Семейства шрифтов.
- **Font Size**: Размеры шрифтов (например, `text-xs`, `text-lg`).
- **Font Weight**: Начертания шрифтов (например, `font-regular`, `font-bold`).
- **Line Height**: Высота строки.
- **Letter Spacing**: Интервал между буквами.

### Пространственные токены

- **Spacing**: Отступы и интервалы (например, `space-xs`, `space-md`).
- **Border Radius**: Скругления углов (например, `radius-sm`, `radius-full`).
- **Shadow**: Тени (например, `shadow-sm`, `shadow-lg`).
- **Z-index**: Порядок наложения элементов.

### Токены анимаций

- **Duration**: Длительность анимаций.
- **Easing**: Функции плавности анимаций.

## Архитектура токенов

### Файловая структура

```
tokens/
├── colors/
│   ├── base.json
│   ├── semantic.json
│   └── themes.json
├── typography/
│   ├── font-family.json
│   ├── font-size.json
│   └── font-weight.json
├── spacing/
│   └── spacing.json
├── effects/
│   └── shadows.json
└── tokens.json (агрегированный файл)
```

### Принципы именования

Токены именуются по принципу `category-item-modifier`, например:
- `color-primary-base`
- `spacing-padding-small`
- `font-size-heading-large`

## Инструменты для работы с токенами

### Style Dictionary

- Популярный инструмент от Amazon для генерации токенов в различные форматы (CSS, SCSS, JS, JSON и др.).
- Позволяет использовать одни и те же токены для разных платформ и фреймворков.

### Figma Tokens

- Плагины для Figma, позволяющие экспортировать токены из дизайна в код.
- Обеспечивают синхронизацию между дизайном и разработкой.

### Custom Properties (CSS Variables)

- Нативная поддержка в CSS для динамического изменения значений токенов.
- Используются для темизации и персонализации.

## Практические рекомендации

### 1. Создание токенов

- Начинайте с базовых значений (атомарные цвета, размеры).
- Создавайте семантические токены на основе базовых.
- Используйте числовые шкалы для цветов (50, 100, 200... 900).

### 2. Использование токенов

- Использование токенов через CSS custom properties в стилях компонентов.
- Использование токенов в JavaScript для динамических стилей.
- Интеграция с системами стилизации (styled-components, Emotion).

### 3. Темизация

- Создание отдельных файлов токенов для светлой и тёмной темы.
- Использование CSS-медиа-запросов `prefers-color-scheme` или JavaScript для переключения тем.

### 4. Документирование

- Автоматическая генерация документации на основе файлов токенов.
- Примеры использования в [[Storybook]] и других инструментах документации.

## Российские особенности

### Соответствие стандартам

- Учет требований ГОСТ Р 50597-2019 и других отечественных стандартов.
- Обеспечение контрастности цветов в соответствии с [[WCAG]] уровня AA и AAA.

### Локализация

- Возможность адаптации токенов для разных языков (например, изменение размеров шрифтов для кириллических шрифтов).

## Заключение

Токены дизайна — это фундамент для создания консистентной, масштабируемой и доступной дизайн-системы. Их правильное использование позволяет упростить поддержку стилей, обеспечить единообразие интерфейсов и ускорить разработку. В условиях российского рынка 2025 года особое внимание уделяется соответствию отечественным стандартам и требованиям доступности.
