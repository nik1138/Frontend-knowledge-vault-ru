# Системы-сборки

## Обзор

Системы сборки - это инструменты, которые автоматизируют процесс преобразования исходного кода в готовый для использования продукт. Они управляют зависимостями, запускают транспиляцию, объединяют файлы, минимизируют код и выполняют другие задачи, необходимые для подготовки фронтенд-приложений к развертыванию.

## Основные функции систем сборки

- **Управление зависимостями** - загрузка и установка пакетов
- **Транспиляция** - преобразование современного JavaScript в совместимый код
- **Объединение файлов** - бандлинг модулей в один или несколько файлов
- **Минификация** - уменьшение размера файлов
- **Оптимизация** - улучшение производительности
- **Автоматизация** - выполнение задач по триггеру

## Популярные системы сборки

### 1. npm Scripts

> [!tip] npm Scripts
> Простейшая система сборки, встроенная в npm. Используется в большинстве современных проектов.

```json
{
  "scripts": {
    "build": "webpack --mode production",
    "dev": "webpack serve --mode development",
    "lint": "eslint src/",
    "test": "jest"
  }
}
```

### 2. Webpack

> [!info] Webpack
> Мощная система сборки с поддержкой модулей, плагинов и лоадеров.

```javascript
// webpack.config.js
const path = require('path');

module.exports = {
  entry: './src/index.js',
  output: {
    filename: 'bundle.js',
    path: path.resolve(__dirname, 'dist')
  },
  module: {
    rules: [
      {
        test: /\.js$/,
        exclude: /node_modules/,
        use: {
          loader: 'babel-loader'
        }
      },
      {
        test: /\.css$/,
        use: ['style-loader', 'css-loader']
      }
    ]
  },
  plugins: [
    // Плагины для оптимизации и других задач
  ]
};
```

### 3. Vite

> [!info] Vite
> Современная система сборки, обеспечивающая быструю загрузку и HMR (Hot Module Replacement).

```javascript
// vite.config.js
import { defineConfig } from 'vite';
import vue from '@vitejs/plugin-vue';

export default defineConfig({
  plugins: [vue()],
  build: {
    outDir: 'dist',
    rollupOptions: {
      output: {
        assetFileNames: (assetInfo) => {
          if (assetInfo.name.endsWith('.css')) {
            return 'css/[name].[hash].css';
          }
          return '[name].[hash].[ext]';
        }
      }
    }
  }
});
```

## Практические рекомендации

### Оптимизация сборки

1. **Разделение кода** - используйте динамические импорты для разделения приложения на чанки:

```javascript
// Lazy loading компонента
const LazyComponent = () => import('./LazyComponent.vue');
```

2. **Оптимизация зависимостей** - исключайте тяжелые библиотеки из основного бандла:

```javascript
// webpack.config.js
module.exports = {
  externals: {
    jquery: 'jQuery'
  }
};
```

3. **Кэширование** - используйте хэши для имен файлов:

```javascript
// webpack.config.js
output: {
  filename: '[name].[contenthash].js',
  path: path.resolve(__dirname, 'dist')
}
```

### Лучшие практики

- **Используйте кэширование** для ускорения повторных сборок
- **Разделяйте сборки** для development и production
- **Мониторьте размер бандла** с помощью инструментов анализа
- **Оптимизируйте изображения** и статические ресурсы
