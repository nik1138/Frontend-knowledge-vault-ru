# Статический деплой Vue.js приложений

## Обзор

Статический деплой — это наиболее распространенный способ развертывания Vue.js приложений, при котором приложение компилируется в статические файлы HTML, CSS и JavaScript, которые затем размещаются на веб-сервере. Этот подход идеально подходит для приложений, не требующих серверной логики для генерации контента в реальном времени.

## Подготовка приложения к статическому деплою

### Сборка проекта

Для подготовки Vue.js приложения к статическому деплою используется команда сборки:

```bash
npm run build
```

или

```bash
yarn build
```

Эта команда генерирует оптимизированные статические файлы в директории `dist/`, которые можно разместить на любом веб-сервере.

### Конфигурация сборки

В файле `vue.config.js` можно настроить параметры сборки:

```javascript
module.exports = {
  // Установка публичного пути для размещения на подпапке
  publicPath: process.env.NODE_ENV === 'production' ? '/my-app/' : '/',
  
  // Выходная директория
  outputDir: 'dist',
  
  // Директория для ассетов
  assetsDir: 'assets',
  
  // Использование хэшей в именах файлов для кэширования
  filenameHashing: true,
  
  // Минимизация CSS
  css: {
    extract: process.env.NODE_ENV === 'production' ? {
      ignoreOrder: true
    } : false
  }
}
```

## Серверные конфигурации

### Nginx

Пример конфигурации Nginx для обслуживания статических Vue-приложений:

```nginx
server {
    listen 80;
    server_name example.com;
    root /var/www/vue-app;
    index index.html;

    # Обслуживание статических файлов
    location / {
        try_files $uri $uri/ /index.html;
    }

    # Кэширование статических ассетов
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2|ttf|eot)$ {
        expires 1y;
        add_header Cache-Control "public, immutable";
    }

    # Заголовки безопасности
    add_header X-Frame-Options "SAMEORIGIN" always;
    add_header X-XSS-Protection "1; mode=block" always;
    add_header X-Content-Type-Options "nosniff" always;
}
```

### Apache

Конфигурация .htaccess для Apache:

```apache
Options -MultiViews
RewriteEngine On

# Перенаправление всех запросов на index.html
RewriteCond %{REQUEST_FILENAME} !-f
RewriteRule ^ index.html [QSA,L]

# Кэширование ассетов
<IfModule mod_expires.c>
    ExpiresActive on
    ExpiresByType text/css "access plus 1 year"
    ExpiresByType application/javascript "access plus 1 year"
    ExpiresByType image/png "access plus 1 year"
    ExpiresByType image/jpg "access plus 1 year"
    ExpiresByType image/jpeg "access plus 1 year"
    ExpiresByType image/gif "access plus 1 year"
    ExpiresByType image/svg+xml "access plus 1 year"
    ExpiresByType application/font-woff "access plus 1 year"
    ExpiresByType application/font-woff2 "access plus 1 year"
    ExpiresByType application/vnd.ms-fontobject "access plus 1 year"
    ExpiresByType font/ttf "access plus 1 year"
    ExpiresByType font/otf "access plus 1 year"
</IfModule>
```

## Популярные платформы для статического хостинга

### Vercel

Vercel предоставляет отличную поддержку для Vue.js приложений:

1. Создайте репозиторий с вашим Vue-приложением
2. Добавьте в корень проекта файл `vercel.json`:

```json
{
  "builds": [
    {
      "src": "package.json",
      "use": "@vercel/static-build",
      "config": {
        "distDir": "dist"
      }
    }
  ],
  "routes": [
    {
      "src": "/(.*)",
      "dest": "/index.html"
    }
  ]
}
```

### Netlify

Netlify также отлично подходит для хостинга Vue-приложений:

1. Подключите ваш репозиторий к Netlify
2. Укажите команду сборки: `npm run build`
3. Укажите публичную директорию: `dist`

### Яндекс.Облако (Static Files)

В 2025 году Яндекс.Облако активно развивает сервис Static Files для размещения статических сайтов:

1. Загрузите собранные файлы в бакет Object Storage
2. Включите публичный доступ
3. Настройте CDN для ускорения доставки контента
