# Глобальные стили в Vue

## Введение

Глобальные стили в Vue - это CSS-правила, которые применяются ко всему приложению, а не ограничены конкретным компонентом. Они используются для определения базовых стилей, сброса стилей браузера, тем оформления и других общих визуальных характеристик.

## Основы глобальных стилей

Глобальные стили можно определить несколькими способами:

### 1. Внешний CSS-файл

Самый простой способ - подключить CSS-файл через `main.js` или в HTML-шаблоне:

```javascript
// main.js
import './assets/css/global.css'
import { createApp } from 'vue'
import App from './App.vue'

createApp(App).mount('#app')
```

```html
<!-- public/index.html -->
<!DOCTYPE html>
<html>
  <head>
    <link rel="stylesheet" href="/src/assets/css/global.css">
  </head>
  <body>
    <div id="app"></div>
  </body>
</html>
```

### 2. Внутри компонента без атрибута scoped

```vue
<template>
  <div class="my-component">
    <h1>Заголовок</h1>
  </div>
</template>

<style>
/* Эти стили будут применяться глобально */
h1 {
  color: #333;
  font-family: Arial, sans-serif;
}

/* Глобальный класс для всего приложения */
.app-container {
  max-width: 1200px;
  margin: 0 auto;
  padding: 20px;
}
</style>
```

## Структура глобальных стилей

Рекомендуется структурировать глобальные стили следующим образом:

```

├── assets/
│   └── css/
│       ├── base/
│       │   ├── reset.css
│       │   ├── typography.css
│       │   └── utilities.css
│       ├── components/

│       │   └── forms.css
│       ├── themes/
│       │   ├── light.css
│       │   └── dark.css
│       └── global.css
```

### Переключение тем

```javascript
// utils/theme.js
export function setTheme(themeName) {
  document.documentElement.className = themeName;
  localStorage.setItem('theme', themeName);
}

export function initTheme() {
  const savedTheme = localStorage.getItem('theme');
  const prefersDark = window.matchMedia('(prefers-color-scheme: dark)').matches;
  
  const theme = savedTheme || (prefersDark ? 'dark-theme' : 'light-theme');
  setTheme(theme);
}
```

```vue
<!-- App.vue -->
<template>
  <div id="app" :class="currentTheme">
    <header>
      <button @click="toggleTheme">Переключить тему</button>
    </header>
    <main>
      <router-view />
    </main>
  </div>
</template>

<script>
import { initTheme, setTheme } from './utils/theme.js'

export default {
  name: 'App',
  data() {
    return {
      currentTheme: 'light-theme'
    }
  },
  mounted() {
    initTheme();
    this.currentTheme = localStorage.getItem('theme') || 'light-theme';
  },
  methods: {
    toggleTheme() {
      this.currentTheme = this.currentTheme === 'light-theme' ? 'dark-theme' : 'light-theme';
      setTheme(this.currentTheme);
    }
  }
}
</script>
```

## Практические рекомендации

### 1. Использование CSS-переменных

CSS-переменные позволяют легко управлять глобальными стилями:

```css
:root {
  /* Цвета */
  --color-primary: #42b983;
  --color-success: #2ecc71;
  --color-warning: #f39c12;
  --color-danger: #e74c3c;
  
  /* Размеры */
  --border-radius: 4px;
  --border-radius-lg: 8px;
  --border-radius-sm: 2px;
  
  /* Тени */
  --shadow-sm: 0 1px 2px 0 rgba(0, 0, 0, 0.05);
  --shadow: 0 1px 3px 0 rgba(0, 0, 0, 0.1), 0 1px 2px 0 rgba(0, 0, 0, 0.06);
  --shadow-lg: 0 10px 15px -3px rgba(0, 0, 0, 0.1), 0 4px 6px -2px rgba(0, 0, 0, 0.05);
  
  /* Анимации */
  --transition-base: all 0.2s ease-in-out;
  --transition-fast: all 0.1s ease-in-out;
}
```

### 2. Медиа-запросы для адаптивности

```css
/* assets/css/base/responsive.css */
@media (min-width: 640px) {
  .container {
    max-width: 640px;
  }
}

@media (min-width: 768px) {
  .container {
    max-width: 768px;
  }
}

@media (min-width: 1024px) {
  .container {
    max-width: 1024px;
  }
}

@media (min-width: 1200px) {
  .container {
    max-width: 1200px;
  }
}

/* Адаптивные шрифты */
@media (max-width: 768px) {
  h1 { font-size: var(--font-size-3xl); }
  h2 { font-size: var(--font-size-2xl); }
  h3 { font-size: var(--font-size-xl); }
}
```

### 3. Адаптация под российские реалии

```css
/* assets/css/base/localization.css */
/* Учет особенностей русского языка */
body[lang="ru"] {
  font-family: 'Roboto', 'Arial', 'Helvetica', sans-serif;
  /* Увеличенный интерлиньяж для кириллических шрифтов */
  line-height: 1.7;
}

/* Поддержка направления текста */
[dir="rtl"] {
  text-align: right;
}

/* Стили для печати */
@media print {
  body {
    color: black;
    background: white;
  }
  
  .no-print {
    display: none !important;
  }
}
```

## Загрузка глобальных стилей

### В Vue CLI проекте

```javascript
// vue.config.js
module.exports = {
  css: {
    loaderOptions: {
      sass: {
        additionalData: `@import "@/assets/css/variables.scss";`
      }
    }
  }
}
```

### В Vite проекте

```javascript
// vite.config.js
import { defineConfig } from 'vite'
import vue from '@vitejs/plugin-vue'

export default defineConfig({
  plugins: [vue()],
  css: {
    preprocessorOptions: {
      scss: {
        additionalData: `@import "@/assets/css/variables.scss";`
      }
    }
  }
})
```

## Лучшие практики

1. **Минимизация глобальных стилей** - использовать как можно меньше глобальных стилей, чтобы избежать конфликтов
2. **Использование CSS-переменных** - для унификации цветов, размеров и других значений
3. **Модульная структура** - разделять стили на логические блоки
4. **Документирование** - описывать назначение глобальных классов и переменных
5. **Тестирование** - проверять стили на разных устройствах и браузерах

## Совместимость с инструментами

Глобальные стили поддерживаются всеми основными инструментами сборки Vue:

- Vue CLI
- Vite
- Nuxt.js
- Webpack с vue-loader

## Заключение

Глобальные стили - важная часть архитектуры стилизации в Vue-приложениях. Они позволяют задать единые стандарты оформления для всего приложения и обеспечивают согласованность пользовательского интерфейса. При правильной организации глобальные стили значительно упрощают разработку и поддержку больших приложений.
