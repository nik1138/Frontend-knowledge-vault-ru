# Transition

## Обзор

Компонент `<transition>` в Vue предоставляет возможность анимировать появление, исчезновение и изменение DOM-элементов. Это ключевой инструмент для создания плавных пользовательских интерфейсов с визуальной обратной связью. В российской веб-разработке в 2025 году анимации переходов становятся стандартом для повышения пользовательского опыта и конкурентоспособности веб-приложений.

## Основы использования

Компонент `<transition>` оборачивает элемент, который нужно анимировать. Vue автоматически добавляет CSS-классы в определенные моменты жизненного цикла перехода:

- `v-enter-from` - начальное состояние перед появлениями
- `v-enter-active` - активное состояние во время появления
- `v-enter-to` - конечное состояние появления
- `v-leave-from` - начальное состояние перед исчезновением
- `v-leave-active` - активное состояние во время исчезновения
- `v-leave-to` - конечное состояние исчезновения

### Пример базового использования

```vue
<template>
  <div>
    <button @click="show = !show">Переключить</button>
    <transition name="fade">
      <p v-if="show">Текст с анимацией</p>
    </transition>
  </div>
</template>

<script>
export default {
  data() {
    return {
      show: true
    }
  }
}
</script>

<style>
.fade-enter-active, .fade-leave-active {
  transition: opacity 0.5s ease;
}
.fade-enter-from, .fade-leave-to {
  opacity: 0;
}
</style>
```

## Практические рекомендации

### 1. Оптимизация производительности

В условиях ограниченной пропускной способности интернета в некоторых регионах России, важно оптимизировать анимации для плавной работы на слабых устройствах:

- Используйте `transform` и `opacity` вместо изменения геометрических свойств
- Ограничьте количество одновременных анимаций
- Учитывайте предпочтения пользователей относительно анимаций (media query `prefers-reduced-motion`)

### 2. Совместимость с доступностью

С учетом требований российского законодательства об accessibility, рекомендуется:

- Обеспечить возможность отключения анимаций через настройки
- Использовать атрибуты ARIA для описания анимационных состояний
- Проверять контрастность цветов во время анимаций

### 3. Кроссбраузерность

В России до сих пор значительное количество пользователей использует устаревшие браузеры, поэтому:

- Тестировать анимации в различных браузерах
- Обеспечивать деградацию функциональности без анимаций
- Использовать CSS-префиксы при необходимости

## Продвинутые возможности

### События переходов

Компонент `<transition>` предоставляет JavaScript-события для более сложной логики:

- `@before-enter` - перед началом появления
- `@enter` - во время появления
- `@after-enter` - после завершения появления
- `@before-leave` - перед началом исчезновения
- `@leave` - во время исчезновения
- `@after-leave` - после завершения исчезновения

```vue
<template>
  <transition
    @before-enter="beforeEnter"
    @enter="enter"
    @leave="leave"
    :css="false"
  >
    <div v-if="show" class="animated-element">Анимированный элемент</div>
  </transition>
</template>

<script>
export default {
  data() {
    return {
      show: true
    }
  },
  methods: {
    beforeEnter(el) {
      // Подготовка элемента перед анимацией
      el.style.opacity = 0
    },
    enter(el, done) {
      // Анимация появления
      let opacity = 0
      const timer = setInterval(() => {
        opacity += 0.1
        el.style.opacity = opacity
        if (opacity >= 1) {
          clearInterval(timer)
          done()
        }
      }, 50)
    },
    leave(el, done) {
      // Анимация исчезновения
      let opacity = 1
      const timer = setInterval(() => {
        opacity -= 0.1
        el.style.opacity = opacity
        if (opacity <= 0) {
          clearInterval(timer)
          done()
        }
      }, 50)
    }
  }
}
</script>
```

## Заключение

Компонент `<transition>` - мощный инструмент для создания анимаций в Vue-приложениях. Его правильное использование может значительно улучшить пользовательский опыт, особенно в условиях российского рынка, где важна как производительность, так и доступность. Важно учитывать специфику российских пользователей и требования законодательства при реализации анимационных эффектов.
