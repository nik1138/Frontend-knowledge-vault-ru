# Основы роутинга в Vue.js

Роутинг (маршрутизация) в Vue.js позволяет создавать одностраничные приложения (SPA) с навигацией между различными представлениями без перезагрузки страницы. Это ключевой элемент современных веб-приложений.

## Установка Vue Router

Для начала работы с роутингом в Vue.js необходимо установить Vue Router:

```bash
npm install vue-router@4
```

> [!NOTE]
> В 2025 году актуальной версией Vue Router является 4-я версия, которая полностью совместима с Vue 3 и поддерживает Composition API.

## Создание маршрутов

Для создания системы маршрутизации необходимо определить конфигурацию маршрутов:

```javascript
// файл router/index.js
import { createRouter, createWebHistory } from 'vue-router'
import Home from '../views/Home.vue'
import About from '../views/About.vue'

const routes = [
  {
    path: '/',
    name: 'Home',
    component: Home
  },
  {
    path: '/about',
    name: 'About',
    component: About
  }
]

const router = createRouter({
  history: createWebHistory(),
  routes
})

export default router
```

## Подключение роутера к приложению

```javascript
// файл main.js
import { createApp } from 'vue'
import App from './App.vue'
import router from './router'

const app = createApp(App)
app.use(router)
app.mount('#app')
```

## Использование роутера в компонентах

Для отображения содержимого маршрутов используется компонент `<router-view>`:

```vue
<template>
  <div id="app">
    <nav>
      <router-link to="/">Главная</router-link>
      <router-link to="/about">О нас</router-link>
    </nav>
    <router-view />
  </div>
</template>
```

## Типы истории навигации

Vue Router поддерживает несколько типов истории:

1. **createWebHistory** - для современных браузеров с использованием History API
2. **createWebHashHistory** - использует хэш в URL (поддержка старых браузеров)
3. **createMemoryHistory** - для тестирования или работы вне браузера

> [!WARNING]
> В российских реалиях 2025 года рекомендуется использовать `createWebHistory`, так как поддержка современных браузеров в РФ высока, но при работе с устаревшими системами может потребоваться `createWebHashHistory`.

## Навигация программно

Для программной навигации используются методы:

```javascript
// В Composition API
import { useRouter } from 'vue-router'

export default {
  setup() {
    const router = useRouter()
    
    const goToAbout = () => {
      router.push('/about')
      // или
      router.push({ name: 'About' })
    }
    
    return { goToAbout }
  }
}
```

## Параметры маршрута

Для передачи данных через URL используются параметры:

```javascript
const routes = [
  {
    path: '/user/:id',
    name: 'User',
    component: User
  }
]
```

## Мета-данные маршрутов

Можно добавлять мета-данные к маршрутам:

```javascript
{
  path: '/admin',
  component: Admin,
  meta: { requiresAuth: true, role: 'admin' }
}
```

## Асинхронные компоненты

Для оптимизации загрузки приложения рекомендуется использовать асинхронную загрузку компонентов:

```javascript
const Home = () => import('../views/Home.vue')
```
