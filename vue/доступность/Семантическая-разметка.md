# Семантическая разметка в Vue-приложениях

## Введение

Семантическая разметка - это использование HTML-элементов по их прямому назначению для передачи смысла и структуры контента. В контексте Vue-приложений семантическая разметка особенно важна для обеспечения доступности интерфейсов для пользователей с ограниченными возможностями, использующих вспомогательные технологии (скринридеры, клавиатурная навигация и т.д.).

## Основные принципы семантической разметки

### Использование правильных HTML-элементов

Вместо использования универсальных элементов типа `<div>` и `<span>`, следует использовать семантически значимые теги:

```vue
<template>
  <!-- Правильно -->
  <header>
    <nav>
      <ul>
        <li><router-link to="/">Главная</router-link></li>
        <li><router-link to="/about">О нас</router-link></li>
      </ul>
    </nav>
  </header>

  <main>
    <article>
      <h1>{{ title }}</h1>
      <p>{{ content }}</p>
    </article>
  </main>

  <footer>
    <p>&copy; {{ year }} Все права защищены</p>
  </footer>

  <!-- Неправильно -->
  <div>
    <div>
      <div>
        <div>Главная</div>
        <div>О нас</div>
      </div>
    </div>
  </div>

  <div>
    <div>
      <div>{{ title }}</div>
      <div>{{ content }}</div>
    </div>
  </div>

  <div>
    <div>&copy; {{ year }} Все права защищены</div>
  </div>
</template>
```

### Иерархия заголовков

Заголовки должны следовать логической структуре документа:

```vue
<template>
  <article>
    <!-- Основной заголовок страницы -->
    <h1>{{ mainTitle }}</h1>
    
    <section>
      <h2>{{ sectionTitle }}</h2>
      <p>Контент секции</p>
      
      <subsection>
        <h3>{{ subsectionTitle }}</h3>
        <p>Контент подсекции</p>
      </subsection>
    </section>
  </article>
</template>
```

## Семантические элементы в Vue

### Семантические блоки

- `<header>` - шапка страницы или секции
- `<nav>` - навигационные элементы
- `<main>` - основной контент страницы
- `<article>` - независимый контент (статья, пост)
- `<section>` - тематическая группа контента
- `<aside>` - сопутствующий контент
- `<footer>` - нижний колонтитул

```vue
<template>
  <div class="page-wrapper">
    <header class="page-header">
      <h1>Логотип и заголовок сайта</h1>
      <nav aria-label="Основная навигация">
        <ul>
          <li v-for="item in navItems" :key="item.id">
            <router-link :to="item.path">{{ item.name }}</router-link>
          </li>
        </ul>
      </nav>
    </header>

    <main class="main-content">
      <article v-for="post in posts" :key="post.id">
        <header>
          <h2>{{ post.title }}</h2>
          <time :datetime="post.date">{{ formatDate(post.date) }}</time>
        </header>
        <section>
          <p>{{ post.excerpt }}</p>
        </section>
      </article>
    </main>

    <aside class="sidebar">
      <section>
        <h3>Популярные статьи</h3>
        <ul>
          <li v-for="post in popularPosts" :key="post.id">
            <a href="#" @click.prevent="selectPost(post.id)">{{ post.title }}</a>
          </li>
        </ul>
      </section>
    </aside>

    <footer class="page-footer">
      <p>&copy; {{ currentYear }} Название компании</p>
    </footer>
  </div>
</template>
```

## Практические рекомендации

### 1. Избегайте лишней вложенности

```vue
<template>
  <!-- Правильно -->
  <article>
    <h2>{{ title }}</h2>
    <p>{{ content }}</p>
  </article>

  <!-- Избегайте -->
  <div>
    <div>
      <article>
        <div>
          <h2>{{ title }}</h2>
        </div>
        <div>
          <p>{{ content }}</p>
        </div>
      </article>
    </div>
  </div>
</template>
```

### 2. Используйте семантические элементы для форм

```vue
<template>
  <form @submit.prevent="handleSubmit">
    <fieldset>
      <legend>Личная информация</legend>
      
      <label for="name">Имя:</label>
      <input id="name" type="text" v-model="formData.name">
      
      <label for="email">Email:</label>
      <input id="email" type="email" v-model="formData.email">
    </fieldset>
    
    <button type="submit">Отправить</button>
  </form>
</template>
```

### 3. Обработка динамического контента

При работе с динамическим контентом в Vue важно сохранять семантическую структуру:

```vue
<template>
  <div>
    <header v-if="showHeader">
      <h1>{{ pageTitle }}</h1>
    </header>
    
    <main>
      <article v-for="item in items" :key="item.id" :aria-labelledby="`title-${item.id}`">
        <h2 :id="`title-${item.id}`">{{ item.title }}</h2>
        <p>{{ item.description }}</p>
      </article>
    </main>
  </div>
</template>
```

## Особенности для российских разработчиков

### Соответствие требованиям 508 и Веб-контента

Российские веб-сайты, особенно государственные и коммерческие, должны соответствовать требованиям доступности. Семантическая разметка - это первый шаг к соответствию:

- ГОСТ Р 52872-2012 (аналог WCAG 2.0)
- Федеральный закон №419-ФЗ (требования к доступности)

### Локализация заголовков и навигации

```vue
<template>
  <nav aria-label="Навигация по сайту">
    <ul>
      <li><router-link to="/">{{ $t('main') }}</router-link></li>
      <li><router-link to="/about">{{ $t('about') }}</router-link></li>
      <li><router-link to="/services">{{ $t('services') }}</router-link></li>
      <li><router-link to="/contact">{{ $t('contact') }}</router-link></li>
    </ul>
  </nav>
</template>
```
