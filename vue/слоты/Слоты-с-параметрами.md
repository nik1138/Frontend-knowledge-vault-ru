# Слоты с параметрами

Слоты с параметрами (scoped slots) - это мощная возможность Vue.js, позволяющая компоненту передавать данные в слот, который используется родительским компонентом. Это позволяет создавать очень гибкие и переиспользуемые компоненты.

## Основы слотов с параметрами

Слот с параметрами позволяет передавать данные из дочернего компонента в родительский через слот:

```vue
<!-- List.vue -->
<template>
  <ul>
    <li v-for="item in items" :key="item.id">
      <slot :item="item" :index="index" :isEven="index % 2 === 0">
        {{ item.name }}
      </slot>
    </li>
  </ul>
</template>

<script>
export default {
  data() {
    return {
      items: [
        { id: 1, name: 'Элемент 1' },
        { id: 2, name: 'Элемент 2' },
        { id: 3, name: 'Элемент 3' }
      ]
    }
  }
}
</script>
```

Использование компонента:

```vue
<!-- Parent.vue -->
<template>
  <List v-slot="{ item, index, isEven }">
    <span :class="{ even: isEven, odd: !isEven }">
      {{ index + 1 }}. {{ item.name }}
    </span>
  </List>
</template>
```

## Синтаксис слотов с параметрами

В Vue 3 используется синтаксис `v-slot` или его сокращение `#`:

```vue
<!-- Полный синтаксис -->
<List v-slot="{ item, index }">
  <span>{{ item.name }}</span>
</List>

<!-- Сокращенный синтаксис -->
<List #default="{ item, index }">
  <span>{{ item.name }}</span>
</List>
```

## Практические примеры

### Компонент с данными пользователя

```vue
<!-- UserList.vue -->
<template>
  <div class="user-list">
    <div v-for="user in users" :key="user.id" class="user-card">
      <slot 
        :user="user" 
        :formattedName="formatName(user)"
        :canEdit="user.id === currentUserId"
      >
        <h3>{{ formatName(user) }}</h3>
        <p>{{ user.email }}</p>
      </slot>
    </div>
  </div>
</template>

<script>
export default {
  props: ['currentUserId'],
  data() {
    return {
      users: [
        { id: 1, firstName: 'Иван', lastName: 'Петров', email: 'ivan@example.com' },
        { id: 2, firstName: 'Мария', lastName: 'Сидорова', email: 'maria@example.com' }
      ]
    }
  },
  methods: {
    formatName(user) {
      return `${user.firstName} ${user.lastName}`;
    }
  }
}
</script>
```

```vue
<!-- Использование -->
<template>
  <UserList :currentUserId="userId" v-slot="{ user, formattedName, canEdit }">
    <div class="custom-user-card">
      <h3>{{ formattedName }}</h3>
      <p>{{ user.email }}</p>
      <button v-if="canEdit" @click="editUser(user)">Редактировать</button>
    </div>
  </UserList>
</template>
```

### Слоты с параметрами для форматирования

```vue
<!-- DataTable.vue -->
<template>
  <table class="data-table">
    <thead>
      <tr>
        <th v-for="column in columns" :key="column.key">
          {{ column.title }}
        </th>
      </tr>
    </thead>
    <tbody>
      <tr v-for="(row, rowIndex) in data" :key="rowIndex">
        <td v-for="column in columns" :key="column.key">
          <slot 
            :name="`column-${column.key}`"
            :value="row[column.key]"
            :row="row"
            :rowIndex="rowIndex"
            :column="column"
          >
            {{ row[column.key] }}
          </slot>
        </td>
      </tr>
    </tbody>
  </table>
</template>

<script>
export default {
  props: {
    data: Array,
    columns: Array
  }
}
</script>
```

```vue
<!-- Использование -->
<template>
  <DataTable :data="users" :columns="columns">
    <template #column-actions="{ row, rowIndex }">
      <button @click="editUser(row)">Изменить</button>
      <button @click="deleteUser(row.id)">Удалить</button>
    </template>
    <template #column-status="{ value }">
      <span :class="`status-${value}`">{{ value }}</span>
    </template>
  </DataTable>
</template>
```

## Рекомендации по использованию

- Используйте слоты с параметрами для передачи данных из дочернего компонента
- Называйте параметры осмысленно и понятно
- Не передавайте слишком много параметров в один слот
- Слоты с параметрами особенно полезны для компонентов списков и таблиц
- Используйте деструктуризацию в синтаксисе слота для удобства

## Совместимость с российскими реалиями 2025

В российской разработке в 2025 году слоты с параметрами особенно популярны при создании:

- Компонентов для государственных информационных систем
- Административных интерфейсов с сложной логикой отображения
- Финансовых приложений с динамическим форматированием данных
- Систем управления контентом с гибким отображением элементов

> [!tip] Совет
> Слоты с параметрами позволяют создавать очень гибкие компоненты, но не переусердствуйте с количеством передаваемых параметров, чтобы не усложнять использование компонента.
