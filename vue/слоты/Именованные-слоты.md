# Именованные слоты

Именованные слоты (named slots) в Vue.js позволяют создавать более гибкие компоненты, в которых можно контролировать, куда именно будут вставляться дочерние элементы. Это особенно полезно при создании сложных компонентов с несколькими областями для контента.

## Основы именованных слотовов

Именованные слоты создаются с помощью атрибута `name` на элементе `<slot>`:

```vue
<!-- Component.vue -->
<template>
  <div class="card">
    <header>
      <slot name="header"></slot>
    </header>
    <main>
      <slot name="content"></slot>
    </main>
    <footer>
      <slot name="footer"></slot>
    </footer>
  </div>
</template>
```

Использование компонента с именованными слотами:

```vue
<!-- Parent.vue -->
<template>
  <Component>
    <template #header>
      <h1>Заголовок карточки</h1>
    </template>
    <template #content>
      <p>Основной контент карточки</p>
    </template>
    <template #footer>
      <button>Действие</button>
    </template>
  </Component>
</template>
```

## Синтаксис именованных слотовов

В Vue 3 используется сокращенный синтаксис `#` вместо `v-slot:`:

- `v-slot:header` → `#header`
- `v-slot:content` → `#content`
- `v-slot:footer` → `#footer`

## Практические примеры

### Карточка пользователя

```vue
<!-- UserCard.vue -->
<template>
  <div class="user-card">
    <div class="avatar">
      <slot name="avatar"></slot>
    </div>
    <div class="info">
      <slot name="name"></slot>
      <slot name="position"></slot>
    </div>
    <div class="actions">
      <slot name="actions"></slot>
    </div>
  </div>
</template>
```

```vue
<!-- Использование -->
<template>
  <UserCard>
    <template #avatar>
      <img src="avatar.jpg" alt="Аватар" />
    </template>
    <template #name>
      <h3>Иван Петров</h3>
    </template>
    <template #position>
      <p>Frontend разработчик</p>
    </template>
    <template #actions>
      <button>Связаться</button>
    </template>
  </UserCard>
</template>
```

## Рекомендации по использованию

- Используйте именованные слоты для создания многокомпонентных каркасов
- Имена слотов должны быть понятными и описательными
- Избегайте чрезмерного усложнения компонентов с множеством слотов
- Рассмотрите использование [[Слоты-по-умолчанию]] для основного контента

## Совместимость с российскими реалиями 2025

В российской разработке в 2025 году именованные слоты особенно популярны при создании:

- UI-библиотек для государственных проектов
- Административных панелей с ригидной структурой
- Компонентов для корпоративных систем с жесткими требованиями к верстке

> [!tip] Совет
> Используйте именованные слоты для создания переиспользуемых компонентов с фиксированной структурой, но изменяемым содержимым.
