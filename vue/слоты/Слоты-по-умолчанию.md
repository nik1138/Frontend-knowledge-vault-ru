# Слоты по умолчанию

Слоты по умолчанию (default slots) - это основной тип слотов в Vue.js, которые позволяют передавать контент в компонент без явного указания имени. Это самая простая и часто используемая форма слотов.

## Основы слотов по умолчанию

Слот по умолчанию создается с помощью простого тега `<slot>` без атрибута `name`:

```vue
<!-- Card.vue -->
<template>
  <div class="card">
    <div class="card-body">
      <slot></slot>
    </div>
  </div>
</template>
```

Использование компонента:

```vue
<!-- Parent.vue -->
<template>
  <Card>
    <p>Это содержимое будет передано в слот по умолчанию</p>
  </Card>
</template>
```

## Резервный контент (fallback content)

Слоты могут содержать резервный контент, который будет отображаться, если родитель не передаст ничего в слот:

```vue
<!-- Alert.vue -->
<template>
  <div class="alert">
    <slot>
      Сообщение по умолчанию
    </slot>
  </div>
</template>
```

## Слоты с ограниченной областью видимости

Слоты могут получать данные от компонента-владельца:

```vue
<!-- List.vue -->
<template>
  <ul>
    <li v-for="item in items" :key="item.id">
      <slot :item="item" :index="index">
        {{ item.name }}
      </slot>
    </li>
  </ul>
</template>

<script>
export default {
  data() {
    return {
      items: [
        { id: 1, name: 'Элемент 1' },
        { id: 2, name: 'Элемент 2' }
      ]
    }
  }
}
</script>
```

```vue
<!-- Использование -->
<template>
  <List v-slot="{ item, index }">
    <strong>{{ index + 1 }}.</strong> {{ item.name }}
  </List>
</template>
```

## Практические примеры

### Простой компонент-обертка

```vue
<!-- Wrapper.vue -->
<template>
  <div class="wrapper">
    <div class="header">
      <slot name="header"></slot>
    </div>
    <div class="content">
      <slot></slot>
    </div>
    <div class="footer">
      <slot name="footer"></slot>
    </div>
  </div>
</template>
```

```vue
<!-- Использование -->
<template>
  <Wrapper>
    <template #header>
      <h1>Заголовок</h1>
    </template>
    
    <p>Это содержимое будет в слоте по умолчанию</p>
    
    <template #footer>
      <small>Подвал</small>
    </template>
  </Wrapper>
</template>
```

## Рекомендации по использованию

- Слот по умолчанию должен использоваться для основного содержимого компонента
- Не рекомендуется использовать более одного слота по умолчанию в компоненте
- Слоты по умолчанию идеально подходят для создания оборачивающих компонентов
- Используйте резервный контент для улучшения доступности компонента

## Совместимость с российскими реалиями 2025

В российской разработке в 2025 году слоты по умолчанию особенно востребованы при создании:

- Компонентов для государственных порталов с гибким содержимым
- Библиотек компонентов для банковских систем
- Каркасов для внутренних корпоративных приложений

> [!tip] Совет
> Слоты по умолчанию - это основа для создания гибких и переиспользуемых компонентов, особенно подходят для оборачивания контента с дополнительной логикой или стилями.
