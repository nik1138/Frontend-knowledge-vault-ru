# Динамические слоты

Динамические слоты (dynamic slots) в Vue.js позволяют использовать динамические имена для слотов, что дает возможность программно управлять тем, какие части компонента будут заполнены контентом. Это особенно полезно при создании гибких компонентов с изменяющимся интерфейсом.

## Основы динамических слотов

Динамические слоты создаются с использованием квадратных скобок в атрибуте `v-slot` или в сокращенном синтаксисе `#`:

```vue
<!-- DynamicLayout.vue -->
<template>
  <div class="dynamic-layout">
    <header v-if="hasSlot('header')">
      <slot :name="headerSlotName"></slot>
    </header>
    
    <main>
      <slot></slot>
    </main>
    
    <footer v-if="slots.footer">
      <slot name="footer"></slot>
    </footer>
  </div>
</template>

<script>
export default {
  props: {
    headerSlotName: {
      type: String,
      default: 'header'
    }
  },
  methods: {
    hasSlot(name = 'default') {
      return !!this.$slots[name]
    }
  },
  computed: {
    slots() {
      return this.$slots
    }
  }
}
</script>
```

## Синтаксис динамических слотов

В Vue 3 динамические слоты используются следующим образом:

```vue
<!-- Использование динамического имени слота -->
<template>
  <DynamicComponent>
    <template #[dynamicSlotName]>
      <p>Содержимое динамического слота</p>
    </template>
  </DynamicComponent>
</template>

<script>
export default {
  data() {
    return {
      dynamicSlotName: 'content'
    }
  }
}
</script>
```

## Практические примеры

### Компонент с динамическими вкладками

```vue
<!-- TabContainer.vue -->
<template>
  <div class="tab-container">
    <div class="tabs">
      <button 
        v-for="tab in tabNames" 
        :key="tab"
        @click="activeTab = tab"
        :class="{ active: activeTab === tab }"
      >
        {{ tabLabels[tab] }}
      </button>
    </div>
    
    <div class="tab-content">
      <slot :name="activeTab"></slot>
    </div>
  </div>
</template>

<script>
export default {
  props: {
    initialTab: {
      type: String,
      default: 'tab1'
    },
    tabNames: {
      type: Array,
      default: () => ['tab1', 'tab2', 'tab3']
    },
    tabLabels: {
      type: Object,
      default: () => ({
        tab1: 'Первая вкладка',
        tab2: 'Вторая вкладка',
        tab3: 'Третья вкладка'
      })
    }
  },
  data() {
    return {
      activeTab: this.initialTab
    }
  }
}
</script>
```

```vue
<!-- Использование -->
<template>
  <TabContainer 
    :tab-names="dynamicTabNames" 
    :tab-labels="dynamicTabLabels"
    v-slot="{ activeTab }"
  >
    <template v-for="tabName in dynamicTabNames" #[tabName]>
      <div :key="tabName">
        <h3>Содержимое вкладки {{ tabName }}</h3>
        <p>Динамическое содержимое для {{ tabName }}</p>
      </div>
    </template>
  </TabContainer>
</template>

<script>
export default {
  data() {
    return {
      dynamicTabNames: ['profile', 'settings', 'notifications'],
      dynamicTabLabels: {
        profile: 'Профиль',
        settings: 'Настройки',
        notifications: 'Уведомления'
      }
    }
  }
}
</script>
```

### Динамические слоты для макетов

```vue
<!-- DynamicLayout.vue -->
<template>
  <div class="layout">
    <component 
      :is="layoutComponent" 
      v-slot="slotProps"
    >
      <template 
        v-for="slotName in availableSlots" 
        #[slotName]="slotProps"
      >
        <slot :name="slotName" v-bind="slotProps"></slot>
      </template>
    </component>
  </div>
</template>

<script>
export default {
  props: {
    layoutType: {
      type: String,
      default: 'default'
    }
  },
  computed: {
    layoutComponent() {
      return () => import(`./layouts/${this.layoutType}.vue`)
    },
    availableSlots() {
      // Возвращаем список доступных слотов для текущего макета
      const layouts = {
        default: ['header', 'main', 'footer'],
        sidebar: ['header', 'sidebar', 'main', 'footer'],
        card: ['header', 'body', 'footer']
      }
      return layouts[this.layoutType] || layouts.default
    }
  }
}
</script>
```

### Универсальный компонент карточки

```vue
<!-- DynamicCard.vue -->
<template>
  <div class="card">
    <div v-if="$slots[headerSlot]" class="card-header">
      <slot :name="headerSlot"></slot>
    </div>
    
    <div v-if="$slots[bodySlot]" class="card-body">
      <slot :name="bodySlot"></slot>
    </div>
    
    <div v-if="$slots[footerSlot]" class="card-footer">
      <slot :name="footerSlot"></slot>
    </div>
  </div>
</template>

<script>
export default {
  props: {
    headerSlot: {
      type: String,
      default: 'header'
    },
    bodySlot: {
      type: String,
      default: 'body'
    },
    footerSlot: {
      type: String,
      default: 'footer'
    }
  }
}
</script>
```

```vue
<!-- Использование -->
<template>
  <DynamicCard 
    :header-slot="cardHeaderSlot" 
    :body-slot="cardBodySlot"
  >
    <template #[cardHeaderSlot]>
      <h2>{{ cardTitle }}</h2>
    </template>
    
    <template #[cardBodySlot]>
      <p>{{ cardContent }}</p>
    </template>
    
    <template #footer>
      <button>Действие</button>
    </template>
  </DynamicCard>
</template>

<script>
export default {
  data() {
    return {
      cardHeaderSlot: 'title',
      cardBodySlot: 'content',
      cardTitle: 'Заголовок карточки',
      cardContent: 'Содержимое карточки'
    }
  }
}
</script>
```

## Рекомендации по использованию

- Используйте динамические слоты для создания гибких и универсальных компонентов
- Обеспечьте понятные имена для динамических слотов
- Документируйте возможные имена слотов в компоненте
- Используйте проверки на существование слотов перед их отображением
- Избегайте чрезмерного усложнения компонентов с динамическими слотами

## Совместимость с российскими реалиями 2025

В российской разработке в 2025 году динамические слоты особенно востребованы при создании:

- Универсальных компонентов для государственных порталов
- Гибких макетов для многостраничных приложений
- Интерфейсов с изменяющимся содержимым в зависимости от роли пользователя
- Адаптивных компонентов для корпоративных систем

> [!tip] Совет
> Динамические слоты позволяют создавать очень гибкие компоненты, но требуют тщательного тестирования и документирования для обеспечения понятности использования.
