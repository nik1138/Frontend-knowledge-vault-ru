# Reactivity System

Система реактивности Vue 3 - это мощный механизм, который позволяет автоматически обновлять пользовательский интерфейс при изменении данных. Она лежит в основе всей реактивности во Vue 3.

## Основные концепции

### [ref](ref.md)
Создает реактивную ссылку для примитивных значений и объектов. Используется с `.value` для доступа к значению.

### [reactive](reactive.md)
Создает глубоко реактивный объект. Все свойства объекта становятся реактивными рекурсивно.

### [computed](computed.md)
Создает вычисляемое свойство, которое автоматически отслеживает зависимости и кэширует результат.

### [watch](watch.md)
Отслеживает изменения реактивных значений и выполняет побочные эффекты при их изменении.

### [watchEffect](watchEffect.md)
Выполняет функцию и автоматически отслеживает зависимости, перезапуская функцию при их изменении.

## Вспомогательные функции

### [toRefs](toRefs.md)
Преобразует реактивный объект в набор ref'ов, позволяя деструктурировать объект с сохранением реактивности.

### [toRef](toRef.md)
Создает ref для одного свойства реактивного объекта, синхронизированный с этим свойством.

## Поверхностные реактивные функции

### [shallowReactive](shallowReactive.md)
Создает поверхностно реактивный объект - реактивны только корневые свойства.

### [shallowRef](shallowRef.md)
Создает поверхностно реактивную ссылку - реактивно только изменение значения, не внутренних свойств.

## Архитектурная схема

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│   ref/     │    │   reactive  │    │   computed  │
│  reactive  │◄──►│    state    │◄──►│   values    │
└─────────────┘    └─────────────┘    └─────────────┘
       │                   │                   │
       ▼                   ▼                   ▼
┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│   watch/    │    │   DOM       │    │   watch-    │
│ watchEffect │───►│  updates    │◄───┤   effect    │
└─────────────┘    └─────────────┘    └─────────────┘
```

## Практическое применение

При построении приложений используйте:
- `ref` для примитивных значений и локального состояния
- `reactive` для объектов со сложной структурой
- `computed` для производных значений с кэшированием
- `watch` для побочных эффектов при изменении данных
- `toRefs` для безопасной деструктуризации возвращаемых значений из setup()
