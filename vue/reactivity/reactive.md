# reactive

## Определение
`reactive` - это функция в Composition API Vue 3, которая создает глубоко реактивный объект. В отличие от `ref`, `reactive` используется исключительно для объектов и делает все их свойства рекурсивно реактивными.

## Синтаксис
```javascript
import { reactive } from 'vue'

const state = reactive({
  // данные объекта
})
```

## Особенности
- Создает глубоко реактивный объект
- Все вложенные свойства становятся реактивными
- Не нуждается в `.value` для доступа к свойствам
- Возвращает прокси объекта

## Примеры использования

### Простой реактивный объект
```javascript
import { reactive } from 'vue'

export default {
  setup() {
    const state = reactive({
      count: 0,
      user: {
        name: 'John',
        age: 30
      }
    })
    
    const increment = () => {
      state.count++
    }
    
    return {
      state,
      increment
    }
  }
}
```

### В шаблоне
```vue
<template>
  <div>
    <p>Счетчик: {{ state.count }}</p>
    <p>Имя: {{ state.user.name }}</p>
    <button @click="increment">Увеличить</button>
  </div>
</template>
```

### Работа с массивами
```javascript
const list = reactive([1, 2, 3])

// Все методы массива будут реактивными
list.push(4)
list.splice(0, 1)
```

## Отличия от ref
| reactive                           | ref                                                        |     |
| ---------------------------------- | ---------------------------------------------------------- | --- |
| Только для объектов (не примитивы) | Для любых значений                                         |     |
| Без `.value`                       | Требует `.value`                                           |     |
| Глубокая реактивность              | Аналогично, глубокая рективность, оборачивает в reactive() |     |
| Возвращает прокси                  | Возвращает объект с `.value`                               |     |

## Ограничения
- Не может использовать примитивные значения
- Не может разворачиваться в шаблоне как ref
- Необходимо использовать как свойство объекта в возвращаемом значении setup

## Связь с другими концепциями
- [[ref]] - для примитивов и поверхностной реактивности
- [[shallowReactive]] - поверхностная реактивность
- [[toRefs]] - преобразование реактивного объекта в refs
- [[computed]] - вычисляемые значения

## Примечания
- Для деструктуризации реактивного объекта используйте [[toRefs]]
- `reactive` не может быть создан из getter/setter функции
- Используйте `ref` для примитивных значений, `reactive` для объектов
