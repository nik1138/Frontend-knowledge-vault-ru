# State Management

Управление состоянием в Vue.js - это подход к хранению, управлению и передаче данных между компонентами. Это критически важно для масштабируемых приложений.

## Основные подходы

### [Reactivity System](../reactivity/Reactivity System)
Встроенная система реактивности Vue для локального состояния

### [Vuex](vuex.md)
Официальная библиотека управления состоянием для Vue (устаревшая рекомендация для новых проектов)

### [Pinia](pinia.md)
Современная библиотека управления состоянием для Vue (рекомендуемая)

### [Provide/Inject](provide-inject.md)
Встроенный механизм для передачи данных глубоко в дереве компонентов

### [Composables](../composables/Composables)
Функции для логики управления состоянием в Composition API

## Паттерны управления состоянием

### Локальное состояние
- Используется `ref`, `reactive` в пределах одного компонента
- Подходит для простого состояния компонента

### Подъем состояния (Lifting State)
- Состояние поднимается к общему родителю
- Подходит для состояния, разделяемого между несколькими компонентами

### Глобальное состояние
- Централизованное хранилище для всего приложения
- Подходит для сложных приложений с множеством компонентов

## Выбор подхода

| Подход | Когда использовать |
|--------|-------------------|
| Локальное состояние | Простое состояние компонента |
| Provide/Inject | Глубокая передача данных в дереве |
| Pinia | Сложное приложение с глобальным состоянием |
| Composables | Повторно используемая логика состояния |

## Best Practices

### Структура хранилища (на примере Pinia)
```javascript
import { defineStore } from 'pinia'

export const useMainStore = defineStore('main', {
  // Состояние
  state: () => ({
    user: null,
    count: 0
  }),
  
  // Геттеры
  getters: {
    doubleCount: (state) => state.count * 2
  },
  
  // Действия
  actions: {
    async fetchUser() {
      this.user = await api.getUser()
    }
  }
})
```

### Использование в компонентах
```vue
<script setup>
import { useMainStore } from '@/stores/main'

const store = useMainStore()
</script>
```
