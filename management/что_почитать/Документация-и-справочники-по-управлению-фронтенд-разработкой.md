# Документация и справочники по управлению фронтенд-разработкой

## Обзор

В 2025 году качественная документация и справочники играют ключевую роль в эффективном управлении фронтенд-разработкой. Ниже представлены основные документы, справочники и руководства, которые должны быть под рукой у менеджера фронтенд-разработки с учетом российских реалий.

## Официальная документация фреймворков

### 1. React

**Документация:** https://react.dev

React остается ведущим фреймворком для фронтенд-разработки. Документация содержит важную информацию для менеджеров:

- Архитектурные паттерны
- Рекомендации по структуре проектов
- Подходы к управлению состоянием
- Рекомендации по производительности

**Ключевые разделы:**
- "Thinking in React" - важен для понимания архитектурного подхода
- "Performance" - критично для масштабных проектов
- "Testing" - для обеспечения качества

**Связанные темы:** [[Компоненты]], [[Управление состоянием]], [[Тестирование]]

### 2. Vue.js

**Документация:** https://vuejs.org

Vue.js популярен в российских компаниях благодаря своей простоте и гибкости. Важные разделы:

- Архитектура приложений
- Управление состоянием (Vuex/Pinia)
- Инструменты для командной работы

**Преимущества:**
- Четкая документация
- Хорошо подходит для постепенного внедрения
- Активное русскоязычное сообщество

**Связанные темы:** [[Компоненты]], [[Архитектура фронтенд-приложений]]

### 3. Angular

**Документация:** https://angular.io

Angular подходит для корпоративных решений. Документация охватывает:

- Архитектурные принципы
- Управление зависимостями
- Тестирование
- Производительность

**Преимущества:**
- Полнофункциональный фреймворк
- Подходит для масштабных проектов
- Хорошая поддержка корпоративных требований

**Связанные темы:** [[Архитектура фронтенд-приложений]], [[Контроль качества]]
