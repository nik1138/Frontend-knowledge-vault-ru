# Технические навыки

Технические навыки (hard skills) - это конкретные, измеримые компетенции, которые можно приобрести через обучение и практику. В сфере IT и других технических профессий они играют ключевую роль при подборе персонала в 2025 году.

## Основные принципы указания технических навыков в резюме

### 1. Актуальность и релевантность
- Указывайте только те навыки, которые действительно требуются для позиции
- Приоритизируйте навыки в соответствии с требованиями вакансии
- Регулярно обновляйте список в соответствии с изменениями на рынке труда

### 2. Уровень владения
В 2025 году рекрутеры и работодатели обращают внимание на уровень владения навыками:

- **Начальный (Junior)** - знакомство с технологией, способность выполнять простые задачи с наставничеством
- **Средний (Middle)** - самостоятельная работа с технологией, способность решать типовые задачи
- **Продвинутый (Senior)** - глубокое понимание, способность принимать архитектурные решения
- **Эксперт** - признанный специалист, способный обучать других, разрабатывать стратегию использования технологии

### 3. Классификация технических навыков

#### Программирование
- Языки программирования (см. [[Языки-программирования]])
- Фреймворки и библиотеки (см. [[Фреймворки]])
- Алгоритмы и структуры данных
- Паттерны проектирования
- Архитектурные подходы

#### Системы и инфраструктура
- Операционные системы (Linux, Windows, macOS)
- Сетевые технологии
- Облачные платформы (AWS, Azure, GCP, Yandex Cloud)
- Контейнеризация (Docker, Kubernetes)
- CI/CD

#### Базы данных
- Реляционные (PostgreSQL, MySQL, Oracle)
- NoSQL (MongoDB, Redis, Cassandra)
- Инструменты анализа данных (SQL, BigQuery, Snowflake)

#### Инструменты разработки
- Системы контроля версий (Git, SVN)
- IDE и редакторы кода
- Сборщики и пакетные менеджеры
- Тестирование (unit, integration, e2e)

## Рекомендации по оформлению

### Использование ключевых слов
В 2025 году многие компании используют ATS-системы (Applicant Tracking Systems) для первичной фильтрации кандидатов. Важно включать в резюме ключевые слова из вакансии:

- Копируйте термины и технологии, используемые в описании вакансии
- Указывайте альтернативные названия технологий (например, "JavaScript/JS", "PostgreSQL/Postgres")

### Количественная оценка
Где это возможно, добавляйте количественные показатели:

- "Разработка на Python более 3 лет"
- "Оптимизация производительности веб-приложения на 40%"
- "Работа с базами данных объемом более 10 ГБ"

## Примеры формулировок

### Для начинающих специалистов
- "Базовое знание JavaScript и фреймворка React"
- "Опыт работы с Git для контроля версий кода"
- "Знание основных принципов ООП на примере Java"

### Для опытных специалистов
- "Проектирование и разработка микросервисной архитектуры на Spring Boot и Node.js"
- "Оптимизация SQL-запросов, улучшение производительности баз данных на 50%"
- "Внедрение CI/CD пайплайнов с использованием GitHub Actions и Jenkins"

## Особенности российского рынка 2025

### Локализованные технологии
- Акцент на отечественные и альтернативные решения (например, ClickHouse, 1С-платформа)
- Знание специфики работы с российскими государственными системами (СМЭВ, ГИС и др.)
- Понимание требований по защите информации (ФСТЭК, ФСБ)

### Языки программирования
В 2025 году в России особенно востребованы:
- Python (для Data Science, AI/ML, backend)
- Java (корпоративная разработка)
- C++ (системное программирование, игры)
- JavaScript/TypeScript (frontend и backend)
- Go (высоконагруженные системы)

### Облачные платформы
- Yandex Cloud - растущая популярность в российском сегменте
- AWS и Azure - по-прежнему востребованы, особенно в международных компаниях
- Alibaba Cloud - альтернатива в азиатском регионе

## Ошибки при указании технических навыков

### Чрезмерное перечисление
Не стоит перечислять все известные технологии - это создает впечатление поверхностного знания. Лучше указать 5-10 наиболее релевантных навыков с указанием уровня владения.

### Отсутствие обновления
Технологии быстро развиваются, особенно в IT. Регулярно обновляйте список навыков, добавляя новые и убирая устаревшие.

### Неправдоподобные утверждения
Избегайте необоснованно высокой оценки своего уровня владения навыками - это может быть проверено на техническом интервью.
