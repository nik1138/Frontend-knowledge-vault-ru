# Резюме для позиции Junior разработчика - Рекомендации для 2025 года

## Общие принципы подготовки резюме на позицию Junior

Резюме на позицию Junior разработчика в России 2025 года должно акцентировать внимание на обучаемости, базовых навыках программирования и стремлении к профессиональному росту. Работодатели на этих позициях ищут кандидатов с потенциалом, а не с обширным опытом.

## Структура резюме для Junior позиции

### 1. Контактная информация
- Полное имя
- Номер телефона
- Email
- Ссылка на профиль GitHub (обязательно!)
- Ссылка на LinkedIn (по желанию)
- Город проживания

### 2. Краткое описание (Summary)
Краткое резюме (2-3 предложения), описывающее вас как начинающего специалиста, с указанием технологий, в которых вы имеете базовые навыки, и сферы интересов.

> [!tip] Совет
> Не пишите "ищу первую работу", а лучше "начинающий специалист с опытом в ... и стремлением развиваться в ...".

### 3. Навыки (Skills)
Разделите навыки на категории:
- Языки программирования (например, JavaScript, Python, Java)
- Фреймворки и библиотеки (например, React, Express, Django)
- Инструменты и технологии (например, Git, SQL, Docker)
- Языки (указать уровни по CEFR)

### 4. Образование
- Университет, факультет, специальность (если соответствует IT)
- Курсы повышения квалификации
- Сертификаты (если есть)

### 5. Проекты
Это наиболее важная часть для Junior позиции:
- Описание проекта (2-3 предложения)
- Использованные технологии
- Ссылка на GitHub
- Ваша роль (если командный проект)
- Результат/достижение

> [!important] Важно
> Добавьте как минимум 2-3 проекта, демонстрирующих разные аспекты ваших навыков.

### 6. Опыт работы (если есть)
Даже если работа не связана с IT, укажите:
- Навыки, которые пригодились в программировании
- Примеры решения задач
- Работу в команде

## Особенности российского рынка 2025

### Локализация
- Используйте русский язык для резюме при подаче на российские позиции
- Указывайте готовность к переезду или локации, где вы можете работать
- Учитывайте особенности российского трудового законодательства

### Технологии в тренде
- Акцент на Cloud технологии (AWS, Azure, GCP)
- Базы данных (SQL, NoSQL)
- Основы DevOps (Docker, CI/CD)
- Основы тестирования

## Примеры формулировок

### Для раздела "Навыки":
- "Базовые знания JavaScript и HTML/CSS"
- "Опыт работы с Git и GitHub"
- "Понимание принципов ООП"
- "Знание основных паттернов проектирования"

### Для раздела "Проекты":
- "Создал веб-приложение для управления задачами с использованием React и Node.js"
- "Реализовал REST API для системы учета студентов с использованием Python и Flask"
- "Участвовал в разработке игры на Python с графическим интерфейсом"

## Частые ошибки

- Избегайте указания "желаемой зарплаты" на позиции Junior
- Не преувеличивайте уровень владения технологиями
- Не используйте слишком сложную терминологию без понимания
- Не указывайте ложную информацию

## Рекомендации по оформлению

- Используйте чистый, читаемый шрифт
- Сделайте резюме объемом не более 1-2 страниц
- Используйте единый стиль форматирования
- Проверьте орфографию и пунктуацию
