# Документация и справочники регулярных выражений

## Обзор официальной документации и справочных материалов по регулярным выражениям

В 2025 году доступно множество официальных документов, справочников и руководств по регулярным выражениям для различных языков программирования и инструментов. Ниже представлены лучшие источники документации, которые должны быть в закладках каждого разработчика.

## Официальная документация языков программирования

### 1. Python - Модуль `re`

Официальная документация модуля регулярных выражений в Python.

- **Адрес**: [docs.python.org](https://docs.python.org/3/library/re.html)
- **Особенности**:
  - Полное описание всех функций
  - Примеры использования
  - Таблица сопоставления синтаксиса
  - Рекомендации по производительности

#### Основные функции:
- `re.match()` - проверка соответствия в начале строки
- `re.search()` - поиск по всей строке
- `re.findall()` - поиск всех совпадений
- `re.sub()` - замена совпадений
- `re.compile()` - компиляция регулярного выражения

```python
import re

# Пример использования
pattern = r'\b[A-Za-z0-9._%+-]+@[A-Za-z0-9.-]+\.[A-Z|a-z]{2,}\b'
text = "Контакт: user@example.com"
match = re.search(pattern, text)
if match:
    print(f"Найден email: {match.group()}")
```

### 3. Java - java.util.regex

Официальная документация API регулярных выражений в Java.

- **Адрес**: [docs.oracle.com](https://docs.oracle.com/en/java/javase/21/docs/api/java.base/java/util/regex/package-summary.html)
- **Классы**:
  - `Pattern` - представление скомпилированного регулярного выражения
  - `Matcher` - движок для выполнения сопоставлений
  - `PatternSyntaxException` - исключение при синтаксической ошибке

```java
import java.util.regex.Pattern;
import java.util.regex.Matcher;

public class RegexExample {
    public static void main(String[] args) {
        String regex = "\\b[A-Za-z0-9._%+-]+@[A-Za-z0-9.-]+\\.[A-Za-z]{2,}\\b";
        String text = "Контакт: user@example.com";
        
        Pattern pattern = Pattern.compile(regex);
        Matcher matcher = pattern.matcher(text);
        
        if (matcher.find()) {
            System.out.println("Найден email: " + matcher.group());
        }
    }
}
```

## Структура типичной документации regex

### Основные разделы:
1. **Введение** - базовые понятия
2. **Синтаксис** - метасимволы и их значения
3. **Квантификаторы** - повторения и диапазоны
4. **Группы** - захват и незахват
5. **Флаги** - модификаторы поведения
6. **Примеры** - практические применения
7. **Оптимизация** - советы по производительности

### Общие элементы синтаксиса:
- `.` - любой символ (кроме новой строки)
- `^` - начало строки
- `$` - конец строки
- `*` - ноль или более
- `+` - один или более
- `?` - ноль или один
- `{n,m}` - от n до m повторений
- `[abc]` - любой из указанных символов
- `(abc)` - группа захвата
- `\d` - цифра
- `\w` - слово
- `\s` - пробельный символ
