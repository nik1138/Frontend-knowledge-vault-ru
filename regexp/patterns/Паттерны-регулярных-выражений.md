# Паттерны регулярных выражений

В этой статье рассмотрим распространенные паттерны регулярных выражений, которые часто используются в веб-разработке. Эти паттерны помогут вам решать типичные задачи по валидации и обработке строк.

## Часто используемые паттерны

### Валидация email

```javascript
const emailRegex = /^[^\s@]+@[^\s@]+\.[^\s@]+$/;

// Пример использования
function validateEmail(email) {
  return emailRegex.test(email);
}

validateEmail("user@example.com"); // true
validateEmail("invalid.email");    // false
```

### Проверка номера телефона

```javascript
// Простой паттерн для международного формата
const phoneRegex = /^\+?[\d\s\-\(\)]{7,15}$/;

// Более строгий паттерн для российских номеров
const ruPhoneRegex = /^(\+7|8)?[\s\-]?\(?[0-9]{3}\)?[\s\-]?[0-9]{3}[\s\-]?[0-9]{2}[\s\-]?[0-9]{2}$/;

function validatePhone(phone) {
  return phoneRegex.test(phone);
}
```

### Валидация пароля

```javascript
// Пароль: минимум 8 символов, одна заглавная, одна строчная, одна цифра
const passwordRegex = /^(?=.*[a-z])(?=.*[A-Z])(?=.*\d)[a-zA-Z\d@$!%*?&]{8,}$/;

function validatePassword(password) {
  return passwordRegex.test(password);
}
```

### Проверка URL

```javascript
const urlRegex = /^(https?:\/\/)?([\da-z\.-]+)\.([a-z\.]{2,6})([\/\w \.-]*)*\/?$/;

function validateUrl(url) {
  return urlRegex.test(url);
}
```

### Валидация даты (формат DD/MM/YYYY)

```javascript
const dateRegex = /^(0[1-9]|[12][0-9]|3[01])\/(0[1-9]|1[0-2])\/\d{4}$/;

function validateDate(dateString) {
  return dateRegex.test(dateString);
}
```

## Паттерны для веб-разработки

### Поиск HTML-тегов

```javascript
// Нахождение всех HTML-тегов
const htmlTagRegex = /<[^>]*>/g;

const htmlString = '<div class="example">Текст</div><p>Параграф</p>';
const tags = htmlString.match(htmlTagRegex);
console.log(tags); // ['<div class="example">', '</div>', '<p>', '</p>']
```

### Извлечение параметров из URL

```javascript
// Извлечение параметров из строки запроса
const queryStringRegex = /[?&]([^=#]+)=([^&#]*)/g;

const url = 'https://example.com/page?name=John&age=30&city=Moscow';
const params = {};
let match;

while (match = queryStringRegex.exec(url)) {
  params[decodeURIComponent(match[1])] = decodeURIComponent(match[2]);
}

console.log(params); // { name: 'John', age: '30', city: 'Moscow' }
```

### Валидация цвета в формате HEX

```javascript
const hexColorRegex = /^#([A-Fa-f0-9]{6}|[A-Fa-f0-9]{3})$/;

function validateHexColor(color) {
  return hexColorRegex.test(color);
}

validateHexColor("#ff0000"); // true
validateHexColor("#f00");    // true
validateHexColor("red");     // false
```

### Проверка формата IP-адреса

```javascript
const ipRegex = /^(?:(?:25[0-5]|2[0-4][0-9]|[01]?[0-9][0-9]?)\.){3}(?:25[0-5]|2[0-4][0-9]|[01]?[0-9][0-9]?)$/;

function validateIP(ip) {
  return ipRegex.test(ip);
}

validateIP("192.168.1.1"); // true
validateIP("256.1.1.1");   // false
```

## Практические примеры

### Форматирование номера кредитной карты

```javascript
function formatCreditCard(cardNumber) {
  // Убираем все нецифровые символы
  const cleanNumber = cardNumber.replace(/\D/g, '');
  
  // Добавляем пробелы каждые 4 цифры
  return cleanNumber.replace(/(\d{4})/g, '$1 ').trim();
}

formatCreditCard("1234567890123456"); // "1234 5678 9012 3456"
```

### Валидация формы регистрации

```javascript
class FormValidator {
  static patterns = {
    email: /^[^\s@]+@[^\s@]+\.[^\s@]+$/,
    phone: /^\+?[\d\s\-\(\)]{7,15}$/,
    password: /^(?=.*[a-z])(?=.*[A-Z])(?=.*\d)[a-zA-Z\d@$!%*?&]{8,}$/,
    username: /^[a-zA-Z0-9_]{3,20}$/
  };
  
  static validate(field, value) {
    if (!this.patterns[field]) {
      throw new Error(`Неизвестное поле: ${field}`);
    }
    
    return this.patterns[field].test(value);
  }
}

// Использование
console.log(FormValidator.validate('email', 'user@example.com')); // true
console.log(FormValidator.validate('password', 'MyPass123'));     // true
```

## Повышение безопасности

При использовании регулярных выражений для обработки пользовательского ввода, особенно в веб-приложениях, важно учитывать безопасность:

- Избегайте сложных регулярных выражений, которые могут быть уязвимы к ReDoS (Regular Expression Denial of Service) атакам
- Всегда валидируйте и эскейпите пользовательский ввод перед использованием
- Используйте специализированные библиотеки для сложной валидации, когда это возможно
