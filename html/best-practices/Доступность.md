# Доступность HTML: руководство по лучшим практикам

## Введение

Доступность (accessibility) веб-сайтов - это практика создания веб-страниц, которые могут быть использованы всеми людьми, включая тех, кто имеет ограничения по здоровью. В России это особенно актуально в связи с Федеральным законом №419-ФЗ от 27.12.2023 о доступной среде.

## Основные принципы доступности

### 1. Понятность (Perceivable)
Информация и элементы пользовательского интерфейса должны быть представлены способами, которые пользователи могут воспринимать.

#### Альтернативный текст для изображений
```html
<!-- Хорошо: информативное описание -->
<img src="chart.png" alt="График роста продаж за 2024 год показывает увеличение на 25%">

<!-- Плохо: бессмысленное описание -->
<img src="chart.png" alt="Картинка">

<!-- Для декоративных изображений -->
<img src="decoration.png" alt="" role="presentation">
```

#### Заголовки и структура
```html
<!-- Правильная иерархия заголовков -->
<h1>Основной заголовок</h1>
<h2>Подраздел</h2>
<h3>Подподраздел</h3>
```

### 2. Управляемость (Operable)
Компоненты пользовательского интерфейса должны быть управляемыми, интерфейс должен быть удобен для управления.

#### Клавиатурная навигация
```html
<!-- Все интерактивные элементы должны быть доступны с клавиатуры -->
<a href="/page">Ссылка</a>
<button type="button">Кнопка</button>
<input type="text" placeholder="Текстовое поле">
```

#### Порядок фокуса
```html
<!-- Убедитесь, что порядок фокуса логичен -->
<form>
  <input type="text" name="name" placeholder="Имя">
  <input type="email" name="email" placeholder="Email">
  <button type="submit">Отправить</button>
</form>
```

### 3. Понятность (Understandable)
Информация и работа пользовательского интерфейса должны быть понятными.

#### Ясные заголовки и метки
```html
<form>
  <label for="email">Email адрес</label>
  <input type="email" id="email" name="email">
  
  <label for="phone">Телефон (в формате +7 XXX XXX-XX-XX)</label>
  <input type="tel" id="phone" name="phone" placeholder="+7 (___) ___-__-__">
</form>
```

### 4. Надежность (Robust)
Контент должен быть надежным, чтобы различные вспомогательные технологии могли его понять.

## Практические рекомендации

### 1. Используйте ARIA-атрибуты при необходимости

#### ARIA-роль для определения типа элемента
```html
<div role="button" tabindex="0" onclick="doSomething()">Кнопка</div>
```

#### ARIA-атрибуты для описания
```html
<button aria-label="Закрыть окно" onclick="closeModal()">×</button>

<div role="alert" aria-live="assertive">
  Ошибка: неправильный формат email
</div>
```

#### ARIA для областей
```html
<div role="main" aria-label="Основной контент">
  <!-- Основной контент страницы -->
</div>

<nav aria-label="Навигация по сайту">
  <ul>
    <li><a href="/">Главная</a></li>
    <li><a href="/about">О нас</a></li>
  </ul>
</nav>
```

### 2. Обеспечьте контрастность цветов

```css
/* Пример стилей с хорошей контрастностью */
.text {
  color: #000000;
  background-color: #ffffff;
}

.button {
  color: #ffffff;
  background-color: #0066cc;
}
```

### 3. Обработка ошибок форм

```html
<form>
  <div class="form-group">
    <label for="email">Email *</label>
    <input type="email" id="email" name="email" required>
    <span class="error-message" id="email-error" role="alert">Введите корректный email</span>
  </div>
  
  <button type="submit">Отправить</button>
</form>
```

### 4. Семантические элементы

```html
<!-- Используйте семантические элементы для структуры -->
<header>
  <h1>Название сайта</h1>
  <nav aria-label="Основная навигация">
    <ul>
      <li><a href="/">Главная</a></li>
    </ul>
  </nav>
</header>

<main>
  <article>
    <h2>Заголовок статьи</h2>
    <p>Содержимое статьи</p>
  </article>
</main>

<footer>
  <p>Контактная информация</p>
</footer>
```

### 5. Таблицы с доступностью

```html
<table>
  <caption>Ежемесячная статистика продаж</caption>
  <thead>
    <tr>
      <th scope="col">Месяц</th>
      <th scope="col">Продажи</th>
      <th scope="col">Прибыль</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <th scope="row">Январь</th>
      <td>120</td>
      <td>24000 ₽</td>
    </tr>
  </tbody>
</table>
```

## Требования российского законодательства

В соответствии с Федеральным законом №419-ФЗ от 27.12.2023:
- Веб-сайты государственных и муниципальных учреждений должны быть доступны
- Коммерческие организации обязаны обеспечить доступность своих сайтов для инвалидов
- Необходимо соблюдать стандарты WCAG 2.1 AA

## Проверка доступности

### Инструменты для проверки
- axe DevTools
- WAVE
- Lighthouse
- Встроенные инструменты разработчика в браузерах

### Ручная проверка
1. Проверка с клавиатуры (Tab, Shift+Tab)
2. Использование скринридера (NVDA, JAWS, VoiceOver)
3. Проверка контрастности цветов
4. Тестирование с увеличенным масштабом

## Пример комплексной формы с доступностью

```html
<form action="/submit" method="post">
  <fieldset>
    <legend>Контактная информация</legend>
    
    <div class="form-row">
      <label for="full-name">Полное имя *</label>
      <input type="text" id="full-name" name="full-name" required 
             aria-describedby="name-help">
      <small id="name-help">Введите ваше полное имя</small>
    </div>
    
    <div class="form-row">
      <label for="email">Email *</label>
      <input type="email" id="email" name="email" required 
             aria-describedby="email-error email-help">
      <small id="email-help">Мы не передаем email третьим лицам</small>
      <div id="email-error" class="error" role="alert" aria-live="polite">
        <!-- Ошибки будут отображаться здесь -->
      </div>
    </div>
    
    <div class="form-row">
      <label for="phone">Телефон</label>
      <input type="tel" id="phone" name="phone" 
             placeholder="+7 (___) ___-__-__">
    </div>
  </fieldset>
  
  <fieldset>
    <legend>Дополнительная информация</legend>
    
    <div class="form-row">
      <label for="message">Сообщение</label>
      <textarea id="message" name="message" rows="5" cols="50"></textarea>
    </div>
    
    <div class="form-row">
      <input type="checkbox" id="consent" name="consent" required>
      <label for="consent">Я согласен с <a href="/privacy">политикой конфиденциальности</a> *</label>
    </div>
  </fieldset>
  
  <button type="submit">Отправить</button>
</form>
```

## Заключение

Создание доступных веб-сайтов - это не только правовая обязанность в России, но и важная часть инклюзивного дизайна. Соблюдение принципов доступности делает веб-контент доступным для всех пользователей, независимо от их возможностей.
