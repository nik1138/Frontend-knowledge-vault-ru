# Семантическая разметка в HTML: руководство по лучшим практикам

## Введение

Семантическая разметка - это использование HTML-элементов для выражения смысла и структуры контента, а не просто для визуального оформления. Правильное применение семантических тегов улучшает доступность, SEO и поддерживаемость кода.

## Основные семантические элементы

### Блочные элементы

#### `<header>`
Используется для обозначения заголовка секции или всей страницы. Может содержать логотип, навигацию, заголовки.

```html
<header>
  <h1>Название сайта</h1>
  <nav>
    <ul>
      <li><a href="#home">Главная</a></li>
      <li><a href="#about">О нас</a></li>
    </ul>
  </nav>
</header>
```

#### `<nav>`
Обозначает навигационные ссылки. Рекомендуется использовать внутри `header`, `footer` или как отдельный элемент.

```html
<nav aria-label="Основная навигация">
  <ul>
    <li><a href="/">Главная</a></li>
    <li><a href="/services">Услуги</a></li>
    <li><a href="/contacts">Контакты</a></li>
  </ul>
</nav>
```

#### `<main>`
Представляет основной контент страницы, уникальный для конкретной страницы и не повторяющийся в других частях сайта.

```html
<main>
  <article>
    <h2>Заголовок статьи</h2>
    <p>Основной контент статьи...</p>
  </article>
</main>
```

#### `<article>`
Представляет собой независимую, самодостаточную часть контента, которую можно понять вне контекста всей страницы.

```html
<article>
  <header>
    <h2>Заголовок статьи</h2>
    <p>Опубликовано: <time datetime="2025-01-15">15 января 2025</time></p>
  </header>
  <p>Содержимое статьи...</p>
  <footer>
    <p>Автор: Иван Иванов</p>
  </footer>
</article>
```

#### `<section>`
Группирует связанные по смыслу элементы. Обычно содержит заголовок.

```html
<section>
  <h2>Наши услуги</h2>
  <p>Описание наших услуг...</p>
</section>
```

#### `<aside>`
Содержит контент, косвенно связанный с основным содержимым.

```html
<aside>
  <h3>Реклама</h3>
  <p>Рекламный блок</p>
</aside>
```

#### `<footer>`
Обозначает нижний колонтитул секции или всей страницы.

```html
<footer>
  <p>&copy; 2025 Компания ООО "Рога и копыта". Все права защищены.</p>
  <address>
    Москва, ул. Тверская, д. 1, стр. 2<br>
    Телефон: <a href="tel:+74951234567">+7 (495) 123-45-67</a>
  </address>
</footer>
```

### Структурные элементы

#### `<h1> - <h6>`
Заголовки документа. Важно соблюдать иерархию заголовков без пропусков уровней.

```html
<h1>Основной заголовок страницы</h1>
<p>Введение к основной теме</p>

<h2>Подраздел 1</h2>
<p>Содержимое подраздела 1</p>

<h3>Подподраздел 1.1</h3>
<p>Дополнительная информация</p>

<h2>Подраздел 2</h2>
<p>Содержимое подраздела 2</p>
```

#### `<figure>` и `<figcaption>`
Используются для группировки медиа-контента с описанием.

```html
<figure>
  <img src="chart.png" alt="График роста продаж">
  <figcaption>Рост продаж по месяцам в 2024 году</figcaption>
</figure>
```

## Практические рекомендации

### 1. Используйте правильные заголовки
- На странице должен быть только один `<h1>`
- Соблюдайте иерархию заголовков
- Заголовки должны быть информативными

### 2. Применяйте элементы по назначению
- Не используйте `<div>` там, где подходит семантический элемент
- Используйте `<article>` для независимого контента
- Применяйте `<section>` для логических разделов

### 3. Связь с доступностью
Семантическая разметка улучшает работу скринридеров и других вспомогательных технологий. Это особенно важно в свете российского законодательства о доступности цифровой среды (Федеральный закон №419-ФЗ от 27.12.2023).

### 4. SEO преимущества
Поисковые системы лучше понимают структуру контента с помощью семантических элементов, что может положительно сказаться на ранжировании.

## Пример комплексной семантической разметки

```html
<!DOCTYPE html>
<html lang="ru">
<head>
  <meta charset="UTF-8">
  <title>Пример семантической разметки</title>
</head>
<body>
  <header>
    <h1>Название сайта</h1>
    <nav aria-label="Навигация по сайту">
      <ul>
        <li><a href="/">Главная</a></li>
        <li><a href="/about">О компании</a></li>
        <li><a href="/services">Услуги</a></li>
        <li><a href="/contacts">Контакты</a></li>
      </ul>
    </nav>
  </header>

  <main>
    <article>
      <header>
        <h1>Заголовок статьи</h1>
        <p>Опубликовано: <time datetime="2025-01-15">15 января 2025</time></p>
      </header>
      
      <section>
        <h2>Введение</h2>
        <p>Вступительный текст статьи...</p>
      </section>
      
      <section>
        <h2>Основная часть</h2>
        <p>Основное содержание статьи...</p>
        
        <figure>
          <img src="diagram.png" alt="Схема процесса">
          <figcaption>Схема основного процесса</figcaption>
        </figure>
      </section>
      
      <footer>
        <p>Автор: <a href="/author/ivanov">Иванов И.И.</a></p>
      </footer>
    </article>
    
    <aside>
      <h2>Похожие статьи</h2>
      <ul>
        <li><a href="/article/1">Статья 1</a></li>
        <li><a href="/article/2">Статья 2</a></li>
      </ul>
    </aside>
  </main>

  <footer>
    <p>&copy; 2025 ООО "Рога и копыта". Все права защищены.</p>
    <nav aria-label="Дополнительная навигация">
      <ul>
        <li><a href="/privacy">Политика конфиденциальности</a></li>
        <li><a href="/terms">Условия использования</a></li>
      </ul>
    </nav>
  </footer>
</body>
</html>
```

## Заключение

Семантическая разметка - это фундамент для создания доступных, SEO-оптимизированных и поддерживаемых веб-страниц. В условиях российского законодательства и современных требований к веб-доступности, правильное использование семантических элементов становится не просто рекомендацией, а необходимостью.
