# Лучшие практики HTML

Лучшие практики HTML помогают создавать семантически правильные, доступные и поддерживаемые веб-страницы. Эти принципы важны для разработчиков всех уровней.

## Общие принципы написания HTML

HTML (HyperText Markup Language) — основа веб-страниц. При написании HTML-кода важно придерживаться следующих принципов:

- **Семантичность**: Используйте теги по их предназначению, а не для стилизации
- **Читаемость**: Пишите понятный код с правильной структурой
- **Поддерживаемость**: Код должен быть легко изменяемым и понятным другим разработчикам
- **Совместимость**: Код должен корректно работать в различных браузерах

```html
<!-- Хорошо: семантически правильная структура -->
<header>
  <nav>
    <ul>
      <li><a href="/">Главная</a></li>
      <li><a href="/about">О нас</a></li>
    </ul>
  </nav>
</header>

<!-- Плохо: использование div вместо семантических тегов -->
<div>
  <div>
    <ul>
      <li><a href="/">Главная</a></li>
      <li><a href="/about">О нас</a></li>
    </ul>
  </div>
</div>
```

## Семантическая структура

Семантические теги улучшают доступность и SEO. Используйте теги, которые описывают структуру документа:

- `<header>` — заголовок секции или документа
- `<nav>` — навигационные ссылки
- `<main>` — основное содержимое
- `<article>` — независимый контент
- `<section>` — тематическая группа контента
- `<aside>` — сопутствующая информация
- `<footer>` — нижний колонтитул

```html
<!DOCTYPE html>
<html lang="ru">
<head>
  <meta charset="UTF-8">
  <title>Пример семантической структуры</title>
</head>
<body>
  <header>
    <h1>Мой сайт</h1>
  </header>
  
  <nav>
    <ul>
      <li><a href="/">Главная</a></li>
      <li><a href="/blog">Блог</a></li>
    </ul>
  </nav>
  
  <main>
    <article>
      <header>
        <h2>Заголовок статьи</h2>
      </header>
      <section>
        <p>Содержимое статьи...</p>
      </section>
    </article>
  </main>
  
  <aside>
    <h3>Боковая панель</h3>
  </aside>
  
  <footer>
    <p>&copy; 2023 Все права защищены</p>
  </footer>
</body>
</html>
```

## Доступность (Accessibility)

Доступность делает веб-сайты пригодными для использования людьми с ограниченными возможностями. Основные принципы:

- Используйте теги заголовков (`<h1>`-`<h6>`) в правильном порядке
- Добавляйте атрибут `alt` к изображениям
- Используйте ARIA-атрибуты при необходимости
- Обеспечьте возможность навигации с клавиатуры

```html
<!-- Хорошо: изображение с описанием -->
<img src="logo.png" alt="Логотип компании">

<!-- Хорошо: кнопка с ARIA-описанием -->
<button aria-label="Закрыть модальное окно">✕</button>

<!-- Хорошо: правильная структура заголовков -->
<h1>Основной заголовок</h1>
<h2>Подраздел</h2>
<h3>Подпункт</h3>
```

Для более подробной информации см. [[accessibility]].

## Производительность

Оптимизация HTML-кода влияет на производительность веб-сайта:

- Минимизируйте количество DOM-элементов
- Избегайте вложенности более 3-4 уровней
- Используйте ленивую загрузку изображений (`loading="lazy"`)
- Подключайте CSS и JS в правильных местах

```html
<!-- Хорошо: ленивая загрузка изображений -->
<img src="image.jpg" loading="lazy" alt="Описание изображения">

<!-- Хорошо: подключение CSS в head -->
<head>
  <link rel="stylesheet" href="styles.css">
</head>
<body>
  <!-- контент -->
  <script src="script.js"></script>
</body>
```

## Безопасность

HTML-код должен быть защищен от XSS-атак:

- Экранируйте пользовательский ввод
- Не используйте `innerHTML` для вставки данных
- Используйте CSP (Content Security Policy)

```html
<!-- Плохо: уязвимость к XSS -->
<div id="content"></div>
<script>
  // Потенциально опасный код
  document.getElementById('content').innerHTML = userInput;
</script>

<!-- Лучше: безопасное вставление данных -->
<div id="content"></div>
<script>
  document.getElementById('content').textContent = userInput;
</script>
```

## Поддержка

HTML-код должен быть совместим с различными браузерами:

- Используйте современные, но поддерживаемые теги
- Проверяйте совместимость с помощью [Can I Use](https://caniuse.com)
- Используйте полифилы для старых браузеров при необходимости

## Форматирование кода

Читаемый код облегчает поддержку:

- Используйте последовательный отступ (обычно 2 или 4 пробела)
- Правильно вкладывайте теги
- Комментируйте сложные участки
- Используйте осмысленные имена классов и ID

```html
<!-- Хорошо: отформатированный код -->
<div class="product-card">
  <img class="product-image" src="product.jpg" alt="Описание товара">
  <div class="product-info">
    <h3 class="product-title">Название товара</h3>
    <p class="product-price">1000 руб.</p>
  </div>
</div>
```

## Современные возможности

Используйте современные возможности HTML5:

- Новые теги: `<article>`, `<section>`, `<time>`, `<mark>` и др.
- Встроенные элементы форм: `<input type="date">`, `<input type="email>` и др.
- Встроенная валидация форм
- Встроенная аудио/видео поддержка

```html
<!-- Хорошо: использование современных элементов -->
<form>
  <label for="email">Email:</label>
  <input type="email" id="email" name="email" required>
  
  <label for="birthdate">Дата рождения:</label>
  <input type="date" id="birthdate" name="birthdate">
  
  <button type="submit">Отправить</button>
</form>

<!-- Хорошо: встроенная поддержка медиа -->
<video controls>
  <source src="movie.mp4" type="video/mp4">
  Ваш браузер не поддерживает видео.
</video>
```

## Примеры хороших практик

```html
<!DOCTYPE html>
<html lang="ru">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Хорошая практика HTML</title>
  <link rel="stylesheet" href="styles.css">
</head>
<body>
  <header>
    <h1>Мой сайт</h1>
    <nav aria-label="Основная навигация">
      <ul>
        <li><a href="/" aria-current="page">Главная</a></li>
        <li><a href="/about">О нас</a></li>
      </ul>
    </nav>
  </header>
  
  <main>
    <article>
      <header>
        <h2>Интересная статья</h2>
        <time datetime="2023-01-01">1 января 2023</time>
      </header>
      <p>Содержимое статьи...</p>
    </article>
  </main>
  
  <footer>
    <p>&copy; 2023 Все права защищены</p>
  </footer>
  
  <script src="script.js"></script>
</body>
</html>
```

## Примеры плохих практик

```html
<!-- Плохо: отсутствие семантики -->
<div class="header">
  <div class="nav">
    <ul>
      <li><a href="/">Главная</a></li>
    </ul>
  </div>
</div>

<!-- Плохо: неправильное использование тегов -->
<div><h1>Заголовок</h1></div>
<span><div>Блок элемент</div></span>

<!-- Плохо: отсутствие атрибута alt -->
<img src="image.jpg">

<!-- Плохо: встроенные стили -->
<div style="color: red; font-size: 16px;">Текст</div>
```
