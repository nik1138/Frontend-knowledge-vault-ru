# Канонические-ссылки

Каноническая ссылка (rel="canonical") - это элемент HTML, который помогает веб-мастерам предотвратить дублирование контента, указывая поисковым системам основную версию страницы с одинаковым или похожим содержимым. Это важный элемент SEO-стратегии.

## Основы канонических ссылок

### Базовый синтаксис
```html
<link rel="canonical" href="https://example.com/page">
```
Этот тег указывает основной URL для страницы, избегая проблем с дублированием контента.

### Местоположение тега
Каноническая ссылка должна находиться в секции `<head>` документа:
```html
<head>
  <title>Заголовок страницы</title>
  <link rel="canonical" href="https://example.com/page">
  <!-- Другие метатеги -->
</head>
```

## Сценарии использования

### 1. Параметры в URL
```html
<link rel="canonical" href="https://example.com/product/123">
```
Для страниц с различными параметрами:
- `https://example.com/product/123?ref=homepage`
- `https://example.com/product/123?ref=category`
- `https://example.com/product/123?color=red&size=large`

### 2. Сортировка и фильтрация
```html
<link rel="canonical" href="https://example.com/catalog/electronics">
```
Для отсортированных или отфильтрованных версий одной и той же категории.

### 3. Печатные версии
```html
<link rel="canonical" href="https://example.com/article/123">
```
На странице `https://example.com/print/article/123` указывает на основную версию.

### 4. Различные домены
```html
<link rel="canonical" href="https://www.example.com/page">
```
Для версий без www, указывает на версию с www.

## Российские особенности 2025

В 2025 году в России:

- **Яндекс** и **Google** по-прежнему активно учитывают канонические ссылки
- Рост популярности мобильных версий сайтов
- Увеличение числа мультирегиональных сайтов
- Рост значимости локальных поисковых систем

### Яндекс и канонические ссылки

Яндекс также поддерживает атрибут `rel="alternate canonical"`:
```html
<link rel="alternate canonical" href="https://example.com/page">
```
Хотя стандартный `rel="canonical"` также полностью поддерживается.

### Мультирегиональные сайты
Для российских сайтов с различными региональными версиями:
```html
<!-- На странице https://example.com/moscow/page -->
<link rel="canonical" href="https://example.com/moscow/page">

<!-- Альтернативные версии -->
<link rel="alternate" hreflang="ru-moscow" href="https://example.com/moscow/page">
<link rel="alternate" hreflang="ru-spb" href="https://example.com/spb/page">
<link rel="alternate" hreflang="ru" href="https://example.com/page">
```

## Расширенные сценарии

### 1. Кросс-доменные канонические ссылки
```html
<link rel="canonical" href="https://main-site.com/original-content">
```
На зеркальных или партнерских сайтах (редко используется и требует осторожности).

### 2. Динамические канонические ссылки
Для CMS и интернет-магазинов:
```php
<!-- Пример для PHP -->
<link rel="canonical" href="<?php echo getCurrentPageUrl(); ?>">
```

### 3. Канонические ссылки для пагинации
```html
<!-- На странице 1 -->
<link rel="canonical" href="https://example.com/category">

<!-- На странице 2 -->
<link rel="canonical" href="https://example.com/category?page=2">
<link rel="prev" href="https://example.com/category">
<link rel="next" href="https://example.com/category?page=3">
```

## Практические рекомендации

### Правила использования
1. **Одна каноническая ссылка на страницу** - избегайте нескольких тегов
2. **Каноническая ссылка на существующую страницу** - не указывайте на несуществующий URL
3. **Полные URL** - используйте полные URL, включая протокол (https://)
4. **Постоянные ссылки** - канонические ссылки должны указывать на постоянные URL
5. **Консистентность** - используйте одинаковые канонические ссылки на всех дубликатах

### Примеры правильного использования

**Хорошо:**
```html
<link rel="canonical" href="https://www.example.com/product/awesome-gadget">
```

**Плохо:**
```html
<link rel="canonical" href="/product/awesome-gadget">  <!-- Относительный URL -->
<link rel="canonical" href="https://example.com/product/awesome-gadget/">  <!-- Добавлен слэш -->
```

> [!tip] Совет
> Используйте инструменты Яндекс.Вебмастер и Google Search Console для проверки корректности канонических ссылок.

## Проверка канонических ссылок

### Инструменты для анализа
- Яндекс.Вебмастер
- Google Search Console
- Screaming Frog SEO Spider
- Ahrefs
- SEMrush

### Команды для проверки
```bash
# Проверка канонической ссылки с помощью curl
curl -I "https://example.com/page" | grep -i link

# Или через консоль браузера
document.querySelector('link[rel="canonical"]').href
```

## Ошибки при использовании канонических ссылок

1. **Циклические ссылки** - когда страницы указывают друг на друга
2. **Указание на несуществующую страницу** - 404 ошибки
3. **Непоследовательность** - разные дубликаты указывают на разные канонические URL
4. **Игнорирование HTTPS/HTTP** - разные протоколы для канонических URL
5. **Использование относительных URL** - должны быть абсолютными
