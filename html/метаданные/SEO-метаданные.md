# SEO-метаданные

SEO-метаданные - это элементы HTML, которые помогают поисковым системам понимать содержимое веб-страницы и влияют на её позиции в поисковой выдаче. Эффективные SEO-метаданные являются ключевым фактором видимости сайта в поисковых системах.

## Основные SEO-метатеги

### Title
```html
<title>Релевантный заголовок страницы (50-60 символов)</title>
```
Самый важный SEO-элемент. Появляется в поисковой выдаче и вкладке браузера.

> [!tip] Рекомендация
> Оптимальная длина заголовка: 50-60 символов. В России в 2025 году Яндекс и Google по-прежнему отдают предпочтение качественным заголовкам.

### Description
```html
<meta name="description" content="Краткое описание страницы (150-160 символов)">
```
Краткое описание содержимого страницы, которое отображается в поисковой выдаче.

### Keywords
```html
<meta name="keywords" content="ключевые, слова, страницы">
```
> [!warning] Важно
> Поисковые системы больше не учитывают этот тег для ранжирования, но может быть полезен для внутреннего использования.

## Дополнительные SEO-теги

### Canonical URL
```html
<link rel="canonical" href="https://example.com/page">
```
Указывает основной URL для страницы с одинаковым или похожим контентом, предотвращая дублирование контента.

### Robots
```html
<meta name="robots" content="index, follow">
```
Инструкции поисковым роботам:
- `index/follow` - индексировать и переходить по ссылкам (по умолчанию)
- `noindex/nofollow` - не индексировать и не переходить по ссылкам
- `noindex, follow` - не индексировать, но переходить по ссылкам
- `index, nofollow` - индексировать, но не переходить по ссылкам

### Alternate Language
```html
<link rel="alternate" hreflang="ru" href="https://example.com/ru/">
<link rel="alternate" hreflang="en" href="https://example.com/en/">
```
Помогает поисковым системам понимать языковые и региональные версии страницы.

## Российские особенности 2025

В 2025 году в России:

- **Яндекс** остается ведущей поисковой системой
- **Google** сохраняет значительную долю рынка
- Важно учитывать региональные особенности и локализованные запросы
- Рост популярности голосового поиска и поиска с использованием ИИ
- Увеличение значимости локального SEO

### Яндекс-специфичные теги

```html
<meta name="yandex-verification" content="verification_code">
```
Подтверждение прав на сайт в Яндекс.Вебмастере.

```html
<meta name="yandex-tableau-widget" content="logo=https://example.com/logo.png, title=Название, description=Описание">
```
Для виджета Яндекс.Табло.

## Локализованные SEO-практики

### Использование кириллических доменов
```html
<link rel="alternate" hreflang="ru" href="https://пример.рф/">
```
Поддержка кириллических доменов в России.

### Региональные настройки
```html
<meta name="geo.region" content="RU-MOS">
<meta name="geo.placename" content="Москва">
<meta name="geo.position" content="55.7558;37.6176">
```
Для локального SEO и географической релевантности.

### Semantic SEO
Использование структурированных данных для улучшения понимания контента:

### Mobile-First Indexing
```html
<meta name="viewport" content="width=device-width, initial-scale=1.0">
```
Учитывайте, что поисковые системы в первую очередь индексируют мобильную версию сайта.

### Core Web Vitals
Хотя не являются метатегами, влияют на SEO:
- Время загрузки
- Интерактивность
- Визуальная стабильность

## Практические рекомендации

1. **Уникальные заголовки**: Каждая страница должна иметь уникальный заголовок
2. **Оптимизация описаний**: Описание должно быть информативным и побуждать к действию
3. **Ключевые слова**: Используйте релевантные ключевые слова естественным образом
4. **Структура URL**: Используйте понятные URL с ключевыми словами
5. **Скорость загрузки**: Оптимизируйте изображения и ресурсы

> [!tip] Совет
> Используйте инструменты Яндекс.Вебмастер и Google Search Console для мониторинга SEO-показателей вашего сайта.

## Проверка SEO-метаданных

Инструменты для анализа:

- Яндекс.Вебмастер
- Google Search Console
- Ahrefs
- SEMrush
- Screaming Frog SEO Spider

## Ошибки, которых следует избегать

- Дублирование заголовков и описаний
- Слишком длинные или короткие заголовки
- Использование ключевых слов в заголовках ненатуральным образом
- Отсутствие canonical тегов на страницах с похожим контентом
- Игнорирование mobile-оптимизации
