# Альтернативные-версии

Альтернативные версии - это метатеги, которые помогают поисковым системам понимать различные версии одной и той же страницы для разных языков, регионов или форматов. Основные типы альтернативных версий включают hreflang, alternate и различные форматы представления контента.

## Hreflang (Language and Region)

Hreflang теги помогают поисковым системам показывать пользователям правильную языковую и региональную версию контента.

### Базовый синтаксис
```html
<link rel="alternate" hreflang="ru" href="https://example.com/ru/">
<link rel="alternate" hreflang="en" href="https://example.com/en/">
<link rel="alternate" hreflang="ru-ru" href="https://example.com/ru/">
<link rel="alternate" hreflang="x-default" href="https://example.com/">
```

### Языковые коды
- `ru` - русский язык
- `en` - английский язык
- `de` - немецкий язык
- `fr` - французский язык
- `es` - испанский язык

### Региональные коды
- `ru-ru` - русский для России
- `ru-kz` - русский для Казахстана
- `en-us` - английский для США
- `en-gb` - английский для Великобритании

## Российские особенности 2025

В 2025 году в России:

- Рост многоязычного контента (русский, английский, тюркские, кавказские языки)
- Увеличение значимости регионального контента
- Развитие отечественных поисковых решений
- Акцент на локализацию контента

### Пример для российского сайта
```html
<link rel="alternate" hreflang="ru" href="https://example.com/">
<link rel="alternate" hreflang="ru-moscow" href="https://example.com/moscow/">
<link rel="alternate" hreflang="ru-spb" href="https://example.com/spb/">
<link rel="alternate" hreflang="ru-kazan" href="https://example.com/kazan/">
<link rel="alternate" hreflang="en" href="https://example.com/en/">
<link rel="alternate" hreflang="x-default" href="https://example.com/">
```

### Поддержка языков меньшинств
```html
<link rel="alternate" hreflang="tt" href="https://example.com/tt/">  <!-- Татарский -->
<link rel="alternate" hreflang="ba" href="https://example.com/ba/">  <!-- Башкирский -->
<link rel="alternate" hreflang="cv" href="https://example.com/cv/">  <!-- Чувашский -->
```

## Типы альтернативных версий

### 1. Языковые версии
```html
<link rel="alternate" hreflang="ru" href="https://example.com/ru/">
<link rel="alternate" hreflang="uk" href="https://example.com/uk/">
<link rel="alternate" hreflang="kk" href="https://example.com/kk/">
```

### 2. Альтернативные форматы
```html
<link rel="alternate" type="application/rss+xml" title="RSS" href="https://example.com/feed.xml">
<link rel="alternate" type="application/json" href="https://example.com/data.json">
<link rel="alternate" media="only screen and (max-width: 640px)" href="https://m.example.com/">
```

### 3. Предпочтительный язык
```html
<link rel="alternate" hreflang="x-default" href="https://example.com/">
```
Указывает версию по умолчанию для пользователей, чей язык/регион не указан в других hreflang тегах.

## Практические рекомендации

### Правила использования hreflang

1. **Взаимная ссылка**: Если страница A указывает на страницу B, то страница B должна указывать на страницу A
2. **Полные URL**: Всегда используйте полные URL, включая протокол
3. **Консистентность**: Используйте одинаковые языковые коды повсюду
4. **Канонические ссылки**: Используйте вместе с [[Канонические-ссылки]]

### Пример полного набора тегов
```html
<head>
  <title>Главная страница</title>
  <link rel="canonical" href="https://example.com/ru/">
  <link rel="alternate" hreflang="ru" href="https://example.com/ru/">
  <link rel="alternate" hreflang="en" href="https://example.com/en/">
  <link rel="alternate" hreflang="de" href="https://example.com/de/">
  <link rel="alternate" hreflang="x-default" href="https://example.com/ru/">
</head>
```

> [!tip] Совет
> Для российских сайтов с региональными версиями обязательно используйте hreflang теги, чтобы Яндекс и Google правильно распределяли трафик по регионам.

## Альтернативные форматы представления

### RSS Feed
```html
<link rel="alternate" type="application/rss+xml" title="Новости сайта" href="/feed.xml">
```

### JSON Feed
```html
<link rel="alternate" type="application/json" title="JSON Feed" href="/feed.json">
```

### Мобильная версия
```html
<link rel="alternate" media="only screen and (max-width: 640px)" href="https://m.example.com/">
```

```html
<link rel="amphtml" href="https://example.com/amp/page">
```

### Проверка через консоль
```javascript
// Проверка hreflang тегов в браузере
document.querySelectorAll('link[rel="alternate"][hreflang]').forEach(link => {
  console.log(link.getAttribute('hreflang'), link.href);
});
```

## Ошибки при использовании

1. **Неполные пары**: Когда одна страница указывает на другую, но не наоборот
2. **Неправильные языковые коды**: Использование несуществующих или неправильных кодов
3. **Указание на несуществующие URL**: Ссылки на страницы, возвращающие 404
4. **Игнорирование HTTPS/HTTP**: Разные протоколы для канонических и альтернативных URL
5. **Отсутствие x-default**: Для международных сайтов рекомендуется указывать версию по умолчанию

## XML Sitemap для hreflang

Кроме тегов в HTML, hreflang можно указать в XML Sitemap:
```xml
<?xml version="1.0" encoding="UTF-8"?>
<urlset xmlns="http://www.sitemaps.org/schemas/sitemap/0.9"
        xmlns:xhtml="http://www.w3.org/1999/xhtml">
  <url>
    <loc>https://example.com/ru/</loc>
    <xhtml:link rel="alternate" hreflang="ru" href="https://example.com/ru/"/>
    <xhtml:link rel="alternate" hreflang="en" href="https://example.com/en/"/>
    <xhtml:link rel="alternate" hreflang="x-default" href="https://example.com/ru/"/>
  </url>
</urlset>
```
