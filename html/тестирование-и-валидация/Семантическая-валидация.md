# Семантическая-валидация

## Обзор

Семантическая валидация HTML - это процесс проверки правильности использования HTML элементов в соответствии с их предполагаемым значением и назначением. В условиях российской веб-разработки 2025 года, когда требования к доступности и SEO постоянно усиливаются, семантическая валидация становится критически важной частью процесса создания веб-сайтов.

## Понятие семантической валидации

Семантическая валидация отличается от синтаксической тем, что она проверяет не только правильность написания кода, но и правильность использования HTML элементов в контексте их смысла и назначения. Это включает:

- Правильное использование семантических элементов HTML5
- Соответствие структуры документа логической иерархии
- Корректное применение тегов в зависимости от контекста
- Соблюдение принципов доступности через семантику

## Основные принципы семантической валидации

### Использование правильных элементов

Каждый HTML элемент имеет свое семантическое значение, и использование правильного элемента важно для:

- Правильной интерпретации контента браузерами и поисковыми системами
- Доступности для пользователей вспомогательных технологий
- SEO оптимизации
- Поддержки кода в долгосрочной перспективе

```html
<!-- Неправильно -->
<div class="button">Нажми меня</div>
<div class="header">Заголовок страницы</div>
<div class="navigation">...</div>

<!-- Правильно -->
<button>Нажми меня</button>
<header>Заголовок страницы</header>
<nav>...</nav>
```

### Иерархия заголовков

Правильная иерархия заголовков критически важна для семантики:

- Используйте заголовки по иерархии: h1, h2, h3 и т.д.
- Не пропускайте уровни (например, сразу переходя от h2 к h4)
- Один h1 на страницу (в большинстве случаев)
- Заголовки должны отражать структуру контента

```html
<!-- Правильно -->
<main>
    <h1>Основной заголовок страницы</h1>
    <section>
        <h2>Раздел 1</h2>
        <article>
            <h3>Подраздел 1.1</h3>
            <p>Контент...</p>
        </article>
        <article>
            <h3>Подраздел 1.2</h3>
            <p>Контент...</p>
        </article>
    </section>
    <section>
        <h2>Раздел 2</h2>
        <p>Контент...</p>
    </section>
</main>
```

### Семантические элементы HTML5

Использование семантических элементов HTML5 улучшает структуру документа:

- `<header>` - для заголовков секций или страницы
- `<nav>` - для навигационных элементов
- `<main>` - для основного контента страницы
- `<section>` - для логически связанных разделов
- `<article>` - для самостоятельных, переносимых частей контента
- `<aside>` - для сопутствующего контента
- `<footer>` - для нижних колонтитулов
- `<figure>` и `<figcaption>` - для медиа-контента с подписями

## Инструменты семантической валидации

### Валидатор W3C

W3C валидатор проверяет не только синтаксис, но и семантическую корректность:

- Проверяет правильное вложение элементов
- Выявляет нарушения семантики
- Указывает на несоответствие спецификациям

### Axe-core

Библиотека для проверки доступности, которая включает проверки семантики:

```javascript
const axe = require('axe-core');

// Проверка семантики и доступности
axe.run(document, {
    runOnly: {
        type: 'tag',
        values: ['wcag2a', 'wcag2aa', 'section508']
    }
}, (err, results) => {
    if (err) throw err;
    console.log(results);
});
```

### Кастомные скрипты валидации

Пример скрипта для проверки семантики:

```javascript
function validateSemantics(htmlDocument) {
    const errors = [];
    
    // Проверка наличия одного h1 на страницу
    const h1Elements = htmlDocument.querySelectorAll('h1');
    if (h1Elements.length !== 1) {
        errors.push(`Найдено ${h1Elements.length} элементов h1, должно быть 1`);
    }
    
    // Проверка правильной иерархии заголовков
    const headers = htmlDocument.querySelectorAll('h1, h2, h3, h4, h5, h6');
    let lastLevel = 0;
    
    headers.forEach(header => {
        const level = parseInt(header.tagName.charAt(1));
        if (level > lastLevel + 1) {
            errors.push(`Нарушена иерархия заголовков: ${header.tagName} после ${lastLevel}`);
        }
        lastLevel = level;
    });
    
    return errors;
}
```

## Практические правила семантической валидации

### Использование тегов по назначению

1. **Теги для структуры**:
   - Используйте `<div>` только для структурных целей, когда нет подходящего семантического тега
   - Используйте `<section>` для логически связанных разделов
   - Используйте `<article>` для самостоятельных частей контента

2. **Теги для содержимого**:
   - Используйте `<p>` для абзацев текста
   - Используйте `<ul>`, `<ol>`, `<dl>` для списков
   - Используйте `<figure>` и `<figcaption>` для изображений с подписями

3. **Теги для интерактивных элементов**:
   - Используйте `<button>` для кнопок
   - Используйте `<a>` для ссылок
   - Используйте `<form>`, `<input>`, `<textarea>` для форм

### Проверка контекста использования

- Убедитесь, что элементы используются в правильном контексте
- Проверьте, соответствует ли содержимое семантике элемента
- Убедитесь, что вложенные элементы семантически совместимы

### Проверка доступности через семантику

- Используйте теги, которые улучшают доступность
- Проверьте, правильно ли вспомогательные технологии интерпретируют разметку
- Убедитесь, что семантика поддерживает навигацию с клавиатуры

## Российские особенности семантической валидации

В российской веб-разработке 2025 года особое внимание уделяется:

- Соответствию требованиям доступности (ГОСТ Р 52872-2019)
- Поддержке кириллического контента в семантических элементах
- Учету особенностей российских поисковых систем (Yandex)
- Локализации семантических меток и описаний

### Yandex и семантика

Yandex в 2025 году активно использует семантическую разметку для:

- Понимания структуры страницы
- Определения основного контента
- Улучшения показателей релевантности

## Проверка семантики в разных контекстах

### Мобильные устройства

- Убедитесь, что семантическая структура сохраняется на мобильных устройствах
- Проверьте, как вспомогательные технологии интерпретируют адаптированную разметку

### Печатные версии

- Используйте семантические теги, которые корректно отображаются при печати
- Убедитесь, что логическая структура сохраняется в печатной версии

### Вспомогательные технологии

- Проверьте, как разметка интерпретируется скринридерами
- Убедитесь, что семантика помогает навигации для пользователей с ограниченными возможностями

## Лучшие практики

### Структура документа

```html
<!DOCTYPE html>
<html lang="ru">
<head>
    <meta charset="UTF-8">
    <title>Заголовок страницы</title>
</head>
<body>
    <header>
        <h1>Название сайта</h1>
        <nav aria-label="Основная навигация">
            <!-- Навигация -->
        </nav>
    </header>
    
    <main>
        <h2>Основной заголовок контента</h2>
        <article>
            <h3>Заголовок статьи</h3>
            <p>Контент статьи...</p>
        </article>
    </main>
    
    <aside>
        <h3>Сопутствующая информация</h3>
        <!-- Боковая панель -->
    </aside>
    
    <footer>
        <p>Информация о сайте</p>
    </footer>
</body>
</html>
```

### Использование ARIA в сочетании с семантикой

- Используйте ARIA атрибуты для дополнительного уточнения семантики
- Не перегружайте разметку лишними ARIA атрибутами
- Следуйте принципу "семантика первична, ARIA вторична"

### Регулярная проверка

- Внедрите автоматическую проверку семантики в процесс разработки
- Проводите регулярные аудиты семантической корректности
- Обучайте команду правилам семантической разметки
