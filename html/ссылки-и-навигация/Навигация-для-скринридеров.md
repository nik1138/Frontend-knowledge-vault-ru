# Навигация для скринридеров

## Обзор

Навигация для скринридеров - это важная часть доступного веб-дизайна, обеспечивающая возможность пользователям с нарушениями зрения эффективно перемещаться по веб-странице. Правильная реализация позволяет скринридерам корректно интерпретировать структуру сайта и предоставить пользователям понятные инструменты навигации.

## Основы доступной навигации

### Семантические элементы

Использование семантически правильных HTML-элементов:

```html
<header>
  <nav aria-label="Основная навигация">
    <ul>
      <li><a href="/">Главная</a></li>
      <li><a href="/about">О нас</a></li>
      <li><a href="/services">Услуги</a></li>
    </ul>
  </nav>
</header>

<main>
  <article>
    <h1>Заголовок статьи</h1>
    <p>Содержимое статьи...</p>
  </article>
</main>

<aside>
  <nav aria-label="Боковая навигация">
    <h2>Популярные статьи</h2>
    <ul>
      <li><a href="/article1">Статья 1</a></li>
      <li><a href="/article2">Статья 2</a></li>
    </ul>
  </nav>
</aside>

<footer>
  <nav aria-label="Дополнительная навигация">
    <ul>
      <li><a href="/privacy">Политика конфиденциальности</a></li>
      <li><a href="/sitemap">Карта сайта</a></li>
    </ul>
  </nav>
</footer>
```

### ARIA-атрибуты для навигации

```html
<nav role="navigation" aria-label="Основное меню">
  <ul>
    <li><a href="/" aria-current="page">Главная</a></li>
    <li><a href="/products">Продукты</a></li>
    <li><a href="/services">Услуги</a></li>
  </ul>
</nav>
```

## Типы навигации для скринридеров

### 1. Основная навигация

```html
<nav aria-label="Основная навигация" role="navigation">
  <ul>
    <li><a href="/">Главная</a></li>
    <li><a href="/about">О нас</a></li>
    <li><a href="/services">Услуги</a></li>
    <li><a href="/products">Продукты</a></li>
    <li><a href="/contact">Контакты</a></li>
  </ul>
</nav>
```

### 2. Пропуск навигации

Позволяет пользователям с клавиатурой и скринридерами пропустить повторяющиеся элементы навигации:

```html
<a href="#main-content" class="skip-link">Перейти к основному содержимому</a>

<!-- Повторяющаяся навигация -->
<nav aria-label="Основная навигация">
  <ul>
    <li><a href="/">Главная</a></li>
    <li><a href="/about">О нас</a></li>
  </ul>
</nav>

<main id="main-content">
  <h1>Основной контент страницы</h1>
  <p>Содержимое страницы...</p>
</main>
```

```css
.skip-link {
  position: absolute;
  top: -40px;
  left: 6px;
  background: #000000;
  color: white;
  padding: 8px;
  text-decoration: none;
  border-radius: 4px;
  z-index: 1000;
}

.skip-link:focus {
  top: 6px;
}
```

### 3. Навигация по заголовкам

Скринридеры могут перемещаться по заголовкам разного уровня:

```html
<main>
  <h1>Заголовок первого уровня</h1>
  <p>Введение...</p>
  
  <h2>Раздел 1</h2>
  <p>Содержимое раздела 1...</p>
  
  <h3>Подраздел 1.1</h3>
  <p>Содержимое подраздела 1.1...</p>
  
  <h2>Раздел 2</h2>
  <p>Содержимое раздела 2...</p>
</main>
```

### 4. Ландмарки (обозначения областей)

```html
<header role="banner">
  <h1>Заголовок сайта</h1>
</header>

<nav role="navigation" aria-label="Основная навигация">
  <ul>
    <li><a href="/">Главная</a></li>
  </ul>
</nav>

<main role="main">
  <h1>Основной контент</h1>
  <p>Содержимое страницы...</p>
</main>

<aside role="complementary">
  <h2>Боковая панель</h2>
  <p>Дополнительная информация...</p>
</aside>

<footer role="contentinfo">
  <p>© 2025 Все права защищены</p>
</footer>
```

## Атрибуты доступности

### 1. `aria-label` и `aria-labelledby`

```html
<!-- aria-label -->
<nav aria-label="Основная навигация">
  <ul>
    <li><a href="/">Главная</a></li>
    <li><a href="/products">Продукты</a></li>
  </ul>
</nav>

<!-- aria-labelledby -->
<h2 id="nav-heading">Навигация по категориям</h2>
<nav aria-labelledby="nav-heading">
  <ul>
    <li><a href="/tech">Технологии</a></li>
    <li><a href="/design">Дизайн</a></li>
  </ul>
</nav>
```

### 2. `aria-current`

Указывает текущую активную страницу или элемент:

```html
<nav aria-label="Основная навигация">
  <ul>
    <li><a href="/">Главная</a></li>
    <li><a href="/products" aria-current="page">Продукты</a></li>
    <li><a href="/services">Услуги</a></li>
  </ul>
</nav>
```

### 3. `aria-describedby`

```html
<nav aria-label="Навигация по статье">
  <ul>
    <li><a href="#intro" aria-describedby="intro-desc">Введение</a></li>
    <li><a href="#content" aria-describedby="content-desc">Содержание</a></li>
  </ul>
</nav>

<p id="intro-desc">Раздел с вводной информацией</p>
<p id="content-desc">Основное содержание статьи</p>
```

## Структура меню для скринридеров

### Простое меню

```html
<nav aria-label="Простое меню">
  <ul>
    <li><a href="/">Главная</a></li>
    <li><a href="/about">О нас</a></li>
    <li><a href="/contact">Контакты</a></li>
  </ul>
</nav>
```

### Многоуровневое меню

```html
<nav aria-label="Многоуровневое меню" role="navigation">
  <ul>
    <li>
      <a href="/products">Продукты</a>
      <ul>
        <li><a href="/products/electronics">Электроника</a></li>
        <li><a href="/products/clothing">Одежда</a></li>
      </ul>
    </li>
    <li>

        <li><a href="/services/consulting">Консалтинг</a></li>
        <li><a href="/services/support">Поддержка</a></li>
      </ul>
    </li>
  </ul>
</nav>
```

## Навигация по контенту

### Оглавление

```html
<aside class="table-of-contents">
  <h2>Содержание</h2>
  <nav aria-label="Оглавление">
    <ul>
      <li><a href="#section1">Раздел 1</a></li>
      <li><a href="#section2">Раздел 2</a>
        <ul>
          <li><a href="#subsection2-1">Подраздел 2.1</a></li>
          <li><a href="#subsection2-2">Подраздел 2.2</a></li>
        </ul>
      </li>
      <li><a href="#section3">Раздел 3</a></li>
    </ul>
  </nav>
</aside>
```

### Якорные ссылки

```html
<nav aria-label="Навигация по странице">
  <ul>
    <li><a href="#top">Наверх</a></li>
    <li><a href="#content">К содержимому</a></li>
    <li><a href="#comments">К комментариям</a></li>
  </ul>
</nav>
```

## Практические рекомендации

### 1. Использование заголовков

```html
<!-- Правильная иерархия заголовков -->
<h1>Заголовок страницы</h1>
<h2>Раздел 1</h2>
<h3>Подраздел 1.1</h3>
<h2>Раздел 2</h2>
```

### 2. Осмысленные тексты ссылок

```html
<!-- Хорошо -->
<a href="/products/smartphones">Смартфоны</a>

<!-- Не рекомендуется -->
<a href="/products/smartphones">Нажмите здесь</a>
```

### 3. Визуальное выделение фокуса

```css
a:focus,
button:focus,
input:focus {
  outline: 2px solid #007cba;
  outline-offset: 2px;
}
```

### 4. Язык содержимого

```html
<html lang="ru">
<head>
  <meta charset="UTF-8">
  <title>Пример страницы</title>
</head>
<body>
  <!-- Содержимое -->
</body>
</html>
```

## Проверка доступности

### Инструменты для тестирования

1. **NVDA** (Windows) - бесплатный скринридер
2. **VoiceOver** (macOS) - встроенный скринридер Apple
3. **JAWS** - коммерческий скринридер для Windows
4. **WAVE** - инструмент для проверки доступности
5. **axe DevTools** - расширение для браузеров

### Ручная проверка

```html
<!-- Проверьте, что все элементы имеют правильные ARIA-атрибуты -->
<nav aria-label="Основная навигация" role="navigation">
  <ul>
    <li><a href="/" aria-current="page">Главная</a></li>
    <li><a href="/products">Продукты</a></li>
  </ul>
</nav>
```

## Современные подходы

### Инклюзивная разработка

```html
<!-- Обеспечение доступности для всех пользователей -->
<nav aria-label="Основная навигация" role="navigation">
  <ul>
    <li><a href="/" aria-current="page" aria-describedby="home-desc">Главная</a></li>
  </ul>
</nav>

<p id="home-desc" hidden>Текущая страница - главная</p>
```

### Адаптивная навигация

```html
<nav aria-label="Адаптивное меню">
  <button 
    aria-expanded="false" 
    aria-controls="mobile-menu" 
    aria-label="Открыть/закрыть меню">
    Меню
  </button>
  <ul id="mobile-menu" hidden>
    <li><a href="/">Главная</a></li>
    <li><a href="/products">Продукты</a></li>
  </ul>
</nav>
```

## Лучшие практики

### 1. Последовательность навигации

Сохраняйте логичный порядок навигации по всему сайту.

### 2. Контекст для ссылок

Обеспечьте достаточный контекст для понимания назначения ссылки:

```html
<!-- Хорошо -->
<a href="/article1">Прочитать статью о веб-доступности</a>

<!-- Не рекомендуется -->
<a href="/article1">Подробнее</a>
```

### 3. Альтернативные способы навигации

Предоставьте несколько способов доступа к информации:

- Поиск
- Карта сайта
- Оглавление
- Фильтры и теги

### 4. Тестирование с реальными пользователями

Регулярно тестируйте навигацию с пользователями, использующими скринридеры.

## Заключение

Навигация для скринридеров - ключевой элемент доступного веб-дизайна. Правильное использование семантических элементов HTML, ARIA-атрибутов и соблюдение принципов доступности позволяет создать инклюзивный опыт для всех пользователей, включая тех, кто использует вспомогательные технологии.
