# HTML Атрибуты: Полное Руководство

HTML атрибуты - это специальные параметры, которые предоставляют дополнительную информацию об элементах HTML. Они изменяют поведение или внешний вид элемента и всегда указываются в открывающем теге.

## Что такое HTML атрибуты

HTML атрибуты - это пары имя=значение, которые добавляются к HTML-тегам для изменения их поведения или предоставления дополнительной информации. Атрибуты позволяют управлять стилями, функциональностью, доступностью и семантикой веб-страниц.

```html
<img src="image.jpg" alt="Описание изображения" width="300">
```

В этом примере `src`, `alt` и `width` являются атрибутами элемента `<img>`.

## Глобальные атрибуты

Глобальные атрибуты могут использоваться с любым HTML-элементом. Они предоставляют общие возможности для управления элементами.

| Атрибут | Описание |
|---------|----------|
| `class` | Определяет один или несколько классов CSS |
| `id` | Уникальный идентификатор элемента |
| `style` | Встроенные CSS-стили |
| `title` | Дополнительная информация об элементе |
| `data-*` | Пользовательские атрибуты данных |
| `hidden` | Скрывает элемент от отображения |
| `tabindex` | Управляет порядком фокуса при навигации с клавиатуры |

Пример:
```html
<div id="main-content" class="container" data-role="main" tabindex="0">
  Основное содержимое страницы
</div>
```

## Атрибуты доступности (ARIA)

ARIA (Accessible Rich Internet Applications) атрибуты улучшают доступность веб-контента для пользователей с ограниченными возможностями.

| Атрибут | Описание |
|---------|----------|
| `role` | Определяет тип элемента для вспомогательных технологий |
| `aria-label` | Устанавливает доступную метку элемента |
| `aria-describedby` | Связывает элемент с описанием |
| `aria-hidden` | Скрывает элемент от вспомогательных технологий |
| `aria-expanded` | Указывает состояние раскрытия/свертывания |

Пример:
```html
<button role="button" aria-label="Закрыть окно" aria-expanded="false">X</button>
```

Для более подробного изучения доступности см. [[доступность]].

## Семантические атрибуты

Семантические атрибуты помогают описать структуру и значение содержимого страницы.

| Атрибут | Описание |
|---------|----------|
| `lang` | Язык содержимого элемента |
| `dir` | Направление текста (ltr/rtl) |
| `datetime` | Временная метка в элементах `<time>` |
| `pubdate` | Дата публикации в `<time>` |

Пример:
```html
<html lang="ru" dir="ltr">
  <article>
    <time datetime="2023-11-10" pubdate>10 ноября 2023</time>
  </article>
</html>
```

## Атрибуты форм

Атрибуты форм управляют поведением элементов ввода данных.

| Атрибут | Описание |
|---------|----------|
| `name` | Имя элемента формы |
| `value` | Значение элемента |
| `required` | Обязательное поле |
| `placeholder` | Подсказка в поле ввода |
| `disabled` | Отключенное поле |
| `readonly` | Только для чтения |
| `min`, `max` | Ограничения для числовых значений |
| `pattern` | Регулярное выражение для проверки |

Пример:
```html
<form>
  <input type="email" name="email" required placeholder="Введите email"
         pattern="[a-z0-9._%+-]+@[a-z0-9.-]+\.[a-z]{2,}$">
  <input type="number" name="age" min="18" max="100">
  <button type="submit">Отправить</button>
</form>
```

## Атрибуты мультимедиа

Атрибуты для управления аудио и видео контентом.

| Атрибут | Описание |
|---------|----------|
| `controls` | Отображение элементов управления |
| `autoplay` | Автоматическое воспроизведение |
| `loop` | Повторное воспроизведение |
| `muted` | Заглушение звука |
| `preload` | Стратегия предварительной загрузки |
| `poster` | Изображение-обложка для видео |

Пример:
```html
<video controls autoplay muted preload="metadata"
       poster="preview.jpg" width="640" height="480">
  <source src="movie.mp4" type="video/mp4">
  <source src="movie.webm" type="video/webm">
  Ваш браузер не поддерживает видео.
</video>
```

## Атрибуты безопасности

Атрибуты, обеспечивающие защиту веб-приложений.

| Атрибут | Описание |
|---------|----------|
| `sandbox` | Ограничения для содержимого iframe |
| `integrity` | Проверка целостности внешнего ресурса |
| `crossorigin` | Политика CORS для внешних ресурсов |
| `referrerpolicy` | Политика отправки заголовка Referer |

Пример:
```html
<iframe src="https://example.com" sandbox="allow-scripts allow-same-origin"></iframe>
<script src="https://cdn.example.com/library.js"
        integrity="sha384-..." crossorigin="anonymous"></script>
```

## Современные атрибуты

Новейшие атрибуты, добавленные в HTML5 и последующих спецификациях.

| Атрибут | Описание |
|---------|----------|
| `async`, `defer` | Управление загрузкой скриптов |
| `loading` | Ленивая загрузка изображений и iframe |
| `fetchpriority` | Приоритет загрузки ресурсов |
| `inert` | Элемент не взаимодействует с пользователем |

Пример:
```html
<img src="image.jpg" loading="lazy" alt="Изображение">
<script src="script.js" async fetchpriority="high"></script>
```

## Примеры использования

### Пример 1: Семантическая статья с доступностью
```html
<article role="article" aria-labelledby="article-title" lang="ru">
  <header>
    <h1 id="article-title">Заголовок статьи</h1>
    <time datetime="2023-11-10">10 ноября 2023</time>
  </header>
  <main>
    <p>Содержимое статьи с <mark>выделенными</mark> фрагментами.</p>
  </main>
  <footer>
    <p>Автор: <span property="author">Имя Фамилия</span></p>
  </footer>
</article>
```

### Пример 2: Интерактивная форма с валидацией
```html
<form action="/submit" method="post">
  <fieldset>
    <legend>Регистрация пользователя</legend>

    <label for="username">Имя пользователя:</label>
    <input type="text"
           id="username"
           name="username"
           required
           minlength="3"
           maxlength="20"
           placeholder="Введите имя пользователя">

    <label for="password">Пароль:</label>
    <input type="password"
           id="password"
           name="password"
           required
           pattern="(?=.*\d)(?=.*[a-z])(?=.*[A-Z]).{8,}"
           title="Пароль должен содержать не менее 8 символов, включая заглавную, строчную буквы и цифру">

    <button type="submit" disabled>Зарегистрироваться</button>
  </fieldset>
</form>
```

## Лучшие практики

1. **Используйте семантические элементы** - предпочитайте семантические элементы div'ам и span'ам
2. **Обеспечьте доступность** - используйте ARIA-атрибуты при необходимости
3. **Не забывайте атрибуты безопасности** - особенно при работе с внешними ресурсами
4. **Используйте логические имена атрибутов** - они должны быть понятными и описательными
5. **Проверяйте валидность** - используйте валидаторы HTML для проверки кода
6. **Минимизируйте использование style в атрибутах** - предпочитайте CSS-классы
