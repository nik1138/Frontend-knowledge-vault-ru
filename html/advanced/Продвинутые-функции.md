# Продвинутые функции HTML

## Обзор

HTML (HyperText Markup Language) - это основа веб-разработки, но помимо базовых элементов, таких как заголовки, параграфы и ссылки, язык предоставляет множество продвинутых функций, которые позволяют создавать более сложные, интерактивные и доступные веб-страницы. В 2025 году, с развитием веб-стандартов и увеличением требований к доступности и производительности, знание продвинутых возможностей HTML становится критически важным для современных разработчиков.

Этот документ охватывает ключевые продвинутые функции HTML, включая семантические элементы, формы с расширенной валидацией, мультимедийные возможности, а также интеграцию с JavaScript и CSS для создания современных веб-приложений.

## Семантические элементы HTML5

### Основные семантические элементы

Семантические элементы позволяют описывать структуру документа более точно, что улучшает доступность и SEO. Вот основные элементы:

- `<header>` - определяет заголовок секции или документа
- `<nav>` - содержит навигационные ссылки
- `<main>` - основное содержимое документа
- `<article>` - независимый фрагмент контента
- `<section>` - логически сгруппированный контент
- `<aside>` - боковая панель с дополнительной информацией
- `<footer>` - определяет нижний колонтитул секции или документа

```html
<!DOCTYPE html>
<html lang="ru">
<head>
    <meta charset="UTF-8">
    <title>Пример семантической структуры</title>
</head>
<body>
    <header>
        <h1>Мой веб-сайт</h1>
        <nav>
            <ul>
                <li><a href="#home">Главная</a></li>
                <li><a href="#about">О нас</a></li>
                <li><a href="#contact">Контакты</a></li>
            </ul>
        </nav>
    </header>

    <main>
        <section>
            <h2>Основная информация</h2>
            <p>Это основной контент страницы.</p>
        </section>
        
        <article>
            <h3>Новость</h3>
            <p>Содержимое новости...</p>
        </article>
    </main>

    <aside>
        <h3>Боковая панель</h3>
        <p>Дополнительная информация.</p>
    </aside>

    <footer>
        <p>&copy; 2025 Мой сайт. Все права защищены.</p>
    </footer>
</body>
</html>
```

### Роль в доступности

Семантические элементы играют важную роль в создании доступных веб-сайтов, особенно для пользователей с ограниченными возможностями, использующих вспомогательные технологии, такие как скринридеры. В России, в соответствии с требованиями [Федерального закона №416-ФЗ](https://www.consultant.ru/document/cons_doc_LAW_170029/), веб-сайты государственных учреждений и организации, оказывающие услуги населению, обязаны соответствовать требованиям доступности.

## Расширенные формы

### Валидация на стороне клиента

HTML5 предоставляет встроенные механизмы валидации форм без использования JavaScript:

```html
<form>
    <label for="email">Email:</label>
    <input type="email" id="email" name="email" required>
    
    <label for="age">Возраст:</label>
    <input type="number" id="age" name="age" min="18" max="100" required>
    
    <label for="website">Веб-сайт:</label>
    <input type="url" id="website" name="website" required>
    
    <label for="phone">Телефон:</label>
    <input type="tel" id="phone" name="phone" pattern="[0-9]{10}" placeholder="1234567890">
    
    <button type="submit">Отправить</button>
</form>
```

### Кастомные валидационные сообщения

Используйте JavaScript для настройки валидационных сообщений:

```javascript
const form = document.querySelector('form');
const emailInput = document.getElementById('email');

emailInput.addEventListener('invalid', function(e) {
    if (e.target.validity.valueMissing) {
        e.target.setCustomValidity('Пожалуйста, введите ваш email');
    } else if (e.target.validity.typeMismatch) {
        e.target.setCustomValidity('Пожалуйста, введите корректный email адрес');
    }
});

emailInput.addEventListener('input', function(e) {
    e.target.setCustomValidity(''); // Сброс кастомного сообщения при вводе
});
```

### Новые типы input

HTML5 предоставляет специфические типы input для улучшения UX на мобильных устройствах:

- `date` - выбор даты
- `time` - выбор времени
- `datetime-local` - выбор даты и времени
- `month` - выбор месяца
- `week` - выбор недели
- `color` - выбор цвета
- `range` - ползунок для числовых значений

```html
<form>
    <label for="birthday">Дата рождения:</label>
    <input type="date" id="birthday" name="birthday">
    
    <label for="color">Любимый цвет:</label>
    <input type="color" id="color" name="color" value="#ff0000">
    
    <label for="volume">Громкость:</label>
    <input type="range" id="volume" name="volume" min="0" max="100" value="50">
</form>
```

## Мультимедийные элементы

### Элемент `<video>`

```html
<video controls width="800" height="600" poster="thumbnail.jpg">
    <source src="movie.mp4" type="video/mp4">
    <source src="movie.webm" type="video/webm">
    <source src="movie.ogg" type="video/ogg">
    Ваш браузер не поддерживает тег video.
    <a href="movie.mp4">Скачать видео</a>
</video>
```

#### Атрибуты видео

- `controls` - показывает элементы управления
- `autoplay` - автовоспроизведение (ограничено политиками браузеров)
- `muted` - отключает звук (позволяет использовать autoplay)
- `loop` - зацикливает воспроизведение
- `preload` - указывает, как загружать видео (`none`, `metadata`, `auto`)
- `poster` - изображение-заглушка до начала воспроизведения

### Элемент `<audio>`

```html
<audio controls>
    <source src="audio.mp3" type="audio/mpeg">
    <source src="audio.ogg" type="audio/ogg">
    Ваш браузер не поддерживает тег audio.
</audio>
```

### Поддержка форматов

В 2025 году рекомендуется использовать следующие форматы:

- **Видео**: MP4 (H.264), WebM (VP9), AV1
- **Аудио**: MP3, OGG, WAV, Opus

Для российских разработчиков важно учитывать, что пользователи могут использовать разные браузеры, включая отечественные, такие как Яндекс.Браузер, Opera, и необходимо тестировать совместимость.

## Канвас и SVG

### Элемент `<canvas>`

Canvas позволяет рисовать графику с помощью JavaScript:

```html
<canvas id="myCanvas" width="400" height="200"></canvas>

<script>
const canvas = document.getElementById('myCanvas');
const ctx = canvas.getContext('2d');

// Рисуем прямоугольник
ctx.fillStyle = '#FF0000';
ctx.fillRect(10, 10, 150, 80);

// Рисуем круг
ctx.beginPath();
ctx.arc(100, 150, 40, 0, 2 * Math.PI);
ctx.fillStyle = '#0000FF';
ctx.fill();
</script>
```

### SVG (Scalable Vector Graphics)

SVG - это XML-базированный формат для векторной графики:

```html
<svg width="200" height="200" xmlns="http://www.w3.org/2000/svg">
    <circle cx="100" cy="100" r="50" fill="red" />
    <rect x="50" y="50" width="100" height="100" fill="blue" />
    <text x="100" y="150" text-anchor="middle" fill="white">SVG</text>
</svg>
```

## Web Components

### Custom Elements

Позволяет создавать собственные HTML-элементы:

```javascript
class MyCard extends HTMLElement {
    constructor() {
        super();
        
        // Создаем shadow DOM
        const shadow = this.attachShadow({mode: 'open'});
        
        const wrapper = document.createElement('div');
        wrapper.setAttribute('class', 'wrapper');
        
        const style = document.createElement('style');
        style.textContent = `
            .wrapper {
                border: 1px solid #ccc;
                padding: 16px;
                margin: 8px;
                border-radius: 8px;
            }
        `;
        
        wrapper.innerHTML = `
            <h3><slot name="title">Заголовок</slot></h3>
            <p><slot name="content">Содержимое</slot></p>
        `;
        
        shadow.appendChild(style);
        shadow.appendChild(wrapper);
    }
}

customElements.define('my-card', MyCard);
```

Использование:

```html
<my-card>
    <h3 slot="title">Моя карточка</h3>
    <p slot="content">Содержимое карточки</p>
</my-card>
```

### Shadow DOM

Shadow DOM изолирует стили и DOM-структуру компонента:

```javascript
class MyButton extends HTMLElement {
    connectedCallback() {
        const shadow = this.attachShadow({mode: 'closed'});
        
        shadow.innerHTML = `
            <style>
                button {
                    background-color: #007bff;
                    color: white;
                    border: none;
                    padding: 8px 16px;
                    border-radius: 4px;
                    cursor: pointer;
                }
                
                button:hover {
                    background-color: #0056b3;
                }
            </style>
            <button>
                <slot name="text">Кнопка</slot>
            </button>
        `;
    }
}

customElements.define('my-button', MyButton);
```

## Доступность (Accessibility)

### ARIA атрибуты

ARIA (Accessible Rich Internet Applications) помогает сделать веб-контент доступным для людей с ограниченными возможностями:

```html
<div role="alert" aria-live="assertive">
    Сообщение об ошибке
</div>

<button aria-label="Закрыть" aria-describedby="close-desc">
    ×
</button>
<p id="close-desc">Закрывает текущее окно</p>

<input type="text" 
       aria-label="Поиск" 
       aria-describedby="search-help"
       autocomplete="off">
<p id="search-help">Введите ключевые слова для поиска</p>
```

### Семантические альтернативы

Вместо использования `<div>` и `<span>` для структурирования контента, используйте семантические элементы:

```html
<!-- Плохо -->
<div class="navigation">
    <div class="nav-item">Главная</div>
    <div class="nav-item">О нас</div>
</div>

<!-- Хорошо -->
<nav>
    <a href="/">Главная</a>
    <a href="/about">О нас</a>
</nav>
```

## Современные возможности HTML

### Элемент `<dialog>`

Для создания модальных окон:

```html
<dialog id="modal">
    <h2>Заголовок модального окна</h2>
    <p>Содержимое модального окна</p>
    <button id="closeModal">Закрыть</button>
</dialog>

<button id="openModal">Открыть модальное окно</button>

<script>
const modal = document.getElementById('modal');
const openBtn = document.getElementById('openModal');
const closeBtn = document.getElementById('closeModal');

openBtn.addEventListener('click', () => {
    modal.showModal();
});

closeBtn.addEventListener('click', () => {
    modal.close();
});
</script>
```

### Элемент `<details>` и `<summary>`

Для создания интерактивных аккордеонов:

```html
<details>
    <summary>Часто задаваемые вопросы</summary>
    <p>Ответ на первый вопрос...</p>
</details>

<details>
    <summary>Как использовать этот сайт?</summary>
    <p>Инструкция по использованию...</p>
</details>
```

### Элемент `<data>`

Для связывания содержимого с машинно-читаемыми данными:

```html
<p>Текущая температура: <data value="25">25°C</data></p>
<p>Цена: <data value="1299">1 299 ₽</data></p>
```
