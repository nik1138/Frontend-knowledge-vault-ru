# Svelte

## Обзор

Svelte - это прогрессивный фреймворк для создания пользовательских интерфейсов, который отличается от других фреймворков тем, что переносит большую часть работы из браузера в процесс сборки. Это означает, что Svelte создает чистый, эффективный JavaScript во время компиляции, а не работает с виртуальным DOM в браузере. SvelteKit - это фреймворк для создания полнофункциональных веб-приложений на основе Svelte.

## Основные концепции

### Компиляция вместо выполнения

В отличие от React, Vue и Angular, которые работают в браузере, Svelte компилируется в чистый JavaScript во время сборки. Это позволяет создавать более легкие и быстрые приложения без необходимости в библиотеке фреймворка в браузере.

### Компоненты

Компоненты в Svelte - это просто файлы с расширением .svelte, содержащие HTML, CSS и JavaScript в одном файле. Синтаксис максимально приближен к стандартным веб-технологиям.

```svelte
<script>
  let name = 'мир';
  let count = 0;

  function handleClick() {
    count += 1;
  }
</script>

<h1>Привет, {name}!</h1>
<p>Счетчик: {count}</p>
<button on:click={handleClick}>Нажми меня</button>

<style>
  h1 {
    color: blue;
  }
  
  button {
    padding: 10px;
    margin: 5px;
  }
</style>
```

### Реактивность

Svelte предоставляет встроенную реактивность, которая автоматически обновляет DOM при изменении переменных. Для этого используется простое присваивание значений переменным.

```svelte
<script>
  let count = 0;
  
  $: doubled = count * 2; // Реактивное вычисление
  
  function increment() {
    count += 1; // При изменении count автоматически обновляется doubled
  }
</script>

<p>Счетчик: {count}</p>
<p>Удвоенное значение: {doubled}</p>
<button on:click={increment}>+</button>
```

### Специальные теги

Svelte предоставляет специальные теги для управления поведением компонентов:

- `<svelte:head>` - для управления содержимым тега head
- `<svelte:options>` - для настройки компонента
- `<svelte:self>` - для рекурсивного рендеринга
- `<svelte:component>` - для динамического рендеринга компонентов

## Функции и директивы

### Директивы

Svelte использует директивы для управления DOM:

- `on:` - для обработки событий
- `bind:` - для двустороннего связывания данных
- `class:` - для условного добавления классов
- `style:` - для условного добавления стилей
- `animate:` - для анимаций
- `use:` - для директив пользовательских хуков

```svelte
<input bind:value={name} placeholder="Введите имя" />
<div class:highlighted={isHighlighted}>Текст с условным классом</div>
<button on:click={() => visible = !visible}>Переключить видимость</button>
{#if visible}
  <p>Этот текст появляется/исчезает</p>
{/if}
```

### Жизненный цикл

Svelte предоставляет функции жизненного цикла:

- `onMount` - вызывается после монтирования компонента
- `beforeUpdate` - вызывается перед обновлением DOM
- `afterUpdate` - вызывается после обновления DOM
- `onDestroy` - вызывается перед уничтожением компонента

```svelte
<script>
  import { onMount, onDestroy } from 'svelte';

  let intervalId;

  onMount(() => {
    intervalId = setInterval(() => {
      console.log('Тик');
    }, 1000);
  });

  onDestroy(() => {
    clearInterval(intervalId);
  });
</script>
```

## SvelteKit

SvelteKit - это фреймворк для создания полнофункциональных веб-приложений на основе Svelte. Он предоставляет:

- Роутинг на основе файловой системы
- Серверный рендеринг (SSR)
- Статическую генерацию (SSG)
- API маршруты
- Интеграцию с бэкендом

```javascript
// src/routes/index.svelte
<script>
  import { onMount } from 'svelte';
  import { page } from '$app/stores';
  
  let data;
  
  onMount(async () => {
    const response = await fetch('/api/data');
    data = await response.json();
  });
</script>

<h1>Добро пожаловать!</h1>
{#if data}
  <pre>{JSON.stringify(data, null, 2)}</pre>
{/if}
```

## Преимущества и недостатки

### Преимущества

- Маленький размер бандла
- Высокая производительность
- Простой синтаксис
- Отсутствие виртуального DOM
- Встроенная реактивность
- Хорошая поддержка TypeScript

### Недостатки

- Меньшее сообщество по сравнению с React/Vue
- Меньше готовых решений и библиотек
- Меньше рабочих мест, требующих Svelte
- Меньше документации на русском языке
