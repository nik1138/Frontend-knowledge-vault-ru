# Семантические атрибуты HTML

Семантические атрибуты HTML определяют смысловое значение элементов на веб-странице. Они помогают браузерам, поисковым системам и вспомогательным технологиям лучше понимать структуру и содержание документа.

## Атрибуты структуры документа

### `role`
Атрибут `role` определяет семантическую роль элемента в соответствии со спецификацией ARIA.

```html
<div role="banner">Шапка сайта</div>
<div role="main">Основное содержимое</div>
<div role="contentinfo">Футер сайта</div>
```

### `itemprop`
Атрибут `itemprop` используется для разметки микроданных Schema.org.

```html
<div itemscope itemtype="http://schema.org/Person">
    <span itemprop="name">Иван Иванов</span>
    <span itemprop="jobTitle">Веб-разработчик</span>
</div>
```

## Атрибуты навигации

### `aria-current`
Атрибут `aria-current` указывает текущий элемент в навигационной структуре.

```html
<nav>
    <a href="/home" aria-current="page">Главная</a>
    <a href="/about">О нас</a>
    <a href="/contact">Контакты</a>
</nav>
```

## Атрибуты таблиц

### `scope`
Атрибут `scope` определяет область действия заголовка таблицы.

```html
<table>
    <thead>
        <tr>
            <th scope="col">Имя</th>
            <th scope="col">Фамилия</th>
            <th scope="col">Возраст</th>
        </tr>
    </thead>
    <tbody>
        <tr>
            <td scope="row">Иван</td>
            <td>Иванов</td>
            <td>30</td>
        </tr>
    </tbody>
</table>
```

### `headers`
Атрибут `headers` связывает ячейку данных с соответствующими заголовками.

```html
<table>
    <tr>
        <th id="name">Имя</th>
        <th id="city">Город</th>
    </tr>
    <tr>
        <td headers="name">Алексей</td>
        <td headers="city">Москва</td>
    </tr>
</table>
```

## Атрибуты списков

### `start`
Атрибут `start` задает начальное значение для нумерованного списка.

```html
<ol start="5">
    <li>Пункт 5</li>
    <li>Пункт 6</li>
    <li>Пункт 7</li>
</ol>
```

### `reversed`
Атрибут `reversed` указывает, что нумерованный список должен отображаться в обратном порядке.

```html
<ol reversed start="10">
    <li>Пункт 10</li>
    <li>Пункт 9</li>
    <li>Пункт 8</li>
</ol>
```

## Атрибуты изображений и медиа

### `alt`
Атрибут `alt` предоставляет альтернативный текст для изображений, что критически важно для доступности.

```html
<img src="logo.png" alt="Логотип компании 'ТехноСофт'">
```

### `longdesc`
Атрибут `longdesc` указывает URL-адрес с подробным описанием изображения.

```html
<img src="chart.png" alt="Диаграмма роста компании" longdesc="description.html">
```

## Атрибуты ссылок

### `rel`
Атрибут `rel` определяет отношение между текущим документом и связанным документом.

```html
<!-- Для SEO -->
<link rel="canonical" href="https://example.com/page">
<link rel="alternate" hreflang="ru" href="https://example.com/ru/page">

<!-- Для безопасности -->
<a href="https://external-site.com" rel="noopener noreferrer">Внешняя ссылка</a>
```

### `download`
Атрибут `download` указывает, что ссылка предназначена для скачивания файла.

```html
<a href="/files/manual.pdf" download="руководство.pdf">Скачать руководство</a>
```

## Атрибуты времени

### `datetime`
Атрибут `datetime` задает машинно-читаемую дату/время для элементов `<time>`.

```html
<time datetime="2025-01-15T10:30:00+03:00">15 января 2025, 10:30</time>
```

## Атрибуты цитирования

### `cite`
Атрибут `cite` указывает источник цитаты или выделенного текста.

```html
<blockquote cite="https://example.com/article">
    <p>Это важная цитата из статьи</p>
</blockquote>
```

## Атрибуты определений

### `title`
Атрибут `title` может использоваться с элементом `<dfn>` для дополнительного описания термина.

```html
<p>Термин <dfn title="HyperText Markup Language">HTML</dfn> означает...</p>
```

## Атрибуты для доступности

### `aria-label`
Атрибут `aria-label` предоставляет доступную метку для элемента.

```html
<button aria-label="Закрыть окно">×</button>
```

### `aria-labelledby`
Атрибут `aria-labelledby` связывает элемент с другим элементом, который содержит его метку.

```html
<h2 id="section-title">Раздел настроек</h2>
<div aria-labelledby="section-title">
    <!-- содержимое раздела -->
</div>
```

### `aria-describedby`
Атрибут `aria-describedby` указывает на элемент, содержащий описание текущего элемента.

```html
<input type="text" 
       aria-describedby="help-text"
       placeholder="Введите имя">
<p id="help-text">Имя должно содержать только кириллические буквы</p>
```

## Атрибуты для SEO

### `property` и `content`
Атрибуты `property` и `content` используются для мета-данных Open Graph.

```html
<meta property="og:title" content="Заголовок страницы">
<meta property="og:description" content="Описание страницы">
<meta property="og:image" content="https://example.com/image.jpg">
```

## Практические рекомендации

1. Всегда используйте атрибут `alt` для изображений, даже если он пустой (`alt=""` для декоративных изображений).
2. Применяйте семантические элементы и атрибуты для улучшения структуры документа.
3. Используйте атрибуты доступности (`aria-*`) для улучшения UX с вспомогательными технологиями.
4. Указывайте атрибут `datetime` в элементе `<time>` для машинно-читаемой даты.
5. Используйте атрибут `rel` с правильными значениями для SEO и безопасности.

> [!tip] Лучшая практика
> Семантические атрибуты улучшают доступность, SEO и общее качество кода. Используйте их последовательно и осмысленно.

## Совместимость с российскими стандартами

В 2025 году в России продолжает действовать ГОСТ Р 52872-2019 по обеспечению доступности веб-сайтов. При разработке веб-сайтов необходимо использовать семантические атрибуты для соответствия требованиям доступности.
