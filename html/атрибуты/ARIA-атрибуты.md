# ARIA атрибуты

ARIA (Accessible Rich Internet Applications) - это набор атрибутов, которые помогают сделать веб-контент и веб-приложения более доступными для людей с ограниченными возможностями. ARIA-атрибуты обеспечивают дополнительную информацию вспомогательным технологиям, таким как программы чтения с экрана.

## Основы ARIA

ARIA-атрибуты делятся на несколько категорий:
- **Роли** - определяют тип элемента (кнопка, меню, диалог и т.д.)
- **Свойства** - описывают состояние элемента (активный, выбран, скрыт и т.д.)
- **Состояния** - указывают текущее состояние элемента (расширен, свернут, выбран и т.д.)

## Основные ARIA-роли

### Ландшафтные роли
- `banner` - шапка сайта
- `main` - основное содержимое
- `navigation` - навигационная область
- `complementary` - дополнительная информация
- `contentinfo` - футер сайта

```html
<header role="banner">
    <h1>Шапка сайта</h1>
</header>

<nav role="navigation">
    <ul>
        <li><a href="/">Главная</a></li>
        <li><a href="/about">О нас</a></li>
    </ul>
</nav>

<main role="main">
    <h1>Основное содержимое</h1>
</main>

<footer role="contentinfo">
    <p>© 2025 Все права защищены</p>
</footer>
```

### Структурные роли
- `article` - самостоятельный фрагмент содержимого
- `section` - раздел содержимого
- `group` - логическая группа элементов
- `heading` - заголовок

```html
<article role="article">
    <h2 role="heading">Заголовок статьи</h2>
    <div role="group">
        <p>Содержимое статьи</p>
    </div>
</article>
```

### Виджетные роли
- `button` - кнопка
- `checkbox` - флажок
- `combobox` - комбинированный список
- `link` - ссылка
- `menu` - меню
- `menuitem` - элемент меню
- `progressbar` - индикатор выполнения
- `slider` - ползунок
- `tab` - вкладка
- `tabpanel` - панель вкладки

```html
<div role="button" tabindex="0" onclick="doSomething()">
    Нажми меня
</div>

<input type="checkbox" role="checkbox" id="agree">
<label for="agree">Согласен с условиями</label>
```

## Основные ARIA-свойства и состояния

### `aria-label`
Предоставляет доступную метку элементу.

```html
<button aria-label="Закрыть">✕</button>
<div role="img" aria-label="Логотип компании"></div>
```

### `aria-labelledby`
Связывает элемент с другим элементом, который содержит его метку.

```html
<h2 id="dialog-title">Подтверждение действия</h2>
<div role="dialog" aria-labelledby="dialog-title">
    <!-- содержимое диалога -->
</div>
```

### `aria-describedby`
Указывает на элемент, содержащий описание текущего элемента.

```html
<input type="text" 
       aria-describedby="help-block"
       placeholder="Введите email">
<div id="help-block">Email должен содержать символ @</div>
```

### `aria-hidden`
Скрывает элемент от вспомогательных технологий.

```html
<div aria-hidden="true">Этот текст не будет прочитан скринридером</div>
```

### `aria-expanded` и `aria-collapsed`
Указывают состояние расширения/свертывания элемента.

```html
<button aria-expanded="false" onclick="toggleMenu()">
    Меню
</button>
<ul id="menu" hidden>
    <li>Пункт 1</li>
    <li>Пункт 2</li>
</ul>
```

### `aria-selected`
Указывает, выбран ли элемент в группе.

```html
<ul role="tablist">
    <li role="tab" aria-selected="true" aria-controls="panel1">Вкладка 1</li>
    <li role="tab" aria-selected="false" aria-controls="panel2">Вкладка 2</li>
</ul>
```

### `aria-checked`
Указывает состояние флажка или переключателя.

```html
<input type="checkbox" role="checkbox" aria-checked="true">
```

### `aria-disabled`
Указывает, отключен ли элемент.

```html
<button aria-disabled="true">Недоступная кнопка</button>
```

### `aria-required`
Указывает, что поле обязательно для заполнения.

```html
<input type="text" aria-required="true" placeholder="Обязательное поле">
```

## ARIA-атрибуты для списков и деревьев

### `aria-level`
Указывает уровень вложенности элемента в иерархической структуре.

```html
<ul role="tree">
    <li role="treeitem" aria-level="1">Уровень 1</li>
    <ul>
        <li role="treeitem" aria-level="2">Уровень 2</li>
    </ul>
</ul>
```

### `aria-posinset` и `aria-setsize`
Указывают позицию элемента в наборе и размер набора.

```html
<ul role="listbox">
    <li role="option" aria-posinset="1" aria-setsize="5">Опция 1</li>
    <li role="option" aria-posinset="2" aria-setsize="5">Опция 2</li>
</ul>
```

## ARIA-атрибуты для живых регионов

### `aria-live`
Указывает, что содержимое региона может изменяться динамически.

```html
<div aria-live="polite" id="notifications">
    <!-- сюда будут добавляться уведомления -->
</div>
```

### `aria-atomic`
Определяет, следует ли вспомогательной технологии читать весь регион при изменении.

```html
<div aria-live="polite" aria-atomic="true" id="status">
    Статус: Загрузка...
</div>
```

### `aria-relevant`
Определяет, какие изменения должны быть объявлены.

```html
<div aria-live="polite" 
     aria-relevant="additions text" 
     id="chat">
    <!-- новые сообщения будут объявлены -->
</div>
```

## Практические рекомендации

1. **Используйте семантические HTML-элементы в первую очередь** - ARIA следует использовать только когда семантические элементы недостаточны.

2. **Не переопределяйте семантику** - избегайте использования ARIA-ролей, которые противоречат встроенной семантике HTML-элементов.

```html
<!-- НЕПРАВИЛЬНО -->
<h1 role="button">Заголовок как кнопка</h1>

<!-- ПРАВИЛЬНО -->
<button>Кнопка с заголовком</button>
```

3. **Обеспечьте клавиатурную навигацию** - элементы с ARIA-ролями, которые должны быть интерактивными, должны быть доступны с клавиатуры.

4. **Используйте `tabindex` осознанно** - значение `0` делает элемент доступным в порядке табуляции, `-1` позволяет программно фокусироваться на элементе.

5. **Проверяйте совместимость** - разные вспомогательные технологии по-разному поддерживают ARIA-атрибуты.

> [!warning] Важно
> ARIA - это мощный инструмент, но его неправильное использование может ухудшить доступность. Используйте ARIA только когда это необходимо и после тщательного тестирования.

## Современные подходы

В 2025 году веб-разработчики все чаще используют ARIA в сочетании с современными фреймворками и библиотеками доступности, такими как:
- React ARIA (от Adobe)
- Reach UI
- Headless UI

Эти библиотеки обеспечивают правильную реализацию ARIA-атрибутов в сложных интерактивных компонентах.

## Тестирование доступности

Для проверки корректности использования ARIA рекомендуется:
- Использовать инструменты разработчика браузера (например, axe DevTools)
- Тестировать с программами чтения с экрана (NVDA, JAWS, VoiceOver)
- Проверять клавиатурную навигацию
- Использовать автоматизированные инструменты (Lighthouse, axe-core)

## Совместимость с российскими стандартами

В 2025 году в России продолжает действовать ГОСТ Р 52872-2019 по обеспечению доступности веб-сайтов. Использование ARIA-атрибутов является важной частью соответствия этому стандарту, особенно при создании сложных интерактивных интерфейсов.
