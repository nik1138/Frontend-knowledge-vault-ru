# Parcel

Parcel - это современный сборщик, который работает "из коробки" с минимальной настройкой. В 2025 году Parcel остается популярным выбором для быстрой разработки и прототипирования, особенно в стартапах и небольших командах в России.

## Основные особенности

Parcel отличается следующими ключевыми особенностями:

- **Zero configuration** - работает без настройки для большинства проектов
- **Быстрая сборка** - использование многопоточности и кеширования
- **Встроенная поддержка** - CSS, LESS, SASS, TypeScript, JSX, Flow и т.д.
- **Автоматическая оптимизация** - минификация, объединение файлов и tree-shaking
- **Hot Module Replacement** - мгновенное обновление изменений в браузере

## Архитектура Parcel

Parcel использует плагин-архитектуру с асинхронным графом зависимостей. Это позволяет ему обрабатывать различные типы файлов параллельно, что значительно ускоряет процесс сборки. Parcel строит граф зависимостей и применяет трансформации к каждому узлу.

## Применение в российских реалиях 2025

В российской разработке Parcel особенно популярен для:

- Быстрого прототипирования и стартап-проектов
- Сайтов с небольшой сложностью
- Образовательных проектов и тренингов
- Проектов с ограниченными сроками разработки
- Небольших команд, где важна скорость настройки

## Конфигурация Parcel

Parcel изначально работает без конфигурации, но при необходимости можно использовать файлы:

- `.parcelrc` - для настройки плагинов и трансформаций
- `.postcssrc` - для настройки PostCSS
- `babel.config.js` - для настройки Babel
- `tsconfig.json` - для настройки TypeScript

## Плагины и расширения

Parcel поддерживает плагины для расширения функциональности:

- `@parcel/transformer-*` - трансформеры для различных типов файлов
- `@parcel/bundler-*` - бандлеры для изменения способа сборки
- `@parcel/namer-*` - для настройки именования выходных файлов
- `@parcel/runtime-*` - для добавления runtime кода
- `@parcel/resolver-*` - для изменения способа разрешения зависимостей

## Производительность

Parcel использует:
- Многопоточность для параллельной обработки файлов
- Умное кеширование для ускорения повторных сборок
- Инкрементальную сборку для обработки только измененных файлов
- Бинарное представление графа зависимостей для эффективности

## Сравнение с другими инструментами

Parcel отличается от [[Webpack]] своей простотой и отсутствием необходимости в ручной настройке. В отличие от [[Rollup]], Parcel ориентирован на сборку приложений, а не библиотек. По сравнению с [[Vite]], Parcel имеет более ограниченные возможности для разработки в реальном времени, но проще в настройке.

## Поддержка HTML

Parcel изначально поддерживает HTML как точку входа. При указании HTML-файла как входной точки, Parcel автоматически:
- Разрешает все зависимости (CSS, JS, изображения и т.д.)
- Встраивает их в HTML
- Оптимизирует и минимизирует ресурсы
- Создает оптимальную структуру файлов

## Практические рекомендации

1. Используйте Parcel для прототипов и небольших проектов
2. Включите кеширование для ускорения сборки в CI/CD
3. Используйте `--dist-dir` для указания директории вывода
4. Рассмотрите использование `--public-url` для развертывания на поддоменах
5. Используйте `--target` для настройки целевых браузеров

## Безопасность и изоляция

В российских реалиях 2025 года Parcel может быть предпочтительным выбором для:
- Проектов с требованиями к быстрой изоляции сред
- Разработки в условиях ограниченного доступа к внешним ресурсам
- Сборки приложений с минимальными зависимостями от внешних систем
