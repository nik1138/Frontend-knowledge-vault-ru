# Структурированные данные в HTML

**Структурированные данные** — это стандартизированный формат для предоставления информации о странице, который помогает поисковым системам лучше понимать содержимое и отображать его в результатах поиска в расширенном виде (рич-сниппеты).

## Форматы структурированных данных

### JSON-LD (рекомендуемый)
```html
<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "Article",
  "headline": "Структурированные данные в HTML",
  "author": {
    "@type": "Person",
    "name": "Автор"
  }
}
</script>
```

### Microdata
```html
<div itemscope itemtype="https://schema.org/Article">
  <h1 itemprop="headline">Структурированные данные в HTML</h1>
  <span itemprop="author" itemscope itemtype="https://schema.org/Person">
    <span itemprop="name">Автор</span>
  </span>
</div>
```

### RDFa
```html
<div vocab="https://schema.org/" typeof="Article">
  <h1 property="headline">Структурированные данные в HTML</h1>
  <span property="author" typeof="Person">
    <span property="name">Автор</span>
  </span>
</div>
```

## Схемы данных

[[schema-org]] — основной словарь структурированных данных, определяющий типы и свойства. Наиболее распространенные типы:

- **Organization** — организация или компания
- **Article** — статья или публикация
- **Product** — товар или продукт
- **LocalBusiness** — местный бизнес
- **Event** — событие
- **Person** — человек
- **Place** — место или локация
- **Review** — обзор или отзыв

## Реализация в HTML

Структурированные данные внедряются в HTML через тег `<script>` для JSON-LD или через атрибуты элементов для Microdata и RDFa. Рекомендуется использовать JSON-LD, так как он не зависит от разметки и проще в реализации.

## Влияние на SEO

Структурированные данные:
- Повышают видимость в поисковых результатах
- Позволяют отображать расширенные сниппеты (рейтинги, цены, даты)
- Улучшают понимание контента поисковыми системами
- Могут увеличить CTR за счёт более информативных результатов

## Лучшие практики

1. Используйте JSON-LD вместо Microdata или RDFa
2. Валидируйте данные перед публикацией
3. Обновляйте данные при изменении контента
4. Применяйте только релевантные схемы
5. Избегайте дублирования данных
6. Используйте [[semantic-html]] для улучшения доступности
