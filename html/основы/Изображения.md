# Изображения в HTML

Изображения играют важную роль в веб-дизайне и пользовательском опыте. В HTML изображения вставляются с помощью элемента `<img>`, и правильное их использование критически важно для доступности, производительности и SEO.

## Основы работы с изображениями

### Базовый синтаксис

Элемент `<img>` — это одиночный тег (самозакрывающийся), который вставляет изображение в документ:

```html
<img src="path/to/image.jpg" alt="Описание изображения">
```

### Обязательные атрибуты

- `src` — путь к изображению (обязательный)
- `alt` — альтернативный текст (обязательный для доступности)

```html
<img 
    src="photo.jpg" 
    alt="Описание изображения для доступности и SEO"
>
```

## Атрибуты изображений

### Основные атрибуты

```html
<img 
    src="image.jpg"                    <!-- Путь к изображению -->
    alt="Описание изображения"         <!-- Альтернативный текст -->
    width="300"                       <!-- Ширина в пикселях -->
    height="200"                      <!-- Высота в пикселях -->
    title="Всплывающая подсказка"      <!-- Подсказка при наведении -->
>
```

### Атрибуты доступности

```html
<img 
    src="chart.png" 
    alt="График роста продаж за 2025 год, показывающий увеличение на 15%"
    role="img"                        <!-- Роль элемента для скринридеров -->
    aria-describedby="chart-desc"     <!-- Связь с описанием -->
>
<div id="chart-desc">Детальное описание графика...</div>
```

### Современные атрибуты производительности

```html
<img 
    src="image.jpg"
    alt="Описание"
    loading="lazy"                    <!-- Ленивая загрузка -->
    decoding="async"                  <!-- Асинхронная декодировка -->
>
```

## Типы изображений и форматы

### Поддерживаемые форматы

- **JPEG (.jpg/.jpeg)** — для фотографий, сжатие с потерями
- **PNG (.png)** — для графики с прозрачностью, сжатие без потерь
- **GIF (.gif)** — для анимированных изображений
- **WebP (.webp)** — современный формат с высоким качеством и маленьким размером
- **SVG (.svg)** — векторные изображения, масштабируемые без потерь качества

### Использование нескольких форматов

```html
<picture>
    <source srcset="image.webp" type="image/webp">
    <source srcset="image.avif" type="image/avif">
    <img src="image.jpg" alt="Альтернативное описание" loading="lazy">
</picture>
```

## Пути к изображениям

### Относительные пути

```html
<!-- Относительно текущего документа -->
<img src="images/photo.jpg" alt="Фото">

<!-- Относительно корня сайта -->
<img src="/images/photo.jpg" alt="Фото">

<!-- На уровень выше -->
<img src="../images/photo.jpg" alt="Фото">
```

### Абсолютные пути

```html
<!-- Полный URL -->
<img src="https://example.com/images/photo.jpg" alt="Фото">

<!-- Относительно протокола -->
<img src="//example.com/images/photo.jpg" alt="Фото">
```

## Семантическое использование изображений

### Изображение как часть статьи

```html
<article>
    <h2>Новости компании</h2>
    <p>В этом месяце мы запускаем новую линейку продуктов.</p>
    
    <figure>
        <img src="new-products.jpg" alt="Новая линейка продуктов компании">
        <figcaption>Новые продукты, запущенные в этом месяце</figcaption>
    </figure>
</article>
```

### Логотип сайта

```html
<header>
    <h1>
        <img src="logo.svg" alt="Название компании" width="150" height="50">
    </h1>
</header>
```

### Иконки и декоративные элементы

```html
<!-- Для декоративных изображений alt должен быть пустым -->
<img src="decoration.png" alt="" role="presentation">

<!-- Или использовать CSS для декоративных изображений -->
<div class="decorative-element" style="background-image: url('decoration.png');"></div>
```

## Ленивая загрузка (Lazy Loading)

Современный атрибут `loading="lazy"` позволяет откладывать загрузку изображений до тех пор, пока они не понадобятся:

```html
<img 
    src="image.jpg" 
    alt="Описание" 
    loading="lazy"
    width="800" 
    height="600"
>
```

### Когда использовать lazy loading:

- Изображения ниже "сгиба" (части страницы, видимой при первоначальной загрузке)
- Изображения в каруселях или вкладках
- Изображения в длинных списках

## Адаптивные изображения

### Атрибут `srcset` для разных разрешений

```html
<img 
    src="image-800.jpg"
    srcset="image-400.jpg 400w, image-800.jpg 800w, image-1200.jpg 1200w"
    sizes="(max-width: 600px) 400px, (max-width: 1000px) 800px, 1200px"
    alt="Адаптивное изображение"
>
```

### Использование элемента `<picture>` для разных изображений

```html
<picture>
    <!-- Для мобильных устройств -->
    <source media="(max-width: 768px)" srcset="mobile-image.jpg">
    <!-- Для планшетов -->
    <source media="(max-width: 1024px)" srcset="tablet-image.jpg">
    <!-- По умолчанию -->
    <img src="desktop-image.jpg" alt="Адаптивное изображение">
</picture>
```

## Российские особенности работы с изображениями

В 2025 году в России при работе с изображениями в HTML важно учитывать:

- **Скорость загрузки** — учитывая возможные ограничения на доступ к иностранным ресурсам
- **Хранение изображений** — предпочтение локальным или российским CDN
- **Размер файлов** — оптимизация для пользователей с ограниченной пропускной способностью
- **Соответствие требованиям РКН** — корректное размещение и маркировка изображений

### Рекомендуемые форматы для России

- **WebP** — для современных браузеров, отличное соотношение качества и размера
- **JPEG** — для фотографий, хорошая поддержка
- **PNG** — для графики с прозрачностью
- **SVG** — для логотипов и иконок, масштабируемые без потерь

## Практические рекомендации

1. **Всегда используйте атрибут `alt`** — это критически важно для доступности
2. **Оптимизируйте изображения** — используйте правильные форматы и размеры
3. **Используйте ленивую загрузку** для изображений ниже "сгиба"
4. **Указывайте размеры изображений** — предотвращает сдвиг контента при загрузке
5. **Используйте семантически правильные элементы** (`<figure>`, `<figcaption>`)
6. **Проверяйте доступность** — убедитесь, что изображения понятны для пользователей со скринридерами

## Пример комплексного использования изображений

```html
<!DOCTYPE html>
<html lang="ru">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Пример использования изображений</title>
</head>
<body>
    <header>
        <h1>
            <img src="logo.svg" alt="Название компании" width="200" height="60">
        </h1>
        <nav>
            <ul>
                <li><a href="/">Главная</a></li>
                <li><a href="/about">О нас</a></li>
            </ul>
        </nav>
    </header>
    
    <main>
        <article>
            <h2>Новости компании</h2>
            
            <!-- Изображение в статье -->
            <figure>
                <picture>
                    <source media="(max-width: 768px)" srcset="news-mobile.jpg">
                    <source media="(max-width: 1024px)" srcset="news-tablet.jpg">
                    <img 
                        src="news-desktop.jpg" 
                        alt="Команда компании празднует юбилей" 
                        width="800" 
                        height="500"
                        loading="lazy"
                    >
                </picture>
                <figcaption>Команда празднует 10-летие компании</figcaption>
            </figure>
            
            <p>В этом году мы достигли важного里程碑... (контент статьи)</p>
            
            <!-- Галерея изображений -->
            <section>
                <h3>Фото с мероприятия</h3>
                <div class="gallery">
                    <img 
                        src="event1.jpg" 
                        alt="Участники мероприятия" 
                        width="300" 
                        height="200"
                        loading="lazy"
                    >
                    <img 
                        src="event2.jpg" 
                        alt="Выступление спикера" 
                        width="300" 
                        height="200"
                        loading="lazy"
                    >
                    <img 
                        src="event3.jpg" 
                        alt="Сеттинг на мероприятии" 
                        width="300" 
                        height="200"
                        loading="lazy"
                    >
                </div>
            </section>
        </article>
    </main>
    
    <aside>
        <h3>Реклама</h3>
        <a href="https://example.com">
            <img 
                src="ad-banner.jpg" 
                alt="Реклама продукта" 
                width="300" 
                height="250"
            >
        </a>
    </aside>
    
    <footer>
        <p>&copy; 2025 Все права защищены</p>
    </footer>
</body>
</html>
```
