# Заголовки и иерархия в HTML

Правильное использование заголовков в HTML-документе играет ключевую роль в структурировании контента, доступности и SEO-оптимизации. В российской веб-разработке 2025 года соблюдение иерархии заголовков особенно важно для соответствия требованиям поисковых систем и стандартам доступности.

## Типы заголовков

HTML предоставляет шесть уровней заголовков от `<h1>` до `<h6>`, где `<h1>` — самый важный (наивысший уровень), а `<h6>` — наименее важный (наиболее глубокий уровень):

- `<h1>` — основной заголовок страницы
- `<h2>` — подзаголовки основных разделов
- `<h3>` — подзаголовки подзаголовков
- `<h4>` — подзаголовки третьего уровня
- `<h5>` — подзаголовки четвертого уровня
- `<h6>` — наименее значимые подзаголовки

### Пример иерархии заголовков:

```html
<h1>Основной заголовок страницы</h1>

<p>Введение к основной теме.</p>

<h2>Первый раздел</h2>
<p>Содержимое первого раздела.</p>

<h3>Подраздел 1.1</h3>
<p>Детали подраздела 1.1.</p>

<h3>Подраздел 1.2</h3>
<p>Детали подраздела 1.2.</p>

<h2>Второй раздел</h2>
<p>Содержимое второго раздела.</p>

<h3>Подраздел 2.1</h3>
<p>Детали подраздела 2.1.</p>

<h4>Подраздел 2.1.1</h4>
<p>Дополнительные детали.</p>
```

## Правила использования заголовков

### 1. Один заголовок `<h1>` на страницу

Каждая страница должна содержать только один заголовок `<h1>`, который отражает основную тему страницы. Это важно как для SEO, так и для доступности:

```html
<!-- Правильно -->
<h1>Услуги веб-разработки</h1>
<p>Мы предлагаем широкий спектр услуг по созданию веб-сайтов...</p>

<!-- Неправильно -->
<h1>Услуги веб-разработки</h1>
<h1>Дизайн сайтов</h1>
<h1>Разработка приложений</h1>
```

### 2. Логическая иерархия

Заголовки должны следовать логической иерархии без пропусков уровней. Не переходите от `<h1>` к `<h3>`, пропуская `<h2>`:

```html
<!-- Правильно -->
<h1>Основной заголовок</h1>
<h2>Подзаголовок</h2>
<h3>Подзаголовок второго уровня</h3>

<!-- Неправильно -->
<h1>Основной заголовок</h1>
<h3>Подзаголовок (пропущен h2)</h3>
```

### 3. Семантическое значение

Заголовки должны использоваться не только для визуального оформления, но и для передачи семантической структуры документа:

```html
<!-- Правильно: используется для структурирования контента -->
<h2>Преимущества наших услуг</h2>
<ul>
    <li>Высокое качество</li>
    <li>Быстрая доставка</li>
    <li>Гарантия удовлетворенности</li>
</ul>

<!-- Неправильно: используется только для стилизации текста -->
<h2>80% клиентов возвращаются к нам</h2>
```

## Влияние на SEO в России

В 2025 году поисковые системы, особенно Яндекс, уделяют большое внимание структуре заголовков:

- Заголовок `<h1>` должен содержать ключевые слова страницы
- Подзаголовки `<h2>` помогают поисковым системам понять структуру контента
- Использование ключевых слов в заголовках улучшает ранжирование

Пример SEO-оптимизированной иерархии:

```html
<h1>Разработка веб-сайтов в Москве 2025</h1>
<h2>Современные технологии разработки</h2>
<h3>Реактивные фреймворки</h3>
<h3>Адаптивный дизайн</h3>
<h2>Преимущества нашей студии</h2>
<h3>Опыт работы</h3>
<h3>Портфолио проектов</h3>
```

## Доступность и скринридеры

Заголовки играют важную роль в доступности веб-контента:

1. Пользователи скринридеров используют заголовки для навигации по странице
2. Заголовки позволяют быстро ориентироваться в структуре документа
3. Правильная иерархия заголовков улучшает восприятие контента пользователями с ограниченными возможностями

## Практические рекомендации

1. **Используйте только один `<h1>`** на страницу
2. **Следуйте логической иерархии** — не пропускайте уровни
3. **Включайте ключевые слова** в заголовки для SEO
4. **Сделайте заголовки информативными** — они должны точно описывать содержимое
5. **Проверяйте иерархию** с помощью инструментов разработчика

## Пример комплексной структуры заголовков

```html
<!DOCTYPE html>
<html lang="ru">
<head>
    <meta charset="UTF-8">
    <title>Разработка веб-сайтов</title>
</head>
<body>
    <header>
        <h1>Веб-студия "Технологии 2025"</h1>
    </header>
    
    <main>
        <section>
            <h2>Наши услуги</h2>
            <article>
                <h3>Создание корпоративных сайтов</h3>
                <h4>Лендинги</h4>
                <h4>Корпоративные порталы</h4>
            </article>
            <article>
                <h3>Интернет-магазины</h3>
                <h4>Маркетплейсы</h4>
                <h4>Одностраничные магазины</h4>
            </article>
        </section>
        
        <section>
            <h2>Портфолио</h2>
            <h3>Кейсы клиентов</h3>
            <h4>Проект "Компания A"</h4>
            <h4>Проект "Компания B"</h4>
        </section>
    </main>
    
    <footer>
        <h2>Контактная информация</h2>
        <p>г. Москва, ул. Технологическая, 15</p>
    </footer>
</body>
</html>
```

## Инструменты проверки иерархии

Для проверки правильности иерархии заголовков можно использовать:

- Инструменты разработчика в браузере
- SEO-плагины (например, SEOquake)
- Специализированные сервисы проверки доступности
