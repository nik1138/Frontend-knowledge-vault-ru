# Структура HTML страницы

HTML (HyperText Markup Language) — это основа любого веб-сайта. Понимание структуры HTML-страницы критически важно для любого веб-разработчика, особенно в контексте современных требований российского рынка веб-разработки в 2025 году.

## Базовая структура HTML документа

Каждая HTML-страница должна начинаться с объявления <!DOCTYPE html>, которое указывает браузеру, что документ следует стандарту HTML5:

```html
<!DOCTYPE html>
<html lang="ru">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Заголовок страницы</title>
</head>
<body>
    <!-- Содержимое страницы -->
</body>
</html>
```

### Объяснение основных элементов:

- `<!DOCTYPE html>` — объявление типа документа HTML5
- `<html>` — корневой элемент всей HTML-страницы
- `lang="ru"` — атрибут, указывающий язык содержимого (в данном случае — русский)
- `<head>` — содержит метаинформацию о документе
- `<meta charset="UTF-8">` — указывает кодировку документа
- `<meta name="viewport">` — обеспечивает адаптивность на мобильных устройствах
- `<title>` — заголовок страницы, отображаемый во вкладке браузера
- `<body>` — содержит видимое содержимое страницы

## Элементы секционирования

HTML5 предоставляет несколько элементов для логического разделения страницы:

- `<header>` — верхний колонтитул страницы или раздела
- `<nav>` — навигационные ссылки
- `<main>` — основное содержимое документа
- `<section>` — тематическая группа содержимого
- `<article>` — самостоятельный фрагмент контента
- `<aside>` — контент, косвенно связанный с основным содержимым
- `<footer>` — нижний колонтитул страницы или раздела

Пример структуры с использованием семантических элементов:

```html
<!DOCTYPE html>
<html lang="ru">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Пример структуры HTML страницы</title>
</head>
<body>
    <header>
        <h1>Мой веб-сайт</h1>
        <nav>
            <ul>
                <li><a href="#home">Главная</a></li>
                <li><a href="#about">О нас</a></li>
                <li><a href="#contact">Контакты</a></li>
            </ul>
        </nav>
    </header>

    <main>
        <section id="home">
            <h2>Главная страница</h2>
            <p>Это основной контент главной страницы.</p>
        </section>

        <article>
            <h2>Статья</h2>
            <p>Это содержимое статьи.</p>
        </article>

        <aside>
            <h3>Боковая панель</h3>
            <p>Дополнительная информация.</p>
        </aside>
    </main>

    <footer>
        <p>&copy; 2025 Мой сайт. Все права защищены.</p>
    </footer>
</body>
</html>
```

## Важные мета-теги для SEO в России

В 2025 году SEO-оптимизация остается критически важной для российских веб-сайтов. Следующие мета-теги помогают улучшить видимость в поисковых системах, особенно в Яндексе и Google:

```html
<meta name="description" content="Краткое описание содержимого страницы для отображения в поисковой выдаче">
<meta name="keywords" content="ключевые, слова, для, страницы">
<meta name="author" content="Имя автора или компании">
<meta property="og:title" content="Заголовок для социальных сетей">
<meta property="og:description" content="Описание для социальных сетей">
<meta property="og:image" content="URL-адрес изображения для социальных сетей">
<meta name="yandex-verification" content="код верификации для Яндекса">
```

## Доступность (Accessibility)

Согласно требованиям российского законодательства о доступности веб-контента (ГОСТ Р 52872-2012), все веб-сайты должны быть доступны для пользователей с ограниченными возможностями:

```html
<img src="image.jpg" alt="Описание изображения">
<button aria-label="Описание кнопки">Кнопка</button>
<label for="input-field">Описание поля ввода</label>
<input type="text" id="input-field" aria-describedby="help-text">
<div id="help-text">Текст помощи</div>
```

## Практические рекомендации

1. **Используйте семантические элементы** — они улучшают доступность и SEO
2. **Обязательно указывайте атрибут `lang`** — это важно для скринридеров
3. **Добавляйте `charset`** — всегда используйте UTF-8
4. **Оптимизируйте под мобильные устройства** — используйте viewport meta-тег
5. **Следите за вложенностью тегов** — теги должны быть правильно вложены друг в друга
