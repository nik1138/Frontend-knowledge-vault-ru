# Атрибуты HTML

Атрибуты HTML — это дополнительные параметры, которые предоставляют дополнительную информацию об элементах. Они всегда указываются в открывающем теге и обычно появляются в виде пар имя="значение".

## Общие сведения об атрибутах

Атрибуты позволяют:

- Настроить поведение элементов
- Предоставить дополнительную информацию
- Управление стилями и взаимодействием
- Обеспечить доступность контента

### Синтаксис атрибутов

```html
<тег атрибут="значение">Содержимое</тег>
<тег атрибут1="значение1" атрибут2="значение2">Содержимое</тег>
```

## Глобальные атрибуты

Глобальные атрибуты могут использоваться с любым HTML-элементом:

### `id` — уникальный идентификатор

```html
<div id="main-content">Основное содержимое</div>
<p id="disclaimer">Это важное уведомление</p>
```

### `class` — классы для стилизации и скриптов

```html
<p class="highlight">Выделенный абзац</p>
<div class="container main-content">Контейнер с несколькими классами</div>
```

### `style` — встроенные стили

```html
<p style="color: red; font-size: 16px;">Текст красного цвета</p>
```

### `title` — всплывающая подсказка

```html
<a href="https://example.com" title="Перейти на примерный сайт">Ссылка</a>
```

### `data-*` — пользовательские атрибуты данных

```html
<div data-user-id="123" data-user-role="admin">Пользовательские данные</div>
```

## Атрибуты доступности (ARIA)

Атрибуты ARIA (Accessible Rich Internet Applications) улучшают доступность веб-контента:

### `role` — роль элемента

```html
<div role="banner">Шапка сайта</div>
<div role="navigation">Навигация</div>
<div role="main">Основной контент</div>
<div role="contentinfo">Футер</div>
```

### `aria-label` — доступное имя элемента

```html
<button aria-label="Закрыть окно">×</button>
<input type="text" aria-label="Поиск по сайту">
```

### `aria-describedby` — описание элемента

```html
<label for="password">Пароль:</label>
<input type="password" id="password" aria-describedby="password-help">
<div id="password-help">Пароль должен содержать не менее 8 символов</div>
```

### `tabindex` — порядок фокуса

```html
<div tabindex="0">Этот div может получить фокус</div>
<button tabindex="-1">Этот элемент исключен из последовательности табуляции</button>
```

## Атрибуты специфичных элементов

### Атрибуты изображений

```html
<img 
    src="image.jpg" 
    alt="Описание изображения для доступности" 
    width="300" 
    height="200" 
    title="Всплывающая подсказка"
    loading="lazy"  <!-- Ленивая загрузка -->
>
```

### Атрибуты ссылок

```html
<a 
    href="https://example.com" 
    target="_blank"  <!-- Открыть в новой вкладке -->
    rel="noopener noreferrer"  <!-- Безопасность при target="_blank" -->
    download  <!-- Загрузить файл -->
    title="Описание ссылки"
>Ссылка</a>
```

### Атрибуты форм

```html
<form action="/submit" method="post" enctype="multipart/form-data">
    <input 
        type="text" 
        name="username" 
        placeholder="Введите имя пользователя" 
        required 
        minlength="3" 
        maxlength="20"
        autocomplete="username"
    >
    <input type="email" name="email" required>
    <textarea name="message" rows="5" cols="40" placeholder="Ваше сообщение"></textarea>
    <select name="country">
        <option value="ru" selected>Россия</option>
        <option value="ua">Украина</option>
        <option value="by">Беларусь</option>
    </select>
    <input type="submit" value="Отправить">
</form>
```

## Атрибуты мультимедиа

### Атрибуты аудио

```html
<audio 
    src="audio.mp3" 
    controls 
    autoplay 
    loop 
    muted 
    preload="auto"
>
    Ваш браузер не поддерживает аудио.
</audio>
```

### Атрибуты видео

```html
<video 
    src="video.mp4" 
    controls 
    width="640" 
    height="480" 
    poster="poster.jpg"
    preload="metadata"
    autoplay 
    muted
>
    Ваш браузер не поддерживает видео.
    <track kind="subtitles" src="subtitles.ru.vtt" srclang="ru" label="Русские">
</video>
```

## Российские особенности использования атрибутов

В 2025 году в России при использовании HTML-атрибутов важно учитывать:

- **Требования 449-ФЗ** — обязательное использование атрибутов доступности (`alt`, `title`, `aria-*`)
- **Соответствие требованиям РКН** — корректные атрибуты для государственных сайтов
- **Поддержка кириллических атрибутов** — правильная кодировка и локализация
- **Совместимость с российскими браузерами** — проверка работы атрибутов в Яндекс.Браузере, Mail.Ru Browser и др.

## Практические рекомендации

1. **Используйте атрибут `alt`** для всех изображений для доступности
2. **Добавляйте `rel="noopener noreferrer"`** при использовании `target="_blank"`
3. **Используйте семантически значимые атрибуты** (`role`, `aria-*`) для улучшения доступности
4. **Не используйте специальные символы в значениях атрибутов** без экранирования
5. **Проверяйте валидность** с помощью HTML-валидатора

## Пример комплексного использования атрибутов

```html
<!DOCTYPE html>
<html lang="ru">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Пример использования атрибутов</title>
</head>
<body>
    <header role="banner">
        <h1 id="main-title">Пример использования атрибутов</h1>
        <nav role="navigation" aria-label="Основная навигация">
            <ul>
                <li><a href="/" aria-current="page">Главная</a></li>
                <li><a href="/about" title="О нас">О нас</a></li>
                <li><a href="/services">Услуги</a></li>
            </ul>
        </nav>
    </header>
    
    <main role="main">
        <article>
            <header>
                <h2>Интересная статья</h2>
                <time datetime="2025-01-01" pubdate>1 января 2025</time>
            </header>
            
            <section>
                <h3>Изображение с атрибутами</h3>
                <figure>
                    <img 
                        src="image.jpg" 
                        alt="Описание изображения для доступности" 
                        width="300" 
                        height="200" 
                        loading="lazy"
                        data-category="nature"
                    >
                    <figcaption>Подпись к изображению</figcaption>
                </figure>
                
                <p>В этой статье мы рассмотрим <strong>важные атрибуты</strong> HTML, 
                которые <em title="Имеют особое значение">особенно важны</em> в 2025 году.</p>
                
                <form 
                    action="/submit" 
                    method="post" 
                    class="contact-form" 
                    id="contact-form"
                    novalidate
                >
                    <label for="name">Имя:</label>
                    <input 
                        type="text" 
                        id="name" 
                        name="name" 
                        required 
                        aria-describedby="name-help"
                        autocomplete="name"
                    >
                    <div id="name-help" class="help-text">Введите ваше имя</div>
                    
                    <label for="email">Email:</label>
                    <input 
                        type="email" 
                        id="email" 
                        name="email" 
                        required
                        autocomplete="email"
                    >
                    
                    <button type="submit">Отправить</button>
                </form>
            </section>
        </article>
    </main>
    
    <footer role="contentinfo">
        <p>&copy; 2025 Все права защищены</p>
    </footer>
</body>
</html>
```
