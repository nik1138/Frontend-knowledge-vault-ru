# Введение в HTML

HTML (HyperText Markup Language) — это основной язык разметки для создания веб-страниц. Он определяет структуру контента веб-страницы и позволяет вставлять текст, изображения, видео, ссылки и другие элементы.

## Что такое HTML?

HTML — это язык разметки, а не программирования. Он использует теги для обозначения элементов на странице. Браузеры интерпретируют HTML-код и отображают его в виде визуального представления пользователю.

## История HTML

HTML был разработан Тимом Бернерсом-Ли в 1991 году в CERN. С тех пор он прошел через несколько версий:

- HTML 1.0 (1991) — базовая версия
- HTML 2.0 (1995) — первый официальный стандарт
- HTML 3.2 (1997) — добавлены таблицы, математические формулы
- HTML 4.01 (1999) — последняя "классическая" версия
- XHTML (2000) — XML-версия HTML
- HTML5 (2014) — современный стандарт с расширенными возможностями

## Основные принципы HTML

HTML документ состоит из элементов, которые представлены в виде тегов. Каждый тег может иметь:

- Открывающий тег (например, `<p>`)
- Закрывающий тег (например, `</p>`)
- Атрибуты (например, `class="example"`)

```html
<!DOCTYPE html>
<html lang="ru">
<head>
    <meta charset="UTF-8">
    <title>Пример HTML-документа</title>
</head>
<body>
    <h1>Заголовок первого уровня</h1>
    <p>Это абзац текста.</p>
</body>
</html>
```

## Типичная структура HTML-документа

- `<!DOCTYPE html>` — объявление типа документа
- `<html>` — корневой элемент документа
- `<head>` — содержит метаданные
- `<title>` — заголовок страницы
- `<body>` — видимая часть страницы

## Современные особенности HTML5

HTML5 включает в себя множество новых элементов и атрибутов:

- Семантические элементы: `<header>`, `<nav>`, `<main>`, `<section>`, `<article>`, `<aside>`, `<footer>`
- Элементы мультимедиа: `<audio>`, `<video>`
- Формы: улучшенные элементы управления и валидация
- API: доступ к геолокации, локальному хранилищу и другим возможностям

## Практические советы

1. **Всегда используйте `<!DOCTYPE html>`** в начале документа
2. **Указывайте язык документа** с помощью атрибута `lang`
3. **Используйте семантические элементы** для лучшей доступности
4. **Добавляйте атрибут `alt`** к изображениям для доступности
5. **Проверяйте валидность кода** с помощью [HTML-валидатора](https://validator.w3.org/)

## Российские особенности

- **Доступности веб-контента** в соответствии с требованиями 449-ФЗ
- **Совместимости с российскими браузерами** (Яндекс.Браузер, Mail.Ru Browser, etc.)
- **Локализации контента** с поддержкой кириллицы и других символов
- **Соответствию требованиям РКН** к размещению информации
