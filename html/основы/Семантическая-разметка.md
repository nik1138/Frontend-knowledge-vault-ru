# Семантическая разметка HTML

## Общее описание

Семантическая разметка HTML — это использование HTML-элементов в соответствии с их предполагаемым смыслом и назначением. Это позволяет создавать структурированные, доступные и SEO-оптимизированные веб-страницы, которые легко понимаются как людьми, так и машинами (поисковыми роботами, скринридерами и другими вспомогательными технологиями).

## Основные семантические элементы

### Структурные элементы

```html
<!DOCTYPE html>
<html lang="ru">
<head>
  <meta charset="UTF-8">
  <title>Семантическая разметка HTML</title>
</head>
<body>
  <header>
    <h1>Название сайта</h1>
    <nav>
      <ul>
        <li><a href="/">Главная</a></li>
        <li><a href="/about">О нас</a></li>
        <li><a href="/contact">Контакты</a></li>
      </ul>
    </nav>
  </header>

  <main>
    <article>
      <header>
        <h2>Заголовок статьи</h2>
        <p>Опубликовано: <time datetime="2025-01-15">15 января 2025</time></p>
      </header>
      <section>
        <h3>Раздел статьи</h3>
        <p>Содержимое раздела статьи...</p>
      </section>
      <footer>
        <p>Автор: Иван Иванов</p>
      </footer>
    </article>
  </main>

  <aside>
    <h3>Боковая панель</h3>
    <p>Дополнительная информация</p>
  </aside>

  <footer>
    <p>&copy; 2025 Все права защищены</p>
  </footer>
</body>
</html>
```

### Основные структурные элементы

| Элемент | Назначение |
|---------|------------|
| `<header>` | Шапка страницы или раздела |
| `<nav>` | Навигационные ссылки |
| `<main>` | Основное содержимое страницы |
| `<article>` | Независимый контент (статья, пост) |
| `<section>` | Логический раздел контента |
| `<aside>` | Сопутствующий контент |
| `<footer>` | Подвал страницы или раздела |

## Семантические элементы контента

### Элемент `<article>`

Используется для независимого, самодостаточного контента:

```html
<article>
  <h2>Новость дня</h2>
  <p>Опубликовано: <time datetime="2025-01-15T10:30:00+03:00">15 января 2025, 10:30</time></p>
  <p>Текст новости...</p>
  <footer>
    <p>Автор: <address>Иван Петров</address></p>
  </footer>
</article>
```

### Элемент `<section>`

Группирует связанные по смыслу элементы:

```html
<section>
  <h2>Услуги компании</h2>
  <article>
    <h3>Веб-разработка</h3>
    <p>Создание современных веб-сайтов и приложений</p>
  </article>
  <article>
    <h3>Дизайн</h3>
    <p>Проектирование пользовательского интерфейса</p>
  </article>
</section>
```

### Элемент `<aside>`

Содержит контент, косвенно связанный с основным содержимым:

```html
<article>
  <h2>Современные технологии веб-разработки</h2>
  <p>Текст статьи о современных технологиях...</p>
  
  <aside>
    <h3>Интересный факт</h3>
    <p>HTML был создан Тимом Бернерсом-Ли в 1990 году</p>
  </aside>
</article>
```

## Элементы для данных и определений

### Элемент `<figure>` и `<figcaption>`

```html
<figure>
  <img src="chart.png" alt="Диаграмма роста пользователей">
  <figcaption>Динамика роста пользователей за 2024 год</figcaption>
</figure>
```

### Элементы для определений

```html
<dl>
  <dt>HTML</dt>
  <dd>HyperText Markup Language - язык гипертекстовой разметки</dd>
  <dt>CSS</dt>
  <dd>Cascading Style Sheets - каскадные таблицы стилей</dd>
</dl>
```

### Элемент `<time>`

```html
<p>Встреча состоится <time datetime="2025-02-20T14:00:00+03:00">20 февраля в 14:00</time>.</p>
<time datetime="2025-01-01">Новый 2025 год!</time>
```

```html
<article>

    <h1>Современные веб-технологии в России</h1>
    <p>Опубликовано: <time datetime="2025-01-15">15 января 2025 года</time></p>
  </header>

    <aside>
      <h3>Полезная информация</h3>

    </aside>
  </section>
  
  <footer>
    <p>Автор: <address>Иванов И.И.</address></p>
```

### Соответствие требованиям доступности

```html
<main>
  <h1>Главная страница</h1>
  
  <nav aria-label="Основная навигация">
    <ul>
      <li><a href="/">Главная</a></li>
      <li><a href="/services">Услуги</a></li>
      <li><a href="/contacts">Контакты</a></li>
    </ul>
  </nav>
  
  <article aria-labelledby="article-title">
    <h2 id="article-title">Новости компании</h2>
    <p>Текст новости с правильной семантикой...</p>
  </article>
  
  <aside aria-labelledby="sidebar-title">
    <h2 id="sidebar-title">Полезные ссылки</h2>
    <ul>
      <li><a href="/faq">Частые вопросы</a></li>
      <li><a href="/support">Поддержка</a></li>
    </ul>
  </aside>
</main>
```

### Локализация и SEO

```html
<!DOCTYPE html>
<html lang="ru" dir="ltr">
<head>
  <meta charset="UTF-8">
  <title>Семантическая разметка HTML в российских проектах</title>
  <meta name="description" content="Использование семантической разметки HTML в российских веб-проектах 2025 года">
</head>
<body>
  <header role="banner">
    <h1>Российский веб-стандарт</h1>
    <nav role="navigation" aria-label="Навигация по сайту">
      <ul>
        <li><a href="/html">HTML</a></li>
        <li><a href="/css">CSS</a></li>
        <li><a href="/js">JavaScript</a></li>
      </ul>
    </nav>
  </header>
  
  <main role="main">
    <article>
      <h2>Семантическая разметка в 2025 году</h2>
      <section>
        <h3>Преимущества использования</h3>
        <p>Семантическая разметка улучшает доступность и SEO...</p>
      </section>
    </article>
  </main>
</body>
</html>
```

## Современные практики 2025 года

### Семантика и ARIA

```html
<header role="banner">
  <h1>Название сайта</h1>
  <nav role="navigation" aria-label="Основная навигация">
    <ul>
      <li><a href="/" aria-current="page">Главная</a></li>
      <li><a href="/about">О нас</a></li>
      <li><a href="/services">Услуги</a></li>
    </ul>
  </nav>
</header>

<main role="main">
  <section aria-labelledby="services-heading">
    <h2 id="services-heading">Наши услуги</h2>
    <article role="article">
      <h3>Веб-разработка</h3>
      <p>Создание современных веб-решений</p>
    </article>
  </section>
</main>
```

### Микроразметка Schema.org

```html
<article itemscope itemtype="http://schema.org/Article">
  <h2 itemprop="headline">Семантическая разметка HTML</h2>
  <div itemprop="author" itemscope itemtype="http://schema.org/Person">
    <span itemprop="name">Иван Иванов</span>
  </div>
  <time itemprop="datePublished" datetime="2025-01-15">15 января 2025</time>
  <div itemprop="articleBody">
    <p>Содержимое статьи...</p>
  </div>
</article>
```

### Адаптивная семантическая разметка

```html
<div class="layout-container">
  <header class="site-header">
    <h1 class="logo">Название сайта</h1>
    <button class="menu-toggle" aria-expanded="false" aria-controls="main-nav">
      Меню
    </button>
    <nav id="main-nav" class="main-navigation">
      <ul>
        <li><a href="/">Главная</a></li>
        <li><a href="/about">О нас</a></li>
        <li><a href="/services">Услуги</a></li>
      </ul>
    </nav>
  </header>
  
  <main class="main-content">
    <article class="content-article">
      <header class="article-header">
        <h2>Адаптивная верстка в 2025 году</h2>
        <p>Опубликовано: <time datetime="2025-01-10">10 января 2025</time></p>
      </header>
      <div class="article-body">
        <p>Текст статьи...</p>
      </div>
    </article>
  </main>
  
  <aside class="sidebar" aria-label="Дополнительная информация">
    <h3>Популярные статьи</h3>
    <ul>
      <li><a href="/article1">Статья 1</a></li>
      <li><a href="/article2">Статья 2</a></li>
    </ul>
  </aside>
  
  <footer class="site-footer">
    <p>&copy; 2025 Все права защищены</p>
  </footer>
</div>
```

## Практические рекомендации

1. **Используйте правильные теги** для их предназначения, а не только для стилизации
2. **Следите за иерархией заголовков** - не пропускайте уровни
3. **Добавляйте атрибуты доступности** при необходимости
4. **Проверяйте разметку** с помощью инструментов валидации
5. **Учитывайте мобильные устройства** при создании структуры
6. **Обеспечивайте доступность** для пользователей со скринридерами

### Проверка семантической корректности

```html
<!-- Хорошо: Правильная структура -->
<article>
  <header>
    <h2>Заголовок статьи</h2>
  </header>
  <section>
    <h3>Подраздел</h3>
    <p>Содержимое...</p>
  </section>
  <footer>
    <p>Информация в подвале статьи</p>
  </footer>
</article>

<!-- Плохо: Неправильное использование тегов -->
<div>
  <div class="header-style">
    <div class="title-style">Заголовок статьи</div>
  </div>
  <div class="section-style">
    <div class="subtitle-style">Подраздел</div>
    <div>Содержимое...</div>
  </div>
</div>
```
