# Аудио и видео элементы HTML

## Общее описание

HTML5 предоставляет встроенные элементы для встраивания аудио и видео контента без необходимости использования сторонних плагинов. Элементы `<audio>` и `<video>` позволяют воспроизводить мультимедийные файлы прямо в браузере с возможностью настройки внешнего вида и функциональности.

## Элемент `<audio>`

### Базовое использование

```html
<audio controls>
  <source src="audio.mp3" type="audio/mpeg">
  <source src="audio.ogg" type="audio/ogg">
  Ваш браузер не поддерживает аудио элементы. 
  <a href="audio.mp3">Скачать файл</a> для прослушивания.
</audio>
```

### Атрибуты элемента `<audio>`

| Атрибут | Описание |
|---------|----------|
| `controls` | Отображает элементы управления (плейер) |
| `autoplay` | Автоматически запускает воспроизведение |
| `loop` | Повторяет воспроизведение после окончания |
| `muted` | Выключает звук по умолчанию |
| `preload` | Определяет, как загружать аудио (`none`, `metadata`, `auto`) |

### Пример с различными атрибутами

```html
<audio controls autoplay loop muted preload="metadata">
  <source src="podcast.mp3" type="audio/mpeg">
  <source src="podcast.ogg" type="audio/ogg">
  <source src="podcast.wav" type="audio/wav">
  Ваш браузер не поддерживает аудио элементы.
</audio>
```

## Элемент `<video>`

### Базовое использование

```html
<video width="640" height="360" controls poster="preview.jpg">
  <source src="video.mp4" type="video/mp4">
  <source src="video.webm" type="video/webm">
  <source src="video.ogv" type="video/ogg">
  Ваш браузер не поддерживает видео элементы.
  <a href="video.mp4">Скачать видео</a>
</video>
```

### Атрибуты элемента `<video>`

| Атрибут | Описание |
|---------|----------|
| `controls` | Отображает элементы управления |
| `autoplay` | Автоматически запускает воспроизведение |
| `loop` | Повторяет воспроизведение |
| `muted` | Выключает звук по умолчанию |
| `poster` | URL изображения-обложки до начала воспроизведения |
| `preload` | Определяет, как загружать видео |
| `width`, `height` | Размеры видео в пикселях |

## Форматы файлов

### Аудио форматы

- **MP3** (`audio/mpeg`) - наиболее распространенный, широкая поддержка
- **OGG** (`audio/ogg`) - открытый формат, хорошее сжатие
- **WAV** (`audio/wav`) - высокое качество, большие файлы

### Видео форматы

- **MP4** (`video/mp4`) - наиболее распространенный, широкая поддержка
- **WebM** (`video/webm`) - открытый формат от Google
- **OGV** (`video/ogg`) - открытый формат

> [!tip]
> Всегда указывайте несколько форматов для обеспечения совместимости с разными браузерами.

## Российские особенности и практики

### Загрузка контента

В российских веб-проектах важно учитывать скорость загрузки контента:

```html
<video controls preload="metadata" poster="thumbnail.jpg">
  <!-- Указываем высококачественный MP4 как основной формат -->
  <source src="video-hd.mp4" type="video/mp4">
  <!-- Резервный вариант для старых браузеров -->
  <source src="video-low.mp4" type="video/mp4">
  Ваш браузер не поддерживает видео элементы.
</video>
```

### Альтернативные решения

Для критически важных проектов в России рекомендуется предусмотреть альтернативные способы доступа к контенту:

```html
<video controls>
  <source src="video.mp4" type="video/mp4">
  <p>Ваш браузер не поддерживает видео элементы.</p>
  <p>Альтернативы:</p>
  <ul>
    <li><a href="video.mp4">Скачать видео в формате MP4</a></li>
    <li><a href="transcript.txt">Транскрибация видео</a></li>
  </ul>
</video>
```

### Доступность

В 2025 году в России усилены требования к доступности веб-сайтов:

```html
<video controls width="640" height="360">
  <source src="video.mp4" type="video/mp4">
  <!-- Добавляем титры для видео -->
  <track kind="subtitles" src="subtitles-ru.vtt" srclang="ru" label="Русский">
  <track kind="subtitles" src="subtitles-en.vtt" srclang="en" label="English">
  <track kind="captions" src="captions.vtt" srclang="ru" label="Описания">
  Ваш браузер не поддерживает видео элементы.
</video>
```

## Современные практики 2025 года

### Адаптивное видео

```html
<div class="video-container">
  <video controls>
    <source src="video-mobile.mp4" media="(max-width: 768px)" type="video/mp4">
    <source src="video-desktop.mp4" media="(min-width: 769px)" type="video/mp4">
    <source src="video.mp4" type="video/mp4">
    Ваш браузер не поддерживает видео элементы.
  </video>
</div>

<style>
.video-container {
  position: relative;
  width: 100%;
  height: 0;
  padding-bottom: 56.25%; /* 16:9 Aspect Ratio */
}

.video-container video {
  position: absolute;
  top: 0;
  left: 0;
  width: 100%;
  height: 100%;
}
</style>
```

### Lazy loading для видео

```html
<video controls loading="lazy">
  <source src="video.mp4" type="video/mp4">
  Ваш браузер не поддерживает видео элементы.
</video>
```

## JavaScript API

HTML5 медиа элементы предоставляют богатый JavaScript API для программного управления:

```javascript
// Получаем элемент видео
const video = document.querySelector('video');

// События воспроизведения
video.addEventListener('play', function() {
  console.log('Видео началось');
});

video.addEventListener('pause', function() {
  console.log('Видео приостановлено');
});

video.addEventListener('ended', function() {
  console.log('Видео завершено');
});

// Программное управление
video.play();  // Начать воспроизведение
video.pause(); // Приостановить воспроизведение
video.currentTime = 30; // Перейти к 30 секунде
```

## Практические рекомендации

1. **Всегда указывайте несколько форматов** для обеспечения совместимости
2. **Используйте атрибут `preload`** разумно - `metadata` для экономии трафика
3. **Добавляйте атрибут `poster`** для видео для лучшего UX
4. **Обеспечивайте доступность** с помощью титров и альтернативного контента
5. **Оптимизируйте размер файлов** для быстрой загрузки в российских сетях
