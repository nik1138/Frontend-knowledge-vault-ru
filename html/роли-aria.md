# ARIA Роли в HTML

ARIA (Accessible Rich Internet Applications) - это набор атрибутов, которые помогают улучшить доступность веб-приложений для пользователей с ограниченными возможностями. ARIA роли и атрибуты обеспечивают дополнительную семантическую информацию для вспомогательных технологий, таких как скринридеры.

## Что такое ARIA

ARIA - это техническая спецификация, разработанная W3C для улучшения доступности веб-контента и пользовательских интерфейсов. Она позволяет разработчикам добавлять семантическую информацию к элементам, которые не имеют встроенной семантики или имеют сложное поведение.

## Основные ARIA Роли

### Ландшафтные роли
- `banner` - верхняя часть страницы с навигацией
- `main` - основное содержимое страницы
- `navigation` - область навигации
- `complementary` - дополнительная информация
- `contentinfo` - нижняя часть страницы
- `search` - область поиска

### Структурные роли
- `article` - независимый контент
- `section` - раздел документа
- `group` - группа связанных элементов
- `presentation` - элемент без семантической роли

### Виджетные роли
- `button` - интерактивная кнопка
- `checkbox` - флажок
- `dialog` - диалоговое окно
- `menu` - меню
- `menuitem` - элемент меню
- `progressbar` - индикатор выполнения
- `slider` - ползунок
- `tab` - вкладка
- `tabpanel` - панель вкладки
- `tooltip` - подсказка

## Живые регионы (Live Regions)

Живые регионы позволяют вспомогательным технологиям получать информацию о динамических изменениях на странице без фокуса.

### Атрибуты живых регионов:
- `aria-live` - определяет, когда и как читать изменения
  - `off` - не читать (по умолчанию)
  - `polite` - читать, когда нет других действий
  - `assertive` - читать немедленно
- `aria-atomic` - определяет, читать ли весь регион целиком
- `aria-relevant` - определяет, какие изменения читать (additions, removals, text)

### Пример использования живой области:
```html
<div aria-live="polite" aria-atomic="true">
  <span id="status">Загрузка...</span>
</div>

<script>
  // При изменении статуса скринридер прочитает обновление
  document.getElementById('status').textContent = 'Загрузка завершена';
</script>
```

## Атрибуты состояния и свойств

### Атрибуты состояния:
- `aria-checked` - состояние флажка или переключателя
- `aria-disabled` - состояние отключения элемента
- `aria-expanded` - состояние раскрытия/сворачивания
- `aria-hidden` - видимость для вспомогательных технологий
- `aria-invalid` - состояние ошибки
- `aria-pressed` - состояние нажатия кнопки
- `aria-selected` - состояние выбора элемента

### Атрибуты свойств:
- `aria-label` - текстовая метка элемента
- `aria-labelledby` - ссылка на элемент-метку
- `aria-describedby` - ссылка на элемент с описанием
- `aria-controls` - элементы, контролируемые текущим
- `aria-owns` - дочерние элементы в семантическом дереве

## Семантика vs ARIA

Семантические HTML-элементы (как `button`, `nav`, `main`) имеют встроенную доступность и должны использоваться в первую очередь. ARIA используется для дополнения или переопределения семантики, когда семантические элементы недостаточны.

> [!warning] Важно
> Используйте семантические HTML-элементы, когда это возможно. ARIA добавляйте только при необходимости.

## Примеры использования

### Кнопка с ARIA:
```html
<div role="button" tabindex="0" aria-pressed="false" onclick="toggle()">
  Переключатель
</div>
```

### Меню с ARIA:
```html
<ul role="menu" aria-label="Главное меню">
  <li role="menuitem">Файл</li>
  <li role="menuitem">Правка</li>
  <li role="menuitem">Помощь</li>
</ul>
```

### Вкладки с ARIA:
```html
<div role="tablist">
  <button role="tab" aria-selected="true" aria-controls="panel1">Вкладка 1</button>
  <button role="tab" aria-selected="false" aria-controls="panel2">Вкладка 2</button>
</div>

<div id="panel1" role="tabpanel">
  Содержимое первой вкладки
</div>
<div id="panel2" role="tabpanel" hidden>
  Содержимое второй вкладки
</div>
```

## Доступность

Правильное использование ARIA ролей и атрибутов значительно улучшает опыт пользователей с ограниченными возможностями. Важно:

- Проверять доступность с помощью инструментов и скринридеров
- Обеспечивать согласованность в использовании ролей
- Следить за обновлениями спецификации ARIA
- Тестировать с реальными пользователями

## Заключение

ARIA является мощным инструментом для создания доступных веб-приложений. При правильном использовании он позволяет пользователям с ограниченными возможностями эффективно взаимодействовать с интерфейсами. Помните о принципе минимальной необходимости - используйте ARIA только там, где стандартной семантики недостаточно.
