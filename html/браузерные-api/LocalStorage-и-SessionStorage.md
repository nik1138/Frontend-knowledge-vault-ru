# LocalStorage и SessionStorage

## Обзор

`LocalStorage` и `SessionStorage` являются частью Web Storage API, которые позволяют веб-приложениям хранить данные локально в браузере пользователя. Эти API предоставляют простой способ хранения пар ключ-значение, где значения представлены в виде строк.

### LocalStorage

`LocalStorage` сохраняет данные в браузере без срока истечения. Данные будут храниться до тех пор, пока пользователь не очистит их вручную через настройки браузера или пока веб-приложение не удалит их с помощью JavaScript.

### SessionStorage

`SessionStorage` хранит данные только для одной сессии. Данные будут удалены при закрытии вкладки или браузера. Это полезно для временных данных, которые не должны сохраняться между сеансами.

## Основные методы

### LocalStorage

- `setItem(key, value)` - сохраняет пару ключ-значение.
- `getItem(key)` - возвращает значение по ключу.
- `removeItem(key)` - удаляет пару ключ-значение.
- `clear()` - удаляет все пары ключ-значение.

### SessionStorage

Аналогичные методы, но с префиксом `sessionStorage` вместо `localStorage`.

## Практические рекомендации

### Использование LocalStorage

LocalStorage подходит для хранения данных, которые должны сохраняться между сеансами, например, настройки пользователя или состояние приложения. Однако, не стоит хранить конфиденциальные данные, так как они могут быть доступны через JavaScript.

### Использование SessionStorage

SessionStorage полезен для хранения временных данных, таких как содержимое формы, которое должно быть доступно при навигации по вкладке. Также может использоваться для хранения данных, которые не должны сохраняться между сеансами.

## Примеры использования

### Сохранение настроек пользователя

```javascript
// Сохранение настроек
localStorage.setItem('theme', 'dark');
localStorage.setItem('language', 'ru');

// Получение настроек
const theme = localStorage.getItem('theme');
const language = localStorage.getItem('language');
```

### Временное хранение данных формы

```javascript
// Сохранение данных формы
sessionStorage.setItem('formData', JSON.stringify(formData));

// Получение данных формы
const formData = JSON.parse(sessionStorage.getItem('formData'));
```

## Ограничения и особенности

- Объем хранилища ограничен (обычно 5-10 МБ).
- Данные хранятся в виде строк, поэтому для хранения объектов используйте `JSON.stringify()` и `JSON.parse()`.
- Данные доступны только в том же домене и протоколе.
