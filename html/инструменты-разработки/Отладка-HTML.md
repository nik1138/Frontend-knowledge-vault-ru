# Отладка HTML

Отладка HTML - это процесс поиска и устранения ошибок в HTML-разметке, структуре документа и взаимодействии с другими технологиями веб-разработки. В 2025 году отладка HTML требует понимания не только базовых принципов, но и современных подходов к веб-разработке.

## Основные виды ошибок HTML

### Синтаксические ошибки
Синтаксические ошибки - наиболее распространенные проблемы в HTML:

#### Неправильное закрытие тегов
```html
<!-- Неправильно -->
<div>
  <p>Текст параграфа
</div>

<!-- Правильно -->
<div>
  <p>Текст параграфа</p>
</div>
```

#### Неверное вложение тегов
```html
<!-- Неправильно -->
<p><div>Неверное вложение</div></p>

<!-- Правильно -->
<div><p>Правильное вложение</p></div>
```

#### Отсутствие кавычек в атрибутах
```html
<!-- Неправильно -->
<img src=photo.jpg alt=Фотография>

<!-- Правильно -->
<img src="photo.jpg" alt="Фотография">
```

### Семантические ошибки
- Неправильное использование заголовков (h1-h6)
- Использование div вместо семантических тегов
- Нарушение логической структуры документа

### Ошибки доступности
- Отсутствие атрибутов alt у изображений
- Неправильное использование ARIA-атрибутов
- Отсутствие заголовков у таблиц

## Инструменты отладки HTML

### Браузерные инструменты разработчика
Браузерные DevTools - основной инструмент для отладки HTML:

#### Панель Elements
- Просмотр и редактирование DOM-дерева
- Проверка структуры документа
- Анализ применяемых стилей

> [!tip]
> Используйте горячую клавишу `F12` или `Ctrl+Shift+I` (Cmd+Option+I на Mac) для открытия DevTools

#### Панель Console
- Просмотр ошибок и предупреждений
- Взаимодействие с DOM через JavaScript
- Отладка скриптов, взаимодействующих с HTML

#### Панель Accessibility
- Анализ дерева доступности
- Проверка соответствия требованиям WCAG
- Проверка ролей и свойств элементов

### Онлайн-валидаторы
#### W3C Markup Validator
- Официальный валидатор HTML
- Проверка соответствия стандартам
- Поддержка HTML5

#### Nu Html Checker
- Современный валидатор от W3C
- Более строгая проверка
- Поддержка новых стандартов

### Инструменты командной строки
#### HTMLHint
```bash
# Установка
npm install -g htmlhint

# Проверка файла
htmlhint index.html

# Проверка всех HTML-файлов
htmlhint src/**/*.html
```

#### HTML Tidy
```bash
# Установка и использование
tidy -e index.html  # Проверка с выводом ошибок
tidy -f errors.txt index.html  # Запись ошибок в файл
```

## Методы отладки

### Визуальная отладка
#### Использование временных стилей
```css
/* Временные стили для отладки структуры */
* {
  outline: 1px solid red;
}
```

#### Цветовое выделение блоков
```css
/* Выделение различных типов элементов */
header { background-color: rgba(255, 0, 0, 0.1); }
nav { background-color: rgba(0, 255, 0, 0.1); }
main { background-color: rgba(0, 0, 255, 0.1); }
footer { background-color: rgba(255, 255, 0, 0.1); }
```

### Отладка структуры
#### Проверка иерархии заголовков
Используйте DevTools для проверки правильной иерархии:
- Только один h1 на странице
- Последовательность h1 → h2 → h3 и т.д.
- Логическая структура документа

#### Проверка вложенности элементов
- Убедитесь, что блочные элементы не вложены в строчные
- Проверьте корректность вложенности таблиц
- Убедитесь в правильной структуре списков

### Отладка форм
#### Проверка атрибутов
- Правильное использование `for` и `id` для label
- Атрибуты валидации HTML5
- Правильное использование `name` и `id`

#### Тестирование доступности
- Проверка навигации с клавиатуры
- Тестирование с экранного диктора
- Проверка контрастности

## Практические техники отладки

### Техника "разделения и проверки"
1. Изолируйте проблемную часть HTML
2. Создайте минимальный воспроизводимый пример
3. Проверьте каждый элемент по отдельности
4. Постепенно добавляйте элементы обратно

### Использование временных комментариев
```html
<!-- НАЧАЛО: Проблемная секция -->
<div class="container">
  <!-- Содержимое -->
</div>
<!-- КОНЕЦ: Проблемная секция -->
```

### Проверка через разные браузеры
- Chrome, Firefox, Safari, Edge
- Мобильные браузеры
- Альтернативные браузеры (включая российские)

> [!warning]
> В России в 2025 году продолжается использование локальных браузеров. Обязательно тестируйте HTML-контент в Яндекс.Браузере и других локальных решениях.

### Использование инструментов автоматизации
#### Selenium WebDriver
Для автоматизированного тестирования HTML-структур:
```javascript
const { Builder, By } = require('selenium-webdriver');

async function checkHTMLStructure() {
  let driver = await new Builder().forBrowser('chrome').build();
  await driver.get('http://example.com');
  
  // Проверка наличия элементов
  let header = await driver.findElement(By.tagName('header'));
  let main = await driver.findElement(By.tagName('main'));
  
  await driver.quit();
}
```

#### Puppeteer
Для автоматизированной проверки визуала и структуры:
```javascript
const puppeteer = require('puppeteer');

async function checkHTML() {
  const browser = await puppeteer.launch();
  const page = await browser.newPage();
  await page.goto('http://example.com');
  
  // Проверка структуры
  const headings = await page.$$('.header h1, .header h2');
  console.log(`Найдено заголовков: ${headings.length}`);
  
  await browser.close();
}
```
