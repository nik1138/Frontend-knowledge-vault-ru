# Линтеры для HTML

HTML-линтеры - это инструменты, которые проверяют HTML-код на соответствие стандартам, выявляют ошибки и помогают поддерживать высокое качество кода. В 2025 году использование линтеров стало обязательной практикой для профессиональных веб-разработчиков в России и по всему миру.

## Что такое HTML-линтер

HTML-линтер - это инструмент, который анализирует HTML-код и проверяет его на:
- Синтаксические ошибки
- Несоответствие стандартам HTML
- Потенциальные проблемы доступности
- Нарушение семантической структуры
- Использование устаревших элементов

## Популярные HTML-линтеры в 2025 году

### HTMLHint
HTMLHint - один из самых популярных линтеров для HTML, который легко интегрируется в проекты.

**Установка:**
```bash
npm install -g htmlhint
```

**Конфигурационный файл (.htmlhintrc):**
```json
{
  "tagname-lowercase": true,
  "attr-lowercase": true,
  "attr-value-double-quotes": true,
  "doctype-first": true,
  "tag-pair": true,
  "spec-char-escape": true,
  "id-unique": true,
  "src-not-empty": true,
  "attr-no-duplication": true,
  "title-require": true
}
```

### HTML-Validator (W3C)
Официальный валидатор W3C доступен как веб-сервис и через API:
- Онлайн: validator.w3.org
- API для автоматизации процесса проверки

### Pretty Diff
Инструмент, который не только проверяет HTML, но и форматирует код:
- Проверка синтаксиса
- Форматирование
- Сравнение файлов

### ESLint с HTML-плагинами
Некоторые разработчики используют ESLint с плагинами для проверки HTML-фрагментов в JavaScript:
- eslint-plugin-html
- eslint-plugin-lit

### Интеграция с системами сборки

#### Gulp
```javascript
const htmlhint = require('gulp-htmlhint');
const gulp = require('gulp');

gulp.task('htmlhint', function() {
  return gulp.src('src/*.html')
    .pipe(htmlhint('.htmlhintrc'))
    .pipe(htmlhint.reporter('htmlhint-stylish'));
});
```

#### Grunt
```javascript
grunt.initConfig({
  htmlhint: {
    build: {
      options: {
        htmlhintrc: '.htmlhintrc'
      },
      src: ['src/*.html']
    }
  }
});
```

#### Webpack
С использованием html-webpack-plugin и соответствующих лоадеров.

## Правила валидации

### Основные правила
1. **Закрытие тегов** - все парные теги должны быть закрыты
2. **Кавычки в атрибутах** - значения атрибутов должны быть в кавычках
3. **Уникальность ID** - атрибуты id должны быть уникальными
4. **Типы документов** - документ должен начинаться с объявления doctype
5. **Структура заголовков** - правильная иерархия h1-h6

### Правила доступности
1. **Alt для изображений** - все img должны иметь атрибут alt
2. **Семантическая разметка** - использование подходящих тегов (header, nav, main, section и т.д.)
3. **Якоря для ссылок** - все ссылки должны иметь содержимое
4. **Формы** - правильное использование label и атрибутов доступности

> [!tip]
> В России с 2021 года действуют требования к доступности веб-сайтов. HTML-линтеры помогают соблюдать требования ГОСТ Р 52872-2015 и WCAG 2.1.

### Правила производительности
1. **Размер документа** - минимизация количества DOM-элементов
2. **Валидность ссылок** - проверка целостности ссылок
3. **Оптимизация изображений** - правильное использование атрибутов размеров

## Практические рекомендации

3. Используйте линтеры для рефакторинга старого кода
4. Автоматизируйте проверки качества кода

> [!warning]
> Не переусердствуйте с количеством правил - это может замедлить разработку. Выбирайте правила, которые действительно улучшают качество кода.
