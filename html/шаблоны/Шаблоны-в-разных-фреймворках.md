# Шаблоны в разных фреймворках

## Использование HTML-шаблонов в популярных фреймворках

В 2025 году в российской веб-разработке используются различные JavaScript-фреймворки, каждый из которых имеет свои подходы к созданию и использованию HTML-шаблонов. Понимание этих различий позволяет выбирать наиболее подходящий инструмент для конкретного проекта и эффективно использовать шаблоны в каждом из фреймворков.

## React

React использует JSX - синтаксис, который позволяет писать HTML-подобный код внутри JavaScript. Это позволяет создавать компоненты с встроенными шаблонами.

### Базовый компонент в React

```jsx
import React, { useState, useEffect } from 'react';

// Функциональный компонент с хуками
const ProductCard = ({ product }) => {
  const [isFavorite, setIsFavorite] = useState(false);
  const [isLoading, setIsLoading] = useState(false);

  const toggleFavorite = () => {
    setIsFavorite(!isFavorite);
  };

  const handleBuy = () => {
    setIsLoading(true);
    // Симуляция API-запроса
    setTimeout(() => {
      setIsLoading(false);
      alert(`Товар "${product.name}" добавлен в корзину`);
    }, 1000);
  };

  return (
    <div className="product-card">
      <img 
        src={product.image} 
        alt={product.name} 
        width="200" 
        height="200"
        loading="lazy"
      />
      <h3>{product.name}</h3>
      <p className="price">{product.price} ₽</p>
      <p className="description">{product.description}</p>
      
      <div className="product-actions">
        <button 
          className={`favorite-btn ${isFavorite ? 'active' : ''}`}
          onClick={toggleFavorite}
          aria-label={isFavorite ? 'Убрать из избранного' : 'Добавить в избранное'}
        >
          {isFavorite ? '★' : '☆'}
        </button>
        
        <button 
          className="buy-btn"
          onClick={handleBuy}
          disabled={isLoading}
        >
          {isLoading ? 'Загрузка...' : 'Купить'}
        </button>
      </div>
    </div>
  );
};

// Компонент списка товаров
const ProductList = ({ products }) => {
  return (
    <div className="product-list">
      {products.map(product => (
        <ProductCard 
          key={product.id} 
          product={product} 
        />
      ))}
    </div>
  );
};

// Основной компонент приложения
const App = () => {
  const [products, setProducts] = useState([]);
  const [loading, setLoading] = useState(true);

  useEffect(() => {
    // Симуляция загрузки данных
    setTimeout(() => {
      setProducts([
        {
          id: 1,
          name: 'Смартфон',
          price: 29990,
          description: 'Высокопроизводительный смартфон с отличной камерой',
          image: '/images/phone.jpg'
        },
        {
          id: 2,
          name: 'Ноутбук',
          price: 59990,
          description: 'Мощный ноутбук для работы и игр',
          image: '/images/laptop.jpg'
        },
        {
          id: 3,
          name: 'Наушники',
          price: 7990,
          description: 'Беспроводные наушники с шумоподавлением',
          image: '/images/headphones.jpg'
        }
      ]);
      setLoading(false);
    }, 1000);
  }, []);

  if (loading) {
    return <div className="loading">Загрузка товаров...</div>;
  }

  return (
    <div className="app">
      <header>
        <h1>Интернет-магазин</h1>
      </header>
      
      <main>
        <ProductList products={products} />
      </main>
      
      <footer>
        <p>&copy; 2025 Интернет-магазин. Все права защищены.</p>
      </footer>
    </div>
  );
};

export default App;
```

## Vue.js

Vue.js использует декларативный рендеринг и компонентную архитектуру с возможностью создания шаблонов в отдельных файлах (.vue).

### Компонент в Vue.js

```vue
<template>
  <div class="product-card">
    <img 
      :src="product.image" 
      :alt="product.name" 
      width="200" 
      height="200"
      loading="lazy"
    />
    <h3>{{ product.name }}</h3>
    <p class="price">{{ formattedPrice }}</p>
    <p class="description">{{ product.description }}</p>
    
    <div class="product-actions">
      <button 
        :class="['favorite-btn', { active: isFavorite }]"
        @click="toggleFavorite"
        :aria-label="isFavorite ? 'Убрать из избранного' : 'Добавить в избранное'"
      >
        {{ isFavorite ? '★' : '☆' }}
      </button>
      
      <button 
        class="buy-btn"
        @click="handleBuy"
        :disabled="isLoading"
      >
        {{ isLoading ? 'Загрузка...' : 'Купить' }}
      </button>
    </div>
  </div>
</template>

<script>
export default {
  name: 'ProductCard',
  props: {
    product: {
      type: Object,
      required: true
    }
  },
  data() {
    return {
      isFavorite: false,
      isLoading: false
    };
  },
  computed: {
    formattedPrice() {
      return new Intl.NumberFormat('ru-RU', {
        style: 'currency',
        currency: 'RUB'
      }).format(this.product.price);
    }
  },
  methods: {
    toggleFavorite() {
      this.isFavorite = !this.isFavorite;
    },
    handleBuy() {
      this.isLoading = true;
      // Симуляция API-запроса
      setTimeout(() => {
        this.isLoading = false;
        alert(`Товар "${this.product.name}" добавлен в корзину`);
      }, 1000);
    }
  }
};
</script>
```

## Angular

Angular использует TypeScript и компонентный подход с декораторами и шаблонами, которые могут быть определены в отдельных HTML-файлах или встроенными.

### Компонент в Angular

```typescript
// product-card.component.ts
import { Component, Input, Output, EventEmitter } from '@angular/core';

interface Product {
  id: number;
  name: string;
  price: number;
  description: string;
  image: string;
}

@Component({
  selector: 'app-product-card',
  templateUrl: './product-card.component.html',
  styleUrls: ['./product-card.component.css']
})
export class ProductCardComponent {
  @Input() product!: Product;
  @Output() buyEvent = new EventEmitter<Product>();
  
  isFavorite: boolean = false;
  isLoading: boolean = false;

  toggleFavorite(): void {
    this.isFavorite = !this.isFavorite;
  }

  handleBuy(): void {
    this.isLoading = true;
    // Симуляция API-запроса
    setTimeout(() => {
      this.isLoading = false;
      this.buyEvent.emit(this.product);
    }, 1000);
  }

  get formattedPrice(): string {
    return new Intl.NumberFormat('ru-RU', {
      style: 'currency',
      currency: 'RUB'
    }).format(this.product.price);
  }
}
```

```html
</div>
```

## Практические рекомендации

1. **Выбор фреймворка**: Выбирайте фреймворк в зависимости от требований проекта, команды и сроков разработки.

2. **Производительность**: Учитывайте особенности рендеринга и виртуального DOM в каждом фреймворке.

3. **SEO**: Используйте SSR (Server-Side Rendering) или SSG (Static Site Generation) для улучшения SEO.

4. **Размер бандла**: Оптимизируйте размер бандла, используя tree-shaking и код-сплиттинг.

5. **Доступность**: Обеспечивайте доступность в шаблонах, используя ARIA-атрибуты и семантические элементы.

6. **Типизация**: Используйте TypeScript для улучшения надежности и поддержки кода.

7. **Тестирование**: Пишите unit- и интеграционные тесты для компонентов.

8. **Документация**: Поддерживайте актуальную документацию по шаблонам и компонентам.

## Заключение

Каждый фреймворк предлагает свои уникальные подходы к созданию HTML-шаблонов, и выбор подходящего инструмента зависит от конкретных требований проекта. В российских реалиях 2025 года важно учитывать не только технические аспекты, но и доступность, производительность и SEO. Правильное использование шаблонов в различных фреймворках позволяет создавать эффективные, поддерживаемые и пользовательски-ориентированные веб-приложения.
