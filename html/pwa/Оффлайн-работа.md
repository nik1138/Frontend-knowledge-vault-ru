# Оффлайн работа

Оффлайн работа — одна из ключевых возможностей прогрессивных веб-приложений (PWA), позволяющая пользователям продолжать использовать приложение даже при отсутствии подключения к интернету. Это достигается за счет использования [[Service Workers]], [[Кэширование|кэширования]] и других технологий, которые позволяют хранить данные и функциональность локально на устройстве пользователя.

## Принципы оффлайн работы

### 1. Предварительное кэширование

При первом посещении приложения критически важные ресурсы сохраняются в кэше:

```javascript
const CACHE_NAME = 'offline-v1.3.0';
const OFFLINE_URL = '/offline.html';
const OFFLINE_RESOURCES = [
  '/',
  '/styles/main.css',
  '/scripts/main.js',
  '/images/logo.png',
  OFFLINE_URL
];

self.addEventListener('install', (event) => {
  event.waitUntil(
    caches.open(CACHE_NAME)
      .then((cache) => cache.addAll(OFFLINE_RESOURCES))
  );
});
```

### 2. Перехват сетевых запросов

[[Service Workers]] перехватывают все сетевые запросы и возвращают кэшированные ресурсы при отсутствии соединения:

```javascript
self.addEventListener('fetch', (event) => {
  if (event.request.mode === 'navigate') {
    event.respondWith(
      fetch(event.request)
        .catch(() => caches.open(CACHE_NAME)
          .then((cache) => cache.match(OFFLINE_URL)))
    );
  }
});
```

### 3. Синхронизация при восстановлении соединения

При восстановлении интернет-соединения приложение может синхронизировать накопленные данные:

```javascript
// Использование Background Sync API
if ('serviceWorker' in navigator && 'sync' in window.navigator) {
  navigator.serviceWorker.ready
    .then((registration) => {
      registration.sync.register('sync-data');
    });
}

self.addEventListener('sync', (event) => {
  if (event.tag === 'sync-data') {
    event.waitUntil(syncData());
  }
});
```

## Стратегии оффлайн работы

### 1. Полностью автономное приложение

Приложение может функционировать без интернета, используя только локальные данные:

- Кэширование всех необходимых ресурсов
- Хранение данных в IndexedDB
- Локальная обработка всех операций

### 2. Частичная автономная работа

Только определенные функции доступны без интернета:

- Просмотр кэшированного контента
- Локальное редактирование данных
- Очередь на синхронизацию при восстановлении соединения

### 3. Режим ожидания

При отсутствии интернета показывается информационная страница с возможностью продолжить работу при восстановлении соединения.

## Реализация оффлайн функциональности

### 1. Создание оффлайн-страницы

Создайте специальную страницу для отображения при отсутствии соединения:

```html
<!DOCTYPE html>
<html lang="ru">
<head>
  <meta charset="utf-8">
  <title>Нет соединения</title>
  <style>
    body { font-family: Arial, sans-serif; text-align: center; padding: 50px; }
    .offline-icon { font-size: 64px; margin-bottom: 20px; }
  </style>
</head>
<body>
  <div class="offline-icon">✈️</div>
  <h1>Нет соединения с интернетом</h1>
  <p>Приложение работает в автономном режиме</p>
  <p id="status">Попытка восстановления соединения...</p>
  
  <script>
    // Проверка соединения
    window.addEventListener('online', () => {
      document.getElementById('status').textContent = 'Соединение восстановлено! Перезагрузка...';
      setTimeout(() => window.location.reload(), 1000);
    });
  </script>
</body>
</html>
```

### 2. Обработка состояния сети

В JavaScript можно проверять состояние сети:

```javascript
// Проверка доступности интернета
function checkOnlineStatus() {
  return navigator.onLine;
}

// События изменения состояния сети
window.addEventListener('online', () => {
  console.log('Соединение восстановлено');
  // Синхронизация данных
});

window.addEventListener('offline', () => {
  console.log('Соединение потеряно');
  // Переключение в оффлайн-режим
});
```

### 3. Хранение данных в автономном режиме

Использование IndexedDB для хранения данных при отсутствии соединения:

```javascript
class OfflineDataStore {
  constructor() {
    this.dbName = 'PWAOfflineDB';
    this.version = 1;
    this.db = null;
  }

  async init() {
    return new Promise((resolve, reject) => {
      const request = indexedDB.open(this.dbName, this.version);

      request.onerror = () => reject(request.error);
      request.onsuccess = () => {
        this.db = request.result;
        resolve();
      };

      request.onupgradeneeded = (event) => {
        const db = event.target.result;
        if (!db.objectStoreNames.contains('user-data')) {
          const store = db.createObjectStore('user-data', { keyPath: 'id' });
          store.createIndex('timestamp', 'timestamp', { unique: false });
        }
      };
    });
  }

  async saveData(data) {
    const transaction = this.db.transaction(['user-data'], 'readwrite');
    const store = transaction.objectStore('user-data');
    
    return store.add({
      id: Date.now(),
      data: data,
      timestamp: new Date().toISOString()
    });
  }

  async getOfflineData() {
    const transaction = this.db.transaction(['user-data'], 'readonly');
    const store = transaction.objectStore('user-data');
    
    return new Promise((resolve, reject) => {
      const request = store.getAll();
      request.onsuccess = () => resolve(request.result);
      request.onerror = () => reject(request.error);
    });
  }
}
```

## Практические примеры оффлайн функциональности

### 1. Оффлайн-редактор заметок

```javascript
class OfflineNoteEditor {
  constructor() {
    this.db = new OfflineDataStore();
    this.notes = [];
    this.init();
  }

  async init() {
    await this.db.init();
    await this.loadNotes();
    this.setupEventListeners();
  }

  setupEventListeners() {
    // Синхронизация при подключении к интернету
    window.addEventListener('online', () => {
      this.syncNotes();
    });
  }

  async saveNote(note) {
    if (navigator.onLine) {
      // Сохранение на сервере
      await this.saveToServer(note);
    } else {
      // Сохранение в локальное хранилище
      note.status = 'pending';
      await this.db.saveData(note);
    }
  }

  async syncNotes() {
    const offlineNotes = await this.db.getOfflineData();
    
    for (const note of offlineNotes) {
      try {
        await this.saveToServer(note.data);
        await this.db.deleteFromStore(note.id); // Удаление после успешной синхронизации
      } catch (error) {
        console.error('Ошибка синхронизации заметки:', error);
      }
    }
  }
}
```

### 2. Оффлайн-чтение контента

```javascript
class OfflineContentReader {
  constructor() {
    this.cacheName = 'content-cache-v1';
  }

  async prefetchContent(urls) {
    const cache = await caches.open(this.cacheName);
    return cache.addAll(urls);
  }

  async getContent(url) {
    if (navigator.onLine) {
      const response = await fetch(url);
      if (response.ok) {
        // Кэширование полученного контента
        const cache = await caches.open(this.cacheName);
        cache.put(url, response.clone());
        return response.json();
      }
    }

    // Возврат кэшированного контента
    const cache = await caches.open(this.cacheName);
    const cachedResponse = await cache.match(url);
    return cachedResponse ? cachedResponse.json() : null;
  }
}
```

## Особенности оффлайн работы в России 2025

При разработке оффлайн-функциональности для российского рынка необходимо учитывать:

- Нестабильность интернет-соединения в удаленных регионах
- Ограничения на доступ к зарубежным сервисам
- Требования к защите персональных данных (ФЗ-152)
- Возможные ограничения на использование определенных API
- Локализация интерфейса и сообщений об оффлайн-режиме
- Совместимость с российскими браузерами (Яндекс.Браузер, Mail.Ru Browser)

## Лучшие практики

### 1. Индикация оффлайн-режима

Четко информируйте пользователя о текущем состоянии подключения:

```javascript
function updateConnectionStatus() {
  const statusElement = document.getElementById('connection-status');
  
  if (navigator.onLine) {
    statusElement.textContent = 'Подключено';
    statusElement.className = 'online';
  } else {
    statusElement.textContent = 'Оффлайн';
    statusElement.className = 'offline';
  }
}
```

### 2. Экономное использование ресурсов

- Минимизируйте объем кэшируемых данных
- Используйте сжатие изображений и других ресурсов
- Реализуйте стратегию очистки устаревших данных

### 3. Обработка ошибок

Обеспечьте корректную обработку ситуаций при восстановлении соединения:

```javascript
async function handleConnectionRestored() {
  try {
    await syncPendingOperations();
    showNotification('Данные синхронизированы');
  } catch (error) {
    showNotification('Ошибка синхронизации, повтор через 30 секунд');
    setTimeout(handleConnectionRestored, 30000);
  }
}
```

## Заключение

Оффлайн работа — важная особенность PWA, которая значительно улучшает пользовательский опыт, особенно в условиях нестабильного интернет-соединения. Правильная реализация оффлайн функциональности требует тщательного планирования архитектуры кэширования, обработки состояния сети и синхронизации данных.

> [!tip] Совет
> Начинайте с реализации базовой оффлайн-функциональности, а затем постепенно добавляйте более сложные возможности, такие как фоновая синхронизация.

> [!warning] Важно
> В России требования к защите персональных данных строже, чем в других странах, поэтому учитывайте ФЗ-152 при хранении данных в автономном режиме.
