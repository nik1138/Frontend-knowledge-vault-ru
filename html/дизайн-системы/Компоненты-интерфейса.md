# Компоненты интерфейса в HTML дизайн-системе

## Определение компонентов

Компоненты интерфейса — это переиспользуемые элементы пользовательского интерфейса, реализованные в HTML с соответствующей стилизацией и поведением. Они формируют основу для построения пользовательских интерфейсов в рамках дизайн-системы.

## Типы компонентов

### Атомарные компоненты
- [[Токены-дизайна]] - базовые значения (цвета, размеры, шрифты)
- Кнопки, инпуты, чекбоксы
- Текстовые элементы

### Молекулярные компоненты
- Группы атомарных компонентов
- Формы ввода
- Навигационные элементы

### Организмы
- Комплексные структуры из молекул
- Секции интерфейса
- Макеты страниц

## Структура HTML компонентов

### Семантическая разметка
Каждый компонент должен использовать семантические теги HTML5:
- `<header>`, `<nav>`, `<main>`, `<section>`, `<article>`, `<aside>`, `<footer>`
- `<button>`, `<input>`, `<select>`, `<textarea>` для интерактивных элементов
- `<figure>`, `<figcaption>`, `<time>`, `<address>` для специфических данных

### Атрибуты доступности
- `role` для определения типа компонента для скринридеров
- `aria-*` атрибуты для дополнительного описания
- `tabindex` для управления фокусом

## Практические рекомендации по созданию компонентов

### Структура файлов компонентов
```
components/
├── button/
│   ├── button.html
│   ├── button.css
│   ├── button.js
│   └── button.md
├── form/
│   ├── form.html
│   ├── form.css
│   └── form.md
```

### HTML разметка компонента
```html
<!-- Пример кнопки -->
<button class="ds-btn ds-btn--primary" type="button" aria-label="Основное действие">
  <span class="ds-btn__text">Нажми меня</span>
</button>
```

### Классификация компонентов
- `ds-` - префикс дизайн-системы
- `component-name` - имя компонента
- `--modifier` - модификаторы состояния
- `__element` - дочерние элементы компонента

## Российские особенности 2025

### Локализация компонентов
- Поддержка кириллических шрифтов
- Учет особенностей русского языка (падежи, род, склонения)
- Адаптация под требования российского законодательства

### Безопасность
- Валидация вводимых данных на клиенте
- Защита от XSS атак через атрибуты
- Соответствие требованиям отечественных стандартов

## Примеры компонентов

### Кнопка
```html
<button class="ds-btn ds-btn--primary ds-btn--size-m" type="submit">
  <span class="ds-btn__text">Отправить</span>
</button>
```

### Форма ввода
```html
<form class="ds-form">
  <div class="ds-form__group">
    <label class="ds-form__label" for="email">Электронная почта</label>
    <input class="ds-input ds-input--email" type="email" id="email" required>
  </div>
</form>
```

## Заключение

Компоненты интерфейса являются кирпичиками дизайн-системы, обеспечивающими консистентность и масштабируемость интерфейсов. Правильная архитектура компонентов позволяет командам эффективно работать над проектами и поддерживать единый стиль.
