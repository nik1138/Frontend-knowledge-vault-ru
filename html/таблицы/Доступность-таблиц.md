# Доступность HTML-таблиц

## Введение

Доступность HTML-таблиц - это критически важный аспект веб-разработки, особенно в свете российского законодательства о доступности цифровой среды. С 2025 года требования к доступности веб-сайтов становятся все более строгими, и правильная реализация таблиц играет важную роль в обеспечении равного доступа к информации для всех пользователей, включая людей с ограниченными возможностями.

## Законодательная база

В Российской Федерации действуют следующие нормативные акты, регулирующие доступность:
- Федеральный закон №426-ФЗ "О специальной оценке условий труда"
- Технический регламент Таможенного союза "Безопасность информационных систем"
- Рекомендации WCAG 2.1 AA уровня

## Основные принципы доступности таблиц

### Использование семантически правильных элементов

```html
<table>
  <caption>Ежемесячные продажи по регионам в 2025 году</caption>
  <thead>
    <tr>
      <th scope="col">Регион</th>
      <th scope="col">Январь</th>
      <th scope="col">Февраль</th>
      <th scope="col">Март</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <th scope="row">Москва</th>
      <td>1 200 000 ₽</td>
      <td>1 350 000 ₽</td>
      <td>1 420 000 ₽</td>
    </tr>
    <tr>
      <th scope="row">Санкт-Петербург</th>
      <td>800 000 ₽</td>
      <td>890 000 ₽</td>
      <td>920 000 ₽</td>
    </tr>
  </tbody>
</table>
```

### Атрибут scope для заголовков

Атрибут `scope` помогает программам чтения с экрана определить, к каким данным относится заголовок:

- `scope="col"` - заголовок применяется ко всему столбцу
- `scope="row"` - заголовок применяется ко всей строке

```html
<th scope="col">Месяц</th>
<th scope="row">Продажи</th>
```

## Заголовки таблицы

### Элемент caption

Элемент `<caption>` предоставляет описание таблицы и должен быть первым дочерним элементом `<table>`:

```html
<table>
  <caption>Финансовый отчет компании ООО "Рога и Копыта" за 2025 год</caption>
  <!-- содержимое таблицы -->
</table>
```

### Ассоциация ячеек с заголовками

Для сложных таблиц используйте атрибуты `headers` и `id`:

```html
<table>
  <tr>
    <th id="month">Месяц</th>
    <th id="sales">Продажи</th>
    <th id="profit">Прибыль</th>
  </tr>
  <tr>
    <td headers="month">Январь</td>
    <td headers="sales">1 200 000 ₽</td>
    <td headers="profit">300 000 ₽</td>
  </tr>
</table>
```

## Семантическая структура

### Группировка строк

Используйте элементы `<thead>`, `<tbody>` и `<tfoot>` для логической группировки строк:

```html
<table>
  <thead>
    <tr>
      <th>Продукт</th>
      <th>Количество</th>
      <th>Цена</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <th scope="row">Товар 1</th>
      <td>10</td>
      <td>1 000 ₽</td>
    </tr>
    <tr>
      <th scope="row">Товар 2</th>
      <td>5</td>
      <td>2 500 ₽</td>
    </tr>
  </tbody>
  <tfoot>
    <tr>
      <th>Итого</th>
      <td>15</td>
      <td>22 500 ₽</td>
    </tr>
  </tfoot>
</table>
```

## ARIA-атрибуты для таблиц

### Роль таблицы

Для дополнительной ясности можно использовать ARIA-роли:

```html
<table role="table" aria-label="Финансовый отчет за 2025 год">
  <!-- содержимое таблицы -->
</table>
```

### Описание сложных таблиц

Для сложных таблиц используйте `aria-describedby`:

```html
<p id="table-description">Таблица содержит данные о продажах по регионам за первый квартал 2025 года</p>
<table aria-describedby="table-description">
  <!-- содержимое таблицы -->
</table>
```

## Адаптивность и доступность

### Таблицы на мобильных устройствах

Для обеспечения доступности на мобильных устройствах:

```css
/* Адаптация таблицы для мобильных устройств */
@media (max-width: 768px) {
  table, thead, tbody, th, td, tr {
    display: block;
  }
  
  thead tr {
    position: absolute;
    top: -9999px;
    left: -9999px;
  }
  
  tr {
    border: 1px solid #ccc;
    margin-bottom: 10px;
    padding: 10px;
  }
  
  tr td {
    border: none;
    position: relative;
    padding-left: 50%;
    text-align: left;
  }
  
  tr td:before {
    content: attr(data-label) ": ";
    position: absolute;
    left: 6px;
    width: 45%;
    text-align: left;
    font-weight: bold;
  }
}
```

Соответствующая HTML-структура:

```html
<table>
  <thead>
    <tr>
      <th>Регион</th>
      <th>Продажи</th>
      <th>Прибыль</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td data-label="Регион">Москва</td>
      <td data-label="Продажи">1 200 000 ₽</td>
      <td data-label="Прибыль">300 000 ₽</td>
    </tr>
  </tbody>
</table>
```

## Проверка доступности

### Инструменты для проверки

- axe DevTools
- WAVE
- Lighthouse
- Web Accessibility Toolbar

### Ручная проверка

1. Проверьте, читается ли заголовок таблицы программой чтения с экрана
2. Убедитесь, что отношения между заголовками и данными понятны
3. Проверьте, как таблица отображается в режиме высокой контрастности
4. Протестируйте навигацию по таблице с клавиатуры

## Типичные ошибки и как их избежать

### Ошибки структуры

```html
<!-- НЕПРАВИЛЬНО -->
<table>
  <tr>
    <td>Заголовок 1</td>
    <td>Заголовок 2</td>
  </tr>
  <tr>
    <td>Данные 1</td>
    <td>Данные 2</td>
  </tr>
</table>

<!-- ПРАВИЛЬНО -->
<table>
  <tr>
    <th scope="col">Заголовок 1</th>
    <th scope="col">Заголовок 2</th>
  </tr>
  <tr>
    <td>Данные 1</td>
    <td>Данные 2</td>
  </tr>
</table>
```

### Отсутствие заголовков

```html
<!-- НЕПРАВИЛЬНО -->
<table>
  <tr>
    <td>Москва</td>
    <td>1 200 000 ₽</td>
  </tr>
</table>

<!-- ПРАВИЛЬНО -->
<table>
  <caption>Ежемесячные продажи по регионам</caption>
  <tr>
    <th scope="col">Регион</th>
    <th scope="col">Продажи</th>
  </tr>
  <tr>
    <th scope="row">Москва</th>
    <td>1 200 000 ₽</td>
  </tr>
</table>
```

## Современные практики 2025 года

### Использование новых ARIA-ролей

С 2025 года рекомендуется использовать обновленные ARIA-роли:

```html
<table role="grid">
  <thead role="rowgroup">
    <tr role="row">
      <th role="columnheader">Название</th>
      <th role="columnheader">Значение</th>
    </tr>
  </thead>
  <tbody role="rowgroup">
    <tr role="row">
      <td role="gridcell">Продажи</td>
      <td role="gridcell">1 200 000 ₽</td>
    </tr>
  </tbody>
</table>
```

### Поддержка международных стандартов

Обеспечьте совместимость с международными стандартами доступности:

- WCAG 2.1 AA (минимальный уровень)
- WCAG 2.2 (новые критерии, вступившие в силу в 2024-2025)
- Section 508 (для сайтов, работающих с государственными структурами)

## Заключение

Доступность HTML-таблиц в 2025 году требует:
- Соблюдения законодательных норм
- Использования семантически правильной разметки
- Применения ARIA-атрибутов при необходимости
- Тестирования с использованием вспомогательных технологий
- Обеспечения адаптивности на всех устройствах
