# Проверка SEO: пошаговое руководство

## Общие сведения

Проверка SEO (Search Engine Optimization) — это процесс анализа веб-страницы на предмет соответствия требованиям поисковых систем и выявления возможностей для улучшения видимости в поисковой выдаче. В 2025 году, с развитием ИИ-поиска и усилением конкуренции, комплексная проверка SEO становится особенно важной для российских веб-сайтов.

## Основные этапы SEO-проверки

### 1. Технический аудит

#### Проверка HTML-валидности

> [!tip] Совет
> Используйте [[Валидатор-HTML]] для проверки корректности HTML-кода

Каждый HTML-документ должен соответствовать стандартам:

- Правильная структура документа
- Закрытые теги
- Правильное использование семантических тегов
- Соответствие спецификации HTML5

Пример правильной структуры HTML5:

```html
<!DOCTYPE html>
<html lang="ru">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Заголовок страницы</title>
    <meta name="description" content="Описание страницы">
</head>
<body>
    <header>
        <nav>...</nav>
    </header>
    <main>
        <article>
            <h1>Заголовок статьи</h1>
            <p>Контент статьи...</p>
        </article>
    </main>
    <footer>...</footer>
</body>
</html>
```

#### Проверка доступности (accessibility)

- Использование тегов заголовков в правильной иерархии
- Альтернативный текст для изображений (`alt`)
- Соответствие цветового контраста требованиям WCAG
- Клавиатурная навигация

### 2. Проверка контента

#### Качество и уникальность

> [!warning] Важно
> В 2025 году поисковые системы особенно остро реагируют на низкокачественный и дублирующийся контент

Факторы качества контента:
- Релевантность запросу
- Глубина раскрытия темы
- Актуальность информации
- Отсутствие орфографических ошибок
- Авторитетность источника

#### Использование ключевых слов

- Правильная плотность ключевых слов (не более 1-3%)
- Естественное включение в контент
- Использование семантически связанных слов
- Оптимизация заголовков и подзаголовков

### 3. Метатеги и структурированные данные

#### Обязательные метатеги

```html
<title>Заголовок страницы - до 60 символов</title>
<meta name="description" content="Описание страницы - до 160 символов">
<meta name="keywords" content="ключевые, слова, страницы">
<meta name="robots" content="index, follow">
<meta name="viewport" content="width=device-width, initial-scale=1.0">
<link rel="canonical" href="https://example.com/page">
```

> [!note] 
> Метатег `keywords` не влияет на ранжирование в Google, но может использоваться в российских поисковых системах

#### Структурированные данные

См. [[Структурированные-данные]] для детальной информации по реализации.

### 4. Скорость загрузки страницы

#### Проверка производительности

- Время загрузки не должно превышать 3 секунд
- Использование оптимизированных изображений
- Минификация CSS и JavaScript
- Использование CDN для доставки статических ресурсов

#### Инструменты для проверки скорости:

- PageSpeed Insights (Google)
- GTmetrix
- WebPageTest
- Яндекс.Скорость сайта

### 5. Адаптивный дизайн

#### Проверка мобильной версии

- Корректное отображение на устройствах с разными размерами экрана
- Удобство навигации на сенсорных устройствах
- Размер касаемых элементов не менее 48x48 пикселей
- Отсутствие горизонтального скролла

## Проверка для российских веб-сайтов

### Особенности российского сегмента

#### Поисковые системы

- Google (около 40% рынка)
- Яндекс (около 55% рынка)
- Mail.ru, Rambler и другие (около 5%)

> [!tip] Совет
> При оптимизации для российского рынка учитывайте требования как Google, так и Яндекса

#### Региональные особенности

- Использование русского языка и локализованных форматов
- Учет часовых поясов и географических особенностей
- Соответствие требованиям законодательства РФ (например, ФЗ-152)

#### Локальные факторы ранжирования

- Регистрация в Яндекс.Каталоге
- Наличие локальных контактов
- Использование доменов .ru и .рф
- Соответствие требованиям РКН

### Проверка локализованных элементов

#### Языковая локализация

- Правильное указание языка в теге `<html lang="ru">`
- Использование кириллических шрифтов
- Соответствие форматов дат и чисел российским стандартам

#### Контактная информация

- Наличие страницы "Контакты" с полной информацией
- Регистрация в Яндекс.Справочнике
- Интеграция с картами (2GIS, Яндекс.Карты)

## Чек-лист SEO-проверки

### Техническая сторона
- [ ] HTML-документ валиден
- [ ] Используется правильная кодировка (UTF-8)
- [ ] Наличие тега viewport для адаптивности
- [ ] Все изображения имеют атрибут `alt`
- [ ] Нет битых ссылок (404 ошибок)
- [ ] Наличие sitemap.xml
- [ ] Наличие robots.txt
- [ ] Скорость загрузки страницы менее 3 секунд
- [ ] Используется HTTPS

### Контент
- [ ] Заголовок страницы уникален и информативен
- [ ] Мета-описание уникально и информативно
- [ ] Используется правильная иерархия заголовков (h1, h2, h3...)
- [ ] Контент уникален и ценен для пользователя
- [ ] Ключевые слова естественно включены в текст
- [ ] Нет дублирующегося контента

### Структура
- [ ] Наличие семантических тегов (header, nav, main, article, section, footer)
- [ ] Используется канонический URL
- [ ] Структурированные данные правильно реализованы
- [ ] Навигация интуитивно понятна
- [ ] Есть карта сайта для пользователей

### Доступность
- [ ] Сайт работает на мобильных устройствах
- [ ] Цветовой контраст соответствует стандартам
- [ ] Есть альтернативные тексты для изображений
- [ ] Клавиатурная навигация работает корректно

## Автоматизированные инструменты проверки

### Lighthouse

Google Lighthouse включает проверку SEO в дополнение к другим метрикам:

```bash
# Установка Lighthouse CLI
npm install -g @lhci/cli

# Запуск аудита
lhci autorun --collect.url=https://example.com
```

### Axe-core для проверки доступности

```javascript
// Пример использования axe-core для проверки доступности
const axe = require('axe-core');
axe.run(document, { runOnly: ['wcag2a', 'wcag2aa'] }, (err, results) => {
  if (err) throw err;
  console.log(results.violations);
});
```

## Заключение

Проверка SEO в 2025 году требует комплексного подхода, учитывающего как общие принципы оптимизации, так и специфику российского сегмента интернета. Регулярная проверка и оптимизация сайта помогают поддерживать высокую видимость в поисковой выдаче и обеспечивать положительный пользовательский опыт.
