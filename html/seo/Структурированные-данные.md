# Структурированные данные в HTML

## Введение

Структурированные данные (structured data) — это стандартизированный формат для предоставления информации о странице, которая помогает поисковым системам лучше понимать и индексировать контент. В 2025 году структурированные данные остаются важной частью SEO-стратегии, особенно в условиях развития поиска на основе искусственного интеллекта и семантического веба.

## Основные форматы структурированных данных

### JSON-LD (рекомендуемый формат)

JSON-LD (JavaScript Object Notation for Linked Data) — это наиболее предпочтительный формат от Google и других поисковых систем. Он встраивается в HTML-документ с помощью тега `<script>`.

```html
<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "Article",
  "headline": "Руководство по SEO в 2025 году",
  "author": {
    "@type": "Person",
    "name": "Иван Петров"
  },
  "datePublished": "2025-01-15",
  "dateModified": "2025-01-20",
  "publisher": {
    "@type": "Organization",
    "name": "SEO-компания"
  }
}
</script>
```

### Microdata

Microdata — это атрибуты HTML, которые добавляются непосредственно в разметку страницы. Используется реже, чем JSON-LD, но все еще поддерживается.

```html
<div itemscope itemtype="https://schema.org/Article">
  <h1 itemprop="headline">Руководство по SEO в 2025 году</h1>
  <div itemprop="author" itemscope itemtype="https://schema.org/Person">
    <span itemprop="name">Иван Петров</span>
  </div>
  <time itemprop="datePublished" datetime="2025-01-15">15 января 2025</time>
</div>
```

### RDFa

RDFa (Resource Description Framework in Attributes) — это более сложный формат, который редко используется в настоящее время.

## Основные типы схем Schema.org

### Article и NewsArticle

Используются для новостных и информационных статей:

```json
{
  "@context": "https://schema.org",
  "@type": "Article",
  "headline": "SEO-стратегии в 2025 году",
  "author": {
    "@type": "Person",
    "name": "Анна Сидорова"
  },
  "datePublished": "2025-01-10",
  "dateModified": "2025-01-15",
  "image": "https://example.com/image.jpg",
  "description": "Полное руководство по SEO-стратегиям для 2025 года"
}
```

### Organization и LocalBusiness

Для описания организаций и местных бизнесов:

```json
{
  "@context": "https://schema.org",
  "@type": "LocalBusiness",
  "name": "SEO-агентство Москвы",
  "address": {
    "@type": "PostalAddress",
    "streetAddress": "ул. Тверская, 1",
    "addressLocality": "Москва",
    "postalCode": "123456",
    "addressCountry": "RU"
  },
  "telephone": "+7 (495) 123-45-67",
  "openingHours": "Mo-Fr 09:00-18:00",
  "image": "https://example.com/logo.jpg"
}
```

### Product

Для описания товаров в интернет-магазинах:

```json
{
  "@context": "https://schema.org/",
  "@type": "Product",
  "name": "Смартфон Samsung Galaxy S25",
  "image": [
    "https://example.com/photos/1x1/photo.jpg",
    "https://example.com/photos/1x1/photo2.jpg"
   ],
  "description": "Новейший флагманский смартфон от Samsung",
  "sku": "S25-128GB-BLACK",
  "offers": {
    "@type": "Offer",
    "price": "89990",
    "priceCurrency": "RUB",
    "availability": "https://schema.org/InStock"
  }
}
```

### FAQPage

Для создания страниц с часто задаваемыми вопросами:

```json
{
  "@context": "https://schema.org",
  "@type": "FAQPage",
  "mainEntity": [{
    "@type": "Question",
    "name": "Какие факторы влияют на SEO в 2025 году?",
    "acceptedAnswer": {
      "@type": "Answer",
      "text": "Основные факторы: качество контента, скорость загрузки, мобильная адаптация, структурированные данные и пользовательский опыт."
    }
  }]
}
```

## Практические рекомендации для российских веб-сайтов

### Учет особенностей российских поисковых систем

> [!warning] Важно
> В отличие от Google, Яндекс использует собственный формат структурированных данных — Яндекс.Маркет.Райт, хотя и поддерживает Schema.org.

Для Яндекса особенно важно:

1. Использование тега `<meta name="yandex-verification" content="...">` для подтверждения прав на сайт
2. Добавление схемы `@type: "WebSite"` с указанием `alternateName` для брендовых запросов
3. Использование схемы `@type: "BreadcrumbList"` для навигации

### Локализация структурированных данных

При работе с российскими сайтами учитывайте:

- Использование локали `ru-RU` в датах и форматах
- Правильное указание страны и региона в адресах
- Использование российских телефонных форматов (+7 для России)

### Проверка и тестирование

> [!tip] Совет
> Используйте инструменты для проверки структурированных данных перед публикацией

#### Инструменты для проверки:

1. [[Инструменты-для-анализа-HTML]] - для общего анализа
2. Google Rich Results Test - для проверки структурированных данных
3. Яндекс.Вебмастер - для проверки на соответствие требованиям Яндекса
4. Rich Results Test от Google - https://search.google.com/test/rich-results

## Ошибки при реализации

### Распространенные ошибки:

1. **Неправильная вложенность схем** - убедитесь, что свойства вложены правильно
2. **Несоответствие типа данных** - даты должны быть в формате ISO 8601
3. **Отсутствие обязательных свойств** - у каждого типа есть обязательные поля
4. **Неправильные URL** - все URL должны быть абсолютными

> [!note] 
> Структурированные данные не влияют напрямую на ранжирование, но могут улучшить видимость в поисковой выдаче за счет Rich-контента (фрагментов с изображением, рейтингом и т.д.)

## Заключение

Структурированные данные в 2025 году остаются важным элементом SEO-стратегии. Правильное использование схем Schema.org помогает поисковым системам лучше понимать контент и может привести к улучшению видимости сайта в поисковой выдаче. Особенно важно учитывать особенности российских поисковых систем и локализовывать данные под российские реалии.
