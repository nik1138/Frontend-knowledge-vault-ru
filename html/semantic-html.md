# Семантические элементы HTML

## Определение семантической верстки

Семантическая верстка — это подход к созданию HTML-документов, при котором используются элементы, имеющие смысловое значение, соответствующее их содержимому. Такая разметка позволяет лучше понимать структуру документа как людям, так и машинам (поисковым системам, вспомогательным технологиям и т.д.).

Семантические элементы делают код более понятным, улучшают доступность и SEO, а также способствуют лучшему взаимодействию с вспомогательными технологиями, такими как скринридеры.

## Встроенные семантические элементы HTML5

HTML5 предоставляет множество встроенных семантических элементов, которые помогают структурировать документ:

- `<header>` — заголовок раздела или документа
- `<nav>` — навигационные ссылки
- `<main>` — основное содержимое документа
- `<article>` — самостоятельная часть контента
- `<section>` — логически сгруппированный раздел
- `<aside>` — боковая информация
- `<footer>` — нижний колонтитул раздела или документа
- `<time>` — дата и время
- `<figure>` и `<figcaption>` — медиа-контент с подписью
- `<mark>` — выделенный текст
- `<strong>` и `<em>` — важный и акцентированный текст

## Разница между семантическими и несемантическими элементами

Семантические элементы:
- Имеют определенное смысловое значение
- Помогают структурировать документ
- Улучшают доступность и SEO

Пример:
```html
<article>
  <header>
    <h1>Заголовок статьи</h1>
  </header>
  <p>Содержимое статьи...</p>
</article>
```

Несемантические элементы:
- Не несут смысловой нагрузки
- Используются для стилизации и группировки
- Не улучшают структуру документа

Пример:
```html
<div>
  <div class="title">Заголовок статьи</div>
  <div class="content">Содержимое статьи...</div>
</div>
```

## Примеры семантической разметки

### Структура статьи
```html
<article>
  <header>
    <h1>Заголовок статьи</h1>
    <p>Опубликовано: <time datetime="2023-11-10">10 ноября 2023</time></p>
  </header>
  <section>
    <h2>Введение</h2>
    <p>...</p>
  </section>
  <section>
    <h2>Основная часть</h2>
    <p>...</p>
  </section>
  <footer>
    <p>Автор: Имя Фамилия</p>
  </footer>
</article>
```

### Навигация
```html
<nav>
  <ul>
    <li><a href="#home">Главная</a></li>
    <li><a href="#about">О нас</a></li>
    <li><a href="#contact">Контакты</a></li>
  </ul>
</nav>
```

### Боковая панель
```html
<aside>
  <h3>Связанные статьи</h3>
  <ul>
    <li><a href="#">Статья 1</a></li>
    <li><a href="#">Статья 2</a></li>
  </ul>
</aside>
```

## Доступность и SEO преимущества

### Доступность
- Семантические элементы помогают скринридерам правильно интерпретировать структуру документа
- Пользователи с ограниченными возможностями могут лучше ориентироваться по странице
- Правильная структура улучшает навигацию с клавиатуры

### SEO
- Поисковые системы используют семантическую структуру для понимания содержимого
- Правильное использование заголовков помогает определить иерархию информации
- Улучшает ранжирование страниц в поисковой выдаче

## Продвинутая семантика

### Структура веб-страницы
```html
<!DOCTYPE html>
<html lang="ru">
<head>
  <meta charset="UTF-8">
  <title>Семантическая верстка</title>
</head>
<body>
  <header>
    <h1>Мой сайт</h1>
    <nav>...</nav>
  </header>
  
  <main>
    <article>
      <header>...</header>
      <section>...</section>
      <aside>...</aside>
      <footer>...</footer>
    </article>
  </main>
  
  <footer>
    <p>&copy; 2023 Все права защищены</p>
  </footer>
</body>
</html>
```

### Использование ARIA-атрибутов
```html
<button aria-label="Закрыть окно">✕</button>
<div role="banner">Шаппер сайта</div>
<div role="main">Основной контент</div>
```
