# WCAG-стандарты

**WCAG** (Web Content Accessibility Guidelines) - это международные стандарты доступности веб-содержимого, разработанные W3C (World Wide Web Consortium) через рабочую группу WAI (Web Accessibility Initiative). Эти руководящие принципы помогают создавать веб-контент, доступный для людей с ограниченными возможностями.

## История и версии WCAG

WCAG были разработаны для обеспечения единообразного подхода к оценке доступности веб-содержимого. На 2025 год существуют следующие версии:

- **WCAG 1.0** (1999) - первая версия, устаревшая
- **WCAG 2.0** (2008) - широко принятая версия
- **WCAG 2.1** (2018) - дополнение WCAG 2.0 с акцентом на мобильные устройства
- **WCAG 2.2** (2023) - последние дополнения к WCAG 2.x
- **WCAG 3.0** (в разработке) - новое поколение руководящих принципов

На сегодняшний день **WCAG 2.1** является наиболее актуальной версией для обязательного соблюдения в большинстве юрисдикций, включая Российскую Федерацию.

## Четыре основных принципа WCAG (POUR)

Все руководящие принципы WCAG построены на четырех основных принципах, представленных аббревиатурой **POUR**:

### 1. Perceivable (Воспринимаемость)

Информация и компоненты пользовательского интерфейса должны быть представлены способами, которые пользователи могут воспринимать.

**Примеры требований:**
- Предоставление альтернативного текста для изображений
- Предоставление субтитров для аудио- и видео-контента
- Использование контрастных цветов для текста
- Обеспечение совместимости с вспомогательными технологиями

### 2. Operable (Управляемость)

Компоненты пользовательского интерфейса и навигация должны быть управляемыми.

**Примеры требований:**
- Возможность управления всеми функциями с клавиатуры
- Достаточное время для чтения и использования контента
- Предотвращение проблем с эпилепсией (ограничение мигающих элементов)
- Логичная навигация и возможность ориентации

### 3. Understandable (Понятность)

Информация и работа пользовательского интерфейса должны быть понятными.

**Примеры требований:**
- Ясные и понятные инструкции
- Предсказуемое поведение компонентов
- Валидация форм с понятными сообщениями об ошибках
- Последовательное оформление и навигация

### 4. Robust (Надежность)

Контент должен быть надежным, чтобы различные вспомогательные технологии могли его интерпретировать.

**Примеры требований:**
- Совместимость с вспомогательными технологиями
- Правильная разметка HTML
- Использование уникальных ID
- Корректная структура DOM

## Уровни соответствия

WCAG определяют три уровня соответствия, обозначенные как A, AA и AAA:

### Уровень A (Минимальный уровень)

Это минимальный уровень соответствия. Несоответствие требованиям этого уровня создает барьеры, которые делают содержимое недоступным для некоторых пользователей.

**Примеры требований:**
- Все неподвижные изображения имеют альтернативный текст
- Все аудио- и видео-контент имеют альтернативы
- Все формы имеют соответствующие метки
- Все элементы управления доступны с клавиатуры

### Уровень AA (Средний уровень)

Это средний уровень соответствия, который устраняет основные барьеры доступности. Большинство юрисдикций, включая РФ, требуют соответствия именно этому уровню.

**Примеры требований:**
- Соотношение контрастности текста не менее 4.5:1
- Повторяющиеся элементы навигации могут быть пропущены
- Все функции доступны с клавиатуры
- Предоставление дополнительных способов нахождения веб-страницы

### Уровень AAA (Высший уровень)

Это высокий уровень соответствия, который устраняет почти все барьеры доступности. Соответствие этому уровню не всегда возможно для всего содержимого.

**Примеры требований:**
- Соотношение контрастности текста не менее 7:1
- Расширенные альтернативы для аудио- и видео-контента
- Возможность изменения языка страницы
- Увеличение размера текста до 200% без потери функциональности

## WCAG 2.1 и дополнительные критерии

WCAG 2.1 добавляет 17 новых критериев к WCAG 2.0, с акцентом на:

- **Мобильные устройства** - адаптация для сенсорных интерфейсов
- **Пользователи с когнитивными ограничениями** - упрощение взаимодействия
- **Пользователи с нарушениями зрения** - улучшенная поддержка экранных читалок

### Новые критерии в WCAG 2.1:

- 1.3.4 Ориентация - контент не ограничен по ориентации
- 1.3.5 Идентификация цели - указание цели для компонентов пользовательского интерфейса
- 1.4.10 Переполнение содержимого - содержимое не теряется при изменении размера
- 2.1.4 Частичные срабатывания - предотвращение случайных срабатываний жестов
- 2.2.6 Тайм-аути - предупреждение о тайм-аутах
- 2.3.3 Анимация из триггеров - контроль анимации
- 2.5.1 Движение на основе жестов - альтернативы для жестов
- 2.5.2 Управление жестами - управление движением
- 2.5.3 Пометка для жестов - инструкции для жестов
- 2.5.4 Движение для функций - альтернативы для движения
- 2.5.5 Курсор (Уровень AAA) - увеличение размера курсора
- 2.5.6 Консистентность целей - последовательность целей
- 4.1.3 Статус - предоставление статуса
- 4.1.4 Взаимодействие - корректная обработка взаимодействия

## Российские реалии 2025

В 2025 году в Российской Федерации требования к доступности веб-сайтов становятся все более строгими и формализованными.

### Федеральный закон №419-ФЗ от 01.12.2017

Этот закон "О доступной среде" устанавливает обязанность обеспечения доступности информации на сайтах государственных и муниципальных организаций.

**Основные требования:**
- Обеспечение доступности веб-сайтов для людей с ограниченными возможностями
- Соответствие международным стандартам (включая WCAG 2.1 AA)
- Проведение регулярных проверок доступности
- Создание условий для независимой оценки доступности

### Методика оценки

Российские организации используют методику, разработанную ФСБУ "Институт проблем информатизации" РАН, которая основана на WCAG 2.1.

**Аспекты оценки:**
- Визуальная доступность (контрастность, размер шрифта)
- Слуховая доступность (альтернативы аудио-контенту)
- Моторная доступность (управление с клавиатуры)
- Когнитивная доступность (ясность и простота)

### Практическая реализация в РФ

- **Государственные и муниципальные сайты** обязаны соответствовать WCAG 2.1 уровня AA
- **Коммерческие организации** в некоторых случаях также обязаны обеспечивать доступность
- **Образовательные учреждения** должны обеспечивать доступность образовательного контента
- **Медицинские учреждения** обязаны обеспечивать доступность информации о своих услугах

### Примеры требований в российском законодательстве

```html
<!-- Пример соответствия требованиям WCAG 2.1 в российском контексте -->

<!-- 1. Альтернативный текст для изображений -->
<img src="logo.png" alt="Логотип Министерства образования Российской Федерации" />

<!-- 2. Якорные ссылки для навигации -->
<a href="#main-content" class="skip-link">Перейти к основному содержимому</a>

<!-- 3. Ясные метки форм -->
<label for="search-input">Поиск по сайту:</label>
<input type="text" id="search-input" name="search" />

<!-- 4. Контрастность текста -->
<div class="high-contrast-text">
  <!-- Текст с контрастностью не менее 4.5:1 -->
</div>

<!-- 5. Структура заголовков -->
<h1>Главная страница</h1>
<h2>Новости</h2>
<h3>Важное объявление</h3>
```

## Практические рекомендации по внедрению

### 1. Оценка текущего состояния

- Проведите аудит доступности текущего сайта
- Используйте автоматизированные инструменты (axe, WAVE)
- Проведите тестирование с пользователями с ограниченными возможностями

### 2. Разработка плана соответствия

- Определите приоритетные области для улучшения
- Установите сроки для достижения соответствия WCAG 2.1 AA
- Назначьте ответственных за реализацию изменений

### 3. Обучение команды

- Обучите разработчиков принципам доступности
- Обучите дизайнеров созданию доступных интерфейсов
- Обучите контент-менеджеров созданию доступного контента

### 4. Интеграция в процесс разработки

- Включите проверки доступности в процесс тестирования
- Используйте инструменты автоматической проверки в CI/CD
- Проводите регулярные ретроспективы по доступности

## Инструменты для проверки соответствия WCAG

- **axe-core** - библиотека для автоматизированного тестирования
- **WAVE** - инструмент для оценки доступности
- **Lighthouse** - встроенный инструмент Chrome для аудита доступности
- **Accessibility Insights** - инструмент Microsoft
- **Pa11y** - автоматизированный сканер доступности
- **SortSite** - коммерческий инструмент для проверки доступности

## Заключение

WCAG-стандарты являются фундаментальной основой для создания доступного веб-содержимого. В российских реалиях 2025 года соответствие этим стандартам становится не только этической необходимостью, но и юридическим требованием для многих категорий веб-сайтов. Понимание и применение WCAG 2.1 уровня AA должно быть приоритетом для всех разработчиков, создающих веб-сайты для российской аудитории.
