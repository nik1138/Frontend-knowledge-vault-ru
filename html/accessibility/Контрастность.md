# Контрастность

**Контрастность** - это разница в яркости между цветом текста и его фоном, которая делает текст читаемым для пользователей с нарушениями зрения. Правильная контрастность является важным аспектом доступности веб-сайтов и приложений.

## Основы контрастности

Согласно рекомендациям [[WCAG-стандарты]], соотношение контрастности определяется как отношение яркости более светлого цвета к яркости более темного цвета. Это отношение выражается в виде X:1, где X - кратность разницы в яркости.

### Требования к контрастности по WCAG

- **Уровень AA (рекомендуемый)**:
  - Нормальный текст: соотношение контрастности не менее 4.5:1
  - Крупный текст (18pt или 14pt жирный): соотношение контрастности не менее 3:1

- **Уровень AAA (высший)**:
  - Нормальный текст: соотношение контрастности не менее 7:1
  - Крупный текст: соотношение контрастности не менее 4.5:1

## Практические рекомендации

### Выбор цветов

- Используйте инструменты для проверки контрастности, такие как:
  - [WebAIM Contrast Checker](https://webaim.org/resources/contrastchecker/)
  - Встроенные инструменты в браузерах (например, в Chrome DevTools)
  
- Избегайте сочетаний цветов, которые могут быть проблематичны для пользователей с цветовой слепотой:
  - Красный и зеленый
  - Синий и оранжевый (в некоторых случаях)

### Текст на изображениях

- При использовании текста на изображениях убедитесь, что:
  - Контрастность соответствует рекомендациям
  - Используется контрастная рамка или тень для выделения текста
  - Альтернативный текст (альт-тег) содержит ту же информацию

### Интерактивные элементы

- Для кнопок, ссылок и других интерактивных элементов:
  - Обеспечьте контрастность не менее 3:1 для невидимых элементов
  - Убедитесь, что состояние фокуса имеет достаточную контрастность
  - Используйте визуальные индикаторы, отличные от цвета, для передачи информации

## Российские реалии 2025

В 2025 году в Российской Федерации требования к доступности веб-сайтов становятся все более строгими. Федеральный закон №419-ФЗ от 01.12.2017 года продолжает усиливать требования к доступности информационных систем и веб-сайтов государственных и муниципальных организаций.

### Законодательные требования

- Обязательное соответствие требованиям доступности для государственных сайтов
- Проверка соответствия по методике ФСБУ "Институт проблем информатизации" РАН
- Учет рекомендаций WCAG 2.1 на уровне AA как минимального порога

### Практическая реализация

- При разработке государственных и муниципальных сайтов:
  - Использование инструментов автоматической проверки контрастности
  - Обучение дизайнеров и разработчиков принципам доступности
  - Регулярное тестирование с пользователями с ограниченными возможностями

## Примеры кода

### Хороший контраст

```html
<!-- Плохой контраст (не соответствует требованиям) -->
<div style="color: #cccccc; background: #ffffff;">
  Слабый контрастный текст
</div>

<!-- Хороший контраст -->
<div style="color: #000000; background: #ffffff;">
  Хорошо контрастирующий текст
</div>

<!-- Дополнительный контраст для интерактивных элементов -->
<button style="color: #ffffff; background: #0066cc; border: 2px solid #004c99;">
  Кнопка с хорошим контрастом
</button>
```

### CSS-классы для контрастных цветов

```css
.high-contrast-text {
  color: #000000;
  background-color: #ffffff;
  /* Соотношение контрастности: 21:1 */
}

.medium-contrast-text {
  color: #333333;
  background-color: #ffffff;
  /* Соотношение контрастности: 18.6:1 */
}

.accessible-link {
  color: #0066cc;
  background-color: transparent;
  text-decoration: underline;
  /* Добавление подчеркивания для визуального различия */
}
```

## Инструменты для проверки контрастности

- **WebAIM Contrast Checker** - онлайн-инструмент для проверки контрастности
- **axe DevTools** - расширение для браузера с проверкой доступности
- **Chrome DevTools** - встроенные инструменты для проверки контрастности
- **Color Oracle** - симулятор цветовой слепоты
- **Contrast app** - настольное приложение для проверки контрастности

## Заключение

Контрастность является фундаментальным аспектом доступности веб-сайтов. Соблюдение рекомендаций по контрастности позволяет сделать контент доступным для большего числа пользователей, включая людей с нарушениями зрения. В российских реалиях 2025 года это становится не только этической необходимостью, но и юридическим требованием для многих категорий веб-сайтов.
