# ARIA-роли и состояния

**ARIA** (Accessible Rich Internet Applications) - это техническая спецификация, разработанная W3C для повышения доступности веб-содержимого и приложений. ARIA предоставляет дополнительные атрибуты, которые позволяют разработчикам определять значения и отношения, недоступные через семантическую разметку HTML.

## Основы ARIA

ARIA добавляет информацию о семантике, состоянии и структуре пользовательского интерфейса для вспомогательных технологий, таких как скринридеры. Важно понимать, что ARIA - это дополнение к HTML, а не замена его семантических возможностей.

### Основные категории ARIA-атрибутов

- **Роли (Roles)**: определяют тип элемента (например, кнопка, меню, диалог)
- **Свойства (Properties)**: описывают характеристики элемента (например, aria-label, aria-describedby)
- **Состояния (States)**: отражают текущее состояние элемента (например, aria-disabled, aria-checked)

## ARIA-роли

### Ландшафтные роли

Эти роли определяют области страницы и включают:

- `banner` - заголовок сайта
- `main` - основное содержимое
- `navigation` - навигационные элементы
- `complementary` - дополнительное содержимое
- `contentinfo` - информация об авторских правах, контактная информация
- `search` - область поиска

```html
<header role="banner">
  <h1>Заголовок сайта</h1>
</header>

<nav role="navigation">
  <ul>
    <li><a href="/">Главная</a></li>
    <li><a href="/about">О нас</a></li>
  </ul>
</nav>

<main role="main">
  <p>Основное содержимое страницы</p>
</main>

<footer role="contentinfo">
  <p>&copy; 2025 Все права защищены</p>
</footer>
```

### Структурные роли

- `article` - независимая часть содержимого
- `section` - раздел содержимого
- `group` - логическая группа элементов
- `heading` - заголовок (лучше использовать теги h1-h6)

### Виджетные роли

Эти роли определяют интерактивные элементы:

- `button` - кнопка
- `checkbox` - флажок
- `combobox` - комбинированный список
- `link` - ссылка
- `menu` - меню
- `menuitem` - элемент меню
- `progressbar` - индикатор выполнения
- `radio` - переключатель
- `slider` - ползунок
- `tab` - вкладка
- `tabpanel` - панель вкладки
- `textbox` - текстовое поле

## ARIA-состояния и свойства

### Основные ARIA-атрибуты

- `aria-label` - предоставляет текстовую метку элементу
- `aria-labelledby` - связывает элемент с другим элементом, содержащим метку
- `aria-describedby` - связывает элемент с описанием
- `aria-hidden` - скрывает элемент от вспомогательных технологий
- `aria-live` - указывает, что содержимое элемента может обновляться динамически

### Состояния

- `aria-disabled` - элемент недоступен
- `aria-expanded` - элемент расширен/свернут
- `aria-checked` - состояние проверки для чекбоксов и радио-кнопок
- `aria-selected` - элемент выбран
- `aria-pressed` - состояние нажатия для кнопок
- `aria-current` - указывает текущую страницу или элемент

## Практические рекомендации

### Когда использовать ARIA

1. **Когда семантический HTML недостаточен**
   - При создании сложных интерактивных компонентов
   - Для динамического обновления содержимого

2. **Для улучшения доступности пользовательских компонентов**
   - Кастомные выпадающие списки
   - Слайдеры и ползунки
   - Модальные окна

### Когда НЕ использовать ARIA

1. **Когда семантический HTML подходит**
   - Используйте тег `<button>` вместо `div` с `role="button"`
   - Используйте тег `<a>` вместо `div` с `role="link"`

2. **Избегайте дублирования встроенной семантики**
   - Не добавляйте `role="button"` к элементу `<button>`
   - Не добавляйте `role="checkbox"` к элементу `<input type="checkbox">`

### Примеры правильного использования

```html
<!-- Плохо: использование div с ARIA вместо семантического элемента -->
<div role="button" onclick="doSomething()">Нажми меня</div>

<!-- Хорошо: использование семантического элемента -->
<button onclick="doSomething()">Нажми меня</button>

<!-- Хорошо: кастомный компонент с ARIA -->
<div role="button" tabindex="0" aria-pressed="false" onclick="toggleButton()">
  Переключатель
</div>

<!-- Хорошо: использование ARIA для описания -->
<input type="text" aria-label="Имя пользователя" id="username">
<div id="error-message" aria-live="polite" style="display: none;">
  Пожалуйста, введите имя пользователя
</div>

<!-- Хорошо: навигационное меню с ARIA -->
<nav role="navigation" aria-label="Основная навигация">
  <ul>
    <li><a href="/">Главная</a></li>
    <li><a href="/about">О нас</a></li>
    <li><a href="/contact">Контакты</a></li>
  </ul>
</nav>
```

## Российские реалии 2025

В 2025 году в Российской Федерации требования к доступности веб-сайтов становятся все более строгими, особенно для государственных и муниципальных ресурсов.

### Законодательные требования

- Федеральный закон №419-ФЗ от 01.12.2017 требует обеспечения доступности веб-сайтов
- Необходимость соответствия WCAG 2.1 на уровне AA
- Обязательная проверка доступности государственных сайтов

### Практическая реализация

- При разработке государственных и муниципальных сайтов:
  - Обязательное использование семантической разметки
  - Применение ARIA там, где семантический HTML недостаточен
  - Тестирование с использованием вспомогательных технологий
  - Обучение разработчиков принципам доступности

### Примеры для российских сайтов

```html
<!-- Пример формы с ARIA для российского сайта -->
<form aria-labelledby="form-title">
  <h2 id="form-title">Форма регистрации</h2>
  
  <div class="form-group">
    <label for="email">Электронная почта</label>
    <input 
      type="email" 
      id="email" 
      aria-describedby="email-error email-help"
      required
    >
    <div id="email-help" class="help-text">
      Используйте формат example@domain.ru
    </div>
    <div id="email-error" class="error-text" aria-live="polite">
      <!-- Сообщения об ошибках будут отображаться здесь -->
    </div>
  </div>
  
  <button type="submit" aria-describedby="submit-help">
    Зарегистрироваться
  </button>
  <div id="submit-help" class="help-text">
    После нажатия кнопки вы получите письмо с подтверждением
  </div>
</form>
```

## Лучшие практики

### 1. Приоритет семантическому HTML

- Используйте семантические элементы HTML, когда это возможно
- Добавляйте ARIA только когда семантический HTML недостаточен

### 2. Обновление ARIA-атрибутов

- Динамически обновляйте ARIA-состояния при изменении состояния компонента
- Используйте JavaScript для управления ARIA-атрибутами

```javascript
// Пример обновления ARIA-атрибутов
function toggleButton(element) {
  const isPressed = element.getAttribute('aria-pressed') === 'true';
  element.setAttribute('aria-pressed', !isPressed);
  element.classList.toggle('pressed', !isPressed);
}
```

### 3. Тестирование с вспомогательными технологиями

- Проверяйте работу с экранными читалками (NVDA, JAWS, VoiceOver)
- Убедитесь, что все интерактивные элементы доступны с клавиатуры
- Проверьте, что ARIA-атрибуты корректно читаются

## Заключение

ARIA является мощным инструментом для повышения доступности веб-приложений, особенно когда семантический HTML недостаточен. Важно использовать ARIA правильно, приоритизируя семантический HTML и применяя ARIA только там, где это действительно необходимо. В российских реалиях 2025 года знание и применение ARIA становится не только вопросом доступности, но и юридическим требованием для многих веб-сайтов.
