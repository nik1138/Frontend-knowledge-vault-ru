# Доступность веб-контента (HTML)

## Основы доступности веб-контента

Доступность веб-контента означает создание веб-страниц, которые могут использоваться всеми людьми, включая тех, кто имеет ограниченные возможности. Это включает в себя людей с нарушениями зрения, слуха, моторики или когнитивных способностей. HTML предоставляет множество встроенных возможностей для обеспечения доступности, которые помогают пользователям с различными потребностями взаимодействовать с веб-контентом.

> [!tip] 
> Основная цель доступности - обеспечить равный доступ к информации и функциональности для всех пользователей, независимо от их возможностей.

## Семантическая разметка

Семантическая разметка - это использование HTML-элементов по их прямому назначению. Правильная семантика помогает вспомогательным технологиям, таким как скринридеры, понимать структуру и содержание веб-страницы.

```html
<!-- Правильное использование заголовков -->
<h1>Заголовок страницы</h1>
<h2>Подраздел 1</h2>
<h3>Подподраздел 1.1</h3>
<h2>Подраздел 2</h2>

<!-- Семантические элементы -->
<nav>Навигация</nav>
<main>Основное содержимое</main>
<article>Статья</article>
<section>Раздел</section>
<aside>Боковая панель</aside>
<footer>Футер</footer>
```

## ARIA атрибуты

ARIA (Accessible Rich Internet Applications) - это набор атрибутов, которые помогают сделать веб-приложения более доступными. Они добавляют семантическую информацию, когда стандартный HTML недостаточен.

```html
<!-- Роль и состояние -->
<div role="button" aria-pressed="false">Переключатель</div>

<!-- Описание элемента -->
<button aria-describedby="help-text">Помощь</button>
<div id="help-text">Это текст помощи</div>

<!-- Заголовок для элемента -->
<div aria-label="Поиск">...</div>
```

## Клавиатурная навигация

Все интерактивные элементы должны быть доступны с помощью клавиатуры. Это особенно важно для пользователей, которые не могут использовать мышь.

```css
/* Визуальная индикация фокуса */
:focus {
  outline: 2px solid #005fcc;
  outline-offset: 2px;
}

/* Порядок фокуса */
.tabindex { tabindex: 0; }
```

## Скринридеры

Скринридеры - это программы, которые читают содержимое экрана вслух для пользователей с нарушениями зрения. Правильная разметка и атрибуты помогают скринридерам корректно интерпретировать веб-страницу.

> [!note]
> Популярные скринридеры: NVDA, JAWS, VoiceOver (Mac), TalkBack (Android).

## Доступные формы

Формы должны быть доступны для всех пользователей. Каждое поле ввода должно иметь соответствующую метку.

```html
<!-- Правильное использование меток -->
<label for="email">Email:</label>
<input type="email" id="email" name="email" required>

<!-- Ошибки формы -->
<div role="alert" aria-live="assertive">Пожалуйста, введите корректный email</div>
```

## Изображения и альтернативный текст

Изображения должны иметь альтернативный текст (альт-теги), чтобы пользователи с нарушениями зрения могли понимать содержание.

```html
<!-- Декоративные изображения -->
<img src="background.jpg" alt="" role="presentation">

<!-- Информативные изображения -->
<img src="chart.png" alt="График роста продаж за 2023 год">

<!-- Функциональные изображения -->
<a href="/product">
  <img src="product.jpg" alt="Купить продукт">
</a>
```

## Таблицы доступности

Таблицы должны иметь правильную структуру заголовков и описания для доступности.

```html
<table>
  <caption>Продажи по кварталам</caption>
  <thead>
    <tr>
      <th scope="col">Квартал</th>
      <th scope="col">Продажи</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <th scope="row">Q1</th>
      <td>100,000</td>
    </tr>
  </tbody>
</table>
```

## Примеры кода

Примеры кода демонстрируют применение принципов доступности в реальных сценариях.

### Доступная кнопка

```html
<button type="button" 
        aria-label="Увеличить шрифт" 
        onclick="increaseFontSize()">
  A+
</button>
```

### Доступное всплывающее окно

```html
<div role="dialog" 
     aria-labelledby="modal-title" 
     aria-modal="true">
  <h2 id="modal-title">Подтверждение</h2>
  <p>Вы уверены, что хотите удалить запись?</p>
  <button>Да</button>
  <button>Нет</button>
</div>
```

## Проверка доступности

Для проверки доступности можно использовать различные инструменты:

- [[html/testing-accessibility]]
- Axe DevTools
- WAVE
- Lighthouse

## Лучшие практики

1. Используйте семантические HTML-элементы
2. Обеспечьте контрастность текста
3. Проверяйте клавиатурную навигацию
4. Используйте ARIA только при необходимости
5. Тестируйте с реальными пользователями
6. Регулярно проверяйте доступность
