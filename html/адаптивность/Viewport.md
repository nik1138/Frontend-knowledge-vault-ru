# Viewport

Viewport - это видимая область веб-страницы, доступная пользователю на устройстве. Правильная настройка viewport является ключевым фактором при создании адаптивных веб-сайтов, особенно в условиях разнообразия мобильных устройств в 2025 году.

## Основные понятия

Viewport определяет, как веб-страница будет отображаться на различных устройствах. В мобильных браузерах viewport по умолчанию может быть шире, чем экран устройства, что приводит к автоматическому масштабированию и ухудшению пользовательского опыта.

## HTML-тег viewport

Для правильной настройки viewport в HTML-документе необходимо добавить следующий метатег в секцию `<head>`:

```html
<meta name="viewport" content="width=device-width, initial-scale=1.0">
```

### Атрибуты viewport

- `width=device-width` - устанавливает ширину viewport равной ширине экрана устройства
- `initial-scale=1.0` - задает начальный масштаб страницы (1.0 = без масштабирования)
- `minimum-scale` - минимальный уровень масштабирования
- `maximum-scale` - максимальный уровень масштабирования
- `user-scalable` - разрешает или запрещает пользователю масштабирование

## Практические рекомендации

### 1. Базовая настройка

Для большинства адаптивных проектов рекомендуется использовать:

```html
<meta name="viewport" content="width=device-width, initial-scale=1.0, user-scalable=no">
```

> [!note] 
> Отключение масштабирования (`user-scalable=no`) может ухудшить доступность сайта для пользователей с ограниченными возможностями.

### 2. Расширенные параметры

В некоторых случаях могут потребоваться дополнительные параметры:

```html
<meta name="viewport" content="width=device-width, initial-scale=1.0, minimum-scale=1.0, maximum-scale=5.0, user-scalable=yes">
```

## Типичные ошибки

- **Забытый viewport**: Приводит к неправильному отображению на мобильных устройствах
- **Неправильные значения**: Часто встречаются ошибки с параметрами масштабирования
- **Отсутствие тестирования**: Необходимо тестировать на реальных устройствах

## Совместимость с современными устройствами

В 2025 году важно учитывать:
- Разнообразие форматов экранов (включая складные устройства)
- Высокое разрешение экранов (Retina, 4K и выше)
- Различные ориентации (портретная и альбомная)

## Практический пример

```html
<!DOCTYPE html>
<html lang="ru">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Адаптивный сайт</title>
    <style>
        .container {
            width: 100%;
            padding: 20px;
        }
        
        @media (min-width: 768px) {
            .container {
                max-width: 1200px;
                margin: 0 auto;
            }
        }
    </style>
</head>
<body>
    <div class="container">
        <h1>Адаптивный контент</h1>
        <p>Этот текст будет правильно отображаться на всех устройствах.</p>
    </div>
</body>
</html>
```

## Заключение

Правильная настройка viewport - это первый шаг к созданию действительно адаптивного веб-сайта. В условиях российского рынка 2025 года, где мобильный трафик составляет значительную часть посещений, это особенно важно для обеспечения положительного пользовательского опыта и высоких позиций в поисковой выдаче.

> [!tip]
> Всегда проверяйте корректность отображения на реальных устройствах, а не только в эмуляторах разработчика.
