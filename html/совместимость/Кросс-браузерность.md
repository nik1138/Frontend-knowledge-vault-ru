# Кросс-браузерность

## Основные понятия

Кросс-браузерность — это способность веб-сайта или веб-приложения корректно отображаться и функционировать в различных веб-браузерах. В 2025 году это особенно важно для российских разработчиков, учитывая разнообразие используемых браузеров и сетевые условия.

## Поддерживаемые браузеры в России

В 2025 году в России основными браузерами являются:

- [[Yandex Browser]]
- [[Google Chrome]]
- [[Mozilla Firefox]]
- [[Opera]]
- [[Safari]] (на устройствах Apple)
- [[Microsoft Edge]]

## Проблемы кросс-браузерности

### CSS-совместимость

Разные браузеры могут по-разному интерпретировать CSS-свойства. Основные проблемы:

- Неподдержка современных CSS-свойств
- Различия в отображении flexbox и grid
- Проблемы с CSS-анимациями
- Отличия в обработке единиц измерения

### JavaScript-совместимость

- Различия в поддержке ES6+ функций
- Разные реализации встроенных методов
- Проблемы с событиями и DOM-манипуляциями

## Практические рекомендации

### Использование префиксов

```css
.element {
  -webkit-transform: rotate(45deg);
  -moz-transform: rotate(45deg);
  -ms-transform: rotate(45deg);
  transform: rotate(45deg);
}
```

### Проверка поддержки функций

```javascript
if ('serviceWorker' in navigator) {
  // Регистрация Service Worker
} else {
  // Резервная реализация
}
```

### CSS Reset и Normalize

Использование [[CSS Reset]] или [[Normalize.css]] помогает устранить различия в стандартных стилях браузеров.

## Современные подходы

### Progressive Enhancement

Подход, при котором сайт сначала создается с базовой функциональностью, а затем улучшается для более современных браузеров.

### Graceful Degradation

Создание сайта с современными возможностями, но с обеспечением базовой функциональности в старых браузерах.

## Инструменты поддержки

### Can I Use

Сайт [caniuse.com](https://caniuse.com) предоставляет информацию о поддержке различных CSS и JavaScript функций в браузерах.

### Babel

Транспиляция современного JavaScript для поддержки старых браузеров.

### Autoprefixer

Автоматическое добавление вендорных префиксов к CSS-свойствам.

## Российские особенности

### Скорость интернета

В некоторых регионах России скорость интернета остается ограниченной, что влияет на загрузку сложных скриптов и стилей.

### Локализация

Важно учитывать локализацию интерфейса и корректное отображение кириллических шрифтов.

### Законодательные требования

Соответствие требованиям российского законодательства, включая [[ФЗ-152]] (персональные данные) и [[доступность]] для людей с ограниченными возможностями.

## Лучшие практики

1. Использование [[feature detection]] вместо [[browser detection]]
2. Тестирование в минимальных и максимальных версиях браузеров
3. Использование [[polyfill]] для недостающих функций
4. Оптимизация производительности для старых браузеров
5. Регулярное обновление списка поддерживаемых браузеров
