# Современные и устаревшие функции

**Современные и устаревшие функции** — это концепция, отражающая эволюцию веб-технологий, при которой новые возможности постепенно заменяют устаревшие, но при этом необходимо обеспечивать совместимость с более старыми версиями браузеров и стандартов.

## Классификация функций

### Современные функции (2025)
- **HTML5.3 и HTML5.4**: Новые элементы и атрибуты
- **CSS4 и CSS Level 4 Modules**: Продвинутые селекторы, градиенты, анимации
- **ES2024 и ES2025**: Современные возможности JavaScript
- **Web Components**: Пользовательские элементы, Shadow DOM
- **Progressive Web Apps (PWA)**: Автономная работа, push-уведомления
- **WebAssembly**: Высокопроизводительные веб-приложения
- **Web APIs**: Service Workers, WebRTC, WebGL, Web Audio API

### Устаревшие функции
- **HTML4 и XHTML1.0**: Старые структурные элементы
- **CSS2.1**: Ограниченные возможности стилизации
- **ES5**: Устаревшие методы программирования
- **Flash и Silverlight**: Заменены современными технологиями
- **Табличная верстка**: Заменена CSS-методами

## Хронология эволюции HTML

### 1990-е: Основы HTML
- **HTML 1.0-2.0**: Простая разметка документов
- **HTML 3.2**: Таблицы, элементы форм, расширения

### 2000-е: Стандартизация
- **HTML 4.0**: Семантическая разметка, CSS
- **XHTML 1.0**: Строгий синтаксис, XML-совместимость
- **CSS2.1**: Расширенные возможности стилизации

### 2010-е: Веб 2.0 и мобильные технологии
- **HTML5**: Новые элементы, API, мультимедиа
- **CSS3**: Анимации, переходы, градиенты
- **ES5/ES6**: Современные возможности JavaScript

### 2020-е: Современные веб-технологии
- **HTML5.2/5.3**: Улучшения доступности, безопасности
- **CSS4 и модульные спецификации**: Grid, Flexbox, Custom Properties
- **ES2015-ES2024**: Современные возможности JavaScript
- **Web Components**: Пользовательские элементы
- **PWA**: Автономная работа, push-уведомления

### 4. Web Components

```html
<!-- Пользовательский элемент -->
<template id="user-card-template">
  <style>
    .card {
      border: 1px solid #ddd;
      border-radius: 8px;
      padding: 16px;
      margin: 8px;
    }
  </style>
  <div class="card">
    <h3 id="name"></h3>
    <p id="email"></p>
  </div>
</template>

<script>
class UserCard extends HTMLElement {
  constructor() {
    super();
    
    const template = document.getElementById('user-card-template');
    const templateContent = template.content;
    
    const shadowRoot = this.attachShadow({mode: 'open'});
    shadowRoot.appendChild(templateContent.cloneNode(true));
    
    const nameElement = shadowRoot.getElementById('name');
    const emailElement = shadowRoot.getElementById('email');
    
    nameElement.textContent = this.getAttribute('name') || 'Без имени';
    emailElement.textContent = this.getAttribute('email') || 'Без email';
  }
}

customElements.define('user-card', UserCard);
</script>

<!-- Использование пользовательского элемента -->
<user-card name="Иван Петров" email="ivan@example.com"></user-card>
```

## Устаревшие функции и их замены

### 1. Устаревшие HTML-элементы

| Устаревший элемент | Современная замена | Причина устаревания |
|-------------------|-------------------|-------------------|
| `<font>` | CSS | Семантически неправильное форматирование |
| `<center>` | CSS | Семантически неправильное позиционирование |
| `<bgsound>` | `<audio>` | Неподдерживаемый формат |
| `<marquee>` | CSS/JS анимации | Плохая доступность и производительность |
| `<frameset>` | `<iframe>` или модульная архитектура | Плохая доступность и SEO |
| `<applet>` | `<canvas>` или JavaScript | Заменен Java-апплеты |
| `<acronym>` | `<abbr>` | Унификация сокращений |
| `<big>` | CSS | Семантически неправильное форматирование |

### 2. Устаревшие атрибуты

| Устаревший атрибут | Современная замена | Пример |
|-------------------|-------------------|--------|
| `align` | CSS | `text-align`, `float`, `display` |
| `bgcolor` | CSS | `background-color` |
| `border` | CSS | `border` |
| `cellpadding` | CSS | `padding` |
| `cellspacing` | CSS | `border-spacing` |
| `width` и `height` | CSS | `width`, `height` |

### 3. Замена устаревших подходов

```html
<!-- Устаревший подход -->
<table border="1" cellpadding="5" cellspacing="0" bgcolor="#f0f0f0">
  <tr>
    <td align="center" width="200" height="100">
      <font color="red" size="4">Содержимое</font>
    </td>
  </tr>
</table>

<!-- Современный подход -->
<div class="modern-table">
  <div class="table-row">
    <div class="table-cell">
      Содержимое
    </div>
  </div>
</div>

<style>
.modern-table {
  background-color: #f0f0f0;
  border: 1px solid #000;
}

.table-row {
  display: flex;
}

.table-cell {
  flex: 1;
  padding: 5px;
  text-align: center;
  color: red;
  font-size: 1.2em;
  min-width: 200px;
  min-height: 100px;
  display: flex;
  align-items: center;
  justify-content: center;
}
</style>
```

## Практические рекомендации по переходу

### 1. Постепенная миграция

```html
<!-- Пример постепенной миграции -->
<!DOCTYPE html>
<html lang="ru">
<head>
  <meta charset="UTF-8">
  <title>Пример миграции</title>

    /* Современные стили */

      max-width: 1200px;
      margin: 0 auto;
      padding: 0 20px;
    }
    
    .grid {
      display: grid;
      grid-template-columns: repeat(auto-fit, minmax(300px, 1fr));
      gap: 20px;
    }
    
    /* Запасные стили для старых браузеров */
    @supports not (display: grid) {
      .grid {
        display: flex;
        flex-wrap: wrap;
      }
      
      .grid-item {
        flex: 1 1 300px;
        margin: 10px;
      }
    }
  </style>
</head>
<body>
  <div class="container">
    <div class="grid">
      <div class="grid-item">Элемент 1</div>
      <div class="grid-item">Элемент 2</div>
      <div class="grid-item">Элемент 3</div>
    </div>
  </div>
</body>
</html>
```

### 2. Использование полифилов

```html
<!-- Полифилы для поддержки старых браузеров -->
<script>
// Проверка поддержки и подключение полифила для Object.assign
if (typeof Object.assign !== 'function') {
  Object.assign = function(target) {
    if (target == null) {
      throw new TypeError('Cannot convert undefined or null to object');
    }
    
    var to = Object(target);
    
    for (var index = 1; index < arguments.length; index++) {
      var nextSource = arguments[index];
      
      if (nextSource != null) {
        for (var nextKey in nextSource) {
          if (Object.prototype.hasOwnProperty.call(nextSource, nextKey)) {
            to[nextKey] = nextSource[nextKey];
          }
        }
      }
    }
    return to;
  };
}
</script>

<!-- Полифил для Promise в старых браузерах -->
<script>
if (typeof Promise === 'undefined') {
  // Подключение полифила для Promise
  // (например, es6-promise или native-promise-only)
}
</script>
```

## Особенности для российского рынка (2025)

В России в 2025 году особенно важно учитывать:

1. **Широкое использование устаревших браузеров**: В корпоративных сетях и государственных учреждениях до сих пор могут использоваться устаревшие версии браузеров, особенно Internet Explorer и старые версии Яндекс.Браузера.

2. **Разнообразие устройств**: В регионах могут использоваться более старые мобильные устройства с ограниченной поддержкой современных веб-технологий.

3. **Ограничения по скорости интернета**: В некоторых регионах до сих пор актуальны проблемы с качеством интернет-соединения, что делает важным использование lightweight-подходов и оптимизацию загрузки ресурсов.

4. **Специфика отечественных браузеров**: Необходимо тестировать совместимость с отечественными браузерами, такими как Яндекс.Браузер и Mail.ru Browser, которые могут иметь особенности в реализации современных стандартов.

5. **Требования к доступности**: Увеличение внимания к доступности веб-контента требует поддержки устаревших возможностей для обеспечения доступности для всех пользователей.

## Лучшие практики

1. **Используйте прогрессивное улучшение**: Начинайте с базовой функциональности и добавляйте современные возможности для поддерживающих браузеров.

2. **Проверяйте совместимость**: Регулярно тестируйте сайт на различных браузерах и устройствах, особенно популярных в вашем регионе.

3. **Минимизируйте использование устаревших технологий**: Постепенно заменяйте устаревшие элементы и атрибуты на современные аналоги.

4. **Используйте полифилы при необходимости**: Подключайте полифилы для обеспечения поддержки современных возможностей в старых браузерах.

5. **Оптимизируйте производительность**: Современные технологии должны использоваться с учетом производительности, особенно на устройствах с ограниченными возможностями.
