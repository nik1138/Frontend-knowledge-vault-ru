# Адаптивность

## Определение

Адаптивность (Responsive Design) — это подход к веб-дизайну, при котором веб-страницы автоматически адаптируются к размеру экрана устройства, на котором они отображаются. В 2025 году адаптивность стала стандартом, а не дополнительной функцией.

## Принципы адаптивного дизайна

### Гибкая сетка

Использование относительных единиц измерения вместо фиксированных пикселей:

```css
.container {
  width: 100%;
  max-width: 1200px;
  margin: 0 auto;
  padding: 0 1rem;
}

.grid {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(300px, 1fr));
  gap: 1rem;
}
```

### Гибкие изображения

```css
img, video {
  max-width: 100%;
  height: auto;
}
```

### Медиа-запросы

```css
/* Мобильные устройства */
@media screen and (max-width: 768px) {
  .navigation {
    flex-direction: column;
  }
}

/* Планшеты */
@media screen and (min-width: 769px) and (max-width: 1024px) {
  .grid {
    grid-template-columns: repeat(2, 1fr);
  }
}

/* Десктопы */
@media screen and (min-width: 1025px) {
  .grid {
    grid-template-columns: repeat(3, 1fr);
  }
}
```

## CSS Grid и Flexbox

### Flexbox для одномерных макетов

```css
.flex-container {
  display: flex;
  flex-wrap: wrap;
  justify-content: space-between;
}

.flex-item {
  flex: 1 1 300px;
  margin: 1rem;
}
```

### Grid для двумерных макетов

```css
.grid-container {
  display: grid;
  grid-template-areas: 
    "header header"
    "sidebar main"
    "footer footer";
  grid-template-columns: 250px 1fr;
  grid-template-rows: auto 1fr auto;
  min-height: 100vh;
  gap: 1rem;
}
```

## Типичные брейкпоинты

### Мобильные устройства

- 320px - 480px (смартфоны)
- 481px - 768px (планшеты в портретной ориентации)

### Планшеты

- 769px - 1024px (планшеты в ландшафтной ориентации)

### Десктопы

- 1025px - 1200px (маленькие мониторы)
- 1201px+ (широкие экраны)

## Практические рекомендации

### Использование относительных единиц

- `rem` для размеров шрифта и отступов
- `em` для размеров, связанных с контекстом
- `%` для ширины и высоты
- `vw` и `vh` для размеров, пропорциональных окну просмотра

```css
.typography {
  font-size: 1.2rem;
  padding: 1rem;
  margin: 0.5rem 0;
}

.responsive-element {
  width: 80%;
  height: 50vh;
}
```

### Адаптация навигации

```html
<nav class="responsive-nav">
  <div class="nav-toggle" onclick="toggleNav()">☰</div>
  <ul class="nav-menu">
    <li><a href="#">Главная</a></li>
    <li><a href="#">О нас</a></li>
    <li><a href="#">Контакты</a></li>
  </ul>
</nav>
```

```css
.nav-toggle {
  display: none;
}

@media screen and (max-width: 768px) {
  .nav-toggle {
    display: block;
    font-size: 1.5rem;
    cursor: pointer;
  }
  
  .nav-menu {
    display: none;
    flex-direction: column;
  }
  
  .nav-menu.active {
    display: flex;
  }
}
```

### Адаптация форм

```html
<form class="responsive-form">
  <div class="form-group">
    <label for="name">Имя</label>
    <input type="text" id="name" name="name" class="responsive-input">
  </div>
  <div class="form-group">
    <label for="email">Email</label>
    <input type="email" id="email" name="email" class="responsive-input">
  </div>
  <button type="submit" class="responsive-button">Отправить</button>
</form>
```

```css
.responsive-form {
  max-width: 400px;
  margin: 0 auto;
}

.form-group {
  margin-bottom: 1rem;
}

.responsive-input {
  width: 100%;
  padding: 0.75rem;
  border: 1px solid #ccc;
  border-radius: 4px;
  font-size: 1rem;
}

@media screen and (max-width: 480px) {
  .responsive-input {
    font-size: 16px; /* Для лучшего взаимодействия на сенсорных экранах */
  }
}
```

## Адаптивные изображения

### Использование picture элемента

```html
<picture>
  <source media="(max-width: 768px)" srcset="image-mobile.jpg">
  <source media="(max-width: 1024px)" srcset="image-tablet.jpg">
  <img src="image-desktop.jpg" alt="Описание изображения">
</picture>
```

### Использование srcset

```html
<img src="image-medium.jpg" 
     srcset="image-small.jpg 480w, 
             image-medium.jpg 800w, 
             image-large.jpg 1200w"
     sizes="(max-width: 480px) 100vw, 
            (max-width: 800px) 50vw, 
            33vw"
     alt="Описание изображения">
```

## JavaScript для адаптивности

### Обработка изменения размера окна

```javascript
function handleResize() {
  if (window.innerWidth <= 768) {
    // Мобильные стили
    document.body.classList.add('mobile-layout');
  } else {
    // Десктопные стили
    document.body.classList.remove('mobile-layout');
  }
}

// Обработка при загрузке
handleResize();

// Обработка при изменении размера
window.addEventListener('resize', handleResize);
```

### Детектирование устройств

```javascript
const isMobile = window.matchMedia('(max-width: 768px)').matches;
const isTouchDevice = 'ontouchstart' in window || navigator.maxTouchPoints > 0;
```

## Современные подходы

### Container Queries (новый стандарт)

```css
.card {
  container-type: inline-size;
  container-name: card-container;
}

@container card-container (min-width: 400px) {
  .card-content {
    display: grid;
    grid-template-columns: 1fr 1fr;
  }
}
```

### CSS Clamp для адаптивных размеров

```css
.adaptive-text {
  font-size: clamp(1rem, 2.5vw, 1.5rem);
}
```

### Viewport-relative units

- `vw` - ширина окна просмотра
- `vh` - высота окна просмотра
- `vmin` - меньшее из vw или vh
- `vmax` - большее из vw или vh

## Лучшие практики

1. Использование mobile-first подхода
2. Тестирование на минимальных и максимальных размерах экрана
3. Оптимизация изображений для разных плотностей пикселей
4. Учет особенностей сенсорных интерфейсов
5. Проверка читаемости текста на маленьких экранах
6. Упрощение навигации на мобильных устройствах
7. Использование доступных цветовых контрастов

## Заключение

Адаптивность в 2025 году является фундаментальным требованием к современным веб-сайтам. Учитывая разнообразие устройств и сетевых условий в России, адаптивный дизайн обеспечивает доступность контента для максимально широкой аудитории.

> [!tip] Совет
> Используйте CSS Grid и Flexbox в сочетании для создания сложных адаптивных макетов.

> [!warning] Важно
> Не забывайте тестировать адаптивность на реальных устройствах, а не только в инструментах разработчика.
