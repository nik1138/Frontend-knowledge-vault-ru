# Заголовки и навигация в HTML

## Общие сведения

Заголовки и навигация являются фундаментальными элементами семантической структуры HTML-документа. Они обеспечивают логическую иерархию контента, улучшают доступность и способствуют лучшему пониманию структуры страницы как пользователями, так и поисковыми системами.

В 2025 году в российской веб-разработке особое внимание уделяется созданию доступных и семантически правильных веб-страниц, что соответствует требованиям закона о доступности информационных систем.

## Иерархия заголовков

### Основные теги заголовков

HTML предоставляет шесть уровней заголовков от `<h1>` до `<h6>`:

```html
<h1>Основной заголовок страницы</h1>
<h2>Раздел первого уровня</h2>
<h3>Подраздел второго уровня</h3>
<h4>Подраздел третьего уровня</h4>
<h5>Подраздел четвертого уровня</h5>
<h6>Подраздел пятого уровня</h6>
```

### Правильное использование иерархии

- На странице должен быть только один тег `<h1>` — это основной заголовок документа
- Не пропускайте уровни заголовков (например, не переходите сразу от `<h2>` к `<h4>`)
- Используйте заголовки логически, отражая структуру контента

## Семантические элементы навигации

### Элемент `<nav>`

Элемент `<nav>` обозначает раздел навигации по сайту:

```html
<nav>
  <ul>
    <li><a href="/">Главная</a></li>
    <li><a href="/about">О нас</a></li>
    <li><a href="/services">Услуги</a></li>
    <li><a href="/contact">Контакты</a></li>
  </ul>
</nav>
```

### Элемент `<header>`

Элемент `<header>` обычно содержит заголовки, логотип, навигацию и другие вводные элементы:

```html
<header>
  <h1>Название сайта</h1>
  <nav>
    <!-- навигация -->
  </nav>
</header>
```

## Практические рекомендации для российских разработчиков

### SEO-оптимизация

- Используйте ключевые слова в заголовках, особенно в `<h1>`
- Заголовки должны быть информативными и отражать суть контента
- Следите за длиной заголовков — поисковые системы в России (Яндекс, Google) рекомендуют не превышать 60 символов для `<h1>`

### Доступность

- Заголовки помогают пользователям с экранными читалками ориентироваться на странице
- Используйте ARIA-атрибуты при необходимости: `aria-label`, `aria-labelledby`
- В России требования к доступности регулируются ГОСТ Р 52872-2015 и Федеральным законом №419-ФЗ

### Лучшие практики

- Структурируйте контент логически, используя заголовки
- Не используйте заголовки только для стилизации — для этого есть CSS
- Проверяйте структуру заголовков с помощью инструментов веб-разработки

## Влияние на SEO в России

В 2025 году поисковые системы в России (в первую очередь Яндекс) продолжают учитывать правильную семантическую структуру при ранжировании. Использование заголовков помогает поисковым роботам понимать контент и улучшает видимость сайта в поисковой выдаче.

> [!tip] Совет
> Используйте инструменты валидации структуры, такие как HeadingsMap для Chrome, чтобы проверить правильность иерархии заголовков на вашем сайте.
