# Логические-блоки-контента

Логические блоки контента - это семантически значимые элементы HTML, которые определяют структуру и смысловую нагрузку различных частей веб-страницы. В 2025 году правильное использование логических блоков является ключевым фактором доступности, SEO и общей архитектуры веб-сайта, особенно в контексте российских реалий.

## Основные логические блоки

### `<header>` - заголовок секции или документа

Элемент `<header>` представляет собой группу вспомогательных элементов, часто содержащих заголовки, навигацию, логотип или информацию об авторе:

```html
<header>
  <h1>Название сайта</h1>
  <p>Слоган или краткое описание</p>
  <nav>
    <ul>
      <li><a href="/">Главная</a></li>
      <li><a href="/about">О нас</a></li>
    </ul>
  </nav>
</header>
```

> [!note] Примечание
> У одного документа может быть несколько элементов `<header>`, например, один для всей страницы и по одному для каждой статьи.

### `<nav>` - навигационный блок

Элемент `<nav>` предназначен для группировки ссылок навигации:

```html
<nav aria-label="Основная навигация">
  <ul>
    <li><a href="/">Главная</a></li>
    <li><a href="/products">Продукты</a></li>
    <li><a href="/services">Услуги</a></li>
    <li><a href="/contacts">Контакты</a></li>
  </ul>
</nav>

<nav aria-label="Дополнительная навигация">
  <ul>
    <li><a href="/about">О компании</a></li>
    <li><a href="/news">Новости</a></li>
    <li><a href="/careers">Карьера</a></li>
  </ul>
</nav>
```

### `<main>` - основное содержимое

Элемент `<main>` представляет основное содержимое документа или приложения:

```html
<main>
  <h1>Основной заголовок страницы</h1>
  <p>Основное содержимое страницы...</p>
</main>
```

> [!warning] Важно
> В документе должно быть не более одного элемента `<main>`, и он не должен содержать элементы, которые дублируются на других страницах сайта.

### `<footer>` - нижний колонтитул

Элемент `<footer>` содержит информацию о нижнем колонтитуле секции или документа:

```html
<footer>
  <p>&copy; 2025 Компания "ТехноСофт". Все права защищены.</p>
  <address>
    <a href="mailto:info@tehno-soft.ru">info@tehno-soft.ru</a>
  </address>
  <nav aria-label="Навигация в подвале">
    <ul>
      <li><a href="/privacy">Политика конфиденциальности</a></li>
      <li><a href="/terms">Условия использования</a></li>
    </ul>
  </nav>
</footer>
```

## Блоки для структурирования содержимого

### `<article>` - самостоятельный контент

Элемент `<article>` представляет собой самостоятельный фрагмент контента, который может быть распространён независимо:

```html
<article>
  <header>
    <h2>Заголовок статьи</h2>
    <time datetime="2025-01-15">15 января 2025</time>
  </header>
  
  <p>Содержимое статьи...</p>
  
  <footer>
    <p>Автор: <span>Иван Иванов</span></p>
  </footer>
</article>
```

### `<section>` - логическая секция

Элемент `<section>` представляет собой тематическую группу содержимого:

```html
<section>
  <h2>Наши услуги</h2>
  <p>Мы предлагаем широкий спектр услуг по разработке программного обеспечения.</p>
  
  <article>
    <h3>Веб-разработка</h3>
    <p>Создание современных веб-приложений и сайтов.</p>
  </article>
  
  <article>
    <h3>Мобильные приложения</h3>
    <p>Разработка приложений для iOS и Android.</p>
  </article>
</section>
```

### `<aside>` - сопутствующий контент

Элемент `<aside>` содержит контент, косвенно связанный с основным содержимым:

```html
<article>
  <h2>Новости компании</h2>
  <p>Основной текст новости...</p>
  
  <aside>
    <h3>Связанные статьи</h3>
    <ul>
      <li><a href="#">Статья 1</a></li>
      <li><a href="#">Статья 2</a></li>
    </ul>
  </aside>
</article>

<aside class="sidebar">
  <h3>Популярные статьи</h3>
  <ul>
    <li><a href="#">Самая популярная статья</a></li>
    <li><a href="#">Вторая по популярности</a></li>
  </ul>
</aside>
```

## Структурирование сложных документов

### Комбинирование блоков

Комплексная структура документа с использованием различных логических блоков:

```html
<!DOCTYPE html>
<html lang="ru">
<head>
  <meta charset="UTF-8">
  <title>Пример сложной структуры</title>
</head>
<body>
  <header>
    <h1>Название сайта</h1>
    <nav aria-label="Основная навигация">
      <ul>
        <li><a href="/">Главная</a></li>
        <li><a href="/blog">Блог</a></li>
        <li><a href="/about">О нас</a></li>
      </ul>
    </nav>
  </header>

  <main>
    <h1>Блог компании</h1>
    
    <article>
      <header>
        <h2>Заголовок статьи</h2>
        <time datetime="2025-01-15">15 января 2025</time>
      </header>
      
      <section>
        <h3>Введение</h3>
        <p>Вступительная часть статьи...</p>
      </section>
      
      <section>
        <h3>Основная часть</h3>
        <p>Основное содержимое статьи...</p>
      </section>
      
      <aside>
        <h4>Полезная информация</h4>
        <p>Дополнительные сведения, связанные с темой статьи...</p>
      </aside>
      
      <footer>
        <p>Теги: <a href="/tag/technology">Технологии</a>, <a href="/tag/web-development">Веб-разработка</a></p>
      </footer>
    </article>
  </main>

  <aside>
    <h2>Популярные статьи</h2>
    <ul>
      <li><a href="#">Статья 1</a></li>
      <li><a href="#">Статья 2</a></li>
    </ul>
  </aside>

  <footer>
    <p>&copy; 2025 Компания "ТехноСофт"</p>
    <nav aria-label="Дополнительная навигация">
      <ul>
        <li><a href="/privacy">Политика конфиденциальности</a></li>
        <li><a href="/sitemap">Карта сайта</a></li>
      </ul>
    </nav>
  </footer>
</body>
</html>
```

## Специализированные логические блоки

### `<figure>` и `<figcaption>` - медиа-контент

Для группировки изображений, диаграмм, кода и других элементов с описанием:

```html
<figure>
  <img src="diagram.png" alt="Диаграмма роста продаж">
  <figcaption>Диаграмма роста продаж за 2025 год по регионам</figcaption>
</figure>

<figure>
  <pre><code class="language-javascript">
function greet(name) {
  return `Привет, ${name}!`;
}
  </code></pre>
  <figcaption>Пример JavaScript-функции приветствия</figcaption>
</figure>
```

### `<details>` и `<summary>` - интерактивные блоки

Для создания скрытых/раскрывающихся блоков информации:

```html
<details>
  <summary>Часто задаваемые вопросы</summary>
  <div class="faq-content">
    <details>
      <summary>Как оформить заказ?</summary>
      <p>Для оформления заказа...</p>
    </details>
    <details>
      <summary>Какие способы оплаты доступны?</summary>
      <p>Мы принимаем оплату...</p>
    </details>
  </div>
</details>
```

### `<blockquote>` и `<cite>` - цитаты

Для цитирования чужих слов:

```html
<blockquote cite="https://example.com/article">
  <p>Семантическая разметка - это основа современного веб-дизайна</p>
  <footer>
    <cite>— Иван Петров, веб-разработчик</cite>
  </footer>
</blockquote>
```

## Современные практики 2025 года

### Семантические блоки и вспомогательные технологии

```html
<main>
  <h1>Результаты поиска</h1>
  
  <section role="search-results">
    <h2>Найдено 24 результата</h2>
    
    <article class="search-result">
      <h3><a href="/page1">Заголовок результата 1</a></h3>
      <p>Краткое описание содержимого...</p>
      <time datetime="2025-01-15">15 января 2025</time>
    </article>
    
    <article class="search-result">
      <h3><a href="/page2">Заголовок результата 2</a></h3>
      <p>Краткое описание содержимого...</p>
      <time datetime="2025-01-14">14 января 2025</time>
    </article>
  </section>
  
  <aside role="search-filters">
    <h2>Фильтры поиска</h2>
    <form>
      <!-- Фильтры -->
    </form>
  </aside>
</main>
```

### Адаптивные логические блоки

Структура, адаптирующаяся под разные устройства:

```html
<div class="page-layout">
  <header class="page-header">
    <h1>Название сайта</h1>
    <button class="mobile-menu-toggle" aria-expanded="false">
      Меню
    </button>
    <nav class="desktop-nav">
      <ul>
        <li><a href="/">Главная</a></li>
        <li><a href="/services">Услуги</a></li>
        <li><a href="/contacts">Контакты</a></li>
      </ul>
    </nav>
  </header>
  
  <main class="content-area">
    <!-- Основной контент -->
  </main>
  
  <aside class="sidebar">
    <!-- Боковая панель -->
  </aside>
  
  <footer class="page-footer">
    <!-- Подвал -->
  </footer>
</div>
```

## Заключение

Логические блоки контента - это фундамент семантической верстки, обеспечивающий:
- Понятную структуру документа
- Улучшенную доступность для пользователей с ограниченными возможностями
- Лучшую индексацию поисковыми системами
- Простоту поддержки и развития кода

В российских реалиях 2025 года особенно важно учитывать требования законодательства, локализацию контента и специфику отечественных поисковых систем.
