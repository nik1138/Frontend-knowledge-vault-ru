# Семантические теги в HTML

## Общие сведения

Семантические теги HTML — это элементы, которые несут в себе смысловое значение, описывая структуру и назначение контента на веб-странице. В отличие от универсальных элементов типа `<div>` и `<span>`, семантические теги четко определяют роль содержимого, что улучшает доступность, SEO и понимание кода разработчиками.

## Основные семантические элементы HTML5

### Элементы структуры документа

#### `<header>`
Представляет вводную часть раздела или документа:

```html
<article>
  <header>
    <h2>Заголовок статьи</h2>
    <p>Опубликовано: 24 ноября 2025</p>
  </header>
  <p>Основное содержимое статьи...</p>
</article>
```

#### `<nav>`
Обозначает навигационные ссылки:

```html
<nav>
  <ul>
    <li><a href="#главная">Главная</a></li>
    <li><a href="#о-нас">О нас</a></li>
    <li><a href="#контакты">Контакты</a></li>
  </ul>
</nav>
```

#### `<main>`
Представляет основное содержимое документа:

```html
<main>
  <h1>Основной заголовок страницы</h1>
  <p>Это основное содержимое страницы.</p>
</main>
```

#### `<section>`
Группирует связанный контент:

```html
<section>
  <h2>Новости компании</h2>
  <article>
    <h3>Важное объявление</h3>
    <p>Содержимое новости...</p>
  </article>
</section>
```

#### `<article>`
Представляет независимую, самодостаточную часть контента:

```html
<article>
  <header>
    <h2>Заголовок статьи</h2>
    <p>Автор: Иван Петров</p>
  </header>
  <p>Текст статьи...</p>
  <footer>
    <p>Опубликовано: 24 ноября 2025</p>
  </footer>
</article>
```

#### `<aside>`
Содержит контент, косвенно связанный с основным содержимым:

```html
<article>
  <h2>Основная статья</h2>
  <p>Текст статьи...</p>
  <aside>
    <p>Справочная информация или реклама</p>
  </aside>
</article>
```

#### `<footer>`
Представляет нижнюю часть документа или раздела:

```html
<footer>
  <p>&copy; 2025 Компания. Все права защищены.</p>
  <address>
    Контактная информация: <a href="mailto:info@example.com">info@example.com</a>
  </address>
</footer>
</body>
```

### Элементы содержимого

#### `<figure>` и `<figcaption>`
Используются для группировки медиаконтента с подписью:

```html
<figure>
  <img src="chart.png" alt="Диаграмма продаж">
  <figcaption>Диаграмма продаж за 2025 год</figcaption>
</figure>
```

#### `<time>`
Представляет время или дату:

```html
<p>Событие состоится <time datetime="2025-11-24">24 ноября 2025 года</time>.</p>
<time datetime="14:30">в 14:30</time> по московскому времени.
```

#### `<mark>`
Выделяет фрагмент текста для привлечения внимания:

```html
<p>Поиск по запросу "HTML семантика" дал <mark>1500 результатов</mark>.</p>
```

## Практические рекомендации для российских разработчиков

### Соответствие требованиям доступности

В 2025 году в России требования к доступности веб-сайтов регулируются Федеральным законом №419-ФЗ и ГОСТ Р 52872-2015. Использование семантических тегов критически важно для:

- Пользователей с экранными читалками
- Людей с нарушениями зрения
- Пользователей, использующих клавиатурную навигацию

### SEO-оптимизация

Поисковые системы в России (Яндекс, Google) учитывают семантическую структуру при индексировании:

- Правильное использование заголовков улучшает понимание контента
- Семантические элементы помогают определить важность и структуру информации
- Контент в элементах `<main>` воспринимается как приоритетный

### Совместимость с браузерами

Семантические элементы HTML5 поддерживаются всеми современными браузерами в России:

- Chrome, Yandex Browser (наиболее популярные)
- Firefox, Opera, Safari
- Для IE11 и ниже требуется полифилл или стилизация через CSS

## Пример правильной семантической разметки

```html
<!DOCTYPE html>
<html lang="ru">
<head>
  <meta charset="UTF-8">
  <title>Семантическая разметка в HTML</title>
</head>
<body>
  <header>
    <h1>Сайт о веб-разработке</h1>
    <nav aria-label="Основная навигация">
      <ul>
        <li><a href="/">Главная</a></li>
        <li><a href="/html">HTML</a></li>
        <li><a href="/css">CSS</a></li>
      </ul>
    </nav>
  </header>

  <main>
    <article>
      <header>
        <h2>Семантические теги в HTML</h2>
        <time datetime="2025-11-24">24 ноября 2025</time>
      </header>
      
      <section>
        <h3>Основные элементы</h3>
        <p>Семантические теги помогают структурировать контент...</p>
      </section>
      
      <section>
        <h3>Практическое применение</h3>
        <p>В российской практике важно учитывать...</p>
      </section>
      
      <aside>
        <h4>Полезные ресурсы</h4>
        <ul>
          <li><a href="#">Документация HTML</a></li>
          <li><a href="#">ГОСТ по доступности</a></li>
        </ul>
      </aside>
    </article>
  </main>

  <footer>
    <p>&copy; 2025 Веб-разработка. Все права защищены.</p>
    <address>
      <a href="mailto:contact@webdev.ru">contact@webdev.ru</a>
    </address>
  </footer>
</body>
</html>
```

## Распространенные ошибки

### Неправильное использование `<div>` вместо семантических элементов

```html
<!-- Плохо -->
<div class="header">...</div>
<div class="nav">...</div>
<div class="main">...</div>

<!-- Хорошо -->
<header>...</header>
<nav>...</nav>
<main>...</main>
```

### Использование семантических элементов только для стилизации

Семантические элементы не должны использоваться исключительно для CSS-стилей — они должны отражать смысловую структуру документа.
