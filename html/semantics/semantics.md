# HTML Семантика

HTML семантика - это практика использования правильных HTML элементов для передачи смысла и структуры содержимого веб-страницы. Семантические элементы делают код более понятным как для разработчиков, так и для браузеров и вспомогательных технологий.

## Определение семантики HTML

Семантика в HTML означает использование правильных тегов для передачи смысла содержимого. Это позволяет браузерам, поисковым системам и вспомогательным технологиям лучше понимать структуру и значение веб-страницы. Правильная семантика делает веб-сайты более доступными и улучшает SEO.

## Семантические элементы

### Основные структурные элементы

- `<header>` - представлен в [[html/structure/header]] - содержит вводную информацию о разделе или документе
- `<nav>` - представлен в [[html/structure/navigation]] - определяет навигационные ссылки
- `<main>` - основное содержимое страницы
- `<article>` - независимый, самодостаточный контент
- `<section>` - логическая группа контента, обычно с заголовком
- `<aside>` - боковая панель с дополнительной информацией
- `<footer>` - представлен в [[html/structure/footer]] - нижний колонтитул документа или раздела

### Текстовые семантические элементы

- `<h1>`-`<h6>` - заголовки различного уровня
- `<p>` - абзац текста
- `<strong>` и `<b>` - представлен в [[html/text/bold]] - выделение текста
- `<em>` и `<i>` - представлен в [[html/text/italic]] - курсив
- `<time>` - для обозначения времени и дат
- `<mark>` - выделение важного текста
- `<code>` - представлен в [[html/text/code]] - для отображения кода

## Преимущества семантики

1. **Доступность** - вспомогательные технологии, такие как скринридеры, используют семантическую структуру для навигации по контенту
2. **SEO** - поисковые системы учитывают семантическую структуру при индексации и ранжировании страниц
3. **Поддержка кода** - семантически правильный код легче читать и поддерживать
4. **Производительность** - правильная структура может улучшить производительность при парсинге и рендеринге

## Доступность и SEO

Семантические элементы играют ключевую роль в создании доступных веб-сайтов. Они позволяют пользователям с ограниченными возможностями использовать вспомогательные технологии, такие как скринридеры, для навигации по странице. Семантика также улучшает SEO, поскольку поисковые системы анализируют структуру страницы для понимания и индексации содержимого.

## Примеры использования

### Пример семантической разметки статьи:

```html
<article>
  <header>
    <h1>Заголовок статьи</h1>
    <p><time datetime="2023-11-09">9 ноября 2023</time></p>
  </header>
  
  <section>
    <h2>Первый раздел</h2>
    <p>Содержимое первого раздела статьи...</p>
  </section>
  
  <section>
    <h2>Второй раздел</h2>
    <p>Содержимое второго раздела статьи...</p>
  </section>
  
  <footer>
    <p>Автор: Имя Фамилия</p>
  </footer>
</article>
```

### Пример основной структуры страницы:

```html
<!DOCTYPE html>
<html lang="ru">
  <head>
    <meta charset="UTF-8">
    <title>Пример семантической разметки</title>
  </head>
  <body>
    <header>
      <h1>Мой веб-сайт</h1>
      <nav>
        <ul>
          <li><a href="/">Главная</a></li>
          <li><a href="/about">О нас</a></li>
          <li><a href="/contact">Контакты</a></li>
        </ul>
      </nav>
    </header>
    
    <main>
      <section>
        <h2>Основной контент</h2>
        <p>Здесь находится основной контент страницы.</p>
      </section>
      
      <aside>
        <h3>Боковая панель</h3>
        <p>Дополнительная информация.</p>
      </aside>
    </main>
    
    <footer>
      <p>&copy; 2023 Мой веб-сайт. Все права защищены.</p>
    </footer>
  </body>
</html>
```

## Лучшие практики

1. **Используйте заголовки последовательно** - не пропускайте уровни (например, не переходите сразу от `<h1>` к `<h3>`)
2. **Используйте один `<main>` на странице** - только один основной элемент содержимого
3. **Избегайте избыточности** - не используйте семантические элементы просто потому, что можете
4. **Объединяйте с ARIA-ролями** - представлен в [[html/accessibility/aria]] - когда необходимо дополнительное описание
5. **Проверяйте структуру** - используйте инструменты разработчика для проверки семантической структуры

## Сравнение с несемантическими элементами

| Семантический элемент | Несемантический аналог | Преимущество |
|----------------------|------------------------|--------------|
| `<header>` | `<div class="header">` | Ясное обозначение шапки документа |
| `<nav>` | `<div class="navigation">` | Браузеры и скринридеры понимают назначение |
| `<article>` | `<div class="post">` | Поисковые системы распознают независимый контент |
| `<section>` | `<div class="section">` | Логическая группировка контента |
| `<aside>` | `<div class="sidebar">` | Ясное обозначение дополнительного содержимого |

Использование семантических элементов вместо универсальных `<div>` и `<span>` делает код более понятным и функциональным.
