# Структура документа в HTML

## Общие сведения

Структура HTML-документа — это основа, на которой строится вся веб-страница. Правильная структура обеспечивает семантическую ясность, доступность и корректную интерпретацию контента браузерами и поисковыми системами. В 2025 году в российской веб-разработке особое внимание уделяется созданию структурированных документов, соответствующих современным стандартам и требованиям доступности.

## Базовая структура HTML5 документа

Каждый корректный HTML-документ должен начинаться с объявления <!DOCTYPE>:

```html
<!DOCTYPE html>
<html lang="ru">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Заголовок страницы</title>
</head>
<body>
  <!-- Содержимое страницы -->
</body>
</html>
```

### Объявление <!DOCTYPE>

- Объявление `<!DOCTYPE html>` указывает браузеру, что документ следует стандартам HTML5
- Это объявление должно быть первой строкой документа
- Не чувствительно к регистру: `<!DOCTYPE html>` или `<!doctype HTML>` — оба варианта корректны

### Элемент `<html>`

- Корневой элемент всего HTML-документа
- Атрибут `lang` указывает язык документа (в России — `ru` для русского языка)
- Может содержать атрибуты `dir` для указания направления текста (например, `dir="ltr"` для слева направо)

### Элемент `<head>`

Содержит метаданные документа, которые не отображаются непосредственно на странице:

#### Обязательные элементы

```html
<head>
  <!-- Обязательная кодировка -->
  <meta charset="UTF-8">
  
  <!-- Адаптивная верстка -->
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  
  <!-- Заголовок страницы -->
  <title>Заголовок страницы</title>
</head>
```

#### Дополнительные элементы

```html
<head>
  <!-- Кодировка -->
  <meta charset="UTF-8">
  
  <!-- Адаптивность -->
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  
  <!-- Заголовок -->
  <title>Заголовок страницы</title>
  
  <!-- Описание для SEO -->
  <meta name="description" content="Описание страницы для поисковых систем">
  
  <!-- Автор -->
  <meta name="author" content="Имя автора">
  
  <!-- Канонический URL -->
  <link rel="canonical" href="https://example.com/page">
  
  <!-- Стили -->
  <link rel="stylesheet" href="styles.css">
  
  <!-- Favicon -->
  <link rel="icon" href="favicon.ico">
</head>
```

### Элемент `<body>`

Содержит видимое содержимое документа:

```html
<body>
  <header>
    <!-- Заголовок сайта -->
  </header>
  
  <nav>
    <!-- Навигация -->
  </nav>
  
  <main>
    <!-- Основное содержимое -->
  </main>
  
  <footer>
    <!-- Подвал сайта -->
  </footer>
</body>
```

## Семантическая структура документа

### Использование семантических элементов

```html
<body>
  <header>
    <h1>Название сайта</h1>
    <nav>
      <!-- Навигационные ссылки -->
    </nav>
  </header>

  <main>
    <section>
      <h2>Основной раздел</h2>
      <article>
        <h3>Статья</h3>
        <p>Содержимое статьи</p>
      </article>
    </section>
  </main>

  <aside>
    <h3>Боковая панель</h3>
    <p>Дополнительная информация</p>
  </aside>

  <footer>
    <p>Информация в подвале</p>
  </footer>
</body>
```

### Правильная иерархия заголовков

- Используйте только один элемент `<h1>` на странице — это основной заголовок
- Следуйте логической иерархии: h1 → h2 → h3 → h4 → h5 → h6
- Не пропускайте уровни заголовков

## Практические рекомендации для российских разработчиков

### Требования к доступности

В 2025 году в России требования к доступности веб-сайтов регулируются:

- Федеральным законом №419-ФЗ от 27.12.2007
- ГОСТ Р 52872-2015 "Интернет-ресурсы"
- Рекомендациями W3C по доступности (WCAG 2.1)

Для соответствия этим требованиям:

- Обязательно указывайте атрибут `lang` с правильным языком
- Используйте семантические элементы для структурирования контента
- Обеспечьте корректную навигацию с клавиатуры
- Добавляйте альтернативный текст к изображениям

### SEO-рекомендации

Для российских поисковых систем (Яндекс, Google):

- Заголовок `<title>` должен быть информативным и уникальным (не более 60 символов)
- Мета-описание должно точно отражать содержимое страницы (не более 160 символов)
- Используйте теги заголовков логически для структурирования контента

### Локализация

Для российской аудитории:

- Указывайте `lang="ru"` для русскоязычного контента
- Используйте кодировку UTF-8
- Учитывайте культурные особенности при структурировании информации

## Пример полной структуры документа

```html
<!DOCTYPE html>
<html lang="ru">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <meta name="description" content="Пример корректной HTML-структуры документа">
  <title>Пример HTML-документа</title>
  <link rel="stylesheet" href="styles.css">
  <link rel="icon" href="favicon.ico">
</head>
<body>
  <header>
    <h1>Название сайта</h1>
    <nav aria-label="Основная навигация">
      <ul>
        <li><a href="/">Главная</a></li>
        <li><a href="/about">О нас</a></li>
        <li><a href="/services">Услуги</a></li>
        <li><a href="/contacts">Контакты</a></li>
      </ul>
    </nav>
  </header>

  <main>
    <article>
      <header>
        <h2>Заголовок основной статьи</h2>
        <time datetime="2025-11-24">24 ноября 2025</time>
      </header>
      
      <section>
        <h3>Первый раздел статьи</h3>
        <p>Содержимое первого раздела...</p>
      </section>
      
      <section>
        <h3>Второй раздел статьи</h3>
        <p>Содержимое второго раздела...</p>
      </section>
      
      <aside>
        <h4>Сопутствующая информация</h4>
        <p>Дополнительная информация, связанная с основным контентом</p>
      </aside>
    </article>
  </main>

  <aside>
    <h3>Боковая панель</h3>
    <nav aria-label="Дополнительная навигация">
      <ul>
        <li><a href="/blog">Блог</a></li>
        <li><a href="/news">Новости</a></li>
        <li><a href="/faq">FAQ</a></li>
      </ul>
    </nav>
  </aside>

  <footer>
    <p>&copy; 2025 Компания. Все права защищены.</p>
    <address>
      <a href="mailto:info@example.com">info@example.com</a>
    </address>
    <nav aria-label="Навигация в подвале">
      <ul>
        <li><a href="/privacy">Политика конфиденциальности</a></li>
        <li><a href="/terms">Условия использования</a></li>
      </ul>
    </nav>
  </footer>
</body>
</html>
```

## Проверка структуры документа

### Инструменты валидации

- Валидатор W3C: https://validator.w3.org/
- Инструменты разработчика в браузерах (F12)
- Расширения для браузеров (например, HeadingsMap)

### Автоматизированные проверки

Для российских проектов рекомендуется использовать:

- axe-core для проверки доступности
- Lighthouse для комплексной оценки
- Инструменты командной строки для CI/CD процессов

Правильная структура HTML-документа — основа современного веб-развития в России. В 2025 году она особенно важна из-за ужесточающихся требований к доступности, SEO и пользовательскому опыту.
