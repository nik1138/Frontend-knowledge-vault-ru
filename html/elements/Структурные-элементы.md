# Структурные элементы HTML

## Обзор

Структурные элементы HTML (также называемые семантическими элементами) представляют собой основу современной веб-разработки. Эти элементы определяют логическую структуру документа и помогают браузерам, поисковым системам и вспомогательным технологиям понимать содержание веб-страницы. В 2025 году использование правильных структурных элементов становится не просто хорошей практикой, а критически важным аспектом доступности и SEO.

## Основные структурные элементы

### `<header>`

Элемент `<header>` представляет собой контейнер для введения содержимого или навигационной информации, обычно содержащий логотип, заголовки, навигационное меню или поисковую строку.

```html
<header>
  <h1>Мой веб-сайт</h1>
  <nav>
    <a href="#home">Главная</a>
    <a href="#about">О нас</a>
    <a href="#contact">Контакты</a>
  </nav>
</header>
```

> [!tip] Совет
> Используйте `<header>` не только в начале документа, но и в начале секций [[Разделы.md|разделов]] для улучшения семантики.

### `<nav>`

Элемент `<nav>` определяет навигационные ссылки. Обычно содержит список ссылок на другие страницы или разделы текущей страницы.

```html
<nav>
  <ul>
    <li><a href="/">Главная</a></li>
    <li><a href="/products">Продукты</a></li>
    <li><a href="/about">О компании</a></li>
  </ul>
</nav>
```

> [!important] Важно
> Не все группы ссылок должны быть обернуты в `<nav>`. Используйте его только для основных блоков навигации.

### `<main>`

Элемент `<main>` содержит основное содержание документа. Должен использоваться только один раз на странице (за исключением ситуаций с динамической загрузкой контента).

```html
<main>
  <article>
    <h2>Новости компании</h2>
    <p>Содержание статьи...</p>
  </article>
</main>
```

### `<section>`

Элемент `<section>` представляет собой логический раздел документа, обычно с заголовком. Используется для группировки связанных элементов.

```html
<section>
  <h2>Услуги</h2>
  <p>Описание предоставляемых услуг...</p>
</section>
```

> [!warning] Внимание
> Не используйте `<section>` просто для стилизации. Используйте [[div.md|div]] для стилизации и `<section>` для семантики.

### `<article>`

Элемент `<article>` представляет собой независимую, самодостаточную часть контента, которая может быть распространена или переиспользована отдельно от остальной части документа.

```html
<article>
  <h3>Последние новости</h3>
  <p>Текст новости...</p>
  <time datetime="2025-01-15">15 января 2025</time>
</article>
```

### `<aside>`

Элемент `<aside>` содержит контент, косвенно связанный с основным содержанием. Это может быть боковая панель, заметки или реклама.

```html
<aside>
  <h4>Связанные статьи</h4>
  <ul>
    <li><a href="#">Статья 1</a></li>
    <li><a href="#">Статья 2</a></li>
  </ul>
</aside>
```

### `<footer>`

Элемент `<footer>` содержит информацию о нижнем колонтитуле, обычно включающую авторские права, контактную информацию или ссылки на политику конфиденциальности.

```html
<footer>
  <p>&copy; 2025 Компания. Все права защищены.</p>
  <address>
    <a href="mailto:info@example.com">info@example.com</a>
  </address>
</footer>
```

## Пример полной структуры документа

```html
<!DOCTYPE html>
<html lang="ru">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Структурные элементы HTML</title>
</head>
<body>
  <header>
    <h1>Название сайта</h1>
    <nav>
      <ul>
        <li><a href="/">Главная</a></li>
        <li><a href="/services">Услуги</a></li>
        <li><a href="/contact">Контакты</a></li>
      </ul>
    </nav>
  </header>

  <main>
    <section>
      <h2>Основные услуги</h2>
      <article>
        <h3>Веб-разработка</h3>
        <p>Описание услуги...</p>
      </article>
    </section>
    
    <aside>
      <h4>Полезные ссылки</h4>
      <ul>
        <li><a href="#">Документация</a></li>
        <li><a href="#">Техподдержка</a></li>
      </ul>
    </aside>
  </main>

  <footer>
    <p>Контактная информация и права</p>
  </footer>
</body>
</html>
```

## Практические рекомендации

### 1. Использование в российских реалиях 2025

В России в 2025 году особое внимание уделяется доступности веб-сайтов. Использование структурных элементов критически важно для:

- Соответствия требованиям 44-ФЗ и 275-ФЗ (доступность для людей с ограниченными возможностями)
- Улучшения индексации в поисковых системах Яндекс и Google
- Совместимости с голосовыми интерфейсами и читалками с экрана

### 2. SEO преимущества

Поисковые системы активно используют семантические элементы для понимания структуры контента:

- `<h1>` в `<header>` помогает определить тему страницы
- `<article>` указывает на основной контент
- `<nav>` выделяет навигационные элементы

### 3. Доступность

Структурные элементы обеспечивают:

- Лучшую навигацию для пользователей вспомогательных технологий
- Возможность быстрого перехода к основным разделам
- Правильное чтение контента читалками с экрана

## Сравнение с устаревшими элементами

| Современный элемент | Устаревший подход | Преимущества |
|-------------------|------------------|-------------|
| `<header>` | `<div class="header">` | Семантически значимый, доступный |
| `<nav>` | `<div class="menu">` | Распознается вспомогательными технологиями |
| `<main>` | `<div class="content">` | Позволяет пропустить навигацию |
| `<section>` | `<div class="section">` | Определяет логические разделы |
| `<article>` | `<div class="post">` | Самодостаточный контент |

## Заключение

Структурные элементы HTML играют ключевую роль в современной веб-разработке, особенно в условиях ужесточения требований к доступности и SEO в России. Правильное использование этих элементов не только улучшает структуру документа, но и способствует лучшему восприятию контента пользователями и поисковыми системами.
