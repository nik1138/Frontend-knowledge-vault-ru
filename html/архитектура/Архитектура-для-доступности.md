# Архитектура HTML для доступности

## Обзор

Архитектура HTML для доступности (accessibility, или a11y) обеспечивает возможность использования веб-сайтов людьми с ограниченными возможностями. В 2025 году требования к доступности становятся все более строгими, особенно в государственных и коммерческих веб-ресурсах России.

## Основы доступной архитектуры

### Семантическая разметка

Правильная семантическая разметка позволяет вспомогательным технологиям (скринридерам, навигаторам с клавиатуры) понимать структуру и назначение элементов:

```html
<!-- Правильно: использование семантических тегов -->
<header>
  <nav>
    <ul>
      <li><a href="/">Главная</a></li>
      <li><a href="/about">О нас</a></li>
    </ul>
  </nav>
</header>

<main>
  <article>
    <h1>Заголовок статьи</h1>
    <section>
      <h2>Подраздел статьи</h2>
      <p>Содержимое подраздела</p>
    </section>
  </article>
</main>

<aside>
  <h2>Дополнительная информация</h2>
  <p>Связанный контент</p>
</aside>

<footer>
  <p>Информация о сайте</p>
</footer>
```

> [!tip] Совет
> Используйте семантические теги вместо универсального `<div>`, чтобы структура была понятна вспомогательным технологиям.

### Заголовки и иерархия

```html
<!-- Правильная иерархия заголовков -->
<h1>Основной заголовок страницы</h1>
<h2>Раздел 1</h2>
<h3>Подраздел 1.1</h3>
<h3>Подраздел 1.2</h3>
<h2>Раздел 2</h2>
<h3>Подраздел 2.1</h3>
```

## Атрибуты доступности

### ARIA (Accessible Rich Internet Applications)

ARIA-атрибуты дополняют семантическую разметку информацией для вспомогательных технологий:

```html
<!-- Роль элемента -->
<div role="button" tabindex="0" aria-label="Кнопка закрытия">✕</div>

<!-- Описание элемента -->
<button aria-describedby="help-text">Отправить</button>
<div id="help-text">Эта кнопка отправляет форму</div>

<!-- Заголовок для сложных элементов -->
<div role="region" aria-labelledby="section-title">
  <h2 id="section-title">Информационная секция</h2>
  <p>Сложный контент...</p>
</div>

<!-- Состояния -->
<button aria-pressed="false">Переключатель</button>
<input type="text" aria-invalid="true" aria-errormessage="error-message">
<div id="error-message">Поле заполнено неправильно</div>
```

## Формы и доступность

### Правильная разметка форм

```html
<form>
  <fieldset>
    <legend>Личная информация</legend>
    
    <div>
      <label for="name">Имя:</label>
      <input type="text" id="name" name="name" required aria-describedby="name-help">
      <div id="name-help">Введите ваше имя</div>
    </div>
    
    <div>
      <label for="email">Email:</label>
      <input type="email" id="email" name="email" required aria-invalid="false">
      <span role="alert" id="email-error" style="display: none;"></span>
    </div>
    
    <div>
      <label for="message">Сообщение:</label>
      <textarea id="message" name="message" rows="5" cols="40"></textarea>
    </div>
    
    <button type="submit">Отправить</button>
  </fieldset>
</form>
```

### Управление фокусом

```html
<!-- Управление фокусом в модальных окнах -->
<div class="modal" role="dialog" aria-modal="true" aria-labelledby="modal-title">
  <h2 id="modal-title">Заголовок модального окна</h2>
  <button onclick="closeModal()">Закрыть</button>
</div>

<!-- Пример JavaScript для управления фокусом -->
<script>
function trapFocus(element) {
  const focusableElements = element.querySelectorAll('button, [href], input, select, textarea, [tabindex]:not([tabindex="-1"])');
  const firstElement = focusableElements[0];
  const lastElement = focusableElements[focusableElements.length - 1];

  element.addEventListener('keydown', function(e) {
    if (e.key === 'Tab') {
      if (e.shiftKey) {
        if (document.activeElement === firstElement) {
          lastElement.focus();
          e.preventDefault();
        }
      } else {
        if (document.activeElement === lastElement) {
          firstElement.focus();
          e.preventDefault();
        }
      }
    }
  });
}
</script>
```

## Контрастность и визуальная доступность

### Цвет и контраст

Хотя это не архитектура HTML, цветовая схема должна соответствовать требованиям WCAG:

```html
<!-- Использование CSS для обеспечения контрастности -->
<style>
.high-contrast {
  color: #000000;
  background-color: #ffffff;
  /* Соотношение контрастности не менее 4.5:1 для нормального текста */
}

.focus-indicator {
  outline: 2px solid #005fcc;
  outline-offset: 2px;
}
</style>

<button class="high-contrast focus-indicator">Кнопка с высокой контрастностью</button>
```

## Клавиатурная навигация

### Атрибуты tab-index

```html
<!-- Правильное использование tabindex -->
<button tabindex="0">Элемент с естественным фокусом</button>
<div tabindex="0" role="button" onclick="doAction()">Див, как кнопка</div>
<span tabindex="-1" id="skip-link">Пропустить навигацию</span>

<!-- Ссылка для пропуска навигации -->
<a href="#main-content" class="skip-link">Перейти к основному содержимому</a>
<main id="main-content">
  <!-- Основной контент -->
</main>
```

## Язык и локализация

### Атрибуты языка

```html
<!-- Указание языка документа -->
<html lang="ru">
<head>
  <meta charset="UTF-8">
  <!-- Для частей на другом языке -->
  <p>Этот текст на русском, а <span lang="en">this text is in English</span>.</p>
</head>
```

## Мультимедиа и доступность

### Видео и аудио

```html
<!-- Видео с субтитрами и аудиодескрипцией -->
<video controls width="800" height="600" aria-describedby="video-description">
  <source src="video.mp4" type="video/mp4">
  <track kind="subtitles" src="subtitles-ru.vtt" srclang="ru" label="Русские субтитры">
  <track kind="descriptions" src="audio-description.vtt" srclang="ru" label="Аудиодескрипция">
  Ваш браузер не поддерживает видео.
</video>
<div id="video-description">Описание видео для пользователей с нарушениями зрения</div>

<!-- Аудио с транскрипцией -->
<audio controls>
  <source src="audio.mp3" type="audio/mpeg">
  <a href="transcript.txt">Транскрипция аудио</a>
</audio>
```

## Российские требования

### Федеральный закон 419-ФЗ

В России требования к доступности регулируются Федеральным законом №419-ФЗ от 29.12.2014:

- Обеспечение доступности для людей с ограниченными возможностями
- Соответствие международным стандартам WCAG 2.1 AA
- Обязательная проверка доступности государственных сайтов

### Требования к государственным сайтам

```html
<!-- Пример разметки, соответствующей российским требованиям -->
<!DOCTYPE html>
<html lang="ru" class="no-js">
<head>
  <meta charset="UTF-8">
  <title>Доступный веб-сайт - Название</title>
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <meta name="description" content="Описание сайта, доступного для людей с ОВЗ">
  
  <!-- Подключение скрипта для определения возможностей пользователя -->
  <script>
    // Удаление класса no-js при загрузке JavaScript
    document.documentElement.classList.remove('no-js');
    document.documentElement.classList.add('js');
  </script>
</head>
<body>
  <!-- Структура, соответствующая требованиям доступности -->
  <a href="#main-content" class="skip-link">Перейти к основному содержимому</a>
  
  <header role="banner">
    <h1>Название сайта</h1>
    <!-- Навигация -->
  </header>
  
  <nav role="navigation" aria-label="Основная навигация">
    <ul>
      <li><a href="/">Главная</a></li>
      <li><a href="/services">Услуги</a></li>
      <li><a href="/accessibility">Доступность</a></li>
    </ul>
  </nav>
  
  <main id="main-content" role="main">
    <!-- Основной контент -->
  </main>
  
  <footer role="contentinfo">
    <p>© 2025 Организация. Все права защищены.</p>
  </footer>
</body>
</html>
```

## Практические рекомендации

### Тестирование доступности

1. Проверка с помощью скринридера (NVDA, JAWS, VoiceOver)
2. Тестирование клавиатурной навигации (только Tab, Shift+Tab)
3. Использование инструментов:
   - axe DevTools
   - WAVE
   - Lighthouse
   - [[Инструменты-для-тестирования-доступности]]

### Наиболее частые ошибки

- Отсутствие альтернативного текста для изображений
- Неправильная иерархия заголовков
- Недоступные формы без меток
- Недостаточная контрастность
- Отсутствие описания сложных интерактивных элементов

## Проверка соответствия

### WCAG 2.1 AA

Основные критерии, которые должны быть соблюдены:

- 1.1.1 Нетекстовое содержимое (альт-текст для изображений)
- 1.3.1 Информация и отношения (правильная семантическая разметка)
- 1.3.5 Идентификация цели ввода (подсказки для форм)
- 2.1.1 Клавиатура (все функции доступны с клавиатуры)
- 2.4.6 Заголовки и ярлыки (понятные заголовки и метки)
- 4.1.2 Имя, роль, значение (правильное описание элементов)
