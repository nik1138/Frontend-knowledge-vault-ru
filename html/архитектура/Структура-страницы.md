# Структура страницы

## Введение

Структура HTML-страницы - это фундамент, на котором строится вся веб-страница. Правильная структура обеспечивает доступность, SEO-оптимизацию и поддерживаемость кода. В российской веб-разработке 2025 года акцент на семантически правильную структуру становится особенно важным.

## Базовая структура HTML-страницы

Каждая HTML-страница должна начинаться с определенной структуры, которая обеспечивает правильное отображение и интерпретацию браузерами:

```html
<!DOCTYPE html>
<html lang="ru">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Заголовок страницы</title>
    <meta name="description" content="Краткое описание страницы">
    <link rel="canonical" href="https://example.com/page">
    <link rel="stylesheet" href="styles.css">
</head>
<body>
    <!-- Структурные элементы страницы -->
</body>
</html>
```

## Семантические элементы

Семантические элементы HTML5 играют ключевую роль в структурировании страницы:

### `<header>` - Шапка страницы
```html
<header class="page-header">
    <div class="container">
        <h1>Логотип/Название сайта</h1>
        <nav>...</nav>
    </div>
</header>
```

### `<nav>` - Навигация
```html
<nav class="main-navigation">
    <ul>
        <li><a href="/">Главная</a></li>
        <li><a href="/about">О нас</a></li>
        <li><a href="/services">Услуги</a></li>
        <li><a href="/contact">Контакты</a></li>
    </ul>
</nav>
```

### `<main>` - Основной контент
```html
<main class="main-content">
    <article>
        <header>
            <h1>Заголовок статьи</h1>
            <time datetime="2025-01-15">15 января 2025</time>
        </header>
        <section>
            <h2>Подзаголовок секции</h2>
            <p>Содержимое секции...</p>
        </section>
    </article>
</main>
```

### `<aside>` - Сопутствующий контент
```html
<aside class="sidebar">
    <section>
        <h3>Реклама</h3>
        <div>Рекламный блок</div>
    </section>
    <section>
        <h3>Популярные статьи</h3>
        <ul>...</ul>
    </section>
</aside>
```

### `<footer>` - Подвал страницы
```html
<footer class="page-footer">
    <div class="container">
        <p>&copy; 2025 Название компании. Все права защищены.</p>
        <nav>...</nav>
    </div>
</footer>
```

## Иерархия заголовков

Правильная иерархия заголовков критически важна для SEO и доступности:

```html
<h1>Основной заголовок страницы (только один на странице)</h1>
    <h2>Подраздел 1</h2>
        <h3>Подраздел 1.1</h3>
        <h3>Подраздел 1.2</h3>
            <h4>Подраздел 1.2.1</h4>
    <h2>Подраздел 2</h2>
        <h3>Подраздел 2.1</h3>
```

## Структура контента

### Артикульная структура
```html
<article class="post">
    <header>
        <h1>Заголовок статьи</h1>
        <p class="meta">
            Автор: <span class="author">Иван Иванов</span>
            <time datetime="2025-01-15">15 января 2025</time>
        </p>
    </header>
    
    <section class="content">
        <p>Первый абзац статьи...</p>
        <figure>
            <img src="image.jpg" alt="Описание изображения">
            <figcaption>Подпись к изображению</figcaption>
        </figure>
    </section>
    
    <footer>
        <p>Теги: <a href="/tag/html">HTML</a>, <a href="/tag/css">CSS</a></p>
    </footer>
</article>
```

### Секционная структура
```html
<section class="features">
    <h2>Наши преимущества</h2>
    <div class="feature-grid">
        <article class="feature">
            <h3>Быстрая доставка</h3>
            <p>Описание преимущества...</p>
        </article>
        <article class="feature">
            <h3>Качественный сервис</h3>
            <p>Описание преимущества...</p>
        </article>
    </div>
</section>
```

## Адаптивная структура

С учетом российских реалий 2025 года, структура страницы должна быть адаптивной:

```html
<body>
    <div class="page-wrapper">
        <header class="header">
            <div class="container">
                <div class="header-top">
                    <a href="/" class="logo">Название сайта</a>
                    <button class="mobile-menu-toggle">☰</button>
                    <nav class="desktop-nav">...</nav>
                </div>
                <nav class="mobile-nav">...</nav>
            </div>
        </header>
        
        <main class="main">
            <div class="container">
                <div class="content-wrapper">
                    <article class="content">...</article>
                    <aside class="sidebar">...</aside>
                </div>
            </div>
        </main>
        
        <footer class="footer">
            <div class="container">
                <div class="footer-content">...</div>
            </div>
        </footer>
    </div>
</body>
```

## Структура для форм

Специальная структура для форм с учетом доступности:

```html
<form class="contact-form" action="/submit" method="post">
    <fieldset>
        <legend>Контактная информация</legend>
        
        <div class="form-group">
            <label for="name">Имя:</label>
            <input type="text" id="name" name="name" required>
        </div>
        
        <div class="form-group">
            <label for="email">Email:</label>
            <input type="email" id="email" name="email" required>
        </div>
        
        <div class="form-group">
            <label for="message">Сообщение:</label>
            <textarea id="message" name="message"></textarea>
        </div>
        
        <button type="submit">Отправить</button>
    </fieldset>
</form>
```

## Доступность и семантика

Важные аспекты структуры с точки зрения доступности:

- Использование ARIA-атрибутов для улучшения доступности
- Правильная иерархия заголовков
- Описательные alt-атрибуты для изображений
- Логическая последовательность элементов

```html
<div role="main">
    <h1>Заголовок страницы</h1>
    <nav aria-label="Основная навигация">
        <ul>...</ul>
    </nav>
    <article aria-labelledby="article-title">
        <h2 id="article-title">Заголовок статьи</h2>
        ...
    </article>
</div>
```

## Структура для SEO

Оптимизированная структура для поисковой оптимизации:

```html
<!DOCTYPE html>
<html lang="ru">
<head>
    <meta charset="UTF-8">
    <title>SEO-оптимизированный заголовок - до 60 символов</title>
    <meta name="description" content="SEO-описание - до 160 символов">
    <meta name="keywords" content="ключевые, слова, страницы">
    <link rel="canonical" href="https://example.com/page">
    <meta property="og:title" content="Заголовок для соцсетей">
    <meta property="og:description" content="Описание для соцсетей">
    <meta property="og:type" content="website">
    <meta property="og:url" content="https://example.com/page">
</head>
<body itemscope itemtype="http://schema.org/WebPage">
    <header>
        <h1 itemprop="name">Название сайта</h1>
    </header>
    
    <main itemprop="mainContentOfPage">
        <article itemscope itemtype="http://schema.org/Article">
            <h1 itemprop="headline">Заголовок статьи</h1>
            <div itemprop="author" itemscope itemtype="http://schema.org/Person">
                <span itemprop="name">Имя автора</span>
            </div>
            <div itemprop="articleBody">
                <!-- Основное содержимое статьи -->
            </div>
        </article>
    </main>
</body>
</html>
```
