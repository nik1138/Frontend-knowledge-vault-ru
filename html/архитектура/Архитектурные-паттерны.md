# Архитектурные паттерны

## Введение

Архитектурные паттерны в HTML - это проверенные временем решения, которые помогают создавать структурированные, поддерживаемые и масштабируемые веб-страницы. В российской веб-разработке 2025 года знание и применение этих паттернов становится критически важным для создания конкурентоспособных решений.

## Основные архитектурные паттерны

### 1. Паттерн "Модульный компонент"

Этот паттерн заключается в создании независимых, повторно используемых компонентов:

```html
<!-- Карточка пользователя -->
<div class="user-card" data-user-id="123">
    <div class="user-card__avatar">
        <img src="avatar.jpg" alt="Аватар пользователя" class="user-card__img">
    </div>
    <div class="user-card__info">
        <h3 class="user-card__name">Иван Петров</h3>
        <p class="user-card__position">Разработчик</p>
        <div class="user-card__actions">
            <button class="btn btn--primary">Связаться</button>
            <button class="btn btn--secondary">Профиль</button>
        </div>
    </div>
</div>
```

Преимущества:
- Повторное использование
- Легкость тестирования
- Простота поддержки
- Масштабируемость

### 2. Паттерн "Контейнер-компонент"

Разделяет структурную логику (контейнеры) от презентационной (компоненты):

```html
<!-- Контейнер -->
<div class="product-grid" data-component="product-grid">
    <h2>Популярные товары</h2>
    <div class="product-grid__items">
        <!-- Компоненты будут вставлены сюда -->
    </div>
</div>

<!-- Компонент -->
<div class="product-item" data-component="product-item">
    <img src="product.jpg" alt="Описание товара" class="product-item__image">
    <h3 class="product-item__title">Название товара</h3>
    <p class="product-item__price">1000 ₽</p>
    <button class="product-item__button">В корзину</button>
</div>
```

### 3. Паттерн "Шаблон-фрагмент"

Использование HTML-шаблонов для динамического создания компонентов:

```html
<template id="comment-template">
    <article class="comment">
        <header class="comment__header">
            <img class="comment__avatar" src="" alt="">
            <div class="comment__author-info">
                <h4 class="comment__author"></h4>
                <time class="comment__date"></time>
            </div>
        </header>
        <div class="comment__content"></div>
        <footer class="comment__actions">
            <button class="comment__reply">Ответить</button>
        </footer>
    </article>
</template>

<script>
function createComment(data) {
    const template = document.getElementById('comment-template');
    const clone = template.content.cloneNode(true);
    
    clone.querySelector('.comment__author').textContent = data.author;
    clone.querySelector('.comment__date').textContent = data.date;
    clone.querySelector('.comment__content').textContent = data.content;
    
    return clone;
}
</script>
```

## Структурные паттерны

### 1. Паттерн "Макет страницы" (Page Layout)

Определяет общую структуру страницы:

```html
<!DOCTYPE html>
<html lang="ru">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Название страницы</title>
</head>
<body class="page">
    <div class="page__wrapper">
        <header class="page__header header">
            <div class="container">
                <h1 class="header__title">Название сайта</h1>
                <nav class="header__nav">...</nav>
            </div>
        </header>
        
        <main class="page__main main">
            <div class="container">
                <div class="main__content">...</div>
                <aside class="main__sidebar">...</aside>
            </div>
        </main>
        
        <footer class="page__footer footer">
            <div class="container">
                <p class="footer__copyright">© 2025 Название компании</p>
            </div>
        </footer>
    </div>
</body>
</html>
```

### 2. Паттерн "Секционирование" (Sectioning)

Организует контент в логические секции:

```html
<main class="content">
    <section class="section section--hero">
        <div class="container">
            <h1 class="section__title">Заголовок главной секции</h1>
            <p class="section__description">Описание секции</p>
        </div>
    </section>
    
    <section class="section section--features">
        <div class="container">
            <h2 class="section__title">Наши преимущества</h2>
            <div class="features-grid">
                <article class="feature">
                    <h3 class="feature__title">Преимущество 1</h3>
                    <p class="feature__description">Описание</p>
                </article>
                <article class="feature">
                    <h3 class="feature__title">Преимущество 2</h3>
                    <p class="feature__description">Описание</p>
                </article>
            </div>
        </div>
    </section>
</main>
```

## Паттерны доступности

### 1. Паттерн "Инклюзивная разметка"

Обеспечивает доступность для всех пользователей:

```html
<form class="accessible-form" novalidate>
    <div class="form-field">
        <label for="email" class="form-field__label">Email *</label>
        <input 
            type="email" 
            id="email" 
            name="email" 
            required
            aria-describedby="email-error"
            class="form-field__input">
        <div id="email-error" class="form-field__error" role="alert"></div>
    </div>
    
    <fieldset class="form-fieldset" role="group" aria-labelledby="preferences-title">
        <legend id="preferences-title" class="form-fieldset__title">Настройки</legend>
        <div class="form-field">
            <input type="checkbox" id="newsletter" name="newsletter">
            <label for="newsletter">Получать рассылку</label>
        </div>
    </fieldset>
    
    <button type="submit" class="btn btn--primary">Отправить</button>
</form>
```

### 2. Паттерн "Прогрессивное раскрытие"

Управление видимостью контента:

```html
<div class="accordion" data-accordion>
    <div class="accordion__item" data-accordion-item>
        <button 
            class="accordion__header" 
            aria-expanded="false" 
            aria-controls="panel-1"
            data-accordion-trigger>
            Заголовок аккордеона
        </button>
        <div 
            id="panel-1" 
            class="accordion__panel" 
            role="region" 
            aria-labelledby="accordion-1"
            hidden>
            <p>Содержимое аккордеона</p>
        </div>
    </div>
</div>
```

## Паттерны для SEO

### 1. Паттерн "Семантическая структура"

Оптимизация для поисковых систем:

```html
<!DOCTYPE html>
<html lang="ru">
<head>
    <meta charset="UTF-8">
    <title>SEO-оптимизированный заголовок страницы</title>
    <meta name="description" content="Краткое описание содержимого страницы">
    <link rel="canonical" href="https://example.com/page">
    <meta property="og:title" content="Заголовок для социальных сетей">
    <meta property="og:description" content="Описание для социальных сетей">
</head>
<body itemscope itemtype="https://schema.org/WebPage">
    <header>
        <h1 itemprop="name">Название сайта</h1>
    </header>
    
    <main itemprop="mainContentOfPage">
        <article itemscope itemtype="https://schema.org/Article">
            <h1 itemprop="headline">Заголовок статьи</h1>
            <div itemprop="author" itemscope itemtype="https://schema.org/Person">
                <span itemprop="name">Имя автора</span>
            </div>
            <time itemprop="datePublished" datetime="2025-01-15">15 января 2025</time>
            <div itemprop="articleBody">
                <p>Основное содержимое статьи...</p>
            </div>
        </article>
    </main>
</body>
</html>
```

## Паттерны для производительности

### 1. Паттерн "Ленивая загрузка"

Оптимизация загрузки ресурсов:

```html
<!-- Изображения с ленивой загрузкой -->
<article class="post">
    <h2>Заголовок статьи</h2>
    <img 
        src="placeholder.jpg" 
        data-src="actual-image.jpg" 
        alt="Описание изображения" 
        loading="lazy" 
        class="lazy-image">
    <p>Содержимое статьи...</p>
</article>

<!-- Видео с ленивой загрузкой -->
<video 
    preload="none" 
    controls 
    class="lazy-video">
    <source data-src="video.mp4" type="video/mp4">
</video>
```

### 2. Паттерн "Оптимизация критического рендеринга"

```html
<!DOCTYPE html>
<html lang="ru">
<head>
    <!-- Критические стили встроены -->
    <style>
        .header { height: 60px; background: #fff; }
        .hero { height: 400px; background: #f5f5f5; }
    </style>
    
    <!-- Не критические стили отложены -->
    <link rel="preload" href="styles.css" as="style" onload="this.onload=null;this.rel='stylesheet'">
    <noscript><link rel="stylesheet" href="styles.css"></noscript>
    
    <meta charset="UTF-8">
    <title>Оптимизированная страница</title>
</head>
<body>
    <!-- Контент страницы -->
</body>
</html>
```

## Российские особенности реализации

В условиях российского рынка 2025 года важно учитывать:

1. **Региональные особенности** - поддержка различных языков и культурных особенностей
2. **Ограничения по скорости интернета** - оптимизация для медленных соединений
3. **Законодательные требования** - соблюдение законов об персональных данных
4. **Санкционные ограничения** - использование альтернативных решений при недоступности западных сервисов

### Пример с учетом российских реалий:

```html
<!-- Счетчик метрики с российским провайдером -->
<script>
    // Проверка доступности западных сервисов и альтернативы
    if (typeof gtag !== 'undefined') {
        // Использование Google Analytics (если доступен)
        gtag('config', 'GA_TRACKING_ID');
    } else {
        // Использование российского аналога
        // Яндекс.Метрика или Альт-Метрика
        (function(m,e,t,r,i,k,a){
            // Код российской метрики
        })(window,document,'script');
    }
</script>

<!-- Работа с персональными данными -->
<form class="data-processing-form" data-consent-required>
    <div class="form-field">
        <label for="email">Email</label>
        <input type="email" id="email" name="email" required>
    </div>
    
    <div class="form-field">
        <input type="checkbox" id="consent" name="consent" required>
        <label for="consent">Согласен на обработку персональных данных</label>
    </div>
    
    <button type="submit">Отправить</button>
</form>
```
