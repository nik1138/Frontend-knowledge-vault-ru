# Основные концепции React

## Обзор

React — это библиотека JavaScript для создания пользовательских интерфейсов, построенная на нескольких ключевых концепциях, которые обеспечивают гибкость, производительность и удобство разработки. В этом руководстве мы рассмотрим основные принципы, лежащие в основе React.

## Компонентно-ориентированная архитектура

React строится на **компонентах** — независимых, повторно используемых блоках кода, которые отвечают за отдельные части интерфейса. Компоненты могут быть функциональными или классовыми и могут содержать состояние, логику и визуальное представление.

```jsx
function Welcome(props) {
  return <h1>Привет, {props.name}!</h1>;
}
```

> [!info] Подробнее
> Для более глубокого понимания архитектуры компонентов см. [[component-architecture/Component Architecture Overview]]

## Декларативное программирование

React использует **декларативный подход**, при котором разработчики описывают, как должен выглядеть интерфейс в определенном состоянии, а React сам заботится о том, чтобы DOM соответствовал этому описанию. Это делает код более предсказуемым и легким для отладки.

## Одностороннее связывание данных (One-way Data Binding)

В React данные передаются от родительских компонентов к дочерним через **props** в одном направлении. Это упрощает отслеживание изменений и предотвращает нежелательные побочные эффекты.

## Виртуальный DOM

React создает **виртуальный DOM** — легковесное представление реального DOM в памяти. При изменении состояния React сначала обновляет виртуальный DOM, а затем вычисляет минимальный набор изменений, необходимый для обновления реального DOM.

> [!tip] Производительность
> Виртуальный DOM позволяет оптимизировать производительность за счет минимизации манипуляций с реальным DOM.

## JSX и его трансформация

**JSX** — это синтаксическое расширение JavaScript, которое позволяет писать HTML-подобный код внутри JavaScript. JSX трансформируется в вызовы `React.createElement()`.

```jsx
const element = <h1>Привет, мир!</h1>;
// Трансформируется в:
// const element = React.createElement('h1', null, 'Привет, мир!');
```

## Процесс согласования (Reconciliation)

**Reconciliation** — это процесс, с помощью которого React обновляет DOM, сравнивая предыдущее и новое деревья виртуального DOM. React использует эвристику для эффективного обновления, минимизируя количество изменений.

## Механизм рендеринга

React рендерит компоненты в DOM, начиная с корневого компонента. При изменении состояния или props происходит перерендеринг измененных компонентов.

## Односторонний поток данных

В React данные всегда передаются от родительских компонентов к дочерним, что обеспечивает **односторонний поток данных**. Это делает приложение более предсказуемым и легким для понимания.

## Управление состоянием

React предоставляет систему **состояния** (`state`) для хранения и управления изменяемыми данными компонента. Состояние можно управлять с помощью `useState` в функциональных компонентах или `this.state` в классовых компонентах.

```jsx
function Counter() {
  const [count, setCount] = useState(0);

  return (
    <div>
      <p>Счетчик: {count}</p>
      <button onClick={() => setCount(count + 1)}>+</button>
    </div>
  );
}
```

> [!note] Управление сложным состоянием
> Для сложных приложений рекомендуется использовать [[state-management/State Management Solutions]].

## Композиция против наследования

React поощряет использовать **композицию** вместо **наследования** для повторного использования кода между компонентами. Это достигается через `props`, особенно через `children`.

## Неизменяемость в React

React поощряет **неизменяемость** — при изменении данных создается новый объект, а не изменяется существующий. Это помогает избежать непредсказуемых изменений и упрощает отладку.

## Чистые функции

Компоненты в React должны быть **чистыми функциями** по отношению к своим props — они не должны изменять props и должны возвращать одинаковый результат при одинаковых входных данных.

## Изоляция компонентов

Каждый компонент изолирован от других, что позволяет разрабатывать их независимо и повторно использовать в разных частях приложения.

## Жизненный цикл компонентов

У компонентов есть **жизненные циклы**: монтирование, обновление и размонтирование. В функциональных компонентах это достигается с помощью хуков, таких как `useEffect`.

> [!info] Подробнее о жизненном цикле
> См. [[react/hooks/React Hooks Guide]] для изучения хуков и жизненного цикла.

## Система Props

**Props** — это способ передачи данных от родительского компонента к дочернему. Props являются неизменяемыми и обеспечивают одностороннюю связь между компонентами.

## Система State

**State** — это локальное хранилище данных компонента, которое может изменяться. Изменение состояния вызывает перерендеринг компонента.

## Система обработки событий

React предоставляет систему **обработки событий**, похожую на DOM-события, но с использованием camelCase и передачей функций вместо строк.

```jsx
function Button() {
  const handleClick = () => {
    console.log('Кнопка нажата');
  };

  return <button onClick={handleClick}>Нажми меня</button>;
}
```

## Условный рендеринг

React позволяет **условно отображать** компоненты в зависимости от состояния или props с помощью условных операторов, логического оператора `&&` или тернарного оператора.

```jsx
function Greeting(props) {
  const isLoggedIn = props.isLoggedIn;
  if (isLoggedIn) {
    return <h1>Привет, пользователь!</h1>;
  }
  return <h1>Пожалуйста, войдите.</h1>;
}
```

## Рендеринг списков

React может **рендерить списки** элементов с помощью метода `map()` и ключевых атрибутов (`key`) для идентификации элементов.

```jsx
function List({ items }) {
  return (
    <ul>
      {items.map(item => (
        <li key={item.id}>{item.name}</li>
      ))}
    </ul>
  );
}
```

## Границы ошибок (Error Boundaries)

**Границы ошибок** — это компоненты, которые ловят JavaScript-ошибки в дереве компонентов, записывают их и отображают запасной UI вместо падающего компонента.

> [!warning] Ограничения
> Границы ошибок не ловят ошибки в обработчиках событий, асинхронных функциях или на уровне сервера.
