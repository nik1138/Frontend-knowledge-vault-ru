# Обзор React

React — это популярная JavaScript-библиотека с открытым исходным кодом для создания пользовательских интерфейсов. Разработанная и поддерживаемая Facebook, React позволяет разработчикам создавать сложные интерфейсы из небольших, изолированных компонентов.

## История и развитие React

React был впервые представлен в 2013 году на JSConf. Его разработал инженер Facebook Джордан Валке. Изначально React использовался в ленте новостей Facebook и позже в Instagram. С тех пор он стал одной из самых популярных библиотек для разработки интерфейсов.

Ключевые вехи развития:
- 2013 — Первый релиз
- 2015 — Появление React Native
- 2016 — Введение Fiber (переписывание ядра)
- 2017 — Появление функциональных компонентов с хуками (альфа-версия)
- 2018 — Официальный релиз React Hooks
- 2020 — Появление Concurrent Mode (React 18)

Для более подробной информации о хронологии развития см. [[history|React история]].

## JSX и его преимущества

JSX (JavaScript XML) — это синтаксическое расширение JavaScript, которое позволяет писать HTML-подобный код внутри JavaScript. JSX делает код React более читаемым и позволяет использовать всю мощь JavaScript внутри HTML-подобной структуры.

```jsx
const element = <h1>Привет, мир!</h1>;

function Welcome(props) {
  return <h1>Привет, {props.name}!</h1>;
}
```

Преимущества JSX:
- Читаемость кода
- Возможность использования JavaScript внутри HTML
- Более точные сообщения об ошибках
- Лучшая отладка

Для глубокого понимания JSX см. [[jsx-concept|JSX в React]].

## Компоненты и их типы

React строится на компонентах — независимых и повторно используемых частях пользовательского интерфейса. Компоненты могут быть двух типов:

### Функциональные компоненты

```jsx
function Welcome(props) {
  return <h1>Привет, {props.name}!</h1>;
}
```

### Классовые компоненты

```jsx
class Welcome extends React.Component {
  render() {
    return <h1>Привет, {this.props.name}!</h1>;
  }
}
```

Функциональные компоненты стали предпочтительным способом написания компонентов после введения хуков в React 16.8. Они более компактны и легче поддаются тестированию.

Для более подробного изучения компонентов см. [[react-components|React компоненты]] и [[component-types|Типы компонентов в React]].

## Состояние (State) и свойства (Props)

### Props (свойства)

Props — это данные, передаваемые в компонент от родительского компонента. Props неизменяемы (immutable) внутри компонента.

```jsx
function Welcome(props) {
  return <h1>Привет, {props.name}!</h1>;
}

<Welcome name="Алиса" />
```

### State (состояние)

State — это данные, управляемые самим компонентом. State может изменяться и вызывает повторный рендеринг компонента при изменении.

```jsx
class Clock extends React.Component {
  constructor(props) {
    super(props);
    this.state = {date: new Date()};
  }

  render() {
    return (
      <div>
        <h1>Привет, мир!</h1>
        <h2>Сейчас {this.state.date.toLocaleTimeString()}.</h2>
      </div>
    );
  }
}
```

В функциональных компонентах состояние управляется с помощью хука `useState`.

Для более подробного изучения состояния и свойств см. [[react-state|React State]] и [[props-details|React Props]].

## Методы жизненного цикла

Методы жизненного цикла позволяют выполнять действия на различных этапах существования компонента. В классовых компонентах есть следующие основные методы:

- `componentDidMount` — вызывается после монтирования компонента
- `componentDidUpdate` — вызывается после обновления компонента
- `componentWillUnmount` — вызывается перед размонтированием компонента

```jsx
class Timer extends React.Component {
  constructor(props) {
    super(props);
    this.state = {seconds: 0};
  }

  componentDidMount() {
    this.interval = setInterval(() => {
      this.setState(state => ({
        seconds: state.seconds + 1
      }));
    }, 1000);
  }

  componentWillUnmount() {
    clearInterval(this.interval);
  }

  render() {
    return (
      <div>
        Секунд: {this.state.seconds}
      </div>
    );
  }
}
```

В функциональных компонентах эти методы заменяются хуком `useEffect`.

Для подробного изучения жизненного цикла см. [[lifecycle-methods|Методы жизненного цикла React]].

## Введение в хуки

Хуки — это функции, которые позволяют использовать состояние и другие возможности React в функциональных компонентах. Основные хуки:

- `useState` — для управления состоянием
- `useEffect` — для работы с побочными эффектами
- `useContext` — для работы с контекстом
- `useReducer` — альтернатива useState для сложного состояния

```jsx
import React, { useState, useEffect } from 'react';

function Example() {
  const [count, setCount] = useState(0);

  useEffect(() => {
    document.title = `Вы нажали ${count} раз`;
  });

  return (
    <div>
      <p>Вы нажали {count} раз</p>
      <button onClick={() => setCount(count + 1)}>
        Нажми меня
      </button>
    </div>
  );
}
```

Для подробного изучения хуков см. [[react-hooks|React Hooks]].

## Концепция виртуального DOM

React использует виртуальный DOM (Document Object Model) для оптимизации обновлений реального DOM. Виртуальный DOM — это легковесное представление реального DOM в памяти.

Когда состояние компонента изменяется, React создает новый виртуальный DOM и сравнивает его с предыдущим (алгоритм согласования). Затем React обновляет только те части реального DOM, которые действительно изменились.

Этот подход значительно повышает производительность по сравнению с прямым манипулированием DOM.

Для более глубокого понимания см. [[virtual-dom-concept|Виртуальный DOM в React]].

## Основные концепции

React основывается на нескольких ключевых концепциях:

1. **Компонентный подход** — разбиение интерфейса на независимые части
2. **Односторонний поток данных** — данные передаются сверху вниз
3. **Иммутабельность** — состояние изменяется созданием новых объектов
4. **Декларативность** — описываем, что хотим видеть, а не как это реализовать

Для более подробного изучения концепций см. [[core-concepts|Основные концепции React]].

## Обзор экосистемы

React имеет обширную экосистему, включающую:

- **Redux** — для управления глобальным состоянием
- **React Router** — для навигации в приложении
- **Next.js** — фреймворк для серверного рендеринга
- **Create React App** — инструмент для быстрого старта
- **Jest** — фреймворк для тестирования
- **React DevTools** — инструменты разработчика

React остается одной из ведущих библиотек для разработки пользовательских интерфейсов благодаря своей гибкости, производительности и активному сообществу. Понимание основных концепций React позволяет создавать масштабируемые и поддерживаемые приложения.
