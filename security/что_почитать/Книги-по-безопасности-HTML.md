# Книги по безопасности HTML

## Обзор

Книги по безопасности HTML представляют собой ценный ресурс для изучения фундаментальных и продвинутых аспектов веб-безопасности, связанной с HTML-разметкой и клиентской частью веб-приложений. В 2025 году в российском ИТ-ландшафте особенно актуальны знания в области безопасности, учитывая новые вызовы и требования законодательства.

## Основные рекомендуемые книги

### 1. "The Web Application Hacker's Handbook" - Dafydd Stuttard, Marcus Pinto

**Описание:** Один из фундаментальных трудов в области веб-безопасности, охватывающий широкий спектр уязвимостей, включая те, что связаны с HTML и клиентской частью приложений.

**Релевантность для HTML безопасности:**
- Подробное рассмотрение XSS (межсайтовый скриптинг)
- Защита форм и обработка пользовательского ввода
- Использование HTML5 безопасности
- CSP (Content Security Policy) в контексте HTML

**Примечание для российских разработчиков (2025):** В условиях усиления требований к защите персональных данных и ужесточения законодательства в сфере ИБ, знания из этой книги особенно актуальны.

### 2. "HTML and CSS: Design and Build Websites" - Jon Duckett

**Описание:** Хотя это не книга исключительно о безопасности, она содержит важные сведения о правильной разметке HTML и основах безопасности, которые часто упускаются.

**Релевантность для безопасности:**
- Правильная структура HTML-документа
- Безопасная работа с формами
- Понимание атрибутов и их влияния на безопасность
- Основы семантической разметки

### 3. "Secure Coding: Principles and Practices" - Mark Csikos

**Описание:** Книга, охватывающая общие принципы безопасного программирования, включая клиентскую часть и HTML.

**Релевантность для HTML безопасности:**
- Правила обработки пользовательского ввода
- Защита от инъекций (в том числе HTML-инъекций)
- Правильное использование HTML-атрибутов
- Работа с iframe и другими потенциально опасными элементами

### 4. "XSS Attacks: Cross-Site Scripting Exploits and Defense" - Seth Fogie

**Описание:** Специализированная книга, посвященная исключительно XSS-атакам, одной из основных угроз в контексте HTML.

**Ключевые темы:**
- Типы XSS-атак (Reflected, Stored, DOM-based)
- Защита HTML-страниц от XSS
- Правила эскейпинга и валидации
- CSP как защита от XSS

**Особенность для 2025 года:** В условиях развития веб-компонентов и современных фреймворков, понимание DOM-based XSS особенно важно.

### 5. "The Tangled Web: A Guide to Securing Modern Web Applications" - Michal Zalewski

**Описание:** Глубокое погружение в безопасность веб-приложений, включая протоколы, браузеры и HTML.

**Релевантность для HTML:**
- Архитектура браузеров и безопасность HTML
- Same-Origin Policy и её реализация
- Безопасность HTML5 API
- Работа с кросс-доменными запросами

## Современные российские источники (2025)

### 1. "Безопасность веб-приложений" - А.В. Косолапов

Книга, посвященная веб-безопасности с учетом российских реалий, включая:
- Требования ФСТЭК и ФСБ
- Особенности защиты персональных данных по 152-ФЗ
- Практические рекомендации по защите HTML-форм

### 2. "Информационная безопасность веб-приложений" - С.В. Мосин

Особое внимание уделено:
- Локализации угроз в российском сегменте интернета
- Соответствию требованиям ГОСТ Р 57580
- Практическим аспектам защиты HTML-интерфейсов

## Специализированные ресурсы по HTML безопасности

### 1. "HTML5 Security: A Brief Overview" (различные авторы)

Коллективная работа, посвященная безопасности HTML5, включая:
- Новые уязвимости в HTML5 API
- Безопасность Web Storage
- Защита WebSockets и других HTML5 функций

### 2. "Secure HTML Email: Best Practices"

Хотя и не является полноценной книгой, представляет собой важный ресурс по безопасности HTML в контексте email-рассылок, что особенно актуально для маркетинговых и коммуникационных систем.

## Практические рекомендации

### При выборе литературы по безопасности HTML:

1. **Обратите внимание на дату публикации** - в 2025 году особенно важны книги, учитывающие современные веб-стандарты и последние уязвимости
2. **Проверьте соответствие российским стандартам** - в условиях санкций и требований к импортозамещению, предпочтение отдается отечественным источникам
3. **Учитывайте специфику вашей отрасли** - для банковских или государственных систем могут потребоваться специализированные знания
