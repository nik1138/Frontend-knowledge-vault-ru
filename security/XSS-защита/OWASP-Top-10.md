# OWASP-Top-10

## Обзор

OWASP Top 10 - это авторитетный список из десяти наиболее критических веб-уязвимостей, публикуемый Open Web Application Security Project. В контексте XSS-защиты особенно важны категории, связанные с внедрением кода и неправильной обработкой пользовательского ввода.

## Категории, связанные с XSS

### A03:2021-Injection (Инъекции)
Включает XSS как подкатегорию, где вредоносный скрипт внедряется в уязвимое веб-приложение.

### A07:2021-Identification and Authentication Failures (Сбои в идентификации и аутентификации)
Может быть связан с XSS, когда атакующий использует XSS для захвата аутентификационных данных.

### A08:2021-Software and Data Integrity Failures (Сбои целостности программного обеспечения и данных)
Связан с XSS через возможности подделки данных и внедрения вредоносного контента.

## Классификация XSS в OWASP Top 10

В предыдущих версиях OWASP Top 10 XSS имел отдельную категорию:
- **A07:2017-Cross-Site Scripting (XSS)** - прямое упоминание уязвимостей межсайтового скриптинга

## Рекомендации по защите

1. **Валидация и санитизация ввода** - всегда проверяйте и очищайте пользовательский ввод
2. **Кодирование вывода** - правильно кодируйте данные при выводе в HTML, JavaScript, CSS, URL или HTTP заголовки
3. **Использование CSP** - внедрите политику безопасности контента для ограничения выполнения скриптов
4. **Фреймворки безопасности** - используйте проверенные фреймворки, которые автоматически защищают от XSS
